# 功能一览与最短测试清单

> 这份是最短版，只回答三件事：这个项目有什么功能、怎么测、预期结果是什么。

## 1. 功能一览

| 功能 | 测试输入 | 预期结果 |
| --- | --- | --- |
| 后端健康检查 | `curl http://127.0.0.1:8000/api/health` | 返回 `status=ok`，并显示模型/MySQL 状态 |
| 前端健康面板 | 打开 `http://localhost:5173` | 看到健康状态、输入框、事件流和结果区 |
| 数据库查询助手 | “查询心血管药品并输出表格” | 返回数据库结果和 Markdown 表格 |
| 网络搜索助手 | “搜索 2026 年 AI 在电商行业的应用趋势” | 返回搜索结果和摘要 |
| RAGFlow 助手 | “总结内部知识库里的电商 AI 趋势” | 返回知识库问答结果，没配则友好提示 |
| 文件上传与读取 | 先上传文件，再让它总结 | 能读取附件并结合任务回答 |
| 结果文件生成 | “请整理一份报告并保存为 Markdown” | 在 `app/output/session_{thread_id}` 下生成文件 |

## 2. 先测这 5 个

如果你只想确认项目能跑，先测这 5 个：

1. 后端健康检查
2. 前端健康面板
3. 数据库查询助手
4. 文件上传与读取
5. 结果文件生成

## 3. 最短测试例子

### 3.1 后端健康检查

- 输入：
  ```bash
  curl http://127.0.0.1:8000/api/health
  ```
- 预期结果：
  - 返回 JSON
  - `backend=alive`
  - `llm.configured=true`
  - `mysql.configured=true`

### 3.2 前端健康面板

- 输入：
  - 打开 `http://localhost:5173`
- 预期结果：
  - 页面能打开
  - 能看到健康面板

### 3.3 数据库查询助手

- 输入：
  ```text
  请直接查询 drugs 表中 therapeutic_area 为 心血管 的药品，并用 Markdown 表格输出名称、品牌、规格、厂家和简介，直接给出结果，不要向我提问。
  ```
- 预期结果：
  - 返回药品表格
  - 结果和 NAS MySQL 中的数据一致
  - 生成结果文件

### 3.4 网络搜索助手

- 输入：
  ```text
  搜索 2026 年 AI 在电商行业的应用趋势，并总结 3 个重点。
  ```
- 预期结果：
  - 返回搜索结果
  - 给出简要总结

### 3.5 RAGFlow 助手

- 输入：
  ```text
  请根据内部知识库总结电商行业 AI 应用的 3 个核心趋势。
  ```
- 预期结果：
  - 返回知识库问答结果
  - 没配置时给出未配置提示

### 3.6 文件上传与读取

- 输入：
  1. 上传一个 PDF、Word、Excel 或 Markdown
  2. 再输入：
     ```text
     请先读取我上传的文件，然后总结主要内容。
     ```
- 预期结果：
  - 能读到附件内容
  - 回答里会引用文件信息

### 3.7 结果文件生成

- 输入：
  ```text
  请整理一份心血管药品报告，并保存为 Markdown 文件。
  ```
- 预期结果：
  - 生成 `.md` 文件
  - 文件在 `app/output/session_{thread_id}` 下

## 4. 快速排错

1. 先看 `/api/health`
2. 再看前端页面
3. 再确认 `.env`
4. 再确认 MySQL 是否能连
5. 再看结果文件是否生成
