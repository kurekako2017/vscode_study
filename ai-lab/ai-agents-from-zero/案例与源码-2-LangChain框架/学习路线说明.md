# 02-models_io + 05_parser 学习路线说明

> 这份说明专门给 `02-models_io` 和 `05_parser` 这两组案例用。  
> 先学模型接入，再学输出解析，会比直接看零散代码更容易建立完整认知。

## 2. 为什么要先学 02-models_io

`02-models_io` 解决的是“模型怎么接进来”。

你先看这一组，是为了先搞清楚这些基础问题：

- OpenAI 官方 SDK 和 LangChain 的返回值有什么区别
- `ChatOpenAI`、`ChatDeepSeek`、`init_chat_model` 分别适合什么场景
- `temperature`、`max_tokens` 这些参数会怎么影响结果
- 同样是调用模型，为什么有些写法返回 `AIMessage`，有些写法返回原生对象

如果这一层没搞清楚，后面的 Parser 会很容易看懵，因为你会不知道“模型返回的原始结果长什么样”。

## 3. 02-models_io 每个文件该看什么

### 3.1 `ModelIO_OpenAI.py`

- 看点：OpenAI 官方 SDK 的原生调用方式
- 重点：理解“原生返回值”和“LangChain 返回值”不是一回事
- 你要记住：这是最底层、最直接的接法

### 3.2 `ModelIO_ChatOpenAI.py`

- 看点：LangChain 的 `ChatOpenAI`
- 重点：理解如何把兼容接口接入 LangChain
- 你要记住：这里开始返回的是 `AIMessage`

### 3.3 `ModelIO_DeepSeek.py`

- 看点：原生 provider 的接法
- 重点：理解不同厂商的原生 SDK 写法
- 你要记住：原生 provider 和兼容接口是两条并行路线

### 3.4 `ModelIO_Init_chat_model.py`

- 看点：LangChain 1.x 推荐的统一入口
- 重点：理解为什么统一入口更适合做多模型切换
- 你要记住：这里是后面做统一封装时最有价值的一种写法

### 3.5 `ModelIO_Params.py`

- 看点：temperature、max_tokens、返回元数据
- 重点：理解参数如何影响随机性、长度和成本
- 你要记住：这一步是在学“怎么控制模型行为”

### 3.6 `ModelIO_Qwen.py`

- 看点：阿里云百炼 / 通义千问的原生接法
- 重点：理解原生 provider 的 `invoke()` 和 `stream()` 差异
- 你要记住：这一步适合和云端厂商官方接法一起看

## 4. 为什么要再学 05_parser

`05_parser` 解决的是“模型输出怎么变成程序能用的数据”。

你学完 `02-models_io` 之后，再看 Parser，会更自然，因为你已经知道：

- 模型先返回什么
- 原始输出长什么样
- 什么时候只是字符串
- 什么时候需要 JSON
- 什么时候需要强校验对象

Parser 的核心目标就是把“模型能看懂的话”变成“程序能直接处理的数据结构”。

## 5. 05_parser 的推荐阅读顺序

建议按照“从简单到严格”的顺序看：

1. `StrOutputParserDemo.py`
2. `JsonOutputParserDemo.py`
3. `JsonOutputParser_GetFormatInstructions.py`
4. `StructuredOutput_TypedDict.py`
5. `StructuredOutput_Pydantic.py`
6. `AnnotatedTypedDict.py`
7. `AnnotatedPydantic.py`

## 6. 为什么这个顺序最适合初学者

### 6.1 先看 `StrOutputParserDemo.py`

- 先理解最简单的解析：把模型输出转成纯字符串
- 你要先知道：很多时候程序只需要正文，不需要结构化字段

### 6.2 再看 `JsonOutputParserDemo.py`

- 这是从“字符串”过渡到“字典”的第一步
- 你会开始理解：只要模型输出符合 JSON，程序就能进一步处理

### 6.3 再看 `JsonOutputParser_GetFormatInstructions.py`

- 你会理解：光靠一句“请输出 JSON”还不够，最好把 schema 明确告诉模型
- 这一步非常重要，因为它是后面结构化输出的基础

### 6.4 再看 `StructuredOutput_TypedDict.py`

- 你会第一次看到：模型输出可以直接变成字典
- 这里适合建立“结构化输出”的整体直觉

### 6.5 再看 `StructuredOutput_Pydantic.py`

- 你会理解：结构化输出不只是“长得像 JSON”，还可以做字段校验
- 这一步适合训练“程序要对模型输出做质量把关”的思维

### 6.6 最后看 `AnnotatedTypedDict.py` 和 `AnnotatedPydantic.py`

- 这两份是为了对比：
  - `Annotated + TypedDict` 更偏“类型说明”
  - `Annotated + Pydantic` 更偏“运行时校验”
- 你看完之后，会更清楚“元数据”和“校验”到底谁在起作用

## 7. 这两组案例要一起记住的结论

### 7.1 `02-models_io` 的结论

- 你要先会接模型
- 你要知道不同 provider 的写法差异
- 你要理解返回对象是什么

### 7.2 `05_parser` 的结论

- 你要会把模型输出转成程序能用的结构
- 你要知道什么时候用字符串、什么时候用 JSON、什么时候用强类型对象
- 你要知道模型输出不是“看起来像”就够了，程序还要能稳定解析和校验

## 8. 最适合初学者的记忆方式

你可以把这两组案例记成一句话：

```text
02-models_io 负责“把模型接进来”，05_parser 负责“把模型输出变成能用的数据”。
```

如果你先把这句话记住，再去看每个文件，会轻松很多。

## 9. 建议的练习方式

1. 先把 `02-models_io` 全部跑一遍，确认不同接法的返回值差异
2. 再把 `05_parser` 全部跑一遍，确认不同 parser 的输出类型差异
3. 最后对比 `JsonOutputParser`、`PydanticOutputParser` 和 `with_structured_output`
4. 如果你想继续深入，再去看后面的 Prompt、Chain、Agent、Memory

## 10. 适合的学习目标

- 想搞懂模型接入
- 想搞懂 LangChain 统一入口
- 想搞懂结构化输出
- 想为后面的 Agent、RAG、工具调用打基础
