# 案例与源码-1-Coze&Dify工作流智能体

这个目录主要存放 **Coze / Dify 工作流案例的讲解文档、工作流导出文件、样例数据和截图素材**。

先说结论：**这里几乎没有传统意义上的可直接运行代码**。  
你看到的 `.md` 是案例说明，`.txt` / `.yml` 是工作流导出或节点配置，`.csv` / `.xlsx` 是示例数据，图片则放在项目根目录的 `images/` 下配合正文查看。

---

## 1. 先看哪里

推荐顺序：

1. 先看本文件，搞清楚目录分层。
2. 再看每个案例文件夹里的 `*.md`，了解这个案例要做什么。
3. 然后看对应的 `.txt` / `.yml` 工作流导出文件，确认节点和配置。
4. 最后看 `.csv` / `.xlsx` / 图片等数据与素材，补齐案例细节。

如果你是第一次接触 Coze / Dify，建议先从：

- `3.1-Coze案例：一键生成行业调研PPT`
- `3.4-Dify案例：一键生成行业调研报告`

这两个最适合先建立整体概念。

---

## 2. 文件类型说明

| 类型 | 含义 | 你该怎么读 |
| --- | --- | --- |
| `*.md` | 案例说明文档 | 先看，了解目标、流程、用到的插件 / 模型 / 节点 |
| `*.txt` | Coze 工作流或节点导出文本 | 对照文档看，理解具体链路 |
| `*.yml` | Dify 工作流导出文件 | 对照文档看，理解工作流结构与配置 |
| `*.csv` / `*.xlsx` | 样例数据 | 看输入输出样本，理解案例效果 |
| `images/` 下的图片 | 截图素材 | 配合文档看界面和流程 |

## 4. 这个目录里没有什么

- 没有可直接运行的 Python 项目入口
- 没有像 LangChain / LangGraph 那种 `python3 xxx.py` 直接跑的源码
- 主要是案例拆解、导出文件和数据素材

如果你想看“可运行代码”，应该去：

- `案例与源码-2-LangChain框架`
- `案例与源码-3-LangGraph框架`
