## 4. 推荐推进顺序

### 第 1 步：准备 Python 和前端依赖

目标是让本地命令可执行。

- 后端：`uv sync`
- 前端：`cd frontend && pnpm install`

### 第 2 步：准备模型与配置

目标是让后端可以读取到真实配置。

- 新建 `.env`
- 填入 `LLM_API_KEY`
- 下载 `BAAI/bge-large-zh-v1.5` 到 `docker/embedding/bge-large-zh-v1.5`

### 第 3 步：启动 Docker 基础服务

目标是让 MySQL、Qdrant、ES、Kibana、Embedding 先活起来。

```bash
docker compose -f docker/docker-compose.yaml up -d
```

### 第 4 步：构建元数据知识库

目标是把教学数仓、元数据库、向量索引、全文索引都准备好。

```bash
uv run python -m app.scripts.build_meta_knowledge -c conf/meta_config.yaml
```

### 第 5 步：启动后端

目标是把问数接口服务跑起来。

```bash
uv run fastapi dev main.py
```

### 第 6 步：启动前端

目标是打开问数页面并连上后端。

```bash
cd frontend
pnpm dev
```

## 6. 这份环境在项目里负责什么

- `MySQL`
  - 保存教学数仓
  - 保存元数据库
  - 供 SQL 执行和元数据管理使用
- `Qdrant`
  - 保存字段和指标的语义向量
- `Elasticsearch`
  - 保存字段真实取值，支持关键词和值域检索
- `Embedding`
  - 把字段、指标、问题文本转成向量
- `LangGraph`
  - 组织关键词抽取、召回、过滤、SQL 生成、校验、执行等流程
- `FastAPI`
  - 对外提供 `/api/query`
- `SSE`
  - 流式返回执行过程和结果
- `React`
  - 展示前端问数页面和流程信息
