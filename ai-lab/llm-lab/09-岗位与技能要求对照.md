# 岗位与技能要求对照

这一章解决的问题是：把学习内容翻译成岗位、简历和面试能理解的能力表达。

## 1. 常见岗位方向

| 岗位方向 | 常见关键词 | 优先学习 | 推荐作品 |
| --- | --- | --- | --- |
| 生成 AI 应用开发 | `Python`、`RAG`、`FastAPI`、`生成AIアプリ` | 02 到 06 | `structured_output_demo`、`doc_qa_agent`、`rag_api_demo` |
| 社内搜索 / 知识库问答 | `社内検索`、`ナレッジ検索`、`FAQ`、`PDF` | RAG、文档处理、来源返回 | `doc_qa_agent`、`rag_api_demo` |
| 后端 API / 系统集成 | `API連携`、`バックエンド`、`認証`、`運用` | FastAPI、错误处理、配置、日志 | `rag_api_demo` |
| Agent / 自动化进阶 | `Agent`、`Tool Calling`、`Workflow` | 先完成 LLM 主线，再进 Agent | `tool_agent_demo`、`workflow_agent` |

## 3. 简历表达模板

中文表达：

- 基于 Python 和 OpenAI API 实现生成 AI 应用 PoC
- 基于本地文档实现最小 RAG 问答系统
- 使用 FastAPI 将文档问答能力封装为后端接口
- 支持文档切分、关键词检索、来源返回和 mock 测试
- 整理 RAG 评估观点和改进方向

日语表达：

- `Python による生成AIアプリ PoC を実装`
- `RAG を用いた社内検索 / ナレッジ検索の試作`
- `FastAPI による問い合わせ API を実装`
- `構造化出力を利用した分類・計画生成機能を実装`
- `RAG の評価観点、改善課題を整理`

Agent 方向可以写成：

- `構造化出力と Tool Calling を利用した業務支援 Agent PoC`
- `社内ナレッジ検索と連携した問い合わせ支援 Agent`
- `固定ワークフローによる業務補助 Agent を試作`

## 5. 面试准备重点

每个项目至少能说清：

- 解决什么业务问题
- 输入和输出是什么
- 为什么这样设计
- 当前版本限制是什么
- 如果进入企业环境，下一步补什么
- 如何评估效果和失败案例
