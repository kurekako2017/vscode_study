# 云平台与企业环境

这一章解决的问题是：本地 PoC 如果进入企业环境，会多出哪些现实约束。

```text
本地 RAG / API -> 企业模型平台 -> 权限/网络/配置/日志/成本 -> 可运维服务
```

## 1. 学完要会什么

- 知道为什么本地 demo 和企业环境差很多
- 知道日本现场常见的 `Azure OpenAI`、`AWS Bedrock`
- 能解释权限、网络、密钥、成本、日志为什么重要
- 能把 `RAG -> FastAPI -> 云平台` 这条链路讲清楚
- 能说出 [rag_api_demo](../agent-lab/projects/rag_api_demo/README.md) 进入企业环境要补什么

## 2. 本地版和企业版的区别

| 项目 | 本地学习版 | 企业环境版 |
| --- | --- | --- |
| 文档来源 | 本地目录 | 共享目录、对象存储、企业文档库 |
| API Key | 个人环境变量 | 密钥管理、受控配置 |
| 调用方 | 自己的终端 | 前端、内部系统、管理后台 |
| 权限 | 基本不区分 | 用户、部门、文档权限 |
| 网络 | 本机外网 | 代理、内网、VNet/VPC |
| 日志 | 终端输出 | 可追踪、可审计 |
| 成本 | 少量测试 | 调用量、模型选择、预算控制 |

## 3. 常见平台方向

| 平台 | 常见关键词 | 你先理解什么 |
| --- | --- | --- |
| Azure OpenAI | `Azure OpenAI`、`Azure AI Search`、`App Service`、`VNet` | 企业 Microsoft 环境里常见的模型接入方式 |
| AWS Bedrock | `Bedrock`、`S3`、`Lambda`、`ECS`、`API Gateway` | AWS 环境里调用模型并接对象存储、API、后端服务 |

当前阶段不用马上精通云平台。先理解：

```text
云平台是模型接入、权限、网络、部署和监控的企业基础设施。
```

## 5. 企业环境里最常见的限制

| 限制 | 典型问题 |
| --- | --- |
| 网络 | 不能直接访问外网，需要代理或内网出口 |
| 密钥 | API Key 不能写代码，也不能随便放个人机器 |
| 权限 | 不是所有人都能检索所有文档 |
| 日志 | 出错要能追踪，访问要能审计 |
| 成本 | 调用次数、模型选择、文档处理都影响预算 |
| 部署 | 服务要能启动、健康检查、回滚、重载 |

## 6. 推荐学习顺序

1. 先跑通 [04-RAG.md](./04-RAG.md)
2. 再跑通 [05-FastAPI与企业集成.md](./05-FastAPI与企业集成.md)

不要一上来陷进云厂商名词。先把应用链路讲清楚，再学它怎么上云。
