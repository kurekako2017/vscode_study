# 01. 学习路线

这条路线的目标不是“学完所有 AI 技术”，而是尽快建立能做项目的 `LLM 应用开发` 能力。

建议先看：

- [LLM应用框架系统知识.md](./LLM应用框架系统知识.md)

再按本文路线推进。

## 1. 总目标

面向日本 IT 现场和派遣案件，优先建立这些能力：

- 模型调用
- 结构化输出
- `RAG / 社内検索`
- `FastAPI` 后端接口
- 评估与运维意识
- 基础前后端联动

核心路线：

```text
Python -> 模型调用 -> 结构化输出 -> RAG -> FastAPI -> 评估 -> TypeScript -> Agent
```

## 2. 路线原则

先记住 4 条：

- 先做能运行的应用，再学复杂 Agent。
- 先做 `RAG + FastAPI`，再做 Tool Calling。
- 先把 Python 主线跑通，再补 TypeScript。
- 每个阶段都必须有一个可运行输出物。

不要一开始并行硬学太多东西。当前最稳的节奏是：

1. 学一个小知识点
2. 跑一个 demo
3. 自己改一版
4. 写下输入、处理、输出

## 3. 阶段路线

| 阶段 | 主题 | 要学什么 | 输出物 |
| --- | --- | --- | --- |
| 1 | Python 与模型调用 | Python 基础、环境变量、一次模型请求 | `chat_cli` |
| 2 | 结构化输出 | `Pydantic`、schema、结果校验 | `structured_output_demo` |
| 3 | RAG / 社内検索 | 文档读取、切分、检索、引用来源 | `doc_qa_agent` |
| 4 | API 化 | `FastAPI`、请求体、响应、错误处理 | `rag_api_demo` |
| 5 | 评估与运维 | 测试问题集、质量记录、成本延迟 | `rag_eval_notes` |
| 6 | TypeScript 补充 | `async / await`、类型、前端调用 API | 最小页面或调用脚本 |
| 7 | Agent 进阶 | Tool Calling、工作流、多阶段处理 | `tool_agent_demo` / `workflow_agent` |

## 4. 8 周执行路线

### 第 1 周：Python 基础与模型调用

目标：

- 能看懂基础 Python
- 能运行示例
- 能完成一次模型调用或 mock 调用

要学：

- 变量、函数、列表、字典
- `json`
- 文件读取
- 环境变量
- 基础异常处理

输出物：

- 跑通 `chat_cli`
- 修改一次提示词、模型名或输出长度

验收：

- 能解释 `parse_args()`、`build_client()`、`ask_once()` 的作用

### 第 2 周：结构化输出

目标：

- 能让模型返回稳定字段
- 能读懂 `Pydantic BaseModel`

要学：

- `Pydantic`
- schema
- `Field`
- `model_dump()`
- `model_dump_json()`

输出物：

- 跑通 `structured_output_demo`
- 新增一个字段或一个新 schema

验收：

- 能说明“只让模型输出 JSON”和“用 schema 解析”的区别

### 第 3 周：文档处理与最小 RAG

目标：

- 能读取本地资料
- 能做最小文档问答

要学：

- 文档读取
- 文本切分
- 关键词匹配
- 检索结果排序
- 引用来源

输出物：

- 跑通 `doc_qa_agent`
- 替换一批本地资料测试

验收：

- 能说明一次问答里哪些片段被检索到了

### 第 4 周：FastAPI 基础

目标：

- 能把能力封装成接口
- 能处理基本请求和响应

要学：

- `GET / POST`
- 请求体模型
- JSON 响应
- 启动服务
- 基础错误处理

输出物：

- 写一个最小 `/health`
- 写一个最小 `/ask`

验收：

- 能用浏览器或 curl 调用接口

### 第 5 周：RAG API 化

目标：

- 把问答能力包装成可调用服务
- 开始按企业 PoC 的方式组织代码

要学：

- 配置读取
- 日志
- 异常处理
- 文档问答 API

输出物：

- 跑通 `rag_api_demo`
- 增加一个配置项或一个新接口

验收：

- 能说明请求进入 API 后，到返回答案前经历了哪些步骤

### 第 6 周：评估、整理与作品化

目标：

- 不只是能跑，还能说明效果
- 开始整理作品集材料

要学：

- 测试问题集
- 回答质量记录
- 成本与延迟记录
- 需求概要和基本设计

输出物：

- 一页项目说明
- 一页测试记录或评估笔记

验收：

- 能用 3 到 5 个问题说明系统效果和限制

### 第 7 周：补 JavaScript / TypeScript

目标：

- 能看懂基础 TypeScript
- 能写最小 API 调用脚本

要学：

- JavaScript 基础
- `Promise`
- `async / await`
- `type`
- `interface`
- `npm`

输出物：

- 一个 TypeScript 版 API 调用脚本
- 一个读取本地文件并输出结果的脚本

验收：

- 能看懂请求参数和响应类型

### 第 8 周：前后端联动

目标：

- 能做一个简单 AI 页面
- 能理解 Python 后端和 TypeScript 前端怎么配合

要学：

- 前端表单提交
- 调后端接口
- 错误提示
- 基础页面状态

输出物：

- 一个调用 FastAPI 后端的聊天页面
- 或一个调用文档问答接口的页面

验收：

- 页面能输入问题、调用接口、展示回答和错误信息

## 6. 什么时候进入下一阶段

不要按“看完了”判断，要按“能改动”判断。

可以进入下一阶段的标准：

- 能独立运行当前 demo
- 能解释主要函数
- 能改一个字段、参数或接口
- 能定位一个常见错误
- 能写下最小输入和输出示例

## 7. 最重要的取舍

现在优先级最高的是：

```text
可运行 demo > 概念完整性
能改代码 > 只看教程
RAG + FastAPI > 复杂 Agent
作品化说明 > 零散学习笔记
```
