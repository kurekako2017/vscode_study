# AI Learn 整体学习顺序与运行指南

本文是 `ai-learn/` 的唯一总执行索引。下面所有命令都从仓库根目录 `ai-lab/` 执行。

运行分为两种模式：

- **真实模型模式**：依次尝试 OpenRouter、NVIDIA NIM、本地 Ollama；必须先加载 `.env`。
- **Mock 模式**：命令显式带 `--mock`，不访问网络、不消耗额度，也不要求 API Key。

> 配置两个云端 Key 不能保证所有 40 个示例都直接运行。Key 只解决 LLM 调用认证；Qdrant、Chroma、MCP、前端、Java 客户端等示例仍需要各自的依赖或本地服务。Mock/概念版示例不依赖云端 Key。

## 0. 一次性准备

```bash
python3 -m venv ai-learn/.venv
source ai-learn/.venv/bin/activate
python3 -m pip install -U pip
python3 -m pip install -r ai-learn/requirements.txt
```

首次配置环境文件：

```bash
cp ai-learn/.env.example ai-learn/.env
chmod 600 ai-learn/.env
```

编辑 `ai-learn/.env`，填入 `OPENROUTER_API_KEY` 和 `NVIDIA_API_KEY`。真实模型模式每次新开终端后先加载：

```bash
set -a
source ai-learn/.env
set +a
```

确认加载成功，但不打印 Key：

```bash
python3 -c 'import os; print("OpenRouter:", bool(os.getenv("OPENROUTER_API_KEY")), "NVIDIA:", bool(os.getenv("NVIDIA_API_KEY")), "Ollama:", os.getenv("OLLAMA_BASE_URL"))'
```

真实 Qdrant、Chroma 或本地 embedding 放到相应阶段再安装：

```bash
python3 -m pip install -r ai-learn/requirements-optional.txt
```

## 0.1 真实模型与 Mock 启动命令

真实模型调用会按以下顺序自动回退：

```text
OpenRouter -> NVIDIA NIM -> Ollama -> 最终 Mock 兜底
```

本地 Ollama 需要另开终端启动并确保模型存在：

```bash
ollama serve
ollama pull qwen2.5-coder:1.5b
```

以下入口支持统一的 `--real` / `--mock`。执行 Real 命令前必须已执行上面的 `source ai-learn/.env`。

## 每个示例的固定学习方法

1. 先打开表格中的对应文档，只看目标、输入输出和安全边界。
2. 复制全局命令运行默认离线路径。
3. 修改一个输入，观察输出状态或分支变化。
4. 阅读 `main.py`，找到输入层、业务层和输出层。
5. 记录“适用场景、失败路径、生产差距”三项，再进入下一个示例。

真实模型调用统一使用 `OPENROUTER_API_KEY`、`NVIDIA_API_KEY` 和 `OLLAMA_BASE_URL`；不要把 Key 写入代码、README 或提交到仓库。配置文件只放在被 Git 忽略的 `ai-learn/.env`。
