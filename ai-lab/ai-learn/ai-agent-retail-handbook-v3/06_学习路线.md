# 06_学习路线

## 第一章 成长目标

目标是能够在日本 AI Agent 现场说明 Retail Insight AI 的业务背景、系统架构、担当范围、设计决策、Review 观点和运用扩展。

【TL Review】

- 是否能从业务讲到代码。
- 是否能从代码讲到运用。

【后续扩展】

- 增加阶段检查表。

## 第二章 Backend 路线

先掌握 Task API、状态查询、SSE、报告 API、health check，再掌握 TaskService、日志、异常、幂等和测试。

【项目对应】

```text
FastAPI -> TaskService -> API schema -> error handling -> logging -> tests
```

【TL Review】

- 是否能设计异步任务 API。
- 是否能解释 HTTP Timeout 风险。

【后续扩展】

- API Gateway。
- SSO。
- Rate Limit。

## 第三章 Workflow 路线

掌握 LangGraph 的 State、Node、Edge、checkpoint，并能把 route、KPI workflow、Research Agent、Report Generation 组织成可 Review 的流程。

【项目对应】

```text
route -> KPI -> Research -> Report -> checkpoint
```

【TL Review】

- 是否能说明每个 node 的输入输出。
- 是否能说明失败路径。

【后续扩展】

- interrupt / resume。
- workflow metrics。

## 第四章 Research Agent 路线

掌握 Tool Layer、来源保留、Research timeout、内部资料检索和 Report 合成。

【项目对应】

```text
market -> competitor -> internal docs -> sources -> report context
```

【TL Review】

- Research 是否有边界。
- Tool 是否安全。

【后续扩展】

- OpenSearch。
- VectorDB。
- Tool Allowlist。

## 第五章 日本现场路线

掌握需求整理、基本設計、詳細設計、API 設計、単体試験、結合試験、レビュー、部署、保守改修、障害対応。

【项目对应】

```text
业务需求 -> 设计书 -> API -> 开发 -> 测试 -> Review -> 部署 -> 保守
```

【TL Review】

- 是否能写设计理由。
- 是否能做影响調査。
- 是否能处理 Review 指摘。

【后续扩展】

- Jira。
- Confluence。
- Review 票。
