## 第一章 自我介绍

自我介绍脚本集中维护在 `02_日本AI项目介绍与自我介绍.md`。面试时把项目说成：


### Q4. なぜ KPI は固定ワークフローにしましたか。

**面接官質問（日语）**

```text
なぜ KPI は Agent に自由に SQL を作らせず、固定ワークフローにしましたか。
```

**推奨回答（日语）**

```text
売上、粗利、在庫などの KPI は、数値の正確性、再現性、監査性が重要です。
Agent に自由に SQL を生成させると、SQL Injection、存在しない項目の参照、集計口径のブレ、重いクエリ、権限違反のリスクがあります。
そのため、KPI は固定 SQL テンプレートと whitelist workflow で処理し、市場調査や競合調査を Research Agent に任せる設計にしました。
```

**深掘りされた場合（日语）**

```text
正式な運用を考える場合は、SQL AST parser、テーブル・カラム whitelist、行列レベル権限、最大件数、タイムアウト、監査ログを追加します。
```

**NG回答**

```text
Agent の SQL 生成が難しかったから固定にしました。
```

**回答ポイント（中文）**

重点是“经营数字必须治理”，核心是经营数字治理。

### Q9. SSE と WebSocket の使い分けは何ですか。

**推奨回答（日语）**

**NG回答**

```text

```

**回答ポイント（中文）**

SSE 单向，WebSocket 双向。按用途选择。

## 第三章 技术面试速答

### Chunk

```text
Chunk は、長い社内文書を検索、引用、権限制御、評価しやすい単位に分割するために使います。
LLM の context 制限だけでなく、Retriever、Citation、ACL、Evaluation の最小単位として重要です。
```

### TopK

```text
TopK は context に入れる候補数を決めます。
TopK が小さいと召回不足、大きいとノイズと token cost が増えます。
評価セットで recall、引用正確性、latency を見ながら調整します。
```

### LangChain / LangGraph

```text
LangChain は Prompt、Retriever、Model、Tool の組み合わせに向いています。
LangGraph は State、Node、Edge、条件分岐、checkpoint を持つ workflow に向いています。
```

### MCP

```text
MCP は Research Agent の tool layer に入れます。
SharePoint、Confluence、Office、社内検索、承認フローを tool schema、allowlist、権限、audit log と組み合わせて扱います。
```

### Docker

```text
Docker は開発環境と実行環境を揃え、レビュー担当やチームメンバーが同じ手順でシステムを起動できるようにするために使います。
```

### AWS

```text
React は S3 + CloudFront、FastAPI は ECS または EKS、DB は RDS PostgreSQL、ファイルは S3、secrets は Secrets Manager、ログと metrics は CloudWatch または OpenTelemetry に送る構成を考えます。
```
