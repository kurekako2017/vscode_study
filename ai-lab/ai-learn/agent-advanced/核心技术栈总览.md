# 核心技术栈总览

这份总览用来回答两个问题：

1. 现在 `ai-lab` 已经覆盖了什么
2. `agent-advanced` 需要补什么

## 现有覆盖情况

| 技术栈 | 当前状态 | 主要位置 | 备注 |
| --- | --- | --- | --- |
| Python | 已覆盖 | `llm-lab/`、`agent-lab/` | 作为主开发语言 |
| OpenAI SDK | 已覆盖 | `chat_cli`、`tool_agent_demo`、`rag_api_demo` | 模型调用主线 |
| Pydantic | 已覆盖 | `structured_output_demo`、`rag_api_demo` | 数据校验和结构化输出 |
| FastAPI | 已覆盖 | `rag_api_demo` | 服务化和 API 暴露 |
| 基础 RAG | 已覆盖 | `llm-lab/04-RAG.md`、`doc_qa_agent` | 文档切分、检索、来源输出 |
| Tool Calling | 已覆盖 | `tool_agent_demo` | Agent 工具调用基础 |
| Workflow Agent | 已覆盖 | `workflow_agent` | 固定流程式 Agent |
| React | 部分覆盖 | `rag_api_demo/react-client` | 主要作为客户端展示 |
| Java / Spring Boot | 部分覆盖 | `rag_api_demo/spring-client` | 作为企业系统接入示例 |
| Docker / 脚本 | 基础覆盖 | 各 demo 的 `Dockerfile`、`run_demo.sh` | 适合做本地 PoC |
| 测试 / Smoke Check | 基础覆盖 | `rag_api_demo`、`tool_agent_demo` | 还不够系统 |
| LangChain 学习代码 | 已补充 | `agent-advanced/projects/langchain_chain_demo` | 可直接跑的链式编排示例 |
| LangGraph 学习代码 | 已补充 | `agent-advanced/projects/langgraph_workflow_demo` | 可直接跑的状态图示例 |
| 高级 RAG 学习代码 | 已补充 | `agent-advanced/projects/advanced_rag_pipeline_demo` | 可直接跑的检索流水线示例 |
| LlamaIndex 概念代码 | 已补充 | `agent-advanced/projects/llamaindex_index_demo` | 不依赖真实包的概念演示 |
| 多 Agent 学习代码 | 已补充 | `agent-advanced/projects/multi_agent_team_demo` | 可直接跑的协作角色示例 |

## 需要补强的核心栈

| 技术栈 | 学习目标 | 建议落点 |
| --- | --- | --- |
| LangChain | 统一抽象、链式编排、工具封装 | `frameworks/langchain/` |
| LlamaIndex | 数据索引、检索、RAG 组合 | `frameworks/llamaindex/` |
| LangGraph | 状态图、分支、循环、多 Agent 控制 | `frameworks/langgraph/` |
| 高级 RAG | rerank、hybrid search、query rewrite、multi-query | `rag/` |
| 多 Agent | 角色拆分、协作、监督、路由 | `multi-agent/` |
| React 前端 | 对话 UI、流式输出、来源引用展示 | `frontend/` |
| 评估体系 | 离线评估、人工评估、Tracing、日志 | `eval/` |
| 部署与工程化 | Docker、环境变量、CI/CD、发布 | `deployment/` |

## 技术栈分层建议

### 1. 表现层

- React
- Vite
- 组件库

### 2. 服务层

- FastAPI
- Pydantic
- Python 工具函数

### 3. 智能层

- OpenAI SDK
- LangChain
- LlamaIndex
- LangGraph

### 4. 检索层

- 文档切分
- 向量索引
- rerank
- hybrid retrieval

### 5. 运行层

- Docker
- 环境变量
- 日志
- Tracing
- 测试
- CI/CD

## 结论

`ai-lab` 现阶段的主线已经很清楚了：

- `llm-lab`：基础 LLM 应用
- `agent-lab`：最小 Agent 和工作流
- `agent-advanced`：框架化、工程化、多 Agent 和高级 RAG，并且已经补上了对应的可运行代码案例

这三层分开后，后面加内容不会互相污染。
