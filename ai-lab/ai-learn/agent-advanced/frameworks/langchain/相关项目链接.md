## 学习建议

1. 先跑 `langchain_chain_demo` 的 mock 模式，理解最短链路。
2. 回到学习笔记理解 Tool、Memory、Retriever、RAG 与 Agent 的职责。
3. 再进入高级 RAG 管线，观察真实 LangChain 组件如何替换教学实现。
4. 最后看企业混合检索，重点关注 ACL、引用和数据源边界。

## 注意事项

- “与 LangChain 相关”不等于所有项目都直接依赖 LangChain SDK，项目 README 和导入代码才是依赖事实来源。
- 不要把 demo 移进本目录；demo 的 assets、requirements 和命令依赖当前相对路径。
- 没有模型凭据时优先使用 mock 或本地模式，不要把 API Key 写入文档或代码。
