# 交付前检查清单

这是一页版清单，适合你在项目交付前直接逐项勾选。

## 1. 开发完成

- [ ] 代码通过 `py_compile`
- [ ] 主要逻辑有中文注释或说明
- [ ] `README.md` 能说明怎么运行
- [ ] 默认支持 mock 或离线模式

## 2. 单元测试

- [ ] prompt、解析、检索、权限过滤都有固定样本
- [ ] 关键纯函数有稳定输出
- [ ] 正常分支和失败分支都能验证
- [ ] 不依赖真实 API 也能检查核心逻辑

## 3. 集成测试

- [ ] 本地模型或便宜模型可以启动
- [ ] 环境变量可以正确读取
- [ ] 请求能发出，返回能解析
- [ ] 关键链路没有明显报错

## 4. 小量 E2E

- [ ] 3 到 10 个核心问题已经跑过
- [ ] 引用来源能返回
- [ ] 答案没有明显跑题
- [ ] 延迟在可接受范围内

## 5. 部署前

- [ ] Dockerfile 或启动脚本可用
- [ ] `/health` 或等价健康检查可用
- [ ] 目标平台能注入环境变量
- [ ] 关键配置已经和日本项目环境对齐

## 6. 上线前抽样

- [ ] 核心样本再跑一遍
- [ ] 没有明显回归
- [ ] 没有敏感配置泄露
- [ ] 日志和监控能接上

## 7. 日本项目平台判断

- [ ] AWS / Azure / GCP / OCI / 内网平台已确认
- [ ] 容器、虚拟机、K8s 其中哪一种已经定下来
- [ ] 网络出口、权限、审计要求已经明确
