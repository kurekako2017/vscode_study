# 需求与评估标准

## 功能需求

1. 将问题拆为可检索查询。
2. 保存结构化证据并按来源 ID 去重。
3. 每个事实必须引用来源，审校失败可回到检索。
4. 最大轮次、递归深度和外部调用预算必须可配置。

## 评估

| 指标 | 最低标准 |
| --- | --- |
| Citation coverage | 事实句 100% 带来源 ID |
| Citation validity | 来源 ID 100% 存在于来源表 |
| Evidence diversity | 至少 2 个去重来源 |
| Termination | 不超过 `max_rounds` |
| Unsupported claim | 固定样例中为 0 |
