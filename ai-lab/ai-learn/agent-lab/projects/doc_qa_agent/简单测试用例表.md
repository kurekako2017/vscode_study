# doc_qa_agent 简单测试用例表

## 1. 说明

本文件用于给 `doc_qa_agent` 提供一份更接近实际执行的简化测试表，覆盖命令行、文档读取、chunk 切分、关键词检索、来源输出和异常处理。

当前 `chunk` 规则如下：

- chunk 前是原始文件内容
- 先统一换行，再 `strip()`
- 按字符长度切分，默认 `CHUNK_SIZE = 1200`
- 相邻 chunk 重叠 `CHUNK_OVERLAP = 200`
- chunk 后的来源标签格式为 `相对路径#chunkN`

## 2. 测试前准备

- 运行目录：`ai-lab/` 根目录
- 运行方式：`python3 main.py ...`
- 模式建议：优先用 `--mock` 做功能验证，再用 `--real` 做联调验证
- 测试文件建议：准备 3 类文档
  - 短文档：小于 `CHUNK_SIZE`
  - 长文档：大于 `CHUNK_SIZE`，并且中间有可识别的重复关键词
  - 无关文档：内容与测试问题完全不相关

## 4. 推荐执行顺序

1. 先跑 CLI-01 到 CLI-03，确认程序能启动、能回答、能显示来源。
2. 再跑 CHUNK-01 到 CHUNK-03，确认切分和来源标签正确。
3. 再跑 CLI-04 到 CLI-06，确认检索、多文件和无命中场景正常。
4. 最后跑 CLI-07 到 CLI-10，确认异常处理完整。

## 5. 代价与注意点

- 这是简化版测试用例，不等于正式测试规格书。
- 当前不支持 `PDF`，很多日本现场常见资料无法直接纳入测试范围。
- 当前检索方式是关键词重叠，命中结果可能受措辞影响较大。
- 每次问答都会触发实际模型调用，会产生 API 成本。
- 当前没有日志持久化、认证和检索评估，因此测试覆盖范围仍然偏 PoC。
