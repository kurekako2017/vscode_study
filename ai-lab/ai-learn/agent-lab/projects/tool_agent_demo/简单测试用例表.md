# 简单测试用例表

## 1. 说明

本用例表用于对 `tool_agent_demo` 做最小验证。

重点不是覆盖所有复杂场景，而是确认这几个核心点：

- 工具调用闭环是否成立
- 文件访问边界是否受控
- 回答是否基于工具结果
- 常见异常是否能被基础处理

## 3. 建议测试数据

可以准备一个最小测试目录，包含以下内容：

- `README.md`
- `notes.txt`
- 一个子目录
- 一个包含关键字的文本文件
- 一个非 UTF-8 或不可正常读取的文件

这样可以一次覆盖：

- 列目录
- 读文件
- 搜索文本
- 异常系

## 4. 执行结果记录建议

学习阶段可以按最简方式记录：

- 执行日期
- 执行人
- 用例 ID
- 结果
- 备注

如果后续要更像现场资料，可以再扩成：

- 实际输入 prompt
- 模型名
- 命中的工具
- 输出摘要

## 5. 代价与注意点

- 这是简化版用例表，不是正式企业级测试规格书。
- 实际调用模型会产生 API 成本。
- 模型对是否发起工具调用存在一定波动，因此测试时应关注“整体链路是否成立”，不要把每次表述完全一致当成唯一标准。
- 当前案例仍是学习版，只覆盖只读工具调用，不覆盖数据库、外部 API、写操作和复杂 Agent 任务编排。
