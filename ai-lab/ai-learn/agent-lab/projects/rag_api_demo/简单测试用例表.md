# rag_api_demo 简单测试用例表

## 1. 说明

本文件用于给 `rag_api_demo` 提供一份简化版测试用例表，适合 PoC、学习项目或小型交付物使用。

## 2. 测试用例

| 用例ID | 分类 | 测试内容 | 前提条件 | 输入 | 执行命令 | 期望结果 |
|---|---|---|---|---|---|---|
| API-01 | 启动 | 服务正常启动 | 文档目录有效，使用 Mock 模式 | 启动服务 | `cd ai-learn/agent-lab/projects/rag_api_demo && RAG_API_MOCK=1 uvicorn main:app --reload --port 8000 --host 127.0.0.1` | 服务启动成功 |
| API-02 | 健康检查 | `GET /health` 返回状态 | 服务已启动 | 无 | `curl http://127.0.0.1:8000/health` | 返回 `status`、`docs_dir`、`chunk_count` |
| API-03 | 正常问答 | `POST /ask` 返回回答 | 文档目录有相关资料 | 问题文本 | `curl -X POST http://127.0.0.1:8000/ask -H "Content-Type: application/json" -d '{"question":"请总结文档重点"}'` | 返回 `answer` 和 `sources` |
| API-04 | 来源返回 | 问答结果含来源 | 文档目录有相关资料 | 问题文本 | `curl -X POST http://127.0.0.1:8000/ask -H "Content-Type: application/json" -d '{"question":"请指出文档中关于 RAG 的来源"}'` | `source_count > 0`，`sources` 非空 |
| API-05 | 无命中场景 | 无相关片段时返回说明 | 文档目录中无相关资料 | 冷门问题 | `curl -X POST http://127.0.0.1:8000/ask -H "Content-Type: application/json" -d '{"question":"请解释量子纠缠"}'` | 返回依据不足类回答 |
| API-06 | 文档重载 | `POST /reload` 重新加载文档 | 服务已启动 | 无 | `curl -X POST http://127.0.0.1:8000/reload` | 返回新的 `chunk_count` |
| API-07 | PDF读取 | 支持 `pdf` 文档 | 文档目录存在可读 PDF | 启动后提问 | `curl -X POST http://127.0.0.1:8000/ask -H "Content-Type: application/json" -d '{"question":"请总结 PDF 文档重点"}'` | 可命中 PDF 内容 |
| API-08 | 目录异常 | 文档目录无效时返回错误 | `RAG_API_DOCS_DIR` 指向不存在目录，服务已启动 | 请求健康检查 | `curl http://127.0.0.1:8000/health` | 返回错误并提示目录无效 |
| API-09 | Key异常 | API Key 未配置 | 未配置 `OPENAI_API_KEY` 且未启用 Mock，服务已启动 | 请求健康检查 | `curl http://127.0.0.1:8000/health` | 返回错误并提示 Key 缺失 |
| API-10 | 模型异常 | 模型调用失败时返回错误 | 服务已启动 | 非法模型或调用失败 | `curl -X POST http://127.0.0.1:8000/ask -H "Content-Type: application/json" -d '{"question":"请总结文档重点","model":"definitely-not-a-model"}'` | 接口返回 `500` |

## 3. 推荐执行顺序

1. 先测启动与健康检查
2. 再测正常问答和来源返回
3. 再测 PDF
4. 最后测异常系

## 4. 代价与注意点

- 这是简化版测试用例，不等于正式测试规格书。
- 当前检索是关键词重叠方式，命中质量有限，测试通过不代表检索质量已经足够上线。
- `PDF` 抽取质量受原始文件格式影响，扫描版 PDF 可能效果较差。
- 该样例未加入认证、权限、日志审计，因此测试范围也未覆盖这些企业级要求。
- 模型调用会产生实际 API 成本，反复执行问答测试时要注意调用次数。
