# 基本设计

## 1. 系统概要

`chat_cli` 是最小命令行聊天示例。

系统由本地 Python 程序和模型调用两部分组成：

- 本地程序负责读取参数、初始化客户端、输出结果
- 模型负责生成文本回答

## 2. 模块构成

| 模块 | 作用 |
|---|---|
| 参数解析 | 接收 `prompt` 和 `model` |
| Client 初始化 | 读取 `OPENAI_API_KEY` 并创建客户端 |
| 单次提问处理 | 调用模型并返回文本回答 |
| 交互模式 | 持续读取输入并逐次请求模型 |

## 3. 输入输出设计

### 3.1 输入

- `prompt`
  - 可选，一次性提问内容
- `--model`
  - 模型名

### 3.2 输出

- 成功时输出文本回答
- 失败时输出错误信息

## 4. 处理流程

### 4.1 一次性提问

1. 解析命令行参数
2. 初始化 OpenAI Client
3. 调用模型
4. 输出回答

### 4.2 交互模式

1. 启动程序
2. 用户输入问题
3. 调用模型
4. 输出回答
5. 继续下一轮，直到退出

## 5. 错误处理

- 未配置 `OPENAI_API_KEY` 时退出
- 请求失败时输出错误信息
- 交互模式下单次失败不直接退出整个程序

## 6. 设计取舍

- 不保留历史上下文
- 不做工具调用
- 不做结构化返回

这样可以让这个案例专注在“最小模型调用闭环”。

## 7. 成本与注意点

- 当前版本是学习版。
- 每次调用都会产生 API 成本。
- 如果后续要接业务系统，需要继续扩展为结构化输出、RAG 或 Web API。
