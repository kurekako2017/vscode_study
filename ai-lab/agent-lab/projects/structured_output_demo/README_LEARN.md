Structured Output Demo — 学习说明与快速上手

目的
- 演示如何用模型生成受结构化约束的输出，并用 Pydantic 做解析与验证。

业务场景说明
- 谁会用：希望模型输出能够直接写入数据库、表格、工单或 API 的开发人员。
- 现实中的问题：同一个问题，模型有时返回列表，有时返回文章。人能看懂，但程序不知道应该从哪里取标题、步骤和风险。
- 这个例子怎么解决：先定义 `AgentPlan`，再要求模型按这个固定结构返回结果，最后使用 Pydantic 检查字段。
- 现实例子：用户输入“制定一个客服 Agent 开发计划”，程序得到固定格式的项目名称、摘要、步骤和风险，前端可以直接按字段显示。
- 初学者重点：先观察 `AgentPlan` 定义了什么，再看模型结果如何变成 `AgentPlan` 对象。

快速运行
1. Mock 模式：
   ```bash
   RAG_API_MOCK=1 python3 main.py --prompt "请生成计划"
   ```

关键点
- `AgentPlan`：Pydantic 模型，定义输出结构。
- `client.responses.parse()`：演示如何解析模型输出为结构化对象（real 模式）。

学习建议
- 修改 Pydantic 模型以匹配不同任务（如任务列表、时间线），并验证解析失败的处理逻辑。
