# 示例文档：RAG 演示用

本文件为 rag_api_demo 的示例文档，用于本地 smoke-test 与教学演示。

## 业务场景说明

- 适用场景：需要一个最小的教学样本来验证文档检索、来源输出和接口联动是否正常。
- 如果不用这种方式：只看代码很难感受到请求如何在客户端、后端和文档之间流转。
- 解决的问题：用一份可读的示例文档，帮助演示 RAG 的完整链路并作为测试输入。
- 举例说明：例如让用户问“出差报销要哪些材料”，看前端、后端和 RAG 返回是否联动。

## 项目简介

这个示例文档用于展示如何从本地文本中抽取信息并进行检索式问答（RAG）。

主要点：

- 本仓库包含多个教学用的 Agent demo（chat_cli、doc_qa_agent、rag_api_demo 等）。
- rag_api_demo 会把文档切分为多个 chunk，并基于关键词进行简单检索以构建上下文。

## 处理流程

1. 前端页面或命令行客户端提交问题。
2. 后端先检查文档是否已经加载。
3. 后端在 chunk 中检索相关内容。
4. 后端把检索结果整理成上下文，再返回答案和来源。

## 整理流程

1. 把新的教学文档放进当前目录或指定的 docs 目录。
2. 需要更新时先调用 `/reload`。
3. 再用 `/health` 查看 chunk 数量是否变化。
4. 最后用 `/ask` 验证前端和后端是否都拿到了最新内容。

## 使用建议

1. 以 mock 模式运行服务以验证流程。
2. 将检索替换为 embeddings + 向量检索作为进阶任务。
