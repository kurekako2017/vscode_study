# rag_api_demo 测试观点

## 1. 目的

本文件用于整理 `rag_api_demo` 的最小测试观点，确保该 PoC 至少能覆盖：

- 启动
- 接口
- 文档加载
- 检索
- PDF 支持
- 错误处理

## 2. 测试范围

- `GET /health`
- `POST /ask`
- `POST /reload`
- `md / txt / pdf` 文档读取
- 关键词检索
- 来源返回

## 3. 主要测试观点

### 3.1 启动相关

- `OPENAI_API_KEY` 已配置时可正常启动
- `OPENAI_API_KEY` 未配置时启动失败
- `RAG_API_DOCS_DIR` 指向有效目录时可正常启动
- `RAG_API_DOCS_DIR` 无效时启动失败
- 文档目录为空或无可读文件时启动失败

### 3.2 健康检查

- `GET /health` 返回 `status`
- `GET /health` 返回当前 `docs_dir`
- `GET /health` 返回 `chunk_count`

### 3.3 问答接口

- `POST /ask` 正常接收问题
- 指定模型名时正常传递
- 返回 `answer`
- 返回 `source_count`
- 返回 `sources`

### 3.4 文档读取

- 能读取 `md`
- 能读取 `txt`
- 能读取 `pdf`
- 对无法读取的文件能跳过
- 对非支持类型文件不处理

### 3.5 检索相关

- 问题与文档有关时可命中片段
- 命中多个片段时按分数返回
- 无命中时可返回“依据不足”类回答
- 返回的来源标签与真实文件对应

### 3.6 重载相关

- `POST /reload` 可重新加载文档
- 重载后 `chunk_count` 可更新
- 新增文档后重载可生效

### 3.7 错误处理

- 模型调用失败时返回 `500`
- 服务未初始化时返回错误
- 文档目录异常时返回错误

## 4. 优先级建议

### 高优先级

- 服务能启动
- `/health` 正常
- `/ask` 正常
- PDF 可读取
- 来源可返回

### 中优先级

- `/reload` 正常
- 无命中时返回合理说明
- 无效文件可跳过

### 低优先级

- 极长文档处理
- 特殊字符与编码场景

## 5. 后续可扩展测试

- 检索精度测试
- 多文档对比测试
- 大目录性能测试
- PDF 抽取质量测试
- API 并发测试
