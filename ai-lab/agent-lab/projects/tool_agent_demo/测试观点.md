# 测试观点

## 1. 测试目的

本测试观点用于确认 `tool_agent_demo` 作为最小 Tool Calling PoC，是否已经实现以下目标：

- 模型可以根据任务决定是否调用工具
- 本地程序可以正确执行工具
- 工具结果可以正确回填给模型
- 最终回答基于工具结果生成
- 工具访问范围被限制在 `--workdir` 内

## 2. 观点来源

本测试观点主要依据以下资料整理：

- [需求概要.md](需求概要.md)
- [基本设计.md](基本设计.md)
- [main.py](main.py)

## 3. 共通测试观点

### 3.1 启动与参数

- 未设置 `OPENAI_API_KEY` 时是否报错
- `--workdir` 不存在时是否报错
- `--workdir` 不是目录时是否报错
- 默认参数是否可正常使用

### 3.2 工具定义

- `list_files` schema 是否能被模型识别
- `read_file` schema 是否能被模型识别
- `search_text` schema 是否能被模型识别
- 工具参数缺失时是否按预期失败

### 3.3 工具执行

- 列目录时能否返回文件和目录
- 读文件时能否返回文本内容
- 搜索文本时能否返回文件、行号和命中文本
- 不存在的工具名是否返回错误

### 3.4 Tool Calling 闭环

- 模型发起函数调用后，本地程序是否能执行对应工具
- 工具结果是否能以 `function_call_output` 回填
- 多轮工具调用后是否能得到最终回答
- 超过最大轮次时是否停止

### 3.5 回答质量

- 最终回答是否基于工具返回结果
- 不应虚构未读取过的文件内容
- 当搜索无结果时，是否能给出合理说明

### 3.6 安全边界

- 相对路径越界时是否被拦截
- 是否能阻止访问 `--workdir` 外文件
- 是否不支持写文件
- 是否不支持执行命令

### 3.7 文件处理

- UTF-8 文本文件是否能正常读取
- 非 UTF-8 文件是否能返回错误
- 二进制文件或不可读文本是否不会导致程序崩溃

### 3.8 异常处理

- OpenAI 请求异常时是否报错退出
- JSON 参数解析异常时是否返回错误结果
- 文件不存在时是否返回错误
- 目录不存在时是否返回错误

## 4. 功能别测试观点

### 4.1 `list_files`

- 指定 `.` 时能否列出当前工作目录内容
- 指定子目录时能否列出子目录内容
- 指定文件路径时是否返回“不是目录”
- 指定不存在路径时是否返回错误

### 4.2 `read_file`

- 指定正常文本文件时能否读取内容
- 指定目录时是否返回“不是文件”
- 指定不存在文件时是否返回错误
- 指定非 UTF-8 文件时是否返回错误

### 4.3 `search_text`

- 指定关键字后是否返回正确命中结果
- 搜索目录时是否递归检查子文件
- 搜索文件时是否只检查该文件
- 无命中时是否返回空结果而不是异常
- 命中数量很多时是否按上限截断

## 5. 现场视角下特别需要关注的点

这个案例最重要的不是“是否很聪明”，而是下面这些基础能力是否成立：

- 工具调用链路是否闭环
- 文件访问范围是否可控
- 工具结果是否可追溯
- 回答是否建立在工具证据之上

如果这些点不成立，这个案例就不适合作为后续 Agent 学习基础。

## 6. 不在当前测试范围内的内容

当前不重点覆盖以下内容：

- 多 Agent 协作
- 数据库工具
- 外部 API 工具
- 写文件和命令执行
- 长期记忆
- 企业级权限与审计

## 7. 成本与注意点

- 这是学习版 PoC 的测试观点，不是正式企业测试规格书。
- 只要实际调用模型，就会产生 API 成本。
- 模型是否主动选择调用工具，受提示词和模型行为影响，因此测试时要允许一定波动。
- 当前案例重点是验证最小 Tool Calling 闭环，而不是验证复杂业务自动化效果。
