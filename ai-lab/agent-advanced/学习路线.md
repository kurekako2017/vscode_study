# Agent Advanced 学习路线

这条线默认建立在 `llm-lab` 和 `agent-lab` 已经走过一遍的基础上。

## 推荐顺序

1. [核心技术栈总览](./核心技术栈总览.md)
2. [projects/README.md](./projects/README.md)：先跑 5 个最小 demo，建立直觉
3. `frameworks/`：再看 LangChain、LlamaIndex、LangGraph 的定位差异
4. `rag/`：再看高级 RAG 的完整管线
5. `multi-agent/`：理解多 Agent、角色拆分和协作方式
6. `frontend/`：把 Agent / RAG 能力接到 React 客户端
7. `eval/`：补上评估、Tracing、日志、指标
8. `deployment/`：做 Docker、环境变量、CI/CD 和上线

## 章节安排建议

### 1. 框架层

- LangChain 解决什么问题
- 先看 [frameworks/langchain/LangChain 学习笔记](./frameworks/langchain/LangChain学习笔记.md)
- LlamaIndex 解决什么问题
- LangGraph 适合什么场景
- 什么时候自己写，不用框架

### 2. RAG 层

- 文档切分
- 向量化与索引
- 检索策略
- rerank
- query rewrite
- multi-query
- citations / source 回传

### 3. Agent 层

- 工具注册
- 工具调用循环
- 计划与执行分离
- 多 Agent 协作
- 失败恢复和终止条件

### 4. 前后端层

- React 聊天界面
- 流式输出
- 对话历史
- 结果引用展示
- 重新检索 / 重新生成

### 5. 工程化层

- FastAPI
- Pydantic
- 日志
- Trace
- 单测
- 集成测试
- Docker
- CI/CD

## 学习原则

- 每个主题先讲“为什么要用”
- 再讲“怎么落地”
- 最后补“最小 demo”
- 每个 demo 都要有 README、运行方式和依赖说明
- 文档和代码分开写，但入口要能互相跳转
