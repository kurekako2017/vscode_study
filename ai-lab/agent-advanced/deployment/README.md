# deployment

这个目录是 `agent-advanced` 的部署专题区。

这里主要放两类内容：

- 部署说明
- 最小可运行的容器化示例

## 业务场景说明

- 适用场景：当 demo 已经跑通后，需要把它变成可部署、可交付、可复现的服务。
- 如果不用这种方式：代码只能停留在本地开发环境，别人很难复现，也很难接到 CI/CD 或线上环境。
- 解决的问题：集中说明 Docker、环境变量、发布流程和平台判断，帮助把实验结果落到真实部署里。

## 目录角色

- `deployment/` 本身：讲 Docker、环境变量、发布流程、CI/CD
- `deployment/container_demo/`：一个最小的可容器化服务示例

## 你应该怎么用

先看部署说明，再跑 demo，理解：

- 一个最小服务怎么容器化
- 环境变量怎么传进容器
- 暴露端口和健康检查怎么写

如果你想看“开发 -> 测试 -> E2E -> 上线前抽样 -> 部署验收”的完整顺序，可以回到上级目录看 [开发测试部署流程](../开发测试部署流程.md)。

如果你要按日本项目视角来判断平台，优先看这些：

- AWS
- Azure
- GCP
- Oracle Cloud / OCI
- 企业内网服务器、Kubernetes、虚拟机
