# eval

这个目录是 `agent-advanced` 的评估专题区。

这里主要放两类内容：

- 评估方法说明
- 可运行的本地评估 demo

## 业务场景说明

- 适用场景：当你不只想看 demo 能不能跑，而是想判断 RAG / Agent 的质量到底好不好。
- 如果不用这种方式：功能看起来能用，但命中率、引用质量和稳定性的问题很容易被忽略。
- 解决的问题：把“结果评估”单独拎出来，方便做指标、回归和上线前抽样检查。
- 举例说明：例如拿同一批文档分别测试不同切分长度或检索策略，比较哪种配置更容易命中正确答案。

## 目录角色

- `eval/` 本身：讲如何衡量 RAG / Agent 的结果好不好
- `eval/rag_eval_demo/`：一个最小的本地评估脚本

## 你应该怎么用

先看评估文档，再跑 demo，理解：

- 怎么定义“命中”
- 怎么定义“引用正确”
- 怎么定义“答案可接受”
- 为什么“能跑”不代表“质量够”

如果你想把测试、上线前抽样和部署验收串起来，可以再看一眼上级目录里的 [开发测试部署流程](../开发测试部署流程.md)。
