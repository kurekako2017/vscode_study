# eval

这个目录是 `agent-advanced` 的评估专题区。

这里主要放两类内容：

- 评估方法说明
- 可运行的本地评估 demo

## 目录角色

- `eval/` 本身：讲如何衡量 RAG / Agent 的结果好不好
- `eval/rag_eval_demo/`：一个最小的本地评估脚本

## 你应该怎么用

先看评估文档，再跑 demo，理解：

- 怎么定义“命中”
- 怎么定义“引用正确”
- 怎么定义“答案可接受”
- 为什么“能跑”不代表“质量够”

如果你想把测试、上线前抽样和部署验收串起来，可以再看一眼上级目录里的 [开发测试部署流程](../开发测试部署流程.md)。
