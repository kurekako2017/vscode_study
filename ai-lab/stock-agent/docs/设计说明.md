# Stock Agent 第一版设计说明

## 1. 设计目标

第一版的目标不是预测股价，而是用最小、可运行的 Python 项目学习“输入数据、分析数据、生成报告”的完整流程。

```text
股票代码
  ↓
本地 Mock 数据获取
  ↓
规则化分析
  ↓
Markdown 报告生成
```

## 2. 模块划分

### `fetcher.py`

保存三只股票的固定 Mock 数据，并负责校验输入。它不包含 HTTP 请求，也不读取环境变量。

### `analyzer.py`

根据行业和模拟指标生成中性文字。所谓“AI 总结”只是为了演示未来模型输出的位置，第一版完全由确定性规则生成。

### `report_writer.py`

把股票信息和分析结果组装成 Markdown，并保存到 `reports/<股票代码>_report.md`。

### `main.py`

负责连接以上三个模块，并向用户显示成功或错误信息。

## 3. 数据流

| 阶段 | 输入 | 输出 |
| --- | --- | --- |
| 命令行 | `7203` | 股票代码字符串 |
| 数据获取 | 股票代码 | Mock 股票字典 |
| 分析 | Mock 股票字典 | 总结与风险列表 |
| 报告生成 | 股票字典和分析字典 | Markdown 文件 |

## 4. 安全设计

- 项目没有 `.env` 加载代码。
- 项目没有网络请求库。
- 项目没有证券账户、订单或交易模块。
- 报告明确标注 Mock 数据和非投资建议。
- 输入必须是支持的四位数字股票代码，不能被当作输出路径使用。

## 5. 当前边界

第一版只验证项目骨架与数据流。未来即使增加公开信息接口，也应把数据获取、模型分析和报告生成保持为独立模块，并默认保留 Mock 模式用于离线学习和测试。
