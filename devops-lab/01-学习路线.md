# 学习路线

## 1. 总目标

如果目标是补 `DevOps / SRE` 这条线，并且尽量复用当前工作区已有内容，推荐先建立的是：

- `Docker` 基础
- `CI/CD`
- 自动化运维脚本意识
- 日志与排障能力
- `Kubernetes` 入门认知
- `Prometheus / Grafana` 最小监控栈认知
- `docker-compose` 最小监控模板实操

这条线的重点不是一开始学很全，而是先把最常见、最能马上练起来的部分跑通。

## 2. 先说结论

如果只记最重要的路线，可以记成：

- `Docker -> CI/CD -> 自动化运维 -> Kubernetes -> 监控与可观测性 -> Prometheus / Grafana -> docker-compose 最小监控栈`

更现实一点说：

- 先学容器怎么跑
- 再看代码怎么自动构建和部署
- 再看脚本怎么做状态检查和排障
- 最后再补 `Kubernetes`
- 再把监控与可观测性意识接上
- 再进入最小监控栈
- 再把最小模板真正跑起来

## 3. 当前工作区的材料覆盖情况

### 覆盖较好

- `Docker`
- `CI/CD`
- 自动化运维
- 监控与可观测性入口

### 有但不成体系

- 日志与监控
- 部署与安全
- 平台化监控

### 明显缺口

- `Kubernetes`

所以这条线当前最适合的策略不是“全面铺开”，而是：

- 先把已有材料练熟
- 再把缺口单独补上
- 最后把日志、状态检查和监控意识整合成一条线

## 4. 推荐阶段

### 阶段 1：Docker 与容器

要学：

- `Docker Desktop`
- 容器基本命令
- 镜像和容器区别
- 日志查看
- 常见启动失败排查

输出物：

- 能独立检查 Docker 是否正常
- 能看 LocalStack 容器日志

### 阶段 2：CI/CD

要学：

- `GitHub Actions`
- 工作流文件结构
- `CI` 和 `CD` 的区别
- 构建、测试、部署的基本流程

输出物：

- 能读懂一个 `GitHub Actions` 工作流
- 能解释 `jtproject-ci.yml` 或部署工作流在做什么

### 阶段 3：自动化运维

要学：

- `PowerShell` 脚本化操作
- 状态检查
- 日志查看
- 启动 / 停止 / 重试
- 简单故障诊断

输出物：

- 能运行现有状态脚本
- 能自己改一版检查脚本

### 阶段 4：Kubernetes 入门

要学：

- 为什么会有 `Kubernetes`
- `Pod`、`Deployment`、`Service` 的基本概念
- 它和 `Docker` 的关系

输出物：

- 先建立概念，不要求当前工作区内有完整实战

### 阶段 5：监控与可观测性入口

要学：

- 健康检查 `health check`
- 日志查看
- 状态检查
- 基本监控指标意识
- `CI/CD` 日志排障

输出物：

- 能把“状态异常”和“日志异常”对应起来
- 能写出自己的第一版监控清单

### 阶段 6：Prometheus / Grafana 最小入门

要学：

- `Prometheus` 抓取指标的思路
- `Grafana` dashboard 的基本作用
- 最小 `prometheus.yml`
- metrics endpoint 是什么

输出物：

- 能解释最小监控栈
- 能读懂一个最小抓取配置
- 能写出自己想监控的对象列表

### 阶段 7：docker-compose 最小监控栈实操

要学：

- `docker-compose.yml` 怎么组织 3 个服务
- `Prometheus` 怎么抓本地 demo-app
- `Grafana` 怎么连 `Prometheus`
- 最小排障命令怎么用

输出物：

- 能本地启动一套最小监控栈
- 能访问 `/metrics`
- 能确认 target 是 `UP`

## 6. 最重要的原则

- 先把容器和脚本用起来，再学更抽象的编排。
- 先看真实工作流，再看纯概念。
- 先学会排障和日志，再谈平台化。
- `Kubernetes` 要补，但不必抢在 `Docker` 前面。
- 监控与可观测性要补，但先从日志、状态和最小指标开始。
- 监控平台也要补，但先从最小栈入手，不要一上来堆复杂组件。
- 真正学会的关键还是要把模板跑起来，不要停在概念层。
