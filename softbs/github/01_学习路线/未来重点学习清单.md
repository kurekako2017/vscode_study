# 未来重点学习清单

> 排序原则：优先解决 AI-Learn 近期问题，优先能形成可验证项目产物的工具，不按 Star 数或热度排序。

## P0（立即学习）

### OpenHands

**为什么优先**：

- 直接对应 AI-Learn 的自动开发、Bug 修复和测试验证需求。
- 可以用小型任务快速验证实际价值，失败成本可控。
- 能提前建立 Agent 任务合同、Docker 沙箱、审批和回滚标准。

**本阶段产物**：用 OpenHands 完成一个 AI-Learn 可回滚小功能，并由宿主测试与 Codex 独立审查。

## P1（近期学习）

### Codebase Memory MCP

**为什么优先**：

- 为 AI-Learn 提供代码架构、符号和调用链上下文。
- 可与 Codex/OpenHands 直接组合，改善复杂仓库理解和变更影响分析。
- 本身可以只读方式集成，安全边界较易控制。

**本阶段产物**：生成 AI-Learn 代码地图，并手工抽查至少三条图关系。

### RAGFlow

**为什么优先**：

- AI-Learn 已经开始积累工具教程、决策和复盘，需要可追溯的知识检索。
- 可建立检索与答案评估基线，避免知识库变成“只能演示的聊天界面”。

**本阶段产物**：建立 GitHub AI 工具知识库，记录 Recall@5、引用正确率和答案忠实度。

## P2（后续学习）

### browser-use

**为什么后续**：

- 网页自动化对 AI-Learn 有价值，但当前不是代码生产和知识管理的最短板。
- 复杂网页、登录、CAPTCHA 和界面变动会带来额外维护成本。
- 先完成只读公开站点和测试环境的最小实战即可。

### OpenClaw

**为什么后续**：

- 基础安装和学习已进行，当前缺少的是实战与 AI-Learn 边界验证。
- 如过早将它定为多 Agent 主平台，会在其他工具合同未稳定时增加编排复杂度。
- 应先用只读 workspace 和最小工具集完成一个安全实战。

## P3（暂缓）

### Triton Inference Server

**为什么暂缓**：AI-Learn 当前优先验证 Agent 与知识能力，尚未出现需要专门模型服务化、动态批处理和 GPU 吞吐优化的明确瓶颈。

### NVIDIA NeMo

**为什么暂缓**：模型训练、微调和大型 GPU 工程的学习成本高，对当前 AI-Learn 的近期产物贡献低于 RAG 与 Agent 工程。

### Exo

**为什么暂缓**：跨设备分布式本地推理需要多台合适硬件和网络条件，而当前已有 OpenRouter、ChatNVIDIA 和 Ollama 满足学习需要。

### AutoGPT

**为什么暂缓**：它对理解自治 Agent 历史有价值，但当前已选 OpenHands 和 OpenClaw 作为更接近 AI-Learn 目标的实战对象，同时学习会造成框架分心。

## 优先级调整规则

- P0 完成实战和复盘后，才将 P1 提升为 P0。
- 只有出现可量化瓶颈或 AI-Learn 明确需求，才将 P3 工具提前。
- 工具被放弃时不删除记录，同步更新[工具状态看板](../05_学习进度/工具状态看板.md)和[AI 工具决策记录](../06_决策记录/AI工具决策记录.md)。
