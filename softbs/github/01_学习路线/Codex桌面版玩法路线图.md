# Codex 桌面版玩法路线图

这是一条七天入门路线。每天完成一个主要输出；若当天验收未通过，就停在当天继续，不急着进入下一步。

> 每天开始前：读取 `工作区总索引.md`、`AGENTS.md`、`不上传GitHub文件清单.md`、`密钥与敏感信息管理规范.md`。不要读取密钥，不要自动 commit，不要 push。

## 第 1 天：工作区分析

### 目标

理解目录用途、敏感范围、GitHub 同步边界和近期主线。

### 操作

1. 在 `vscode_study` 根目录开启新线程。
2. 让 Codex 读取四份治理文件。
3. 只查看两层目录、README 和普通索引。
4. 生成或复核工作区分析报告。

### 给 Codex 的简短指令

```text
先读取四份工作区治理文件，再对目录做元数据级分析。不要读取 softbs 和 sap-lab 中的敏感内容，不要读取任何密钥。复核 softbs/github/04_AI-Learn集成/工作区AI分析报告.md，列出判断依据。不要 commit，不要 push。
```

### 输出文件

- `softbs/github/04_AI-Learn集成/工作区AI分析报告.md`

### 安全检查

- [ ] 没有打开 `.env`、凭据、税务、投资或客户文件。

## 第 2 天：工具筛选

### 目标

从大量 GitHub 工具中只选三个当前最值得实践的方向。

### 操作

1. 阅读 GitHub 工具学习地图。
2. 评估十个优先工具。
3. 按难度、实用性、设备成本和项目匹配度排序。
4. 选出三个“现在做”，其余放入候选队列。

```text
读取 GitHub 工具学习地图，复核十个优先工具的难度、实用性和当前适配度。最多保留三个“现在做”，本轮不要安装。更新需先获得 S 级目录授权。不要 commit，不要 push。
```

## 第 3 天：browser-use

### 目标

设计一个只访问公开页面、支持模拟数据的日本股票信息抽取实验。

### 操作

1. 选择一个无需登录的公开数据源或本地测试页。
2. 定义 `7203`、`8306`、`9432` 的字段映射。
3. 先生成计划并人工确认。
4. 小范围运行或用 mock 数据生成报告。

### 给 Codex 的简短指令

```text
为 7203、8306、9432 设计公开信息抽取实验。先给出数据源、字段、频率、失败降级和验证计划，不登录、不绕过限制。没有可靠数据时使用明确标注的 mock 数据。不要自动运行大规模抓取，不要 commit，不要 push。
```

### 输出文件

- 首选：数据源与字段映射计划。
- 确认后：`ai-lab/stock-agent/reports/sample_japan_stock_report.md`。

- [ ] 包含公司名、价格、PER、PBR、股息率、新闻、总结和风险点。
- [ ] 每项事实带来源和时间。
- [ ] 缺失数据没有被猜测。
- [ ] 报告标明“不构成投资建议”。

## 第 5 天：AI 副业机会雷达

### 目标

从现有技能中选一个两周内可演示、风险可控的副业最小产品。

### 操作

1. 只查看授权的 GitHub 学习文档和非敏感项目 README。
2. 对七个关键词生成候选方向。
3. 评估技能匹配、验证成本、潜在价值和合规风险。
4. 选择一个方向，明确停止条件。

### 给 Codex 的简短指令

```text
结合 GitHub 工具学习库、web-projects、ai-lab/ai-learn 和 devops-lab 的非敏感内容，生成副业机会雷达。每个方向包含最小产品、目标客户、收费方式、验证方法、风险和停止条件。不要读取客户、税务、投资或私人资料，不要承诺收入。
```

## 第 6 天：Codex vs OpenHands

### 目标

2. 定义正确性、设置成本、耗时、人工干预和可审查性指标。
3. 先做文档比较，未知项标记“待实测”。
4. 给出当前选择和重新评估条件。

### 给 Codex 的简短指令

```text
基于三个相同小任务设计 Codex 桌面版与 OpenHands 的实践对比。区分已验证事实和待实测项，不使用不明版本的性能或价格结论。本轮不安装 OpenHands。
```

## 第 7 天：Codebase Memory MCP

### 目标

用 Markdown 模拟项目知识图谱，并判断 MCP 是否真正解决问题。

1. 从 `ai-lab/ai-learn` 的一个小模块开始。
2. 建立模块、入口、依赖、概念和运行命令表。
3. 定义三到五个知识图谱问题。
4. 比较普通搜索和模拟索引的回答成本。
5. 决定是否进入 MCP 安装评估。

```text
为 ai-lab/ai-learn 设计 Codebase Memory MCP 实验计划。第一阶段只用 Markdown 模拟，不安装 MCP；排除 softbs、sap-lab、.env、凭据、日志、数据导出和虚拟环境。定义索引范围、问题、成功指标和停止条件。
```
