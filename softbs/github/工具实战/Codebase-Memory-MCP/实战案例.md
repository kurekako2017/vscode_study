# Codebase Memory MCP 实战案例：生成可核对的架构报告

## 案例目标

对一个本地测试仓库生成 `docs/codebase-map.md`，内容必须有图索引证据，且不修改业务代码。

## 步骤 1：建立安全分支

- **为什么做**：报告也是仓库变更，需要可审查和回滚。
- **做什么**：确认工作树后创建专用分支。
- **执行命令**：

```bash
git status --short
git switch -c docs/codebase-memory-map
mkdir -p docs
```

- **预期结果**：当前分支为 `docs/codebase-memory-map`，没有覆盖原有修改。

## 步骤 2：采集图证据

- **为什么做**：报告应来自实际索引，不是模型的主观概括。
- **做什么**：要求 Codex 依次调用索引、schema、架构和搜索工具。
- **执行命令**：在 Codex 中输入：

```text
使用 Codebase Memory MCP 完成以下任务：
1. 索引当前仓库并确认状态。
2. 调用 get_graph_schema 和 get_architecture。
3. 用 search_graph 查找入口、路由、核心类和测试。
4. 选一条关键调用链，用 trace_call_path 追踪 3 层。
先只返回证据摘要，不写文件。
```

- **预期结果**：摘要包含节点/边数、真实文件路径和可抽查的调用关系。

## 步骤 3：生成报告

- **为什么做**：把可验证证据变成之后可维护的仓库文档。
- **做什么**：让 Codex 仅创建 `docs/codebase-map.md`，每个结论引用文件或符号。
- **执行命令**：在 Codex 中输入：

```text
基于刚才的 MCP 证据创建 docs/codebase-map.md。包含：技术栈、入口、模块责任、一条 Mermaid 调用链、测试位置、高风险变更点。每项写出证据路径。不修改其他文件。
```

- **预期结果**：只新增一份 Markdown，且 Mermaid 图可在 VSCode 预览中渲染。

## 步骤 4：验收和记录

- **为什么做**：Agent 产生的架构文档必须由人核对。
- **做什么**：检查变更范围，随机核对三条结论。
- **执行命令**：

```bash
git diff --stat
git diff -- docs/codebase-map.md
rg -n '<报告中的符号名>' .
```

- **预期结果**：只有 `docs/codebase-map.md` 变更，三条抽查结论均可在源码中确认。
