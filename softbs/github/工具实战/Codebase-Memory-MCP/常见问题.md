# Codebase Memory MCP 常见问题

## 问题 1：Codex 中看不到 MCP

- **为什么做**：先区分“二进制未安装”和“Codex 未注册”。
- **做什么**：检查命令路径和 Codex MCP 配置。
- **执行命令**：

```bash
command -v codebase-memory-mcp
rg -n 'codebase-memory' ~/.codex/config.toml
codex mcp --help
```

- **预期结果**：命令有绝对路径，配置文件有对应 server；重启 Codex 后 `/mcp` 可见。

## 问题 2：MCP 启动超时

- **为什么做**：默认启动时间可能不足，也可能是命令不在 Codex 的 PATH 中。
- **做什么**：在 `~/.codex/config.toml` 使用绝对命令路径，并适度增加超时。
- **执行命令**：

```bash
realpath "$(command -v codebase-memory-mcp)"
code ~/.codex/config.toml
```

在对应配置中设置 `startup_timeout_sec = 30` 和 `tool_timeout_sec = 120`。

- **预期结果**：重启 Codex 后 server 能初始化；如仍失败，直接执行二进制可看到错误日志。

## 问题 3：索引为空或路径错误

- **为什么做**：Windows 路径与 WSL 路径不可直接混用。
- **做什么**：确认仓库存在于 WSL 中，并传递 Linux 绝对路径。
- **执行命令**：

```bash
cd ~/code/<your-test-repo>
pwd
find . -maxdepth 2 -type f | head
```

- **预期结果**：`pwd` 为 `/home/...`，文件列表不为空；用该 `pwd` 结果重新索引。

## 问题 4：图关系与源码不一致

- **为什么做**：静态分析对动态调用、反射和生成代码存在天然限制。
- **做什么**：重新索引，用 `get_code_snippet` 和文本搜索交叉验证；生产结论标注“已验证/待验证”。
- **执行命令**：

```bash
rg -n '<函数或路由名>' .
git status --short
```

- **预期结果**：能判定是索引过期、静态分析限制还是报告误读，不把未验证的边当作事实。
