# Codebase Memory MCP 安装教程

> 官方仓库：https://github.com/DeusData/codebase-memory-mcp  
> 建议运行位置：WSL Ubuntu。Docker Desktop 非必需。

## 步骤 1：确认安装位置

- **为什么做**：Codex CLI 和 MCP 服务器在同一个 WSL 环境运行时，路径最简单。
- **做什么**：在 VSCode 的 WSL 终端确认系统和架构。
- **执行命令**：

```bash
uname -s
uname -m
command -v codex
```

- **预期结果**：分别输出 `Linux`、`x86_64` 或 `aarch64`，以及 Codex 可执行文件路径。

## 步骤 2：安装官方二进制

- **为什么做**：官方预编译文件是初学者最少依赖的路径。
- **做什么**：先打开 Releases 页核对最新版本和 `checksums.txt`，再运行官方安装脚本。
- **执行命令**：

```bash
curl -fsSL https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/scripts/setup.sh | bash
command -v codebase-memory-mcp
codebase-memory-mcp --help
```

- **预期结果**：可执行文件在 `PATH` 中，`--help` 输出帮助信息。

> Windows 原生备选：在 PowerShell 执行 `irm https://raw.githubusercontent.com/DeusData/codebase-memory-mcp/main/scripts/setup-windows.ps1 | iex`。如 SmartScreen 告警，先用 Release 页的 SHA-256 校验值验证文件，不要盲目放行。

## 步骤 3：注册到 Codex

- **为什么做**：安装二进制不会自动让 Codex 知道它的存在。
- **做什么**：使用 Codex MCP CLI 添加 STDIO 服务器。
- **执行命令**：

```bash
codex mcp add codebase-memory -- codebase-memory-mcp
codex mcp --help
```

- **预期结果**：`~/.codex/config.toml` 出现 `mcp_servers.codebase-memory` 配置，且命令无报错。

## 步骤 4：在 Codex 中验证

- **为什么做**：只有完成 MCP 初始化和工具列表加载，才算安装成功。
- **做什么**：重启 Codex，在 TUI 中查看 MCP 状态。
- **执行命令**：

```bash
codex
# 进入 Codex 后输入：/mcp
```

- **预期结果**：`codebase-memory` 显示已连接，可见 `index_repository`、`get_architecture`、`trace_call_path` 等工具。

## 模型说明

Codebase Memory MCP 本身不调用 LLM，不需要 OpenRouter、ChatNVIDIA 或 Ollama Key。模型优先级由使用它的 Codex/Agent 决定。
