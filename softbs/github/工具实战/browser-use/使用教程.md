# browser-use 使用教程

## 模型配置原则

`browser-use` 需要模型可靠地解析页面状态和调用工具。按以下顺序验证：

1. OpenRouter 上支持 tool calling 的当前编码/视觉模型。
2. ChatNVIDIA 上支持 tools 的当前模型。
3. Ollama `qwen2.5-coder:1.5b` 只做简单页面和短任务冒烟测试。

## 步骤 1：创建本地密钥文件

- **为什么做**：将密钥与脚本分离，避免误提交。
- **做什么**：创建 `.env` 并将它加入 `.gitignore`；用 VSCode 手动填入真实 Key。
- **执行命令**：

```bash
cd ~/code/browser-use-lab
touch .env
printf '.env\n' >> .gitignore
code .env
```

`.env` 变量名：

```dotenv
OPENROUTER_API_KEY=
NVIDIA_API_KEY=
LLM_PROVIDER=openrouter
LLM_MODEL=<当前支持-tools-的模型-ID>
```

- **预期结果**：`git status` 不列出 `.env`，脚本可通过环境读取配置。

## 步骤 2：配置 OpenAI 兼容适配器

- **为什么做**：OpenRouter、ChatNVIDIA 和 Ollama 都可通过 OpenAI 兼容接口使用，只需切换 Base URL、Key 和 Model ID。
- **做什么**：在 `agent.py` 中保留官方生成的 Agent 结构，将 LLM 初始化替换为下面的提供商映射。
- **执行命令**：

```bash
code agent.py
uv run python -c "from browser_use.beta import ChatOpenAI; print('ChatOpenAI import: OK')"
```

参考配置（当前 API 以安装版本的官方模板为准）：

```python
import os
from dotenv import load_dotenv
from browser_use.beta import ChatOpenAI

load_dotenv()
provider = os.getenv("LLM_PROVIDER", "openrouter")

providers = {
    "openrouter": ("https://openrouter.ai/api/v1", os.getenv("OPENROUTER_API_KEY")),
    "nvidia": ("https://integrate.api.nvidia.com/v1", os.getenv("NVIDIA_API_KEY")),
    "ollama": ("http://localhost:11434/v1", "ollama"),
}
base_url, api_key = providers[provider]
llm = ChatOpenAI(model=os.environ["LLM_MODEL"], base_url=base_url, api_key=api_key)
```

- **预期结果**：导入检查输出 `OK`，代码中没有真实密钥。

## 步骤 3：运行限制域名的第一个 Agent

- **为什么做**：域名白名单可防止模型意外跳转到不在任务范围内的站点。
- **做什么**：在 Agent 的 `BrowserProfile` 设置 `headless=False` 和 `allowed_domains=["example.com"]`，任务只读取页面标题。
- **执行命令**：

```bash
LLM_PROVIDER=openrouter LLM_MODEL='<模型-ID>' uv run python agent.py
```

- **预期结果**：出现可视浏览器，只打开 `example.com`，最终结果包含页面标题。

## 步骤 4：切换 ChatNVIDIA 与 Ollama

- **为什么做**：确认工具与模型解耦，并识别小模型的能力边界。
- **做什么**：只替换两个环境变量，重复同一个简单任务。
- **执行命令**：

```bash
LLM_PROVIDER=nvidia LLM_MODEL='<build.nvidia.com-中的模型-ID>' uv run python agent.py
ollama pull qwen2.5-coder:1.5b
LLM_PROVIDER=ollama LLM_MODEL='qwen2.5-coder:1.5b' uv run python agent.py
```

- **预期结果**：ChatNVIDIA 能完成同一任务；Ollama 至少能连通，如工具调用失败，记录为模型能力限制而非安装失败。
