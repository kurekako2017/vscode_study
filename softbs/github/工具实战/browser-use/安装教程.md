# browser-use 安装教程

> 官方仓库：https://github.com/browser-use/browser-use  
> 建议运行位置：WSL Ubuntu + WSLg。Docker Desktop 可选，首次不建议用容器跑可视浏览器。

## 步骤 1：确认 Python 和显示环境

- **为什么做**：当前 browser-use 需要 Python 3.11+，可视浏览器还需要 WSLg 显示能力。
- **做什么**：在 VSCode WSL 终端检查 Python、`uv` 和显示变量。
- **执行命令**：

```bash
python3 --version
uv --version
echo "$WAYLAND_DISPLAY"
```

- **预期结果**：Python 至少为 3.11，`uv` 有版本号，Windows 11 WSLg 中显示变量通常不为空。

## 步骤 2：创建独立项目

- **为什么做**：独立虚拟环境可避免与其他 AI 工具的 Python 依赖冲突。
- **做什么**：创建项目，安装 browser-use 的 native core 扩展。
- **执行命令**：

```bash
mkdir -p ~/code/browser-use-lab
cd ~/code/browser-use-lab
uv init --python 3.12
uv add "browser-use[core]" python-dotenv
```

- **预期结果**：目录中出现 `pyproject.toml`、`uv.lock` 和 `.venv`，安装无报错。

## 步骤 3：安装浏览器运行时

- **为什么做**：Python 包与 Chromium 二进制是两个独立依赖。
- **做什么**：运行 browser-use 安装与诊断命令。
- **执行命令**：

```bash
uv run browser-use install
uv run browser-use doctor
```

- **预期结果**：Chromium 下载完成，`doctor` 的关键检查为通过。

## 步骤 4：做不使用 LLM 的浏览器冒烟测试

- **为什么做**：先排除浏览器安装问题，再排查模型和 Agent 问题。
- **做什么**：打开安全的示例页面，读取状态后关闭。
- **执行命令**：

```bash
uv run browser-use open https://example.com
uv run browser-use state
uv run browser-use close
```

- **预期结果**：能读到 Example Domain 页面状态，关闭后无残留会话。

## 步骤 5：用 VSCode 和 Codex 打开实验项目

- **为什么做**：Codex 可辅助编辑 Agent 脚本、运行测试和解释日志。
- **做什么**：在 WSL VSCode 中打开项目，生成官方默认模板。
- **执行命令**：

```bash
uv run browser-use init --template default --output agent.py
code .
codex
```

- **预期结果**：VSCode 中可见 `agent.py`，Codex 从该项目目录启动。
