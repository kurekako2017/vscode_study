# browser-use 常见问题

## 问题 1：找不到 Chromium

- **为什么做**：包安装成功不代表浏览器运行时已安装。
- **做什么**：重新安装浏览器并运行诊断。
- **执行命令**：

```bash
cd ~/code/browser-use-lab
uv run browser-use install
uv run browser-use doctor
```

- **预期结果**：`doctor` 不再报 browser executable 缺失。

## 问题 2：WSL 中看不到浏览器窗口

- **为什么做**：可视窗口依赖 WSLg；这与 Agent 或 API Key 无关。
- **做什么**：检查 WSLg，更新并重启 WSL；无 GUI 时先使用 headless。
- **执行命令**：

```powershell
wsl --update
wsl --shutdown
```

```bash
echo "$WAYLAND_DISPLAY"
```

- **预期结果**：重开 Ubuntu 后显示变量不为空；或将 `headless=True` 后任务可运行。

## 问题 3：Ollama 能回答但不会操作页面

- **为什么做**：`qwen2.5-coder:1.5b` 参数量小，复杂页面规划和稳定工具调用能力有限。
- **做什么**：用同一任务切换 OpenRouter，区分模型与框架问题。
- **执行命令**：

```bash
LLM_PROVIDER=openrouter LLM_MODEL='<支持-tools-的模型-ID>' uv run python agent.py
```

- **预期结果**：如 OpenRouter 成功而 1.5b 失败，将根因记为模型能力，保留 Ollama 作简单备用。

## 问题 4：被 CAPTCHA 或登录拦截

- **为什么做**：CAPTCHA 是站点的反自动化保护，不应用脚本绕过。
- **做什么**：停止任务，使用官方 API、测试账号、人工登录会话或合规的托管浏览器。
- **执行命令**：

```bash
uv run browser-use close
```

- **预期结果**：自动化会话结束，没有绕过安全控制或反复重试导致账号风险。

## 问题 5：容器内访问不到 Ollama

- **为什么做**：容器内的 `localhost` 指向容器自己。
- **做什么**：把 Base URL 改为 Docker Desktop 宿主别名。
- **执行命令**：

```bash
curl http://host.docker.internal:11434/api/tags
```

- **预期结果**：返回 Ollama 模型列表；在 OpenAI 兼容客户端中使用 `http://host.docker.internal:11434/v1`。
