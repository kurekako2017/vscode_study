# OpenHands 实战案例：在沙箱中修复一个 Python Bug

## 案例成果

创建一个小型、故意失败的 Python 项目，让 OpenHands 修复 Bug，最后用 Codex 和宿主测试双重验收。

## 步骤 1：准备安全测试项目

- **为什么做**：用人工构造的小 Bug 可准确判断 Agent 是否理解任务。
- **做什么**：在新目录初始化 Python 项目，用 VSCode 创建实现和测试，使其只有一个失败。
- **执行命令**：

```bash
mkdir -p ~/code/openhands-bug-lab
cd ~/code/openhands-bug-lab
uv init --python 3.12
uv add --dev pytest
mkdir -p src tests
code .
```

- **预期结果**：`uv run pytest -q` 只有一个预期失败，并已将起始状态提交到 Git。

## 步骤 2：创建任务合同

- **为什么做**：将任务从“聊天请求”变成可验收合同。
- **做什么**：创建 `TASK.md`，写明现象、允许范围、禁止项和测试命令。
- **执行命令**：

```bash
code TASK.md
git add .
git commit -m "test: add reproducible bug"
```

- **预期结果**：`git status --short` 为空，`TASK.md` 可让另一个人重现唯一失败。

## 步骤 3：让 OpenHands 实施修复

- **为什么做**：使用挂载当前目录的沙箱，限制 Agent 能看到的文件。
- **做什么**：启动 GUI，要求 Agent 阅读 `TASK.md`、先重现、再最小修复、最后测试。
- **执行命令**：

```bash
cd ~/code/openhands-bug-lab
openhands serve --mount-cwd
```

输入：

```text
按 TASK.md 处理。先运行失败测试确认现象，再做最小修复。不改变公开 API，不添加依赖。最后运行 uv run pytest -q 和 git diff --check，汇报修改文件。
```

- **预期结果**：OpenHands 重现先失败、修复后通过，没有越界修改或新增依赖。

## 步骤 4：独立验收与回滚演练

- **为什么做**：Agent 日志中的“测试通过”不能替代宿主的独立验证。
- **做什么**：在 WSL 宿主重跑测试和 diff 检查，用 Codex 审查；如不接受，根据 diff 手动回退指定文件。
- **执行命令**：

```bash
uv run pytest -q
git diff --check
git diff --stat
codex review --uncommitted
```

- **预期结果**：宿主测试通过，Codex 无高优先级问题，修改范围符合 `TASK.md`。
