# OpenHands 安装教程

> 官方仓库：https://github.com/OpenHands/OpenHands  
> Windows 11 用户建议在 WSL Ubuntu 中运行 CLI，Docker Desktop 提供默认沙箱。

## 步骤 1：验证 WSL 与 Docker

- **为什么做**：OpenHands CLI 在 Windows 上要求 WSL，默认用 Docker 隔离 Agent 执行的代码和命令。
- **做什么**：在 VSCode WSL 终端检查发行版、Docker Server 和可用端口。
- **执行命令**：

```bash
echo "$WSL_DISTRO_NAME"
docker version
ss -ltn | rg ':3000\b' || true
```

- **预期结果**：显示 Ubuntu 发行版，Docker Client/Server 均可访问，3000 端口没有被占用。

## 步骤 2：安装 OpenHands CLI

- **为什么做**：官方推荐用 `uv tool` 隔离 CLI 与项目 Python 依赖。
- **做什么**：用 Python 3.12 安装 CLI，然后检查命令。
- **执行命令**：

```bash
uv tool install openhands --python 3.12
openhands --help
```

- **预期结果**：`openhands` 命令可用，帮助中包含 CLI、`serve`、headless 等模式。

## 步骤 3：做首次 CLI 配置

- **为什么做**：OpenHands 需要模型提供商和 API Key 才能启动 Agent 循环。
- **做什么**：启动 CLI，第一次优先选 OpenRouter；在设置中填入当前支持 Agent/tool calling 的模型。
- **执行命令**：

```bash
openhands
```

- **预期结果**：配置保存到 `~/.openhands/settings.json`，能进入交互界面；不要将该文件提交到 Git。

## 步骤 4：启动本地 GUI

- **为什么做**：GUI 适合初学者观察 Agent 步骤、工具调用和确认点。
- **做什么**：在测试项目目录中启动 GUI，只挂载当前目录。
- **执行命令**：

```bash
mkdir -p ~/code/openhands-lab
cd ~/code/openhands-lab
git init
openhands serve --mount-cwd
```

- **预期结果**：Docker 自动拉取所需镜像，终端提示 `http://localhost:3000`，浏览器可打开 OpenHands。

## 步骤 5：确认沙箱边界

- **为什么做**：Agent 会执行 Shell 和修改文件，必须知道它能看到什么。
- **做什么**：在 OpenHands 中只请求列出 `/workspace`，不进行写入。
- **执行命令**：在 OpenHands 中输入：

```text
只读取并列出 /workspace 顶层文件，告诉我当前 Git 分支。不修改文件，不安装依赖。
```

- **预期结果**：只看到 `~/code/openhands-lab` 的挂载内容，宿主其他目录不可见。
