# OpenHands 常见问题

## 问题 1：Windows PowerShell 中 `openhands` 无法运行

- **为什么做**：OpenHands CLI 在 Windows 上的官方路径是 WSL，不是原生 PowerShell。
- **做什么**：进入 Ubuntu，在 WSL 中安装和运行。
- **执行命令**：

```powershell
wsl -d Ubuntu
```

```bash
uv tool install openhands --python 3.12
openhands --help
```

- **预期结果**：WSL 中 `openhands` 可用，VSCode 在 `WSL: Ubuntu` 模式下能共享该命令。

## 问题 2：OpenHands 找不到 Docker

- **为什么做**：通常是 Docker Desktop 未启动或未对 Ubuntu 启用 WSL Integration。
- **做什么**：打开 Docker Desktop，启用 WSL 2 Engine 和 Ubuntu Integration，然后重开 WSL。
- **执行命令**：

```powershell
wsl --shutdown
```

```bash
docker version
docker run --rm hello-world
```

- **预期结果**：WSL 中同时能看到 Docker Client 和 Server，`hello-world` 成功。

## 问题 3：本地 Ollama 可聊天但 Agent 不使用工具

- **为什么做**：1.5B 模型的指令跟随、规划和工具调用能力可能不足。
- **做什么**：先确认端点可达，再用同一任务切换 OpenRouter 或 ChatNVIDIA。
- **执行命令**：

```bash
curl http://localhost:11434/api/tags
ollama run qwen2.5-coder:1.5b "只回答 OK"
```

- **预期结果**：Ollama 连通正常；如更强模型完成任务，根因归类为模型能力，不重装 OpenHands。

## 问题 4：项目文件在沙箱中不可见

- **为什么做**：`openhands serve` 不会默认挂载当前目录。
- **做什么**：先进入项目目录，再使用 `--mount-cwd`。
- **执行命令**：

```bash
cd ~/code/<your-test-repo>
pwd
openhands serve --mount-cwd
```

- **预期结果**：当前目录在沙箱内映射为 `/workspace`，Agent 可读取但不会看到未挂载的其他目录。

## 问题 5：3000 端口被占用

- **为什么做**：另一个开发服务或旧 OpenHands 容器可能仍在监听。
- **做什么**：先找到占用者，只停止明确属于本次实验的容器。
- **执行命令**：

```bash
ss -ltnp | rg ':3000\b'
docker ps --format 'table {{.ID}}\t{{.Names}}\t{{.Ports}}'
```

- **预期结果**：确认占用者后再处理；不盲目停止其他项目容器。
