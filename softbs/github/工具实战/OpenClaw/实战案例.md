# OpenClaw 实战案例：只读学习助手

## 目标

构建一个只能读取本地学习文档、输出每日学习清单的 Agent。不连接真实聊天渠道，不运行 Shell，不写原文档。

## 步骤 1：创建专用 workspace

- **为什么做**：专用 workspace 是文件权限和内容边界。
- **做什么**：创建只包含学习文档副本的目录，不放密钥、Git 资凭据和个人数据。
- **执行命令**：

```bash
mkdir -p ~/.openclaw/workspace-learning/docs
cp <github目录>/GitHub_AI_工具学习体系.md ~/.openclaw/workspace-learning/docs/
cp <github目录>/学习路线图/第一批学习路线.md ~/.openclaw/workspace-learning/docs/
rg -n 'sk-|nvapi-' ~/.openclaw/workspace-learning || true
```

- **预期结果**：目录只有两份 Markdown，密钥扫描无命中。

## 步骤 2：应用最小权限

- **为什么做**：即使任务文本说“只读”，工具策略也应技术性拒绝写入和执行。
- **做什么**：在 OpenClaw 配置中为学习 Agent 设置 read-only workspace，allow 仅保留读取，deny 包含 write/edit/apply_patch/exec/process/browser。
- **执行命令**：

```bash
code ~/.openclaw
openclaw doctor
openclaw gateway status
```

- **预期结果**：`doctor` 不报开放写权限，Gateway 重载后 Agent 对禁止工具的请求被拒绝。

## 步骤 3：生成当日学习计划

- **为什么做**：用一个完全可人工核对的任务验证读取、推理和权限。
- **做什么**：在 Control UI 要求从两份文档生成 60 分钟学习计划，但只在聊天中返回。
- **执行命令**：

```text
读取 docs/ 中的两份文档，为我制定今天 60 分钟的学习计划。只选一个工具，包含 3 个可验证步骤和通关标准。不写文件，不执行命令，不访问网络。
```

- **预期结果**：计划只引用现有文档中的工具和标准，workspace 的文件时间戳不变。

## 步骤 4：演练拒绝越权

- **为什么做**：安全策略必须用反向用例验证，不能只看配置文件。
- **做什么**：明确要求 Agent 创建文件和执行 Shell，确认系统拒绝。
- **执行命令**：

```text
请把计划写入 docs/today.md，然后执行 ls -la。
```

- **预期结果**：Agent 无法调用写入与 exec 工具，`docs/today.md` 不存在。

## 步骤 5：用 Codex 完成需要写入的部分

- **为什么做**：将“长期助手的建议”与“仓库中的变更”分离，可审查、可回滚。
- **做什么**：人工复制 OpenClaw 计划到测试仓库，再让 Codex 按明确范围创建记录。
- **执行命令**：

```bash
cd <github目录>
git switch -c docs/daily-learning-plan
codex
```

- **预期结果**：所有写入发生在 Git 分支内，可用 `git diff` 审查，OpenClaw workspace 仍为只读。
