# OpenClaw 安装教程

> 官方仓库：https://github.com/openclaw/openclaw  
> Windows 11 上建议使用 WSL2 Gateway。Docker 可选，适合容器化 Gateway，但不是本地最快入门路径。

## 步骤 1：确认 WSL2 与 systemd

- **为什么做**：WSL2 是 OpenClaw 在 Windows 上最兼容 Linux Gateway 的运行方式，systemd 用于管理长期运行服务。
- **做什么**：检查 WSL 版本和 PID 1。如 PID 1 不是 systemd，按官方 WSL 方式启用后重启 WSL。
- **执行命令**：

```powershell
wsl --status
```

```bash
ps -p 1 -o comm=
```

- **预期结果**：WSL 版本为 2，PID 1 输出 `systemd`。

## 步骤 2：运行官方安装器

- **为什么做**：官方安装器会处理 Node 24/22.19+ 和 OpenClaw CLI 的版本要求。
- **做什么**：在 WSL Ubuntu 中安装，首次先跳过 onboarding，便于单独验证命令。
- **执行命令**：

```bash
curl -fsSL https://openclaw.ai/install.sh | bash -s -- --no-onboard
openclaw --version
openclaw doctor
```

- **预期结果**：`openclaw` 输出版本，`doctor` 能执行并给出配置状态。

## 步骤 3：运行引导向导

- **为什么做**：OpenClaw 需要 Gateway、workspace、模型认证和工具策略，官方建议用 onboarding 统一配置。
- **做什么**：首次选择本地 Gateway，模型认证优先选 OpenRouter，暂不连接 WhatsApp/Telegram 等真实消息渠道。
- **执行命令**：

```bash
openclaw onboard --auth-choice openrouter-oauth
```

如已手动创建 OpenRouter Key，可用：

```bash
openclaw onboard --auth-choice openrouter-api-key
```

- **预期结果**：向导生成本地配置，默认模型可显示为 `openrouter/auto` 或你选择的具体模型。

## 步骤 4：验证 Gateway

- **为什么做**：CLI 安装成功不代表 Gateway 和模型认证已就绪。
- **做什么**：检查 Gateway、模型和健康状态。
- **执行命令**：

```bash
openclaw gateway status
openclaw models status
openclaw doctor
```

- **预期结果**：Gateway 正常运行，默认模型已解析，没有 Missing auth 或高优先级安全告警。

## 步骤 5：可选 Docker 安装路径

- **为什么做**：需要可抛弃、可移植 Gateway 时才选 Docker；它不会自动让 Agent 工具进入沙箱。
- **做什么**：仅在测试目录克隆仓库，使用官方 Docker setup；不与已运行的 WSL Gateway 共用端口。
- **执行命令**：

```bash
git clone https://github.com/openclaw/openclaw.git ~/code/openclaw-docker
cd ~/code/openclaw-docker
export OPENCLAW_IMAGE='ghcr.io/openclaw/openclaw:latest'
./scripts/docker/setup.sh
```

- **预期结果**：Compose 启动 Gateway，Control UI 默认可从 `http://127.0.0.1:18789/` 访问。如端口冲突，停止其中一种安装方式再继续。
