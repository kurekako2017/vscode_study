# OpenClaw 常见问题

## 问题 1：`openclaw` 安装后找不到

- **为什么做**：Node 全局可执行目录可能不在 WSL 的 `PATH` 中。
- **做什么**：检查 Node、npm 前缀和 PATH，再按官方建议加入 shell 配置。
- **执行命令**：

```bash
node -v
npm prefix -g
echo "$PATH"
export PATH="$(npm prefix -g)/bin:$PATH"
openclaw --version
```

- **预期结果**：当前终端可找到 OpenClaw；确认后再将 PATH 配置写入 `~/.bashrc`。

## 问题 2：Gateway 未运行或重复启动

- **为什么做**：可能是 systemd 未启用、端口冲突或旧服务残留。
- **做什么**：先用官方状态和 doctor 识别具体原因，不盲目删配置。
- **执行命令**：

```bash
openclaw gateway status
openclaw doctor
ss -ltnp | rg ':18789\b' || true
```

- **预期结果**：报告指向唯一具体问题；修复后 Gateway 只有一个实例监听。

## 问题 3：OpenRouter 显示 Missing auth

- **为什么做**：模型列表是公开元数据，可见模型不代表已保存认证。
- **做什么**：重新运行 OpenRouter OAuth 或 API Key 认证，然后用 probe 实测。
- **执行命令**：

```bash
openclaw models auth login --provider openrouter --method oauth
openclaw models status --probe
```

- **预期结果**：OpenRouter 不再出现于 Missing auth，probe 成功或返回明确的额度/模型错误。

## 问题 4：Ollama 在 OpenClaw 中输出原始工具 JSON

- **为什么做**：OpenClaw 的 Ollama provider 需要原生 `/api/chat`，使用 `/v1` OpenAI 兼容端点可破坏工具调用。
- **做什么**：将 Ollama Base URL 改为不含 `/v1` 的主机地址。
- **执行命令**：

```bash
curl http://127.0.0.1:11434/api/tags
openclaw models status
```

- **预期结果**：OpenClaw 通过原生 Ollama provider 连接；如仍无法完成复杂工具任务，考虑 1.5b 模型能力限制。

## 问题 5：Docker Gateway 连不上宿主 Ollama

- **为什么做**：容器的 `127.0.0.1` 是容器自己。
- **做什么**：使用 `host.docker.internal:11434`，并确保 Ollama 监听容器可达地址。
- **执行命令**：

```bash
docker run --rm curlimages/curl:latest http://host.docker.internal:11434/api/tags
```

- **预期结果**：容器能读到 Ollama 模型列表，OpenClaw Docker 配置中 Base URL 不包含 `/v1`。

## 问题 6：权限设置后 Agent 仍能写入

- **为什么做**：可能修改了错误的 Agent、Gateway 未重载，或 allow/deny 策略没有作用到当前会话。
- **做什么**：查看 doctor 和当前 Agent 配置，用新会话重跑越权测试。
- **执行命令**：

```bash
openclaw doctor
openclaw gateway status
find ~/.openclaw/workspace-learning -maxdepth 2 -type f -printf '%TY-%Tm-%Td %TH:%TM %p\n'
```

- **预期结果**：新会话中 write/edit/exec 被拒绝，workspace 文件时间戳不变。
