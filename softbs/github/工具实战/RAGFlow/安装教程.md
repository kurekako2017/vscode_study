# RAGFlow 安装教程

> 官方仓库：https://github.com/infiniflow/ragflow  
> 建议运行位置：Windows 11 上的 Docker Desktop，在 WSL Ubuntu 中执行 Compose 命令。

## 步骤 1：检查资源

- **为什么做**：RAGFlow 包含文档引擎、MySQL、MinIO 和 Redis，官方基线是 4 核 CPU、16 GB RAM、50 GB 磁盘。
- **做什么**：确认 Docker/Compose 版本和 WSL 可用资源；在 Docker Desktop 中分配至少 16 GB 内存。
- **执行命令**：

```bash
docker version --format '{{.Server.Version}}'
docker compose version
nproc
free -h
df -h ~
```

- **预期结果**：Docker >= 24、Compose >= 2.26.1，CPU、内存和磁盘满足基线。

## 步骤 2：设置 Elasticsearch 所需内核参数

- **为什么做**：默认文档引擎 Elasticsearch 需要较高的虚拟内存映射上限。
- **做什么**：在 WSL Ubuntu 中检查并临时设置 `vm.max_map_count`。
- **执行命令**：

```bash
sysctl vm.max_map_count
sudo sysctl -w vm.max_map_count=262144
```

- **预期结果**：输出 `vm.max_map_count = 262144` 或更大值。WSL 重启后可能需重新设置。

## 步骤 3：克隆并选择稳定版本

- **为什么做**：Compose 文件必须与容器镜像版本匹配。
- **做什么**：克隆官方仓库，查看最新 Release；初学阶段使用最新稳定 tag，不使用 nightly。
- **执行命令**：

```bash
cd ~/code
git clone https://github.com/infiniflow/ragflow.git
cd ragflow
git tag --sort=-version:refname | head
# 核对 Release 页后执行：git checkout <稳定-tag>
```

- **预期结果**：当前 HEAD 与选定稳定 tag 一致，`docker/.env` 中的镜像版本与之匹配。

## 步骤 4：启动容器

- **为什么做**：官方 Compose 会一次启动 RAGFlow 和所有依赖，比源码开发模式更适合首次学习。
- **做什么**：在 `docker/` 目录中拉取镜像并后台启动。
- **执行命令**：

```bash
cd ~/code/ragflow/docker
docker compose -f docker-compose.yml pull
docker compose -f docker-compose.yml up -d
docker compose -f docker-compose.yml ps
```

- **预期结果**：相关容器状态为 `running` 或 `healthy`，没有持续重启。

## 步骤 5：打开界面并检查日志

- **为什么做**：端口可访问和后端依赖就绪是安装验收标准。
- **做什么**：从 Windows 打开 RAGFlow 网页，并从 WSL 查看最近日志。
- **执行命令**：

```bash
curl -I http://localhost
docker compose -f ~/code/ragflow/docker/docker-compose.yml logs --tail=100
explorer.exe http://localhost
```

- **预期结果**：HTTP 有正常响应，Windows 浏览器显示登录/注册页，日志无循环崩溃。

> 停止使用 `docker compose ... down`。不要加 `-v`，除非你明确要删除所有学习数据。
