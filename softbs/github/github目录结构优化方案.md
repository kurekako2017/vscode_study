# GitHub 目录结构优化方案

## 1. 方案目的

本方案用于把 `softbs/github` 从“根目录堆放文档 + 新旧目录并存”整理为稳定的 01–07 分类知识库。

本文件只记录迁移方案。当前阶段：

- 不移动任何文件。
- 不删除任何文件。
- 不覆盖任何文件。
- 不修改任何现有 README 或相对路径。
- 不执行 `git add`、commit、push 或 PR。
- 等待用户确认后，才分批执行迁移。

`softbs/github` 继承 `softbs` 的 S 级保护。即使用户确认本方案，执行时也只允许操作确认清单中的路径；遇到目标文件已存在、来源路径变化或范围扩大时必须停止并重新询问。

### 阶段 B：执行第一批 10 项迁移

1. 按第 4 节逐文件移动。
2. 不使用通配符批量移动。
3. 每移动一个文件立即检查来源、目标和校验值。
4. 完成后统一更新第一批相关链接。

### 阶段 C：第一批验收

1. 检查 01、02、04、05、06 分类 README。
2. 检查所有指向第一批文件的 Markdown 链接。

4. 运行相对链接有效性检查。
5. 展示文件变化，等待用户确认，不 commit，不 push。

- 其它根目录文档分类。
- 历史资料子目录创建。
