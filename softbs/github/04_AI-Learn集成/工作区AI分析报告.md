# VSCode Study 工作区 AI 分析报告

## 1. 报告范围

本报告依据以下治理文档、顶层目录结构和普通项目目录名称生成：

- `工作区总索引.md`
- `AGENTS.md`
- `不上传GitHub文件清单.md`
- `密钥与敏感信息管理规范.md`

本次没有读取或展示 `.env`、API Key、Token、密码、证券账户、税务文件、客户资料、投资明细或私人笔记。对 S 级目录只依据治理规则和已授权的 `softbs/github` 学习文档进行判断。

## 2. 总体结论

当前工作区已经形成五条主要路线：

1. **个人知识与业务资料**：`softbs`，S 级保护。
2. **AI 学习与 Agent 实验**：`ai-lab`、`ai-lab/ai-learn`。
3. **Web 与通用开发**：`web-projects`、`java-projects`、`python-projects`。
4. **基础设施实验**：`devops-lab`、`localstack-lab`、`localstack`。
5. **工作相关学习**：`sap-lab`，S 级保护。

最值得近期深入的主线是 `ai-lab/ai-learn`。它已经包含 LangGraph、RAG、MCP、多 Agent、评估、部署和业务 Agent 等方向，可以作为其他原型的技术底座。副业原型宜从“文档与公开数据整理”开始，不宜从自动交易或大规模自治 Agent 开始。

## 4. 适合 GitHub 同步的内容

适合在用户确认后同步：

- `ai-lab/ai-learn` 中不含密钥、数据导出和个人资料的教程、示例与测试。
- `web-projects` 中的通用 Web 教程和可复现示例。
- `devops-lab`、`localstack-lab` 中不含云账号和凭据的实验。
- 工作区治理文档中不暴露私人文件内容的部分。

不适合自动同步：

- `softbs` 全部内容，包括其子目录。
- `sap-lab`。
- `.env`、Key、Token、证书、凭据文件、日志和导出数据。
- 税务、freee、投资、客户、面试与私人学习资料。

## 5. 适合 AI 修改的范围

优先允许：

- A 级目录中的新增教程、知识库补充和用户明确要求的小范围实现。
- B 级目录中的常规项目新增与修改。
- C 级目录中的可恢复实验。

必须先询问：

- S 级目录中的任何修改。
- A/B 级目录中的删除、重命名、批量改动和结构重构。
- 独立 Git 仓库边界不清楚的操作。
- 可能接触账号、凭据、客户或金融隐私的操作。

## 7. 建议整理的内容

以下是“整理建议”，不是自动修改许可：

- GitHub 工具学习文档较多，应明确一个总索引，并为每个工具保留标准五文件结构。
- `ai-lab` 内包含多个独立学习仓库，建议建立“项目—用途—运行命令—状态”索引。
- `ai-lab/ai-learn` 主题较多，建议标注入门、进阶、业务实践和暂缓四种状态。
- 大型第三方源码、虚拟环境、构建产物和依赖目录不应混入学习进度统计。
- 新原型应优先放在清楚的项目目录，不把代码散落到知识库文档目录。

任何涉及 `softbs` 的合并、移动、删除、重命名或批量整理都必须先询问用户。
