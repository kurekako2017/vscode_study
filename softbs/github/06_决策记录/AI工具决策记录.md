# AI 工具决策记录

> 目的：记录“为什么关注、为什么现在学、学完后如何决策”，避免重复调研和无目的收藏。

## OpenHands

### 为什么关注

- Devin 类软件开发 Agent 的开源替代方案。
- 同时提供 CLI、本地 GUI、SDK 和 Docker 沙箱。
- 可学习任务规划、代码修改、测试验证和人工审批。

### 为什么现在学习

- AI-Learn 未来需要自动开发和 Bug 修复能力。
- 它是验证 Agent 任务合同、沙箱与评估机制的优先实验对象。

### 预期用途

- 自动开发小型功能。
- 自动复现、修复和验证 Bug。
- 生成可由 Codex 二次审查的代码变更。

### 最终结论

继续学习，优先级 P0。完成一个真实但可回滚的 AI-Learn 小功能后再决定是否深度集成。

---

## Codebase Memory MCP

### 为什么关注

- 为代码库建立持久化知识图谱。
- 可通过 MCP 向 Codex 和其他 Agent 提供架构、符号和调用链上下文。

### 为什么现在学习

- AI-Learn 代码增长后，单纯全文搜索很难回答跨文件影响范围。
- 它可作为 OpenHands 和 Codex 之前的代码理解层。

### 预期用途

- 生成 AI-Learn 架构地图。
- 追踪函数调用链和变更影响范围。
- 降低 Agent 重复读取代码的 Token 开销。

### 最终结论

继续学习，优先级 P1。先做只读集成，不将图查询结果未经核对地当作事实。

---

## RAGFlow

### 为什么关注

- 提供文档解析、切块、混合检索、重排、引用和 Agent 能力。
- 适合学习企业级 RAG 的数据、检索和评估链路。

### 为什么现在学习

- AI-Learn 需要将教程、架构记录、问题复盘和项目文档变成可检索知识。
- 应在数据量继续增长前建立评估基线。

### 预期用途

- AI-Learn 开发和学习文档问答。
- 为 Agent 提供带引用的知识上下文。
- 评估 Recall@k、引用正确率和答案忠实度。

### 最终结论

继续学习，优先级 P1。只有基线评估达标后才进入 AI-Learn 集成。

---

## browser-use

### 为什么关注

- 为 Agent 提供真实浏览器观察和操作能力。
- 适合网页调研、信息采集和端到端测试。

### 为什么现在学习

- AI-Learn 后续需要官方文档更新检查和网页测试。
- 当前优先级低于代码 Agent 和知识库，可先保留教程和安全边界。

### 预期用途

- 只读采集 GitHub 项目公开资料。
- 验证 AI-Learn Web 界面的关键用户流程。
- 为 RAG 增量索引提供经审核的公开数据。

### 最终结论

后续学习，优先级 P2。在域名白名单、无真实账号和无写操作条件下实战。

---

## OpenClaw

### 为什么关注

- 可作为本地长期运行的个人 Agent Gateway。
- 提供模型路由、workspace、工具策略、渠道和多 Agent 扩展能力。

### 为什么现在学习

- 已具备基础安装和使用经验，适合进入只读实战。
- 它能帮助验证 AI-Learn 未来的多 Agent 入口和权限模型。

### 预期用途

- 统一连接模型、MCP 和本地工作区。
- 作为 AI-Learn 多 Agent 实验控制平面。
- 验证最小权限、人工审批与审计日志。

### 最终结论

继续实战，优先级 P2。在 OpenHands、Codebase Memory MCP 和 RAGFlow 的接口边界明确后，再评估是否成为 AI-Learn 的控制平面。
