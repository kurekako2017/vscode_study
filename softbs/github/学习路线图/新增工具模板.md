# 新增 GitHub AI 工具模板

## 新增流程

1. 在 `工具实战/<工具名>/` 下创建四篇文档。
2. 在 `工具实战/README.md` 添加入口。
3. 在 `../05_学习进度/工具状态看板.md` 自动追加 `| <工具名> | ❌ | ❌ | ❌ | ❌ | 未开始 |`。
4. 在 `学习进度/总进度.md` 添加四个检查项。
5. 在 `../06_决策记录/AI工具决策记录.md` 追加选型决策，在 `../01_学习路线/未来重点学习清单.md` 设置优先级。
6. 如计划集成 AI-Learn，在 `../04_AI-Learn集成/AI-Learn集成路线图.md` 追加阶段、合同和通关条件。
7. 在路线图中写明依赖、前置工具和通关成果。
8. 记录官方仓库、文档、Release 页和最后验证日期。
9. 学习完成后复制 `../05_学习进度/学习复盘模板.md`，并回填看板和决策结论。

## 四篇文档的固定边界

| 文档 | 只解决的问题 |
| --- | --- |
| `安装教程.md` | 环境要求、安装、启动、健康检查、平台差异 |
| `使用教程.md` | 最小概念、第一次操作、模型切换、Codex/VSCode 协作 |
| `实战案例.md` | 有输入、步骤、产物、验收和回滚的完整任务 |
| `常见问题.md` | 现象、原因、诊断命令、预期恢复结果 |

## 步骤模板

````md
## 步骤 N：动作名称

- **为什么做**：说明这一步解决什么问题。
- **做什么**：说明会改变什么状态或生成什么文件。
- **执行命令**：

```bash
command --example
```

- **预期结果**：给出可观察、可判定成功/失败的结果。
````

## 必须包含的兼容性项

- Windows 11 中从哪个终端执行。
- WSL Ubuntu 中的路径、权限和端口。
- Docker Desktop 是必需、可选还是不适用。
- VSCode 如何打开工作区和查看日志。
- Codex 是直接使用、MCP 接入，还是仅辅助生成配置。
- OpenRouter、ChatNVIDIA、Ollama 的配置方式与能力限制。
