# 公共环境与模型配置

> 所有工具的公共前置教程。官方界面和版本可能变化，执行前先阅读各工具的安装教程。

## 步骤 1：确认 Windows 11 和 WSL2

- **为什么做**：五个工具的 Linux 支持最稳定，WSL2 可在 Windows 11 上提供 Linux 环境。
- **做什么**：以管理员身份打开 PowerShell，安装 Ubuntu。
- **执行命令**：

```powershell
wsl --install -d Ubuntu
wsl --status
```

- **预期结果**：能看到 WSL 默认版本为 2，并能从开始菜单打开 Ubuntu。

## 步骤 2：配置 Docker Desktop

- **为什么做**：OpenHands 和 RAGFlow 依赖容器，Docker 也能隔离 Agent 执行环境。
- **做什么**：安装 Docker Desktop，启用 `Use the WSL 2 based engine` 和 Ubuntu 的 WSL Integration。
- **执行命令**：

```bash
docker version
docker compose version
docker run --rm hello-world
```

- **预期结果**：前两条输出版本，最后一条输出 `Hello from Docker!`。

## 步骤 3：连接 VSCode 与 WSL

- **为什么做**：将代码保存在 WSL 的 Linux 文件系统中，可减少 I/O、权限和换行符问题。
- **做什么**：安装 VSCode 的 WSL 扩展，然后从 Ubuntu 终端打开项目。
- **执行命令**：

```bash
mkdir -p ~/code
cd ~/code
code .
```

- **预期结果**：VSCode 左下角显示 `WSL: Ubuntu`，集成终端中 `pwd` 输出 `/home/...`。

## 步骤 4：验证 Codex

- **为什么做**：Codex 是本体系中的代码协作入口，并用于连接 MCP 服务。
- **做什么**：在 WSL 中确认 Codex CLI 可用；如未安装，使用官方安装脚本。
- **执行命令**：

```bash
codex --version
# 仅在未安装时执行：
curl -fsSL https://chatgpt.com/codex/install.sh | sh
```

- **预期结果**：`codex --version` 输出版本号，在 VSCode 终端中输入 `codex` 可启动会话。

## 步骤 5：准备密钥环境变量

- **为什么做**：避免 API Key 出现在代码、Markdown 或 Git 历史中。
- **做什么**：在当前终端临时设置密钥；长期使用时改用密钥管理器或本地不入库的 `.env`。
- **执行命令**：

```bash
export OPENROUTER_API_KEY="在此粘贴密钥"
export NVIDIA_API_KEY="在此粘贴密钥"
test -n "$OPENROUTER_API_KEY" && echo "OpenRouter key: OK"
test -n "$NVIDIA_API_KEY" && echo "NVIDIA key: OK"
```

- **预期结果**：终端只显示两个 `OK`，不显示密钥原文。

## 步骤 6：安装 Ollama 备用模型

- **为什么做**：在无云端额度或断网时保留最小可用模型。
- **做什么**：在 Windows 安装 Ollama，或在 WSL 安装 Linux 版，然后拉取指定模型。
- **执行命令**：

```bash
ollama pull qwen2.5-coder:1.5b
ollama run qwen2.5-coder:1.5b "只回答 OK"
```

- **预期结果**：模型下载完成，第二条命令返回 `OK` 或类似简短答复。

## 模型端点速查

| 优先级 | 提供商 | Base URL | 密钥变量 | 模型名 |
| --- | --- | --- | --- | --- |
| 1 | OpenRouter | `https://openrouter.ai/api/v1` | `OPENROUTER_API_KEY` | 从当前模型列表选择支持 tools 的编码模型 |
| 2 | ChatNVIDIA | `https://integrate.api.nvidia.com/v1` | `NVIDIA_API_KEY` | 从 build.nvidia.com 复制当前 Model ID |
| 3 | Ollama | `http://localhost:11434/v1` | 占位值 `ollama` | `qwen2.5-coder:1.5b` |

> Docker 容器访问宿主 Ollama 时，把 `localhost` 换成 `host.docker.internal`。OpenClaw 是例外：它的 Ollama provider 使用原生 API，Base URL 不加 `/v1`。
