# Codex 可实践工具筛选报告

## 1. 筛选目标

本报告从现有 GitHub 工具学习地图中，筛选适合 Codex 桌面版体验、AI-Learn、日本股票信息实验和副业原型的工具。判断重点是：能否小步运行、能否验证、是否适合当前工作区、安全成本是否可控。

> 本轮只做筛选，不安装工具，不读取 API Key，不自动提交或推送。

## 3. 适合 Codex 桌面版快速体验

### 第一名：MarkItDown

- 安装与概念相对简单。
- 输入输出清楚，容易验证。
- 可立即用于 RAG 文档预处理。
- 应只使用公开或自制样例，不处理 S 级敏感资料。

### 第二名：browser-use

- 能直观看到浏览器自动化效果。
- 可与 Codex 应用内浏览器、Playwright 类 MCP 的体验进行比较。
- 应从本地页面或无需登录的公开页面开始。

### 第三名：daily_stock_analysis 的结构阅读

- 不必先运行完整系统，也能学习数据源、分析器、报告和定时任务如何分层。
- 可把其中的设计思想迁移到 `ai-lab/stock-agent` 的 mock 版本。

## 4. 适合 AI-Learn 的工具

| 工具 | 对应 AI-Learn 主题 | 建议练习 |
| --- | --- | --- |
| Codebase Memory MCP | MCP、Agent Memory、代码理解 | 给一个小子项目建立模块关系索引 |
| RAGFlow | RAG、文档理解、检索评估 | 对比自建轻量 RAG 与平台方案 |
| MarkItDown | 文档摄取、RAG 预处理 | 建立文档到 Markdown 的清洗流程 |
| browser-use | 工具调用、浏览器 Agent | 做一个公开页面的读取与字段抽取 |
| Paperclip | 多 Agent、治理、人工审批 | 先用 LangGraph 模拟任务状态与审批 |
| n8n-workflows | 工作流、外部服务集成 | 对比低代码流程和 LangGraph 状态图 |

## 5. 适合副业创业的工具

- **MarkItDown + PaddleOCR**：可做资料数字化、知识库预处理和内容迁移原型。
- **n8n-workflows**：可做中小企业重复工作自动化演示，但必须控制外部账号权限。
- **RAGFlow**：适合知识库产品验证；完整部署成本较高，应先确认客户需求。
- **browser-use**：适合公开网页监测和录入辅助；需遵守网站条款，不绕过登录或限制。
- **Paperclip**：适合展示 Agent 项目治理思想，但当前不是最小产品的首选。

## 6. 适合日本股票分析学习的工具

优先顺序：

1. **daily_stock_analysis**：学习报告链路与数据源降级。
2. **browser-use**：学习公开数据采集和来源记录。
3. **MarkItDown**：整理公开财报或说明资料。
4. **PaddleOCR**：处理公开图片或扫描文档中的文字。
5. **AI-Hedge-Fund**：只研究多角色架构，暂不运行真实策略。

所有实验都必须使用公开数据或模拟数据，不登录证券账户，不自动交易，不把模型输出当作投资建议。

## 7. 暂时不适合完整实践的工具

### RAGFlow

原因：完整平台涉及多个服务和较高资源成本。当前先把轻量 RAG 的摄取、检索、引用和评估跑通，才能判断是否需要平台。

### Paperclip

原因：多 Agent 组织治理的价值建立在已有多个稳定 Agent 之上。当前应先完成一个可验证 Agent。

### AI-Hedge-Fund

原因：金融 Agent 容易让学习实验越界成交易决策。当前只研究模块边界、角色协作和风险说明。

### OpenHands 完整部署

原因：应先定义与 Codex 相同的基准任务，再决定是否值得承担部署和维护成本。

## 8. 模型适配建议

| 优先级 | 模型渠道 | 适合任务 | 降级策略 |
| --- | --- | --- | --- |
| 1 | OpenRouter | 复杂分析、长报告、Agent 规划 | 不可用时切换 ChatNVIDIA |
| 2 | ChatNVIDIA | RAG、LangGraph、Agent 实验 | 不可用时切换本地 Ollama |
| 3 | Ollama `Qwen2.5-Coder:1.5B` | 简单解释、字段模板、离线测试 | 任务过难时缩小上下文并减少推理要求 |

原型必须保留 mock 模式。模型渠道全部不可用时，仍应输出模板化结果和明确状态，不伪造分析。

## 9. 当前只选择三个

1. **MarkItDown**：先建立一个安全、可验证的文档转换实验。
2. **browser-use**：做一个公开或本地页面的小型抽取实验。
3. **Codebase Memory MCP 的 Markdown 模拟**：先证明知识图谱问题真实存在，再考虑安装。

其余工具保留在候选队列，不同时安装。

## 10. 验收标准

- 十个优先工具都有判断和下一步。
- “现在做”不超过三个主方向。
- 每项都能说明与当前工作区的关系。
- 金融类工具明确限制在学习和公开信息整理。
- 没有要求读取或记录真实 API Key。
