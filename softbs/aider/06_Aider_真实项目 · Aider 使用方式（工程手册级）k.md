# Aider 真实项目使用手册（工程级）

## 核心原则
Aider 不是“全权开发者”，而是：
> 能写代码，但必须被明确指挥的初级工程师

## 推荐启动方式
```powershell
aider src/**/*.py tests/**/*.py
```

## 推荐交互节奏
1. 描述目标（做什么）
2. 描述约束（不能做什么）
3. 指定文件范围
4. 要求先给方案再改代码

## 示例
```text
我们要新增一个缓存层。
约束：
- 不允许重构现有接口
- 不允许改 tests
- 只修改 src/cache.py
先给出实现思路，不要直接写代码。
```

## 高成功率技巧
- 一次只做一个 commit 级别的改动
- 多用 /ask 而不是直接改
- 经常 /diff 查看
