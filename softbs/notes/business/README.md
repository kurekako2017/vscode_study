# Business 副业项目资料库

本目录用于统一管理副业方向规划、市场与工具研究、未来项目方案及受保护的客户项目入口。

## 目录结构

| 目录 | 用途 | 收录标准 |
| --- | --- | --- |
| [`roadmap/`](roadmap/) | 副业路线图与阶段规划 | 回答“先做什么、按什么顺序推进” |
| [`research/`](research/) | 市场、行业、工具与可行性研究 | 回答“有哪些机会、方案如何比较” |
| [`projects/`](projects/) | 待启动或正在规划的副业项目 | 回答“具体做什么、如何执行和验收” |
| [`clients/`](clients/) | 客户项目独立保护区 | 只建立项目入口，不摘录客户正文 |

完整分类入口请查看 [INDEX.md](INDEX.md)，本次迁移验证结果请查看 [迁移执行报告](迁移执行报告.md)。

## 使用方法

新增资料前，先判断它的主要用途：

1. 以阶段目标和行动顺序为主，放入 `roadmap/`。
2. 以信息收集、比较和结论为主，放入 `research/`。
3. 以具体产品、服务或交付计划为主，放入 `projects/`。
4. 客户资料按客户名称整目录保存到 `clients/`，不与通用研究资料混放。
5. 同时符合多个分类时，只保留一个主文件，通过 `INDEX.md` 建立入口，避免重复副本。

## 安全边界

- 不在导航中记录 API Key、Token、密码或账户凭据。
- 不读取或收录个人工作记录。
- 客户项目只允许保存在 `clients/` 高保护区；导航不摘录报价、客户信息或正文。
- 公司决算资料不进入本目录，本次也未处理。
- 不自动删除、覆盖、提交或推送文件。
