## 一、整体架构说明

ATBIO 网站采用常见的 VPS + Nginx + PHP CMS + MySQL + 模板文件架构。访问链路如下：

```text
访客浏览器
  ↓
域名 atmbio.com
  ↓
Hostinger VPS
  ↓
Nginx
  ↓
PHP CMS 程序
  ↓
MySQL 数据库
  ↓
Template 模板 + static 静态资源
  ↓
最终网页展示
```

| 层级 | 作用 | 常见维护内容 |
| --- | --- | --- |
| 域名 | 指向网站服务器 | DNS 解析、HTTPS 访问 |
| Hostinger VPS | 承载网站运行环境 | 重启、快照、系统资源、SSH |
| Nginx | 网站访问服务 | 域名绑定、伪静态、SSL、转发规则 |
| PHP CMS | 网站后台和业务程序 | 内容发布、栏目管理、页面渲染 |
| MySQL | 保存网站数据 | 产品、新闻、页面内容、询盘 |
| Template | 控制页面结构 | 首页、产品页、新闻页、单页布局 |
| static | 控制静态资源 | CSS、JS、图片、字体 |
| uploads | 保存上传文件 | 产品图、新闻图、Banner、PDF |

小皮面板 / 服务器管理面板：

```text
http://187.127.119.240:21710/O9da5b
```

Hostinger hPanel：

```text
https://hpanel.hostinger.com/
```

VPS 是网站运行的底座，主要负责运行环境和服务器资源。

| 项目 | 说明 |
| --- | --- |
| 服务器平台 | Hostinger VPS |

### 小皮面板是什么

小皮面板是安装在 VPS 上的服务器可视化管理面板。它本质上是“服务器管理工具”，不是网站 CMS 后台。

| 项目 | 小皮面板负责什么 |
| --- | --- |
| 网站管理 | 绑定域名、设置网站根目录、管理站点配置 |
| 文件管理 | 上传、下载、编辑、压缩、解压网站文件 |
| 数据库管理 | 创建数据库、导入 SQL、导出 SQL、修改数据库密码 |
| PHP 管理 | 切换 PHP 版本、查看 PHP 配置 |
| Nginx 管理 | 管理站点配置、伪静态、重启服务 |
| SSL 管理 | 申请和续期 HTTPS 证书 |
| 计划任务 | 设置定时备份、日志清理等任务 |

### 小皮面板和 CMS 后台的区别

| 对比项 | 小皮面板 | 网站 CMS 后台 |
| --- | --- | --- |
| 访问地址 | `http://187.127.119.240:21710/O9da5b` | `https://www.atmbio.com/login.php?s=Admin/login` |
| 管理对象 | 服务器、网站文件、数据库、Nginx、PHP、SSL | 产品、新闻、页面内容、栏目、SEO、询盘 |
| 使用人员 | 运维人员、开发人员 | 网站管理员、运营人员 |
| 风险级别 | 较高，误操作可能影响整个网站 | 中等，主要影响内容展示 |
| 是否用于发布代码 | 可以，用于上传和管理网站文件 | 不用于发布代码 |
| 是否用于改产品内容 | 不推荐 | 推荐 |

### VPS / 小皮面板适合处理

| 操作 | 说明 |
| --- | --- |
| 查看服务器状态 | CPU、内存、硬盘、端口 |
| 管理网站文件 | 查看、上传、下载、编辑、压缩、解压 |
| 管理数据库 | 创建、备份、导入、导出、改密码 |
| 配置 SSL | 申请、续期、绑定证书 |
| 管理 Nginx | 域名绑定、伪静态、配置测试、重启 |
| 服务器备份 | 网站目录备份、数据库备份、服务器快照 |

### VPS 操作风险

| 操作 | 风险 | 建议 |
| --- | --- | --- |
| 删除网站目录 | 网站可能直接不可用 | 删除前必须完整备份 |
| 修改 Nginx 配置 | 可能导致网站打不开 | 修改后先执行 `nginx -t` |
| 修改 PHP 版本 | 可能导致 CMS 不兼容 | 先确认程序要求 |
| 修改数据库密码 | 后台和网站可能连接失败 | 同步修改程序配置 |
| 重装系统 | 会清空服务器环境 | 仅在确认备份完整后操作 |

## 六、域名与 Hostinger hPanel 说明

Hostinger hPanel 地址：

```text
https://hpanel.hostinger.com/
```

从当前资料看，VPS 是在 Hostinger 购买的，因此服务器实例、续费、快照、重启等通常在 Hostinger hPanel 管理。域名是否也在 hPanel 管理，需要看域名注册商和 DNS Nameserver 当前指向。

| 项目 | 常见管理位置 | 说明 |
| --- | --- | --- |
| VPS 购买和续费 | Hostinger hPanel | 服务器套餐、续费、资源升级 |
| VPS 重启和快照 | Hostinger hPanel | 服务器级别操作 |
| 域名注册 | 可能是 Hostinger，也可能是其他注册商 | 以实际购买平台为准 |
| DNS 解析 | 当前 Nameserver 所在平台 | 如果 Nameserver 指向 Hostinger，则在 hPanel 管 |
| 网站域名绑定 | 小皮面板 / Nginx | 把域名绑定到 VPS 上的网站目录 |
| HTTPS 证书 | 小皮面板 / Nginx | 申请和续期 SSL 证书 |

### 域名是否在 `hpanel.hostinger.com` 管理

可以按下面规则判断：

| 情况 | 结论 |
| --- | --- |
| 域名是在 Hostinger 购买，且 Nameserver 使用 Hostinger | 域名和 DNS 大概率在 hPanel 管 |
| 域名在其他平台购买，但 DNS 切到 Hostinger | DNS 在 hPanel 管，域名续费仍在原注册商 |
| 域名在其他平台购买，Nameserver 也在其他平台 | DNS 不在 hPanel 管，hPanel 只管 VPS |

> 本文档暂按“VPS 在 Hostinger hPanel 管理”记录。域名和 DNS 的实际管理位置，建议登录 hPanel 后查看 Domains / DNS Zone，或查询域名 Nameserver 后确认。

## 七、项目发布与管理方式

当前网站是 PHP CMS 项目，运行目录在 VPS 上的网站根目录：

```text
/www/www/atmbio.com
```

### 整个网站是否存储在 VPS 上

是。当前网站的程序文件、模板文件、静态资源和上传文件，都存储在 Hostinger VPS 的磁盘目录中：

```text
/www/www/atmbio.com
```

数据库内容不直接存在这个目录里，而是存储在 MySQL 数据库中。

| 内容 | 存储位置 | 说明 |
| --- | --- | --- |
| PHP CMS 程序 | VPS 文件目录 | 例如 `application`、`core`、`index.php`、`login.php` |
| 网页模板 | VPS 文件目录 | 例如 `template` |
| CSS / JS | VPS 文件目录 | 例如 `static/css`、`static/js` |
| 上传图片和附件 | VPS 文件目录 | 例如 `uploads` |
| 产品、新闻、页面正文 | MySQL 数据库 | 通过 CMS 后台管理 |
| 域名解析记录 | DNS 管理平台 | 可能在 Hostinger hPanel，也可能在其他域名平台 |

### 网站文件通过什么上传

网站发布或维护时，文件通常通过以下方式上传到 VPS：

| 上传方式 | 用途 | 适合场景 |
| --- | --- | --- |
| 小皮面板文件管理 | 通过浏览器上传、下载、编辑、解压文件 | 初次上传压缩包、临时小改 |
| SSH / SFTP | 通过远程连接传输文件 | 开发人员维护、批量上传 |
| VS Code Remote SSH | 直接打开 VPS 目录编辑文件 | 修改模板、CSS、JS |
| Git | 从代码仓库拉取或同步代码 | 后续规范化版本管理 |

> 从现有资料看，当前项目很可能是通过“小皮面板文件管理上传程序包 + 导入数据库 + 绑定域名”这种传统 PHP 网站方式发布的。是否使用过 Git，需要登录服务器检查 `.git` 目录或部署记录后确认。

### 网站是怎么发布到小皮面板的

通常流程如下：

```text
准备网站程序文件
  ↓
登录小皮面板
  ↓
创建网站或选择已有网站 atmbio.com
  ↓
设置网站根目录 /www/www/atmbio.com
  ↓
上传 PHP CMS 程序文件
  ↓
创建或导入 MySQL 数据库
  ↓
配置数据库连接文件
  ↓
绑定域名 atmbio.com / www.atmbio.com
  ↓
配置 Nginx / 伪静态 / PHP 版本
  ↓
申请 SSL 证书
  ↓
访问前台和后台验证
```

```text

  ↓ 购买和管理
```

| CMS 后台 | CMS 程序的一部分 | 给管理员改产品、新闻、页面内容 |
| Hostinger hPanel | 云服务平台 | 管 VPS、续费、快照、可能也管域名 DNS |

### 推荐发布方式

| 场景 | 推荐方式 | 说明 |
| --- | --- | --- |

| 小范围改 CSS 或模板 | VS Code Remote SSH 修改服务器文件 | 修改前备份对应文件 |
| 批量改模板或功能 | 本地编辑后上传到小皮面板或用 SSH 同步 | 建议先在测试目录验证 |

## 八、开发修改工具建议

## 九、软件性质与开发语言说明

### 这个网站是用什么开发的

从当前目录结构和访问入口判断，ATBIO 网站本身是 PHP CMS 网站，通常由 PHP 程序 + MySQL 数据库 + 模板文件组成。

| 组成 | 技术判断 | 说明 |
| --- | --- | --- |
| 网站程序 | PHP | 存在 `index.php`、`login.php` 等入口文件 |
| 后台管理 | PHP CMS 的后台模块 | 地址为 `login.php?s=Admin/login` |
| 数据库 | MySQL | 保存产品、新闻、页面内容、询盘 |
| 页面模板 | HTML/PHP 模板体系 | 常见目录为 `template` |
| 样式和交互 | CSS / JavaScript | 常见目录为 `static` |

### 小皮面板是否是免费软件

小皮面板是服务器管理面板，不是 ATBIO 网站本身。它是否免费、是否有商业版或授权限制，需要以服务器当前安装版本和官方授权说明为准。

| 软件 / 平台 | 是否免费 | 说明 |
| --- | --- | --- |
| Hostinger VPS | 付费 | VPS 是购买的云服务器资源 |
| Hostinger hPanel | 随 Hostinger 服务提供 | 用于管理 Hostinger 账户、VPS、域名等 |
| 小皮面板 | 需要按当前安装版本确认 | 通常作为服务器管理面板使用，是否免费或是否有商业功能要看版本 |
| ATBIO CMS 程序 | 需要按源码或授权确认 | 可能是开源 CMS、商业 CMS 或定制程序 |
| MySQL / Nginx / PHP | 常见开源软件 | 具体版本以服务器实际安装为准 |

> 保守写法：不要在交付资料中直接承诺“小皮面板永久免费”或“CMS 免费商用”。如果涉及商业使用、转交客户或长期维护，应登录面板查看版本和授权，或确认程序来源。

### 小皮面板是不是用 PHP 开发的

这个问题要分开看：

| 对象 | 是否 PHP | 说明 |
| --- | --- | --- |
| ATBIO 网站 | 是，按现有结构判断是 PHP CMS | 日常二次开发主要围绕 PHP、模板、CSS、JS、MySQL |
| 网站 CMS 后台 | 是网站 PHP CMS 的一部分 | 用于管理内容，不是服务器面板 |
| 小皮面板 | 不建议仅凭当前资料判断 | 它是服务器管理软件，内部实现语言不影响 ATBIO 网站维护 |
| VPS | 不是开发语言 | VPS 是运行网站的云服务器 |

对 ATBIO 项目维护来说，最重要的是：网站本身按 PHP CMS 维护，小皮面板只负责运行环境和文件管理。

## 十、网站目录结构说明

当前网站根目录按原始记录为：

```text
/www/www/atmbio.com
```

常见目录结构如下：

```text
/www/www/atmbio.com
├── application
├── core
├── data
├── extend
├── public
├── static
├── template
├── uploads
├── vendor
├── weapp
├── index.php
└── login.php
```

| 路径 | 主要作用 | 修改建议 |
| --- | --- | --- |
| `template` | 页面模板、页面结构 | 可按需求修改，修改前备份 |
| `static` | CSS、JS、图片、字体 | 可按需求修改，注意缓存 |
| `uploads` | 后台上传资源 | 谨慎删除，避免图片丢失 |
| `application` | 应用业务逻辑 | 非必要不改 |
| `core` | CMS 核心程序 | 不建议直接改 |
| `data` | 缓存、配置、临时数据 | 修改前确认用途 |
| `extend` | 扩展功能 | 由开发人员维护 |
| `public` | 公共资源或入口资源 | 按实际引用谨慎处理 |
| `vendor` | 第三方依赖 | 不建议手动改 |
| `weapp` | 插件或扩展模块 | 修改前确认功能来源 |
| `index.php` | 网站入口 | 不建议随意修改 |
| `login.php` | 后台入口 | 不建议随意修改 |

## 十七、安全与操作原则

| 原则 | 说明 |
| --- | --- |
| 先备份，再修改 | 所有模板、CSS、JS、数据库操作前先备份 |
| 小步修改 | 一次只改一类问题，方便回滚和排查 |
| 不随意改核心目录 | `core`、`vendor`、`index.php`、`login.php` 谨慎处理 |
| 不直接删上传目录 | `uploads` 可能存放大量页面图片 |
| 不随意重装系统 | 重装会影响整个网站环境 |
| 修改后检查手机端 | 企业网站移动端访问占比通常较高 |
| 记录改动位置 | 方便后续接手人员维护 |

## 十八、常见故障处理

| 故障 | 优先检查 | 可能原因 |
| --- | --- | --- |
| 网站打不开 | VPS、Nginx、域名、PHP、MySQL | 服务器异常、配置错误、服务未启动 |
| 后台打不开 | `login.php`、PHP、数据库 | 后台入口异常、数据库连接失败 |
| 页面错位 | CSS、模板、缓存 | 样式修改错误或缓存未更新 |
| 修改不生效 | 文件路径、浏览器缓存、程序缓存 | 改错文件或缓存未清 |
| 图片不显示 | 图片路径、`uploads`、权限 | 图片不存在、路径错误、权限异常 |
| 表单不能提交 | JS、PHP、数据库、邮件配置 | 前端验证错误或后端处理异常 |
| SSL 失效 | 服务器面板、Nginx、证书有效期 | 证书过期或续期失败 |

## 二十二、交付结论

ATBIO 网站维护可以按四层理解：

| 层级 | 负责内容 | 主要入口 |
| --- | --- | --- |
| 内容层 | 产品、新闻、页面文案、SEO、询盘 | CMS 后台 |
| 页面层 | 首页、导航、页脚、布局、样式、交互 | `template`、`static` |

后续维护时，先判断要处理的是“内容、页面、环境还是平台资源”，再进入对应位置操作。所有涉及模板、数据库、服务器配置、域名解析的修改，都建议先备份或记录原配置，再小步调整，最后检查前台、后台和手机端效果。
