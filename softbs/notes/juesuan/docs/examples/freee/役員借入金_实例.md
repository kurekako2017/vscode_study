# 役員借入金完整案例

## 1. 业务背景

日本一人法人从事 IT・SES 业务。代表董事在公司信用卡尚未开通时，使用个人信用卡支付四项公司云服务。

| 服务 | 日文用途 | 中文解释 | 金额 |
| --- | --- | --- | ---: |
| OpenAI | `生成AI API利用料` | 客户项目的生成式 AI API | 33,000円 |
| Cursor | `開発用AIエディタ` | 程序开发 AI 编辑器 | 3,000円 |
| GitHub | `ソースコード管理` | 私有代码仓库及协作 | 4,000円 |
| AWS | `クラウド利用料` | 测试服务器和存储 | 22,000円 |
| 合计 |  |  | 62,000円 |

本案例假设四项均有公司业务用途和合格账单。消费税税区分不能仅按品牌决定，应根据账单开具主体、适格请求书、国内外交易和电气通信利用役务规则逐笔确认。

## 2. 证憑确认

1. 下载 OpenAI 发票和付款记录。
2. 下载 Cursor 发票和付款记录。
3. 下载 GitHub 发票和付款记录。
4. 下载 AWS 账单、适格请求书信息和付款记录。
5. 确认账单抬头能够说明公司业务用途。
6. 在资料上记录代表董事个人垫付日期。
7. 对每项服务单独判断消费税税区分。
8. 不在会计摘要中记录个人信用卡完整号码。

## 3. 个人垫付登记

### freee 操作步骤

1. 打开 `取引入力` → `収入・支出形式`。
2. 选择 `支出` 和 `決済済み`。
3. 口座选择 `役員資金`。
4. 输入 OpenAI 实际付款日、33,000円和账单确认后的税区分。
5. 勘定科目选择 `通信費`，摘要填写 `OpenAI API利用料`。
6. 关联 OpenAI 发票。
7. 用相同流程登记 Cursor 3,000円。
8. 用相同流程登记 GitHub 4,000円。
9. 用相同流程登记 AWS 22,000円。
10. 打开 `貸借対照表`，确认役員借入金增加 62,000円。

## 4. 公司返还垫付款

假设公司在次月 10 日从普通预金返还代表董事 62,000円。

1. 制作垫付清单，逐项列明服务、日期、金额和证憑。
2. 由公司银行向代表董事转账 62,000円。
3. 同步公司银行明细。
4. 打开 `自動で経理`。
5. 选择 62,000円出金明细。
6. 勘定科目选择 `役員借入金`。
7. 税区分选择対象外，避免在返还时再次计算进项税。
8. 摘要填写 `クラウド利用料立替精算`。
9. 登记交易。
10. 打开役員借入金总账，确认本批垫付款余额归零。

返还分录：

| 借方（日文） | 金额 | 贷方（日文） | 金额 |
| --- | ---: | --- | ---: |
| `役員借入金` | 62,000円 | `普通預金` | 62,000円 |

## 5. 验收结果

1. 四张账单金额合计为 62,000円。
2. 每项费用只有一次费用确认。
3. 个人垫付时役員借入金增加 62,000円。
4. 公司返还后役員借入金减少 62,000円。
5. 返还动作没有重复形成通信费或消费税。
6. OpenAI、Cursor、GitHub、AWS 税区分均有账单依据。
