# 別表四完整填写案例

## 1. 实例目的

別表四把会计上的当期利益调整为法人税法上的所得金额。

中文理解：会计利润并不一定等于税法所得，需要通过“加算”和“减算”进行调整。

## 2. 公司与决算数据

| 日文项目 | 中文解释 | 金额 | 税务处理 |
| --- | --- | ---: | --- |
| `当期利益` | PL 税后当期纯利益 | 699,700円 | 別表四起点 |
| `法人税等` | 法人税、住民税及事业税 | 300,300円 | 加算 |
| `交際費損金不算入` | 交际费不可税前扣除 | 50,000円 | 加算；本案例为社外流出 |
| `減価償却超過額認容` | 前期折旧超额本期认可 | 20,000円 | 减算；留保取崩 |

公司为资本金 5,000,000円、发行股份 5,000,000 股的一人 IT・SES 株式会社，事业年度为 2025-05-01 至 2026-04-30。

## 3. 所得计算

```text
会计当期纯利益                    699,700
加算：法人税、住民税等            300,300
加算：交际费損金不算入             50,000
减算：前期减价償却超额认容         20,000
------------------------------------------
税务所得                        1,030,000
```

## 4. 留保与社外流出

| 调整项 | 本例分类 | 后续影响 |
| --- | --- | --- |
| 法人税、住民税損金不算入 | 按实际表格规则确认 | 与税金科目及別表五（二）核对 |
| 交际费損金不算入 | 社外流出示例 | 通常不形成以后认容项目 |
| 减价償却超额认容 | 留保取崩 | 与別表五（一）减价償却超额项目联动 |

留保、社外流出的判断不能只凭科目名称，应按实际税务调整性质确认。

## 5. 填写顺序

1. 从确定版 PL 取得当期纯利益或纯损失。
2. 列出会计已计入费用但税法不允许損金的项目。
3. 列出会计未计入收益但税法需要益金的项目。
4. 列出税法允许减算或前期留保认容的项目。
5. 为每个项目判断留保或社外流出。
6. 计算加算、减算后的所得或欠损。
7. 把留保增减同步到別表五（一）。
8. 把最终所得转入別表一。

### freee 会计数据抽取步骤

1. 打开 freee `会計帳簿` → `損益計算書`。
2. 确认当期纯利益为 699,700円。
3. 打开 `総勘定元帳`，筛选 `法人税、住民税及び事業税`。
4. 确认当期计提金额为 300,300円。
5. 筛选交际费并打开本案例税务调整明细。
6. 确认損金不算入部分为 50,000円。
7. 打开 `決算申告` → `固定資産台帳`，确认前期折旧超额本期认容 20,000円的依据。
8. 将三项调整资料与振替传票、发票和前期申告书关联保存。

## 6. e-Tax 软件填写步骤

1. 打开目标法人的 `法人税・地方法人税` 申告数据。
2. 选择当年度 `別表四`。
3. 将損益計算書当期利益 699,700円输入起点栏。
4. 在加算栏输入損金经理的法人税等 300,300円。
5. 在交际费相关加算栏输入 50,000円，并按本案例设为社外流出。
6. 在减算栏输入前期减价償却超额认容 20,000円。
7. 将税金和折旧差异按软件提示反映到留保栏。
8. 确认加算合计 350,300円。
9. 确认减算合计 20,000円。
10. 确认所得金额 1,030,000円。
11. 将结果联动至別表一。
12. 将留保增减联动至別表五（一）。

## 8. 验收清单

- [ ] 起点利益与 PL 一致
- [ ] 每项加算、减算都有总账或明细依据
- [ ] 留保和社外流出分类已复核
- [ ] 所得金额与別表一一致
- [ ] 留保项目与別表五（一）一致
- [ ] 没有把永久差异误当成未来认容项目

## 9. 注意

本例未包含寄附金、受取配当、欠损金、役员工资損金不算入等其他调整。实际有这些交易时，需要追加对应别表和明细。
