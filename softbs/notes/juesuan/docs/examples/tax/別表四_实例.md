# 別表四 填写实例（Mock）

## 1. 实例目的

別表四把会计上的当期利益调整为法人税法上的所得金额。

中文理解：会计利润并不一定等于税法所得，需要通过“加算”和“减算”进行调整。

## 2. 共通 Mock 场景

| 项目 | Mock 金额 | 税务处理 |
| --- | ---: | --- |
| PL 当期纯利益 | 900,000円 | 別表四起点 |
| 損金经理的法人税、住民税 | 300,000円 | 加算 |
| 交际费損金不算入 | 50,000円 | 加算；本例假设为社外流出 |
| 前期减价償却超额本期认容 | 20,000円 | 减算；留保取崩 |

## 3. Mock 计算

```text
会计当期纯利益                    900,000
加算：法人税、住民税等            300,000
加算：交际费損金不算入             50,000
减算：前期减价償却超额认容         20,000
------------------------------------------
Mock 税务所得                    1,230,000
```

## 4. 留保与社外流出

| 调整项 | 本例分类 | 后续影响 |
| --- | --- | --- |
| 法人税、住民税損金不算入 | 按实际表格规则确认 | 与税金科目及別表五（二）核对 |
| 交际费損金不算入 | 社外流出示例 | 通常不形成以后认容项目 |
| 减价償却超额认容 | 留保取崩 | 与別表五（一）减价償却超额项目联动 |

留保、社外流出的判断不能只凭科目名称，应按实际税务调整性质确认。

## 5. 填写顺序

1. 从确定版 PL 取得当期纯利益或纯损失。
2. 列出会计已计入费用但税法不允许損金的项目。
3. 列出会计未计入收益但税法需要益金的项目。
4. 列出税法允许减算或前期留保认容的项目。
5. 为每个项目判断留保或社外流出。
6. 计算加算、减算后的所得或欠损。
7. 把留保增减同步到別表五（一）。
8. 把最终所得转入別表一。

## 6. 与其他表的联动

- 减价償却超额留保取崩 → [別表五（一）实例](別表五一_实例.md)

## 7. 验收清单

- [ ] 起点利益与 PL 一致
- [ ] 每项加算、减算都有总账或明细依据
- [ ] 留保和社外流出分类已复核
- [ ] 所得金额与別表一一致
- [ ] 留保项目与別表五（一）一致
- [ ] 没有把永久差异误当成未来认容项目

## 8. 注意

本例未包含寄附金、受取配当、欠损金、役员工资損金不算入等其他调整。实际有这些交易时，需要追加对应别表和明细。
