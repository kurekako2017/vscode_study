# e-Tax・法人税申告手册（完整实战版）

## 1. 目的

本手册用于整理一人法人通过 e-Tax 提交法人税、地方法人税及相关国税资料的准备和检查流程。

> 法人税别表会相互联动。第一次申报、存在亏损结转或复杂税务调整时，建议在发送前咨询税务署或税理士。

## 2. e-Tax主要对象

- 法人税
- 地方法人税
- 消费税（详细参见 [消費税手册](消費税手册.md)）
- 源泉所得税
- 法定调书

法人税确定申告原则上应在事业年度结束之日的次日起两个月内提交。延期、节假日及特殊情况必须查看当年度官方规定。

## 3. 事前准备

| 项目 | 确认内容 |
| --- | --- |
| 利用者識別番号 | 法人对应号码是否可用 |
| 电子证明书 | 是否有效、是否安装 |
| 密码 | 是否能正常签名；不要写入本手册 |
| 软件 | e-Tax软件或对应申报服务是否为最新版本 |
| 前期资料 | 前期申告书、别表五和届出书控件 |
| 当期资料 | 确定版决算书、总分类账、内訳资料 |

## 4. 法人税申告所需基础资料

- 貸借対照表（BS）
- 損益計算書（PL）
- 株主資本等変動計算書
- 個別注記表
- 勘定科目内訳明細書
- 法人事業概況説明書
- 前期法人税申告书全套
- 固定资产台账
- 税金缴纳记录
- 需要的其他证明资料

## 5. 主要别表

### 別表一

汇总课税所得、法人税额、控除税额和中间缴纳额，是法人税申告的中心表。

### 別表四

把会计利润通过加算、减算调整为税法所得。

常见确认对象包括法人税等損金不算入、交际费損金不算入、减价償却超额和前期留保项目认容。

### 別表五（一）

管理利益积立金额和资本金等的税务增减，并与别表四的留保项目以及前期结转联动。

### 別表五（二）

管理法人税、地方法人税、住民税、事业税的发生、缴纳和未缴状态。

### 其他可能需要的资料

- 別表二
- 別表十五
- 適用額明細書
- 勘定科目内訳明細書
- 法人事業概況説明書

不能仅凭上述四份别表判断整套申告已经完整。

## 6. 推荐制作顺序

1. 准备前期申告书、当期决算书和总账。
2. 整理会计与税法之间的调整项目。
3. 制作别表四，确认税法所得。
4. 把留保项目反映到别表五（一）。
5. 在别表五（二）核对税金发生及缴纳状态。
6. 在别表一汇总所得和法人税额。
7. 确认其他必要别表、内訳书和概况书。
8. 对照 BS、PL 和前期申告书进行交叉检查。

## 7. e-Tax发送步骤

1. 打开最新 e-Tax 软件或对应服务。
2. 选择正确法人和事业年度。
3. 输入或导入法人税申告数据。
4. 添加决算书及必要附件。
5. 运行软件内检查并修正错误。
6. 核对申告税额和中间缴纳额。
7. 使用电子证明书签名。
8. 发送申告数据。
9. 打开消息箱，确认接收通知和错误信息。
10. 按期限完成缴纳。
11. 保存发送数据、接收通知和缴纳记录。

发送成功不等于缴纳成功，两项必须分别确认。

## 8. 发送前检查表

- [ ] 法人名称、法人编号、所在地正确
- [ ] 事业年度正确
- [ ] 决算书为最终确认版
- [ ] 前期别表五结转正确
- [ ] 别表四与别表五（一）联动
- [ ] 别表五（二）与税金账簿一致
- [ ] 别表一税额与其他别表一致
- [ ] 必要附件齐全
- [ ] 电子签名有效
- [ ] 提交期限和缴纳期限已确认

## 9. 常见问题

### 只有 freee会計能否完成法人税申告？

能输出决算书不代表能完成法人税别表。应确认 freee申告的当前合同和支持功能，或者使用 e-Tax 手工制作。

### 亏损时是否不用申告？

不能这样判断。即使会计上亏损，法人税申告、地方税均等割和亏损结转等仍可能需要处理。

### 可以照抄前期别表吗？

不可以。前期数据用于结转和核对，当期必须根据实际决算和税务调整更新。

## 12. 一人法人 e-Tax 全流程

### 日文说明

法人の電子申告は、`利用者識別番号の取得`、`電子証明書の準備`、`申告データの作成`、`電子署名`、`送信`、`受信通知の確認`、`納付`、`保存`の順に行う。

### 中文解释

一人法人不能只完成“发送”按钮。完整闭环包括身份准备、申告制作、电子签名、发送结果确认、实际缴税和资料留存。

### 标准执行步骤

1. 确认法人现有的 `利用者識別番号` 和登录状态。
2. 确认代表者可用的 `電子証明書`、有效期限和签名密码。
3. 确认电脑、浏览器、IC 卡读取方式和 e-Tax 软件符合官方当前推荐环境。
4. 准备当期确定版决算书、前期申告书、税务调整资料和中间缴纳资料。
5. 选择正确工具：完整法人申告使用安装版 `e-Taxソフト`、freee申告或其他支持 e-Tax 的法人税软件。
6. 制作法人税或消费税申告数据，并完成软件内校验。
7. 使用代表者电子证明书对确定版申告数据添加电子签名。
8. 使用法人利用者識別番号登录并发送。
9. 立即保存 `即時通知`。
10. 打开 `メッセージボックス`，确认 `受信通知` 没有错误。
11. 按申告结果另行完成纳税，确认银行出账或电子纳税结果。
12. 保存申告数据、申告书 PDF、即時通知、受信通知和纳付记录。

## 13. 利用者識別番号

### 日文说明

`利用者識別番号`は、e-Tax利用者ごとに発行される半角16桁の番号である。既存番号がある法人は同じ番号を継続使用する。

### 中文解释

利用者识别号码相当于法人的 e-Tax 账号。它不是法人编号，也不是マイナンバー。重复提交开始届并取得新号码，会使旧号码无法使用，旧消息箱内容也不会自动继承。

### 取得前检查步骤

1. 检查以前年度申告资料、税务署通知、e-Tax 用户文件或税理士交接资料中是否已有 16 位号码。
2. 尝试使用现有号码和密码登录法人 e-Tax WEB 版。
3. 能正常登录时，继续使用现有号码，不重新提交开始届。
4. 号码存在但密码不明时，使用官方密码再设定或向税务署确认，不新建第二个账号代替。
5. 确认确实从未取得后，才进入新申请流程。

### 在线取得步骤

1. 打开 e-Tax 官方 `ご利用の流れ` 页面。
2. 进入 `開始（変更等）届出書作成・提出コーナー`。
3. 选择法人使用者对应的开始届。
4. 输入法人名称、法人番号、纳税地、代表者、提出先税务署和联络方式。
5. 逐项复核，特别确认法人名称、所在地和税务署无误。
6. 发送开始届。
7. 保存画面显示或后续通知的 16 位 `利用者識別番号`。
8. 设置并妥善保管登录密码，不写入本手册或普通 Markdown。
9. 登录 e-Tax WEB 版 `マイページ`，确认法人基本信息。

### 验证方法

1. 使用 16 位利用者識別番号和密码登录法人 e-Tax WEB 版。
2. 确认 `マイページ` 的法人名称、纳税地和提出先税务署正确。
3. 确认没有因重复取得号码导致旧消息无法查看。

## 14. マイナンバーカード与電子証明書

### 日文说明

法人本人が申告データを送信する場合、原則として法人代表者の電子証明書で電子署名する。代表者個人のマイナンバーカードに格納された`署名用電子証明書`を使用できる。

### 中文解释

法人本身没有“法人マイナンバーカード”。一人法人通常使用代表者个人的マイナンバーカード完成签名。除代表者个人卡外，也可按官方规则使用商業登記認証局或受支持民间认证机构签发的电子证明书。

### 使用前检查步骤

1. 确认マイナンバーカード属于当前法人代表者本人。
2. 检查卡片本体有效期限和 `署名用電子証明書` 有效期限。
3. 确认记得签名用电子证明书密码；该密码通常为 6～16 位英数字，不同于 4 位利用者証明用密码。
4. 确认电脑可使用 IC 卡读卡器，或确认智能手机支持マイナンバーカード读取。
5. 按 e-Tax 当前推荐环境安装或更新事前准备软件、驱动程序和必要组件。
6. 在申告期限前实际执行一次读取测试，不在发送最后一天才首次测试。

### WEB 版登记电子证明书步骤

1. 使用法人利用者識別番号和密码登录 `e-Taxソフト（WEB版）`。
2. 打开 `マイページ`。
3. 进入 `その他の登録情報`。
4. 选择 `電子証明書の登録` 或更新功能。
5. 按画面选择マイナンバーカード、智能手机电子证明书或其他支持方式。
6. 读取电子证明书并输入签名用密码。
7. 确认登记结果、证明书所有者和有效期限。

### 安装版登记或更新步骤

1. 启动最新 `e-Taxソフト` 并打开正确的法人利用者文件。
2. 进入 `利用者情報登録`。
3. 选择 `電子証明書登録・更新`。
4. 选择使用 IC 卡或其他电子证明书介质。
5. 读取证明书并输入正确密码。
6. 发送登记或更新数据。
7. 查看受信通知，确认登记正常完成。

### 安全规则

1. 不在截图、Markdown、邮件正文或聊天记录中保存卡片号码和密码。
2. 连续输错密码可能锁定电子证明书；不确定时先停止尝试。
3. 代表者变更、卡片更新或证明书到期后，先更新 e-Tax 登记再发送申告。

## 15. e-Taxソフト（WEB版）的正确用途

### 日文说明

`e-Taxソフト（WEB版）`はブラウザで利用できるが、作成可能な手続は限定されている。法人税・法人の消費税の完全な申告書作成機能があるとは限らないため、公式の利用可能手続一覧を確認する。

### 中文解释

WEB 版不是安装版的完整替代品。对一人法人而言，它适合登录、管理法人信息、登记电子证明书、办理部分申请或纳税、查看消息与受信通知。完整法人税和法人消费税申告，通常使用安装版 e-Tax 软件或兼容税务软件制作、签名和发送。

### 法人登录步骤

1. 从 e-Tax 官方网站进入 `e-Taxソフト（WEB版）`，不要使用搜索广告或非官方登录链接。
2. 选择 `法人の方`。
3. 输入 16 位 `利用者識別番号`。
4. 输入 e-Tax 登录密码。
5. 点击 `ログイン`。
6. 进入 `マイページ`，确认法人基本信息和电子证明书登记状态。
7. 需要查看接收结果时，进入 `お知らせ・受信通知`。

### WEB 版功能判断步骤

1. 打开官方 `e-Taxソフト（WEB版）で作成可能な手続` 一览。
2. 查找目标税目和目标手续的正式名称。
3. 一览中存在时，按 WEB 版 `申請・納付手続を行う` → `新規作成` 进入。
4. 一览中不存在完整法人申告时，停止在 WEB 版寻找法人税别表入口。
5. 改用安装版 e-Tax 软件、freee申告或其他支持对应年度和税目的法人申告软件。

## 16. 申告数据制作前准备

### 日文说明

申告データ作成前に、対象法人、税目、事業年度、申告区分、会計データ及び前期繰越情報を確定する。

### 中文解释

申告错误往往不是发送动作造成，而是法人、年度、申告类型或前期结转选错。制作前应先冻结会计数据并确定申告条件。

### 准备步骤

1. 确认法人名称、法人番号、纳税地、提出先税务署和代表者。
2. 确认事业年度起止日期。
3. 确认本次是 `確定申告`、`中間申告`、`修正申告` 或其他申告区分。
4. 准备确定版 BS、PL、株主資本等変動計算書和個別注記表。
5. 准备勘定科目内訳明細書和法人事業概況説明書。
6. 准备前期法人税申告书、別表五（一）、別表五（二）和亏损结转资料。
7. 准备当期税务调整、固定资产、交际费、役员工资和税金缴纳资料。
8. 准备中间纳付、源泉税额等控除资料。
9. 确认所用申告软件支持本事业年度、所需别表和最新法改正。
10. 在开始输入前备份原会计数据和申告数据。

## 17. 法人税申告

### 日文说明

法人税申告では、会計上の利益を別表四で税務上の所得へ調整し、別表五（一）・五（二）等と連動させ、別表一で法人税額を確定する。

### 中文解释

法人税申告不能只填写別表一。应从决算书和前期资料出发，完成税务调整、利益积立金和税金缴纳状态等相互联动，再签名发送整套数据。

### 安装版 e-Tax 软件制作步骤

1. 启动最新 `e-Taxソフト`，打开该法人的利用者文件。
2. 确认已安装当年度 `法人税・地方法人税` 对应税目程序和最新更新。
3. 进入 `申告・申請等` 或 `申告・申請等一覧`。
4. 选择 `新規作成`。
5. 选择 `申告`。
6. 税目选择 `法人税・地方法人税`；当年度画面包含其他法定税目时，按实际申告义务选择。
7. 选择正确年度、事业年度和 `確定申告` 等申告区分。
8. 输入法人基本信息、提出先税务署和代表者信息。
9. 选择本法人实际需要的別表、勘定科目内訳明細書和法人事業概況説明書。
10. 导入或制作财务诸表数据；安装版通常按其 XBRL/CSV 等支持格式处理。
11. 输入別表四的加算、减算和所得金额。
12. 输入并核对別表五（一）的利益积立金、资本金等前期结转与当期增减。
13. 输入并核对別表五（二）的法人税、地方法人税等发生、缴纳和未纳状态。
14. 完成別表一及其他必要別表，确认税额和中间缴纳额。
15. 完成勘定科目内訳明細書、法人事業概況説明書和必要附件。
16. 执行软件内形式检查，修正所有错误并逐项确认警告。
17. 对照决算书、前期申告书和税金账簿进行交叉复核。
18. 将确定版申告数据保存为可识别年度和版本的名称。

### 使用 freee申告等税务软件时

1. 确认合同方案支持当前法人、当前年度和所需申告类型。
2. 从确定版会计数据联动或导入决算书。
3. 按软件流程完成別表、内訳书和概况说明书。
4. 执行软件校验和人工交叉检查。
5. 确认最终发送对象是正确的法人利用者識別番号。
6. 按软件提示使用代表者电子证明书签名并发送。
7. 发送后仍按本手册第 20 节确认 e-Tax 受信通知。

### 法人税申告验证方法

1. 別表四所得与別表一课税所得的联动正确。
2. 別表四留保项目与別表五（一）增减一致。
3. 別表五（二）与税金账簿、中间纳付记录一致。
4. 决算书、内訳明细书和概况说明书金额一致。
5. 必要別表和附件没有仅因“不熟悉”而遗漏。

## 18. 消費税申告

### 日文说明

法人の消費税申告では、課税期間、課税事業者区分、一般課税・簡易課税等の計算方法、税区分及び中間納付額を確認し、消費税及び地方消費税を申告する。

### 中文解释

法人消费税申告必须先确定是否为课税事业者、适用一般课税还是简易课税等方法，再使用会计系统的税区分数据制作。WEB 版能否制作目标手续必须按官方当期功能表确认；完整法人消费税申告通常走安装版或兼容税务软件。

### 制作步骤

1. 确认课税期间起止日期与事业年度关系。
2. 确认本法人当期是否为消费税课税事业者。
3. 确认一般课税、简易课税或当年度依法可适用的其他计算方法及届出状态。
4. 在 freee 会计中完成全年税区分、税率、インボイス和课税売上割合等检查。
5. 准备课税売上、课税仕入、非课税、不课税、免税交易和中间纳付资料。
6. 启动安装版 `e-Taxソフト` 或对应法人消费税软件。
7. 新建 `消費税及び地方消費税` 申告数据。
8. 选择正确课税期间、申告区分和计算方法。
9. 选择并填写当前计算方法所需申告书、付表和明细。
10. 输入中间纳付税额和其他控除项目。
11. 确认国税消费税、地方消费税、纳付或还付金额。
12. 执行形式检查并修正错误。
13. 将申告结果与 freee 的未払消費税等期末处理核对。
14. 保存确定版消费税申告数据。

### 消费税验证方法

1. 课税期间和申告区分正确。
2. 计算方法与已提交届出和法定适用条件一致。
3. 税率、税区分和适格请求书处理已检查。
4. 中间纳付额与纳付记录一致。
5. 申告纳付额与会计的未払消費税等余额一致或有明确差异说明。

## 19. 電子署名与送信方法

### 日文说明

確定した申告データに代表者の電子署名を付与し、法人の利用者識別番号でe-Tax受付システムへ送信する。署名後に申告内容を変更した場合は、再度署名が必要となる。

### 中文解释

电子签名绑定的是确定版申告数据。签名后若修改表格或附件，原签名可能失效，应重新检查并签名。税理士代理发送时按照代理发送规则办理，不套用法人本人发送步骤。

### 安装版电子签名步骤

1. 在 `申告・申請等一覧` 选择已完成检查的确定版申告数据。
2. 进入 `電子署名` 功能。
3. 再次确认法人名称、税目、事业年度和申告区分。
4. 选择 `ICカードを利用` 或实际使用的电子证明书方式。
5. 选择マイナンバーカード对应的认证服务或其他受支持证明书。
6. 读取代表者卡片或电子证明书。
7. 输入签名用电子证明书密码。
8. 确认软件显示电子签名已付与。
9. 返回一览，确认数据状态已进入可发送状态。

### 安装版发送步骤

1. 打开 `送信可能一覧`。
2. 选择刚刚签名的目标申告数据。
3. 点击 `送信`。
4. 输入法人 16 位利用者識別番号和 e-Tax 登录密码。
5. 最终确认送信对象、税目和申告数据名称。
6. 执行发送。
7. 查看发送后立即显示的 `即時通知`。
8. 确认受付日時、受付番号、利用者識別番号和是否存在错误。
9. 立即保存或打印即時通知；该画面原则上不能再次显示。
10. 不以即時通知结束作业，继续确认受信通知。

### 发送后的纳税步骤

1. 根据申告书确认法人税、地方法人税和消费税各自纳付金额与期限。
2. 选择ダイレクト納付、インターネットバンキング、信用卡或其他官方支持方式。
3. 需要时制作并发送 `納付情報登録依頼`。
4. 按画面选择目标税目和纳付金额。
5. 完成实际纳付。
6. 保存电子纳税结果或银行扣款记录。
7. 在会计系统登记实际缴纳并核对未払税金余额。

## 20. 受信通知确认

### 日文说明

送信完了後は`即時通知`だけでなく、`メッセージボックス`に格納される`受信通知`を必ず確認する。エラー情報がある場合、正常受付ではない。

### 中文解释

即时通知只表示发送后的第一层检查；受信通知包含法人名称、纳税地、电子证明书等基本检查结果。必须看到无错误的受信通知，才能确认 e-Tax 已正常接收。

### WEB 版确认步骤

1. 打开 e-Tax 官方 `e-Taxソフト（WEB版）`。
2. 选择 `法人の方`。
3. 使用法人利用者識別番号和密码登录。
4. 点击 `お知らせ・受信通知`。
5. 打开 `受信フォルダ` → `メッセージボックス一覧`。
6. 按发送日期、税目和手续名称找到目标消息。
7. 点击消息，打开 `メッセージ詳細（受信通知）`。
8. 确认受付日時、受付番号、税目、事业年度和法人名称。
9. 确认 `エラー情報` 中没有错误。
10. 点击 `帳票を表示する`，选择需要的帐票并执行 `帳票作成`。
11. 显示 PDF 后保存申告书或受信通知资料。
12. 将受信通知与即時通知、发送数据和纳付记录一起归档。

### 安装版确认步骤

1. 在 `e-Taxソフト` 登录受付系统。
2. 打开 `メッセージボックス`。
3. 更新或取得最新消息。
4. 选择本次申告对应的受信通知。
5. 打开详细内容，确认受付结果和错误信息。
6. 保存或打印受信通知及需要的申告帐票。

### 完成判定

1. 即時通知没有发送错误。
2. 受信通知没有错误信息，并显示正确受付信息。
3. 发送的法人、税目、年度和申告区分正确。
4. 申告数据和必要附件可从保存资料中还原。
5. 纳付已单独完成并有付款依据。

## 21. 常见错误与处理

| 错误或误解 | 原因 | 处理方法 |
| --- | --- | --- |
| 把法人番号当作利用者識別番号 | 混淆 13 位法人编号和 16 位 e-Tax 编号 | 从旧申告或 e-Tax 资料确认 16 位号码 |
| 重复取得利用者識別番号 | 未先检查旧账号 | 停止继续申请，确认当前有效号码；旧消息不会自动转移 |
| 在 WEB 版找不到法人税別表 | WEB 版手续范围有限 | 查官方手续一览，改用安装版或兼容法人税软件 |
| 法人使用代表者卡却无法法人方式登录 | 法人 WEB 登录入口只接受利用者識別番号方式 | 用法人号码和密码登录，代表者卡用于电子证明书登记和签名 |
| 4 位密码无法签名 | 输入了利用者証明用密码 | 电子签名使用署名用电子证明书密码，通常为 6～16 位英数字 |
| 电子证明书失效 | 卡片更新、到期或代表者变更后未更新 | 取得有效证明书并执行 `電子証明書登録・更新` 后重新签名发送 |
| 连续输入密码后卡片被锁 | 多次输入错误 | 停止尝试，按行政机关规定办理密码解锁 |
| 签名后修改申告内容 | 数据改变导致签名不再对应 | 重新执行校验并重新电子签名 |
| 即時通知未保存 | 即时通知原则上不能再显示 | 发送后立即保存；同时必须继续确认受信通知 |
| 受信通知显示错误仍认为已申告 | 错误数据未正常接收 | 按错误代码修正并在期限内重新签名发送 |
| 发送成功但没有缴税 | 申告与纳付是两项独立作业 | 按税目和期限另行纳付并保存付款记录 |
| 法人税和地方税都只发 e-Tax | 混淆国税与地方税入口 | 法人税、地方法人税走 e-Tax；法人住民税、法人事业税通常走 eLTAX |
| 直接照抄前期別表 | 前期数据未按当期变化更新 | 只将前期作为结转依据，重新核对当期税务调整和税金 |
| 把接收完成理解为内容审核通过 | `受付完了` 只表示系统正常接收 | 保留申告依据，并应对税务署后续照会或更正要求 |

## 22. 年度资料保存

### 保存步骤

1. 按事业年度建立申告资料保存位置。
2. 保存制作软件的原始申告数据和最终发送版本。
3. 保存申告书、决算书、内訳明细书和概况说明书 PDF。
4. 保存即時通知和受信通知。
5. 保存电子证明书有效性确认记录，但不保存密码。
6. 保存法人税、地方法人税和消费税的纳付记录。
7. 保存发送日期、受付番号和修正版次的对应关系。
8. 对包含マイナンバー或个人信息的资料设置访问权限。
9. 不把申告资料自动上传到公开 GitHub。
