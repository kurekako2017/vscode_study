# freee会計・法人決算 主手册（完整实战版）

## 1. 目的

本手册是一人法人使用 freee会計进行日常记账、月度确认和年度决算的总入口。

本文件以实际画面流程为主，覆盖日常记账、月次检查、年度处理和决算准备。税务申告、社会保险等专业事项继续由各专题手册补充。

> 本资料用于学习和内部作业管理，不替代税理士、社会保险劳务士或行政机关的个别判断。

## 3. freee会計的职责边界

### 主要用途

- 银行及信用卡同步
- 自動で経理
- 收入、费用和振替传票登记
- 请求书及证憑管理
- 总分类账、试算表、PL、BS
- 固定资产台账和决算书基础资料

### 需要另行确认的事项

- 法人税申告书及各种别表是否需要 freee申告的对应方案
- e-Tax、eLTAX的电子签名与发送
- 算定基础届和月额变更届
- 不同年度、申告类型和复杂税务调整的支持范围

“可以从 freee 输出决算书”不等于“法人税申告已经完成”。

## 4. 每月标准流程

### freee操作

1. 同步全部银行和信用卡账户。
2. 处理 `自動で経理` 和未登记明细。
3. 登记当月销售，并确认売掛金。
4. 登记费用，并确认未払金。
5. 把个人垫付款登记为役員借入金。
6. 保存请求书、收据、合同和转账记录。
7. 核对普通预金、现金、売掛金、未払金。
8. 确认工资、源泉所得税、住民税和社会保险。
9. 查看 PL 与 BS 的异常余额。
10. 完成 [月次检查表](../checklist/月次检查表.md)。

## 5. 决算标准流程

### freee操作

1. 确认全年交易已经登记。
2. 将 freee 的银行余额与真实银行余额核对。
3. 确认売掛金、未払金和役員借入金的明细。
4. 确认固定资产台账和减价償却。
5. 确认消费税税区分和未払消费税。
6. 检查 PL 的期间归属和异常费用。
7. 检查 BS 的实际存在性和异常余额。
8. 输出 BS、PL、株主资本等变动计算书和个别注记表。
9. 按 [eTax手册](../tax/eTax手册.md) 和 [eLTAX手册](../tax/eLTAX手册.md) 准备申报。
10. 完成 [年次检查表](../checklist/年次检查表.md)。

## 6. PL 与 BS 的最小理解

### PL（損益計算書）

表示一段期间内的销售、费用和利润。

重点确认：

- 销售是否遗漏或跨期
- 通信费、外注费、交通费、交际费是否异常
- 是否混入私人消费
- 是否登记减价償却

### BS（貸借対照表）

表示决算日的资产、负债和纯资产。

重点确认：

- 普通预金与银行一致
- 现金真实存在且不是负数
- 売掛金和未払金有明细依据
- 役員借入金有垫付和精算记录
- 固定资产台账与账面一致

## 7. freee方案确认

购买或使用 freee申告前，应逐项确认：

1. 当前 freee会計方案是否满足购买条件。
2. 当前事业年度是否在支持范围内。
3. 所需法人税别表、地方税、内訳书和概况书是否支持。
4. 本次属于确定申告、修正申告还是中间申告。
5. 消费税是否需要单独方案或功能。
6. 电子申告应用、电子证明书和操作系统是否可用。

无法使用时，可选择官方 e-Tax/eLTAX 手工申报、升级对应方案，或将申报部分委托税理士。

## 8. 文件保存原则

- 按事业年度建立资料夹。
- 原始证憑、会计账簿、决算书、申告书和接收通知分开保存。
- 不把 API Key、密码或个人编号写入普通 Markdown。
- 电子发送完成后，同时保存接收通知和缴纳记录。
- 税率、期限、freee功能每年从官方网站重新确认。

## 9. 画面名称与使用约定

### 日文说明

freee会計の画面名は日本語で表示される。本手册では、実際のメニュー名を `日本語` で記載し、その直後に中国語の意味を説明する。

### 中文解释

freee 会因方案、权限和界面更新显示略有不同的菜单名称。例如部分账号显示 `取引入力`，部分旧界面或帮助页面显示 `取引`。遇到差异时，以功能名称和所在业务流程为准，不要仅凭按钮位置判断。

### 基本术语

| 日文画面名称 | 中文解释 |
| --- | --- |
| `取引入力` | 交易录入入口，登记收入、支出、振替传票等 |
| `自動で経理` | 对银行、信用卡同步明细进行记账处理 |
| `未登録明細` | 已同步但尚未完成会计处理的明细 |
| `収入・支出形式` | 用收入或支出表单登记交易 |
| `未決済` | 已确认债权或债务，但尚未收款或付款 |
| `決済済み` | 已经收款或付款 |
| `口座振替` | 公司账户之间的资金移动，不属于收入或费用 |
| `振替伝票` | 使用借方、贷方形式直接登记仕訳 |
| `会計帳簿` | 查看仕訳帳、总账、试算表和月次报表 |
| `決算申告` | 固定资产、决算书、年度締め等年度处理入口 |

## 10. 银行同步（銀行口座の同期）

### 日文说明

銀行口座をfreee会計に登録・同期すると、入出金明細を取り込み、`自動で経理`から会計処理できる。同期しただけでは仕訳登録は完了しない。

### 中文解释

银行同步负责把真实银行流水导入 freee。导入后的流水仍需在 `自動で経理` 中逐条确认，不能把“已同步”误认为“已记账”。

### 初次登记与同步步骤

1. 登录正确的 freee 事业所，确认右上角显示的是目标法人和正确事业年度。
2. 打开 `マスタ・口座` → `口座`，中文含义是“主数据与账户”→“账户”。
3. 点击 `＋登録`，选择银行、信用卡或其他需要登记的账户类型。
4. 搜索金融机构名称，选择实际使用的银行及账户类别。
5. 按画面提示完成金融机构认证；认证信息只在官方连接画面输入，不记录到 Markdown、截图或聊天中。
6. 返回 `口座` 一览，打开已登记账户，确认同步状态和最终同步时间。
7. 执行同步；如果当前方案没有一括同步功能，则逐个账户执行同步。
8. 打开 `取引入力`（部分画面为 `取引`）→ `自動で経理`，确认新明细已经出现。
9. 将 freee 显示的账户余额与网银余额核对，并记录差异原因。

### 无法自动同步时

1. 在 `マスタ・口座` → `口座` 中确认该金融机构是否支持同步。
2. 不支持或暂时无法连接时，先登记手工账户。
3. 从银行下载 freee 支持的明细文件，下载前确认日期范围和账户号码。
4. 在对应账户的明细导入画面上传文件。
5. 导入后检查重复日期、重复金额和期首以前的明细。
6. 打开 `自動で経理`，继续完成会计处理。

### 验证方法

1. 确认同步状态没有持续错误。
2. 确认最终同步日期覆盖本次月结日。
3. 确认同一笔流水没有因自动同步与手工导入而重复。
4. 确认期末账面普通预金余额与银行实际余额一致。

## 11. 自動で経理

### 日文说明

`自動で経理`は、同期した明細に勘定科目・税区分・取引先などを設定し、取引登録、未決済取引の消込、口座振替などを行う画面である。

### 中文解释

`自動で経理` 是同步流水的处理中心。每笔明细应判断为普通收入或支出、应收应付的结算、公司账户间转账，或不应记账的对象，而不是机械接受系统建议。

### 标准操作步骤

1. 打开 `取引入力`（部分画面为 `取引`）→ `自動で経理`。
2. 选择需要处理的银行或信用卡账户，并把期间限定为本次作业月份。
3. 查看摘要、日期、金额和系统建议，先与请求书、收据或合同核对。
4. 普通支出选择正确的勘定科目、税区分、取引先、品目、部門和メモタグ。
5. 普通收入选择正确的收入科目、税区分和取引先。
6. 对已登记的売掛金或未払金，选择未決済取引的消込，不要再次登记销售或费用。
7. 对公司银行、信用卡、现金账户之间的移动，选择 `口座振替`。
8. 对私人交易或确实不属于事业所会计的明细，确认理由后使用适当处理；不要用忽略功能掩盖业务交易。
9. 点击登记按钮后，查看处理结果，并继续下一笔。
10. 完成本月后确认未处理件数为零，或每一笔保留明确定义的暂缓理由。

### 自动登记规则

1. 对每月重复且会计处理稳定的明细，先连续人工确认数次。
2. 确认摘要匹配条件不会误命中其他交易。
3. 在画面中建立 `自動登録ルール`，设定勘定科目、税区分和标签。
4. 下月同步后抽查规则结果，不因使用自动规则而省略月次核对。

### 验证方法

1. 从已处理明细打开生成的取引，确认日期、金额、科目和税区分。
2. 打开 `会計帳簿` → `仕訳帳`，以金额或摘要搜索该笔仕訳。
3. 确认账户间转账没有同时形成收入或费用。
4. 确认应收应付收付款没有重复确认收入或费用。

## 12. 未登録明細

### 日文说明

`未登録明細`は、freeeに取り込まれたものの、取引登録・消込・口座振替などの会計処理が完了していない明細である。

### 中文解释

未登记明细表示流水已经进入 freee，但尚未形成完整会计处理。主页通常聚焦当前事业年度，而账户一览可能显示跨年度件数，因此两处数量不一致时应检查期间。

### 检查与处理步骤

1. 打开 `会計帳簿` → `明細の一覧`。
2. 指定目标账户，并把取引日范围设为需要检查的事业年度或月份。
3. 用登记状态筛选未处理明细。
4. 对照银行流水确认明细是否真实存在、是否重复导入。
5. 打开对应明细并进入 `自動で経理`。
6. 根据业务实质完成取引登记、未決済消込或口座振替。
7. 对跨年度明细确认正确的发生日和所属年度，不要为了清零而修改真实日期。
8. 完成后重新执行相同筛选，确认件数变化符合本次处理数量。

### 验证方法

1. 当前月结范围内的未登记明细应为零。
2. 无法立即确定的明细应保留证据和负责人判断记录。
3. 主页与口座一览件数不一致时，确认是否来自不同事业年度或日期范围。

## 13. 振替伝票

### 日文说明

`振替伝票`は、借方・貸方を指定して仕訳形式で記帳する機能である。通常の収入・支出や口座振替で表現しにくい決算整理仕訳などに用いる。

### 中文解释

振替传票适合折旧、期末调整、税额结转等专业仕訳。能够用普通交易或账户转账正确处理时，应优先使用普通流程，以免造成流水与仕訳脱节。

### 操作步骤

1. 准备计算依据，明确借方科目、贷方科目、金额、税区分和发生日。
2. 打开 `取引入力` → `振替伝票`。
3. 在 `発生日` 输入真实发生日或合法的决算整理日。
4. 在借方输入勘定科目、金额、税区分、取引先和必要标签。
5. 在贷方输入对应勘定科目、金额、税区分、取引先和必要标签。
6. 多行仕訳时确认借方合计与贷方合计完全一致。
7. 在摘要或备注写明业务内容和证憑位置，但不写密码、个人编号等敏感信息。
8. 点击登记，保存振替传票。
9. 打开 `会計帳簿` → `仕訳帳`，搜索日期和金额，确认生成结果。

### 使用判断

1. 公司银行 A 转入银行 B 时，使用 `口座振替`，不使用振替传票重复记账。
2. 同步流水对应既有未決済取引时，使用消込，不重新建立振替传票。
3. 期末未払消費税等决算整理需要直接借贷记账时，可使用振替传票。
4. 不理解借贷方向或税区分时，先核对专题手册或请税理士确认。

## 14. 売掛金（应收账款）

### 日文说明

売上を請求日に未決済取引として登録すると、入金前の債権が`売掛金`として計上される。入金時はその未決済取引を消し込む。

### 中文解释

先提供服务、后收款时，应在销售成立日确认收入和売掛金；银行到账时只结清应收，不再次确认销售收入。

### 销售发生时的登记步骤

1. 准备请求书或合同，确认销售成立日、客户、未税或含税金额、税率和付款期限。
2. 打开 `取引入力` → `収入・支出形式`（或 `取引の一覧・登録`）。
3. 选择 `収入`。
4. 输入销售发生日，不使用未来的实际到账日代替。
5. 将决済状态设为 `未決済`，并输入支払期日。
6. 选择销售科目、税区分、取引先和必要标签。
7. 输入金额并关联请求书等证憑。
8. 点击登记，形成收入与売掛金。

### 收款时的消込步骤

1. 银行同步后打开 `自動で経理`。
2. 选择客户汇款对应的入金明细。
3. 选择 `未決済取引の消込`，搜索客户、金额或请求书。
4. 勾选对应的売掛金交易。
5. 有汇款手续费、部分收款或合并收款时，按实际差额项目分别处理，不强行修改原销售额。
6. 登记消込。
7. 打开该客户的未決済一览，确认剩余金额正确。

### 验证方法

1. 在 `会計帳簿` → `貸借対照表` 确认売掛金余额。
2. 按取引先查看未收款明细，并与请求书台账核对。
3. 确认银行入金没有再次计入销售科目。

## 15. 未払金（应付未付款项）

### 日文说明

費用や資産を先に認識し、支払いが後日になる場合は未決済の支出取引として登録する。内容により`買掛金`または`未払金`として管理する。

### 中文解释

费用已经发生但尚未付款时，应先登记未决済交易。购货通常可能使用買掛金，服务费、经费或其他非商品采购常使用未払金，最终应按公司会计方针判断。

### 费用发生时的登记步骤

1. 准备请求书、合同或验收资料，确认费用归属日和付款期限。
2. 打开 `取引入力` → `収入・支出形式`。
3. 选择 `支出`。
4. 输入费用发生日。
5. 将决済状态设为 `未決済`，并输入支払期日。
6. 选择费用或资产科目、税区分、取引先和必要标签。
7. 输入金额并关联证憑。
8. 点击登记，形成费用或资产与未払金等负债。

### 付款时的消込步骤

1. 银行或信用卡明细同步后打开 `自動で経理`。
2. 选择对应的出金明细。
3. 选择 `未決済取引の消込`。
4. 搜索并选择原未決済支出交易。
5. 确认付款日、付款账户和金额。
6. 对部分付款或合并付款，按实际对象分配消込金额。
7. 登记后确认原交易的未决済余额。

### 验证方法

1. 在 `会計帳簿` → `貸借対照表` 确认未払金、買掛金余额。
2. 按取引先核对未付款清单与收到的请求书。
3. 确认实际付款没有重复登记同一费用。

## 16. 役員借入金（法人向董事借款）

### 日文说明

役員が会社経費を個人資金で立て替えた場合、freeeの`役員資金`口座を使用して費用を登録すると、会社が役員へ返済すべき`役員借入金`が計上される。

### 中文解释

董事个人替公司支付费用时，公司形成对董事的负债。`役員借入金` 不是公司的销售收入，也不同于公司把钱借给董事的 `役員貸付金`。

### 个人垫付登记步骤

1. 确认支出确属公司业务，并保存收据或请求书。
2. 打开 `取引入力` → `収入・支出形式`。
3. 选择 `支出`，将决済状态设为 `完了`。
4. 在口座选择 `役員資金`。
5. 输入个人实际支付日期。
6. 选择真实费用科目和税区分，不把费用科目直接写成役員借入金。
7. 输入金额、取引先和必要标签。
8. 点击登记，确认系统形成役員借入金余额。

### 公司返还垫付款步骤

1. 汇总有证憑且已登记的垫付款，不以口头估算金额。
2. 公司银行实际转账给董事后，同步该出金明细。
3. 打开 `自動で経理`，选择该出金明细。
4. 选择勘定科目 `役員借入金`，输入实际返还金额。
5. 确认税区分不会再次形成进项税额。
6. 登记交易。
7. 打开 `会計帳簿` → `貸借対照表`，确认役員借入金减少。

### 验证方法

1. 将役員借入金余额与个人垫付明细表核对。
2. 确认每笔增加都有业务证憑，每笔减少都有公司付款记录。
3. 确认没有把公司借给董事的款项误记为役員借入金。

## 17. 未払消費税等

### 日文说明

決算時に納付すべき消費税額を確定し、法人では通常`未払消費税等`として計上する。処理方法は税込経理・税抜経理で異なるため、設定と申告書の計算結果を一致させる。

### 中文解释

未払消費税等是决算日已经确定、尚未缴纳的消费税负债。含税记账与不含税记账的期末调整流程不同，不能把中间缴纳、期末计提和次年实际缴纳混为一笔。

### 决算确认步骤

1. 打开 `決算申告` → `消費税申告書`，确认课税方式、申告区分和计算期间。
2. 核对全年交易的税区分、税率、适格请求书条件和课税売上割合等基础数据。
3. 确认消费税申告书计算出的国税、地方消费税和合计纳付额。
4. 确认会社采用 `税込経理` 还是 `税抜経理`。
5. 税込経理时，打开 `取引入力` → `振替伝票`。
6. 将発生日设为事业年度末，按确认后的处理登记借方 `租税公課`、贷方 `未払消費税等`。
7. 税抜経理时，打开 `決算申告` → `決算書` → `プレビュー・調整`，确认仮払消費税与仮受消費税调整提示及系统生成结果。
8. 在任何一种方法中，都先确认系统是否已生成调整，避免重复登记。
9. 打开 `会計帳簿` → `貸借対照表`，确认未払消費税等期末余额与申告计算一致。

### 次期实际缴纳步骤

1. 使用公司银行完成实际缴纳，并保留纳付记录。
2. 同步银行出金明细。
3. 打开 `自動で経理`，选择缴纳明细。
4. 选择 `未払消費税等` 作为消除负债的科目。
5. 确认没有再次计入租税公課或进项税额。
6. 登记后检查未払消費税等余额是否按预期归零或减少。

### 验证方法

1. 申告书纳付额、期末仕訳和 BS 的未払消費税等三者一致。
2. 实际缴纳记录与次期消込金额一致。
3. 中间纳付已按公司采用的税込或税抜处理方式单独核对。

## 18. 固定資産

### 日文说明

固定資産を取得した場合は、取得取引の登録と`固定資産台帳`への登録の両方が必要である。台帳登録により減価償却費が計算される。

### 中文解释

购买固定资产不能只登记银行付款，也不能只登记固定资产台账。前者记录取得交易，后者管理使用开始日、耐用年数和折旧，两者必须一致。

### 取得交易登记步骤

1. 准备请求书、合同和付款记录，确认资产名称、取得日、供用开始日和取得价额。
2. 根据资产性质和适用税务规则确认固定资产科目、税区分和处理方法。
3. 打开 `取引入力` → `収入・支出形式`；有同步流水时也可从 `自動で経理` 开始。
4. 选择 `支出`，输入取得日、付款状态和付款账户。
5. 选择正确的固定资产科目，输入金额、税区分、取引先和摘要。
6. 关联请求书等证憑并登记。

### 固定资产台账登记步骤

1. 打开 `決算申告` → `固定資産台帳`。
2. 点击 `＋固定資産の登録`。
3. 输入资产名称、取得日、供用开始日和取得价额。
4. 选择资产分类、償却方法和耐用年数。
5. 按实际情况输入部门、摘要和其他管理信息。
6. 查看系统计算的本期償却额、期末账面价额和月数。
7. 点击登记。
8. 返回台账一览，确认资产已经出现且没有重复。

### 验证方法

1. 固定资产台账取得价额与取得交易一致。
2. `会計帳簿` → `貸借対照表` 的固定资产余额与台账汇总一致。
3. `会計帳簿` → `損益計算書` 的減価償却費与台账本期償却额一致。
4. 处置、报废或出售资产时，确认台账状态和会计仕訳同步处理。

## 19. 給与処理

### 日文说明

freee人事労務で給与計算を確定し、給与取引をfreee会計へ連携する。会計側では未決済取引として連携された給与を、実際の銀行支払明細で消し込む。

### 中文解释

工资计算通常先在 freee 人事劳务完成，再联动至 freee 会计。会计中的工资交易与银行实际付款是两个环节；付款流水应用于结清既有工资交易，不能再次登记工资费用。

### 工资计算与会计联动步骤

1. 在 freee 人事劳务确认员工或役员、工资项目、社会保险、源泉所得税和住民税设置。
2. 输入当月勤怠或固定工资信息，执行工资计算。
3. 检查 `給与明細`，确认支给、控除和差引支給额。
4. 完成当月工资明细确定。
5. 按联动画面将給与取引同步到 freee 会计。
6. 在 freee 会计打开 `取引入力` → `給与取引の登録`，确认联动状态和交易内容。
7. 检查工资、法定福利费、预扣税款和社会保险等科目是否符合公司的会计设定。

### 实际发薪与消込步骤

1. 用公司银行向役员或员工支付差引支給额。
2. 银行同步后打开 `自動で経理`。
3. 选择工资付款的出金明细。
4. 选择对应的未決済給与取引并执行消込。
5. 实际付款日与工资计算上的预定日不同时，保留真实银行付款日。
6. 社会保险、源泉所得税和住民税实际缴纳时，分别用对应负债科目消込。
7. 登记后确认工资未决済余额和各预扣负债余额。

### 验证方法

1. 差引支給额合计与实际银行付款合计一致。
2. 工资总额、社会保险公司负担额与工资台账一致。
3. 源泉所得税、住民税和社会保险负债与尚未缴纳清单一致。
4. 银行明细没有重复形成工资费用。

## 20. 月次处理（月次処理）

### 日文说明

月次処理では、同期・記帳・残高照合・債権債務確認・月次推移確認を一つの流れとして実行し、翌月へ未処理事項を持ち越さない。

### 中文解释

月次处理不是只看利润表，而是把流水、证憑、应收应付、工资税金、资产负债表一起核对。目标是使每个余额都能说明来源。

### 月次标准步骤

1. 确认画面右上角事业所和事业年度正确。
2. 同步全部银行和信用卡账户，记录同步异常。
3. 在 `会計帳簿` → `明細の一覧` 检查本月未登録明細。
4. 在 `自動で経理` 完成普通交易、未決済消込和口座振替。
5. 检查本月销售是否全部登记，并核对売掛金。
6. 检查本月费用和资产取得是否全部登记，并核对未払金、買掛金。
7. 检查个人垫付款和公司返还款，核对役員借入金。
8. 检查工资交易、实际发薪、源泉所得税、住民税和社会保险。
9. 检查本月新增固定资产候选，完成取得交易与台账登记。
10. 确认请求书、收据、合同等证憑已关联或按规定保存。
11. 将普通预金、信用卡、现金等账面余额与外部资料核对。
12. 打开 `会計帳簿` → `損益計算書（月次）`，检查销售、主要费用和利润的月度异常。
13. 打开 `会計帳簿` → `貸借対照表（月次）`，检查现金负数、异常仮払金、长期未清应收应付等。
14. 对异常交易下钻至仕訳和证憑，按有依据的方法修正。
15. 完成 [月次检查表](../checklist/月次检查表.md)，保存未解决事项和处理期限。

### 月次验收标准

1. 目标月份银行和信用卡已同步至月末。
2. 未登録明細为零，或有逐笔书面理由。
3. 银行、信用卡和现金账面余额可与外部资料核对。
4. 売掛金、未払金和役員借入金均有明细依据。
5. PL、BS 的异常波动已经确认原因。

## 21. 年次处理（年次処理）

### 日文说明

年次処理では、1年間の記帳を完了し、実在性・期間帰属・税区分を確認した上で、決算整理仕訳、決算書作成、申告、納付、年度締めを行う。

### 中文解释

年度处理建立在 12 个月月结基础上。先完成记账和余额确认，再做决算调整、决算书、税务申告和缴税，最后才执行 `年度締め`。

### 年次标准步骤

1. 确认全年银行、信用卡和现金交易已登记。
2. 确认所有账户期末余额与银行、信用卡账单和现金盘点一致。
3. 核对売掛金、買掛金、未払金、前受金、前払費用、未収入金和仮勘定明细。
4. 核对库存和在制品；没有库存业务时也记录不适用判断。
5. 核对役員借入金、役員貸付金及关联当事人交易。
6. 核对固定资产台账、取得、处置和减価償却。
7. 检查费用和收入的期间归属，登记前払、未払、前受和未収等调整。
8. 检查消费税税区分、課税方式和申告计算，确认未払消費税等。
9. 检查法人税等税前利润与申告调整所需资料。
10. 打开 `決算申告` → `決算書`，预览貸借対照表、損益計算書等文件。
11. 检查决算书科目显示、负数、异常余额和前期比较。
12. 按 [eTax手册](../tax/eTax手册.md)、[eLTAX手册](../tax/eLTAX手册.md) 和 [消費税手册](../tax/消費税手册.md) 完成申告准备。
13. 完成税务申告和缴纳后，保存申告书、接收通知和纳付记录。
14. 完成 [年次检查表](../checklist/年次检查表.md)。
15. 所有账务与决算书确定后，才执行年度締め。

## 22. 决算准备（決算準備）

### 日文说明

決算準備は、決算書を出力する前に帳簿残高の正確性と根拠資料を確認する工程である。年度締めはこの工程の最後に行う。

### 中文解释

决算准备的核心不是“点击生成决算书”，而是证明资产真实存在、负债没有遗漏、收入费用属于正确年度、税区分合理，并把所有判断资料留存。

### 决算资料准备步骤

1. 固定事业年度末日，列出银行、信用卡、売掛金、未払金、税金和固定资产所需资料。
2. 下载或保存所有银行期末残高资料和信用卡期末账单。
3. 整理期末未收请求书、期末未付请求书和期后收付款记录。
4. 整理役員借入金垫付明细及返还记录。
5. 整理固定资产请求书、使用开始日、处置资料和台账。
6. 整理工资台账、源泉所得税、住民税和社会保险缴纳资料。
7. 整理消费税计算依据、适格请求书判断和中间纳付记录。
8. 打开 `会計帳簿` → `残高試算表`、`貸借対照表`、`損益計算書`，逐科目检查异常。
9. 对每个重要 BS 科目建立“账面余额—外部资料—差异说明”的核对关系。
10. 使用 `振替伝票` 登记经确认的决算整理仕訳，并逐笔复核。
11. 打开 `決算申告` → `決算書` → `プレビュー・調整`，检查决算书显示。
12. 输出或保存确定版 BS、PL、株主資本等変動計算書和個別注記表。

### 年度締め操作步骤

1. 确认决算书、会计账簿和申告使用数据已最终确定。
2. 确认画面右上角显示的是需要关闭的事业年度。
3. 打开 `決算申告` → `年度締め`。
4. 阅读画面提示，确认年度締め会锁定该年度并结转余额。
5. 点击 `年度締めをする`。
6. 切换至下一事业年度，确认期首余额已正确结转。
7. 发现前期需要修正时，按 freee 的解除年度締め流程和公司审批规则处理，并重新核对报表与申告影响。

### 决算验收标准

1. 所有重要资产和负债都有外部资料或明细依据。
2. 期末银行余额、売掛金、未払金、固定资产和税金余额已核对。
3. 决算整理仕訳有计算依据和复核记录。
4. 决算书与申告基础数据一致。
5. 年度締め在决算数据确定后执行，下一年度期首余额正确。

## 23. 常见错误与修正原则

| 常见错误 | 正确判断 |
| --- | --- |
| 银行同步后认为记账已完成 | 进入 `自動で経理` 完成登记、消込或口座振替 |
| 公司账户互转登记成收入和费用 | 使用 `口座振替` |
| 売掛金到账时再次登记销售 | 对原未決済收入执行消込 |
| 未払金付款时再次登记费用 | 对原未決済支出执行消込 |
| 个人垫付直接从公司银行登记 | 使用 `役員資金` 登记原费用，返还时冲减役員借入金 |
| 只登记固定资产付款 | 同时登记取得交易和固定资产台账 |
| 工资银行明细再次登记工资费用 | 消込人事劳务联动的未決済給与取引 |
| 未确认系统调整又手工计提消费税 | 先检查决算书调整和既有仕訳，避免重复 |
| 为清空未登録明細而随意改日期或科目 | 保留真实日期，按证憑和业务实质处理 |
| 决算书未确定就执行年度締め | 完成余额核对、决算调整和决算书确认后再关闭年度 |
