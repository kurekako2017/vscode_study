# freee 勘定科目辞典

## 1. 使用原则

本辞典用于一人 IT・SES 法人选择常用勘定科目。科目只是分类结果，最终必须结合交易实际、合同、支付方式和税区分判断。

### 判断顺序

1. 这笔交易属于公司还是私人。
2. 交易发生在哪个期间。
3. 购买了什么或提供了什么服务。
4. 使用什么支付方式。
5. 是否含消费税及适用税率。
6. 是否构成固定资产或需要期间分配。

## 2. 资产科目

### 現金 / 现金

用途：公司实际持有的纸币和硬币。

禁止：把信用卡、银行支付或个人垫付误记为现金。

### 普通預金 / 普通存款

用途：法人银行账户余额。

每月必须与银行真实余额核对。

### 売掛金 / 应收账款

用途：销售已经发生但尚未收到的款项。

```text
销售发生：借方 売掛金 / 贷方 売上高
收到款项：借方 普通預金 / 贷方 売掛金
```

### 前払費用 / 预付费用

用途：已经支付但属于未来期间的费用。

年度订阅、保险等是否需要期间分配，应根据金额和实际期间判断。

### 工具器具備品・ソフトウェア

用途：符合资产计上条件的设备或软件。

PC、显示器、服务器、购买型软件等需要结合取得价额、使用期间和税务特例判断，不能仅按商品名称决定。

## 3. 负债科目

### 未払金 / 应付款

用途：费用已经发生但尚未支付的款项。

```text
费用发生：借方 通信費等 / 贷方 未払金
实际支付：借方 未払金 / 贷方 普通預金等
```

### 役員借入金 / 董事借款

用途：代表者用个人资金垫付公司费用，公司以后应归还代表者。

```text
个人垫付：借方 费用科目 / 贷方 役員借入金
公司精算：借方 役員借入金 / 贷方 普通預金
```

注意：役員貸付金表示公司借钱给役员，方向相反。

### 預り金 / 代收款

用途：工资中代扣的源泉所得税、住民税、社会保险个人负担等。

必须与工资台账及缴纳记录核对。

### 未払消費税 / 应付消费税

用途：管理确定申告中的消费税及地方消费税缴纳预定额。

具体决算仕訳取决于税込、税抜经理和中间缴纳，不可机械登记。

### 未払法人税等

用途：管理法人税、住民税等确定后的未缴税额。

应与法人税申告书和别表五（二）核对。

## 4. 收益科目

### 売上高 / 销售收入

用途：SES服务、开发、顾问、网站制作和其他主营业务收入。

根据服务完成、验收和合同条件确认计上期间，不能只按收款日判断。

### 雑収入 / 杂项收入

用途：不属于主营销售的零星收入。

不能为了方便把不明入金全部记入杂收入，应先确认来源。

## 5. 费用科目

| 内容示例 | 常用候选科目 | 确认重点 |
| --- | --- | --- |
| AWS、云服务 | 通信費 | 服务内容、期间、消费税 |
| OpenAI、AI SaaS | 通信費 | 业务用途、海外交易税区分 |
| Cursor、GitHub | 通信費 | 订阅期间、业务用途 |
| 手机、网络 | 通信費 | 公司与私人使用比例 |
| 电车、出差 | 旅費交通費 | 日期、目的、路线 |
| 办公用品 | 消耗品費 | 金额、用途、资产判定 |
| 外包开发 | 外注費 | 合同、请求书、源泉征收要否 |
| 银行手续费 | 支払手数料 | 银行明细和税区分 |
| 客户接待 | 交際費 | 对方、人数、目的、证憑 |
| 书籍、资料 | 新聞図書費 | 业务关联性 |
| 税理士等报酬 | 支払報酬料 | 源泉征收和请求书 |

上述只是候选，不能代替个别判断。

## 6. 重点科目

### 通信費

用于网络、电话、云服务和持续性在线服务等。海外 SaaS 还需要单独确认消费税税区分。

### 消耗品費

用于办公用品和符合费用处理条件的低额物品。金额较高或长期使用时应确认固定资产处理。

### 旅費交通費

保存日期、移动区间、出差目的和对象项目。

### 外注費

保存合同、验收资料、请求书和转账记录。还要确认付款对象和业务内容是否涉及源泉征收。

### 交際費

保存发票、参加者、关系和业务目的。不能混入私人饮食。

### 雑費

只用于无法合理归入其他科目的零星费用。长期大量使用会降低账簿可读性。

## 7. 常见修正

### 负现金

1. 打开 `総勘定元帳`。
2. 选择 `現金`。
3. 查找并核对实际支付方式。
4. 个人垫付改为役員借入金。
5. 银行或信用卡支付改为正确口座。
6. 重新确认现金余额。

### 二重登记

1. 比较银行同步交易和手工登记交易。
2. 确认是否为同一证憑、同一金额和日期。
3. 保留正确的原始交易。
4. 删除或修正前必须确认不会破坏消込关系。

### 科目与支付方式混淆

勘定科目回答“买了什么”，决济口座回答“怎么支付”。例如 OpenAI 可以是通信费，但个人垫付时支付口座是役員借入金。

- [ ] 未払金有请求书和支付计划
- [ ] 役員借入金有垫付记录
- [ ] 预り金与工资资料一致
- [ ] 未払税金与申告、缴纳资料一致
- [ ] 杂费和交际费已复核
