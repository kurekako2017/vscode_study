# AI 编程工具节流与低成本开发方案（2026）

## 一、背景

目前：

- GitHub Copilot 免费额度开始限制
- ChatGPT Codex / GPT-5 Agent 消耗额度很快
- Cursor 容易扫描整个项目
- Claude Agent 类工具 token 消耗巨大

因此很多开发者开始采用：

```text
本地模型 + 开源工具 + 少量 API
```

来降低成本。

尤其适合：

- VSCode
- WSL
- Python / TypeScript 开发
- 文档编写
- 日本现场开发

---

# 二、推荐整体方案

|---|---|
| 自动补全 | TabbyML |

| 小型聊天 | Qwen2.5-Coder |
| 高级推理 | ChatGPT / Claude |
| Agent | Codex / OpenHands |

# 三、Continue.dev（最推荐）

GitHub：
https://github.com/continuedev/continue

用途：

- 修改代码
- 修改 Markdown
- 生成 README
- 修改设计书
- 重构
- 分析项目
- 只读取指定文件

优点：

- VSCode 集成优秀
- 支持 Ollama
- 支持 OpenRouter
- 支持 Claude / GPT
- 可以限制上下文
- 能锁定文件
- 非常适合节流

---

## Continue 安装步骤

### 1. 安装 Ollama

官网：
https://ollama.com

安装完成后：

```bash
ollama --version
```

### 2. 下载模型

```bash
ollama pull qwen2.5-coder:1.5b
ollama pull qwen2.5-coder:7b
```

推荐：

| 模型 | 用途 |
|---|---|
| 1.5b | 自动补全 |
| 7b | Chat / 文档修改 |

# 四、Continue 常用技巧

## Edit 模式

快捷键：

```text
Ctrl + I
```

功能：

- 选中文本
- 输入要求
- AI 直接修改

例如：

```text
改成日本IT现场设计书风格
```

---

## 锁定文件

```text
@file auth.ts

只修改这个文件
```

---

## 控制上下文

```text
只根据当前文件回答
不要扫描整个项目
```

# 五、TabbyML（本地 Copilot）

GitHub：
https://github.com/TabbyML/tabby

用途：

- 自动补全
- 本地运行
- 零 token 消耗

优点：

- 完全免费
- 响应快
- 不消耗 GPT 额度

# 六、Aider（CLI 神器）

GitHub：
https://github.com/Aider-AI/aider

特点：

- CLI 模式
- git diff 非常强
- token 控制优秀
- 只上传相关文件

适合：

- WSL
- terminal 开发
- Python 开发

示例：

```bash
aider app.py
```

# 九、真正节流的方法（重点）

## 1. 减少上下文

错误方式：

```text
扫描整个项目
```

正确方式：

```text
只读取当前文件
只读取相关代码
```

---

## 2. 本地自动补全

建议：

```text
TabbyML
+
本地 Qwen
```

---

## 3. Agent 少用

最烧 token：

- 自动遍历项目
- 自动修 bug
- 自动规划
- 多文件分析
