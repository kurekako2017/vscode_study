## 🔍 窗口和显示操作

### 放大和缩小（看不清时用）

#### 快速放大/缩小文本
```
Ctrl + +          ← 放大文本（看不清时用）
Ctrl + -          ← 缩小文本（屏幕太小时用）
Ctrl + 0          ← 恢复默认大小

实用场景：
  • 用投影仪演示代码：Ctrl + + （多按几次）
  • 笔记本屏幕小，内容看不全：Ctrl + - （缩小看更多）
  • 老花眼不适应字体大小：Ctrl + + （调舒服为止）
  • 回到舒适大小：Ctrl + 0
```

#### 永久改变界面缩放
```
方法：菜单设置（永久改，不用每次都按）
  Ctrl + ,               ← 打开设置
  搜索：zoom             ← 输入 zoom
  修改 "Window: Zoom Level"  ← 找到这个选项
  输入数值（如 1.2 = 120%）

数值参考：
  -20 = 80%  （缩小，看更多内容）
  0   = 100% （默认）
  10  = 110% （略微放大）
  20  = 120% （明显放大，投影仪用）
```

```
## 🐍 Python 开发快捷键与导航技巧

下面是面向 Python 开发的实用技巧与快捷键，帮助你在函数/方法间快速跳转、调试并进入/退出函数体检查行为。

- **选择 Python 解释器**：打开命令面板 `Ctrl + Shift + P`，运行 `Python: Select Interpreter`，选择项目虚拟环境（.venv）。
- **在终端运行当前文件**：`Ctrl + F5`（不调试），或右键文件选择 `Run Python File in Terminal`。
- **运行选中代码 / 交互执行**：选中代码后按 `Shift + Enter`（需要 Python 扩展，发送到交互式终端或 REPL）。
- **格式化代码**：`Shift + Alt + F`（或保存时自动格式化，设置 `editor.formatOnSave`）。推荐使用 `black`。
- **静态检查与智能提示**：安装并启用 `Pylance`、`flake8` 或 `pylint` 来获得快速错误提示与重构建议。

### 在文件/项目内快速跳来跳去

- **跳到定义（进入函数实现）**：`F12`。光标放在函数名或调用处按 `F12` 会打开目标定义。
- **Peek（内联预览定义）**：`Alt + F12`，在当前窗口弹出小窗查看定义，不离开当前位置。
- **在侧边打开定义**：`Ctrl + K` 然后按 `F12`（顺序按下）会在侧边打开定义，便于对比查看。
- **查找实现（接口到实现）**：`Ctrl + F12`（Go to Implementation）。
- **查找引用**：`Shift + F12`（显示所有引用，便于查看函数被谁调用）。
- **回到上一个光标位置**：`Alt + ←` 或 `Ctrl + -`（导航后退），用来回到调用处。
- **前进（回到之后的位置）**：`Alt + →` 或 `Ctrl + Shift + -`。
- **文件符号导航（方法列表）**：`Ctrl + Shift + O` 打开当前文件的符号列表（可快速跳到某个函数/类）。
- **全局符号（项目方法搜索）**：`Ctrl + T` 或 `Ctrl + P` 然后输入 `@`/`#`（在不同平台上可用 `Ctrl+T` 搜索整个工作区的符号）。
- **Breadcrumbs（面包屑）显示当前所在类/函数路径**：在编辑器顶部启用面包屑（View → Toggle Breadcrumbs），点击面包屑可以快速导航到同一文件的其他方法/类。

### 调试时进入/退出函数（Step In / Out）

- **设置/取消断点**：点击行号或按 `F9` 切换断点。
- **开始调试**：按 `F5`（启动带调试的运行）。
- **继续 / 恢复执行**：按 `F5`（调试时）或面板中的继续按钮。
- **单步执行（Step Over）**：`F10`，执行当前行，不进入函数内部。
- **进入函数（Step Into）**：`F11`，会进入函数体内部，逐步调试函数内部语句。
- **退出当前函数（Step Out）**：`Shift + F11`，运行到当前函数返回处并暂停。
- **查看调用栈 & 变量**：在调试视图（左侧的 Run and Debug）查看 Call Stack、变量和表达式监视。

### 其他实用技巧

- **快速修复 / 轻量提示**：`Ctrl + .` 在光标位置弹出修复建议（导入、重命名、快速修复）。
- **重命名符号（全项目重命名）**：`F2`（安全重命名变量/函数/类并更新所有引用）。
- **Peek Call Hierarchy / Call Stack**：右键函数名选择 `Peek` → `Peek Call Hierarchy`（查看谁调用了它），或在调试时查看 Call Stack 面板。
- **在编辑器中运行测试**：安装 Python 测试适配器（pytest/unittest），使用测试侧栏运行/调试单元测试。
- **使用虚拟环境（推荐）**：在项目根创建 `.venv`，并在 `settings.json` 或命令面板中选择该解释器，保证依赖在项目隔离环境内。
```

```json
// 打开设置JSON编辑器：Ctrl + Shift + P → "Preferences: Open Settings (JSON)"

{
  // 编辑器基础设置
```

## 🆘 常见问题

### Q1：我找不到我要的文件

**A：** 使用 Ctrl + P 快速打开
- 输入文件名的一部分
- VS Code 会模糊匹配
- 例如输入 "case" 会找到 "case-1-connection.js"

### Q2：我想取消我的修改

**A：** 多个办法
```
1. 单个文件：打开文件 → 右键 → "Revert File"
2. 所有修改：Ctrl + Shift + G → 点击 "Discard All Changes"
3. 撤销最后的修改：Ctrl + Z
```

### Q3：我不小心关闭了文件，怎么找回？

**A：** Ctrl + Shift + T 重新打开最后关闭的标签页

### Q4：终端里的命令输出看不清楚

**A：** 
- 增大终端字体：设置 → Terminal Font Size
- 或按 Ctrl + 鼠标滚轮 调整大小

### Q5：我想让代码自动对齐

**A：** 
```
Ctrl + Shift + I  格式化整个文件
或右键 → "格式化文档"
```

### Q6：怎么找到所有错误？

**A：** Ctrl + Shift + M 打开"问题"面板

### Q7：代码自动补全不工作？

**A：** 
```
检查：
1. 是否装了对应语言的扩展（如 ABAP 支持）
2. Ctrl + Shift + P → "Reload Window" 重新加载
3. 检查设置中的 "autocomplete" 是否启用
```
