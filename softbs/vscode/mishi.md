・移行の流れを説明してください 
設計 → データ準備 → 移行ツール設定 → 実行 → エラー対応 → 検証 
・ IDoc：データを渡す
・ RFC：遠隔で呼び出す
・ BAPI：業務処理を実行する
・ 私の理解では、IDoc、RFC、BAPI はそれぞれ役割が異なります。
・IDoc は、SAP がシステム間で業務データを受け渡すために使う標準フォーマットです。BAPI は SAP が標準提供する業務 API です。正式な業務処理を実行するための入口として使われます。
・ どんな BAPI を使いましたか
販売関連の参照や登録系の BAPI を確認・利用する案件に携わっていました。
・ BDC と何が違いますか
API は標準 API を使う方式で、BDC は画面操作を再現する方式なので、
 一般的には BAPI のほうが保守しやすいと理解しています。


・ 出错时怎么查
実際の現場では、まず設定を確認した上で、
 WE02 / WE05 で状態を見ながら、必要に応じて BD87 や WE19 を使って
 調査・再処理を進めるイメージです。
