# 用 GitHub + AI 打造终极个人管理系统

## 核心理念：输入极简，输出丰富

我们旨在构建一个集任务管理（ToDo）、微型日记和个人知识库于一体的系统。它遵循两个核心原则：

1.  **输入要极快**：无论是任务、灵感还是吐槽，都能在几秒钟内（通过语音或文字）完成记录，最大程度降低心理负担。
2.  **输出要可积累**：所有零散的记录都能被长期、安全地保存、搜索和回顾。通过 AI 定期提炼，将信息转化为结构化的知识。

**最终实现一处管理三件事：**

*   **任务管理 (ToDo)**：快速创建、跟踪和回顾。
*   **微型日记 (Micro-Journaling)**：记录日常想法、情绪和事件。
*   **个人知识库 (Personal Knowledge Management)**：将零散的记录沉淀为系统化的知识。

---

## 整体工作流程

这是一个高度自动化的信息流闭环：

1.  📱 **输入**：通过手机快捷指令（或任何能调用 API 的工具），将任务或日志一键发送到 GitHub Issues。
2.  📋 **整理**：在 GitHub Projects 看板上，通过拖拽对任务进行优先级排序和状态管理。
3.  ⚙️ **归档**：GitHub Actions 在每天深夜自动运行，将当天的所有“日志”类评论抓取出来，汇总成一个独立的 Markdown 日记文件。
4.  🧠 **提炼**：每月或每季度，使用大型语言模型（如 Claude、ChatGPT 等）读取指定时间范围内的日记和 Issue，提炼、总结并生成结构化的知识笔记。
5.  🔄 **回顾**：在年底，回顾全年的知识笔记和数据，进行复盘，并设定下一年的目标。

## 知识分类体系示例

在开始搭建系统之前，一个清晰的分类结构能让你的知识库事半功倍。以下是参照您提供的内容，为您设计的分类体系，您可以在此基础上进行调整：

### 1. 个人成长 (Personal Growth)
这个分类关注您自身的持续发展和内在状态。
- **🎯 目标 (Goals)**: 长期和短期的个人或职业目标。
- **💪 习惯 (Habits)**: 希望养成或戒除的日常习惯追踪。
- **🏋️ 训练 (Training)**: 记录身体训练、健身或任何技能的练习情况 (如 `vo2max.md`)。
- **📔 日记 (Diary)**: 每日的思考、感悟和情绪记录。

### 2. 知识学习 (Knowledge & Learning)
所有外部知识的内化和记录。
- **💡 TIL (Today I Learned)**: 每天学到的小知识点，源自您的 "到" 分类。
- **🔧 技术笔记 (Tech Notes)**: 针对特定技术或工具的深入研究和最佳实践，例如 `claude-code.md`, `amical.md` 等。
- **📚 读书笔记 (Book Notes)**: 对阅读过的书籍进行摘要和思考。

### 3. 生活管理 (Life Management)
处理日常生活的具体事务。
- **✅ 任务管理 (Task Management)**: 具体的、需要执行的待办事项。
- **📱 设备列表 (Device List)**: 管理您的电子设备、软件或实体工具。
- **💳 每月付款 (Monthly Payments)**: 记录订阅服务和其他周期性支出。
- **💰 资产管理 (Asset Management)**: 跟踪个人财务和资产状况。

### 4. 项目与产出 (Projects & Outputs)
您创造和输出的成果。
- **📝 文章 (Articles)**: 您撰写的博客文章或长文。
- **🎤 幻灯片 (Slides)**: 制作的演讲或分享材料。
- **🗣️ 评论 (Comments)**: 在其他地方发表的有价值的评论或观点记录。

### 5. 媒体与资源 (Media & Resources)
收集的外部资料。
- **🖼️ 图片 (Images)**: 收集的有意义的图片、截图或图表。
- **🔗 链接 (Links)**: 待阅读或备忘的网址。

## 步骤 1：创建 GitHub “数字大脑”仓库

这是我们所有数据的大本营。

1.  **新建一个私有仓库**：
    *   登录 GitHub，点击右上角的 `+` 号，选择 `New repository`。
    *   **强烈建议设为私有 (Private)**，保护个人隐私。
    *   推荐命名：`my-life-os`, `second-brain`, `digital-garden`, `personal-os` 等。
    *   勾选 `Add a README file`。

2.  **创建两个核心的 Issue 模板**：
    在你的仓库中，依次点击 `Settings` -> `General` -> `Features` -> `Issues` -> `Set up templates`。你也可以手动在 `.github/ISSUE_TEMPLATE/` 目录下创建以下两个文件。

    *   **`task.md` (任务模板)**：用于创建结构化的待办事项。
    *   **`quick-log.md` (快速记录模板)**：用于记录日记、灵感、吐槽等。

3.  **创建一个“每日日志专用 Issue”**：
    *   在仓库的 `Issues` 页面，点击 `New issue`。
    *   选择 `快速记录` 模板。
    *   **标题**：`Daily Log - 所有日常记录`
    *   **内容**：可以写一句 "这里是我的日常点滴记录处"。
    *   提交后，**请务必记住这个 Issue 的编号** (比如 `#1`)，它将是所有日常记录的“垃圾桶”和“汇总点”。

## 步骤 2：实现「超低摩擦」的手机端输入

这是整个系统中最关键的一环，目标是让记录变得像呼吸一样自然。我们将使用 iPhone 的「快捷指令」App 作为示例。

首先，你需要一个 **Personal Access Token (PAT)**，让快捷指令有权限操作你的仓库。

*   **获取 PAT**：
    1.  访问 GitHub -> `Settings` -> `Developer settings` -> `Personal access tokens` -> `Tokens (classic)`。
    2.  点击 `Generate new token (classic)`。
    3.  **Note**：随便写，比如 `iPhone-Shortcuts`。
    4.  **Expiration**：建议选 `No expiration`（无过期时间）。
    5.  **Select scopes**：只勾选 `repo` 即可。
    6.  点击 `Generate token`，然后**立即复制并保存好这串字符**（`ghp_...`），它只会显示一次。

### 方式 A：快捷指令 - 快速创建新任务

这个指令用于快速创建一个独立的任务 Issue。

1.  打开 iPhone「快捷指令」App。
2.  点击右上角 `+` 新建一个指令。
3.  **添加操作**：
    *   `要求输入`：`问题` 设为 `新任务标题？`。
    *   `URL`：填入以下链接，并**替换 `[你的用户名]` 和 `[你的仓库名]`**。
        ```
        https://github.com/[你的用户名]/[你的仓库名]/issues/new?title=【提供的输入】&labels=task&template=task.md
        ```
    *   `在 Safari 中打开 URL`。
4.  给这个快捷指令命名，比如“新任务”，并添加到主屏幕。

点击它，输入任务标题，就会自动跳转到 GitHub 创建一个预设好模板的新任务页面。

### 方式 B：快捷指令 - 记录日常点滴（最推荐）

这个指令将所有日常的碎片化想法、日记、灵感都通过 API 发送到我们之前创建的 `Daily Log` Issue 下作为评论。这是实现“微型日记”的关键。

1.  打开「快捷指令」App，新建一个指令。
2.  **添加操作**：
    *   `听写文本` 或 `要求输入`：`问题` 设为 `想记录点什么？`。前者可以直接语音输入，更方便。
    *   `获取 URL 内容`：这是核心操作，点击展开进行详细设置：
        *   **URL**：`https://api.github.com/repos/[你的用户名]/[你的仓库名]/issues/[你的DailyLog编号]/comments` (记得替换所有中括号里的内容，包括 Issue 编号的 `#` 号要去掉)。
        *   **方式**：`POST`。
        *   **头**：点击 `添加新`，添加两个。
            1.  `键`: `Accept`, `值`: `application/vnd.github+json`
            2.  `键`: `Authorization`, `值`: `token [你的PAT令牌]` (注意 `token` 和令牌之间有个空格)。
        *   **请求正文**：
            *   `类型`: `JSON`
            *   点击 `添加新字段`：
                *   `键`: `body`
                *   `类型`: `文本`
                *   `值`: `【听写的文本】\n\n—— {{当前日期}} {{当前时间}}` (这里的 `【听写的文本】` 是一个魔法变量，代表上一步的输入内容)
    *   （可选）`播放声音` 或 `显示通知`：`内容` 设为 `已记录！`，给自己一个正反馈。
3.  给快捷指令命名，比如“记一下”，并添加到主屏幕。

现在，你可以通过点击图标或Siri，用语音或打字快速记录任何事情，它们都会被悄无声息地存入你的 `Daily Log` Issue。

## 步骤 5：用 AI 做知识提炼与沉淀

这是将信息升华为知识的关键一步。当你的 `logs/` 目录下积累了足够多的日记文件后，就可以让 AI 帮你总结了。

**推荐工具**：
*   **Claude.ai**：可以直接上传多个 `.md` 文件进行分析。
*   **Cursor / VS Code + Copilot**：可以直接在编辑器里打开整个项目，让 AI 读取和分析。

**核心 Prompt 模板示例**：

> 请阅读我过去一个月（或一个季度）的日记文件（`logs/*.md`）和仓库中相关的 Issue，帮我从以下三个维度，以第一人称的口吻，进行深入的总结与分析。我需要你像一位敏锐的观察者，发现我自己都可能忽略的模式、变化和关联。
> 
> **1. 技术/工作学习：**
> *   我主要在学习和使用哪些新技术？遇到了什么挑战，又是如何解决的？
> *   在工作方法或效率上，我有哪些新的尝试和改进？
> *   哪些话题是我反复提及和思考的？这反映了我的哪些核心关注点？
> 
> **2. 生活习惯与健康：**
> *   我的作息、饮食、运动习惯有哪些变化？这些变化对我的精力状态有什么影响？
> *   有哪些好习惯正在养成？有哪些坏习惯需要警惕？
> *   是否有反复出现的身体或情绪信号？
> 
> **3. 情绪、人际与自我认知：**
> *   这个月我的整体情绪状态是怎样的？哪些事情最能引发我的正面/负面情绪？
> *   在与人交往中，我有哪些新的感悟或困惑？
> *   通过这些记录，我对自己有了哪些新的认识？（例如，我的价值观、我的恐惧、我的渴望）
> 
> **输出要求：**
> *   使用 Markdown 格式，结构清晰。
> *   在每个维度的总结后，附上一个 `[洞察与行动]` 部分，提出 1-2 个具体的、可执行的建议。
> *   将总结报告存为 `knowledge/YYYY-MM-回顾.md` 格式。

## 最小可行起步计划 (MVP)

不要被以上所有步骤吓到，你可以从今天开始，用最小的成本启动这个系统：

1.  **今天**：新建一个 GitHub 仓库，创建一个 `Daily Log` Issue（记住编号）。
2.  **明天**：在手机上设置好“记一下”的快捷指令，然后开始连续记录 7-14 天。感受这种无压力的记录方式。
3.  **两周后**：如果你觉得这个方法对你有用，再回来设置 GitHub Actions 自动归档。
4.  **一个月后**：当你有了足够的数据，再尝试用 AI 做一次总结。

这个系统的美妙之处在于，你的数据永远属于你自己，它们以标准、开放的格式（Markdown 和 Git）存储，你可以随时迁移、搜索、回溯和进行二次开发。祝你玩得开心！
