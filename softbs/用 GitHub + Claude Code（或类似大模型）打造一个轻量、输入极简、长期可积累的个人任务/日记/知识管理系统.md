# 用 GitHub + Claude Code（或类似大模型）打造个人 AI 管理系统（易懂版）

> 这版只做一件事：让你 **今天就能跑通**。先用最小方案，再逐步扩展。

## 先看结论（30 秒）

你只需要 4 步：

1. 建一个私有仓库（例如 `life-manager`）
2. 建一个长期使用的 `Daily Log` Issue（记下编号）
3. 手机把文字/语音发到这个 Issue 评论
4. 每天由 GitHub Actions 自动归档成 `logs/YYYYMMDD.md`

跑通后，再让 AI（Claude Code / 其他模型）每周或每月总结到 `knowledge/`。

---

## 0. 这个系统到底在做什么

把「任务、日记、灵感」统一放在 GitHub，形成一个可追溯系统：

- **输入层**：手机快速记录（低摩擦）
- **存储层**：GitHub Issue + 评论（可检索、可回溯）
- **归档层**：Actions 定时转成 Markdown 文件
- **提炼层**：LLM 定期总结成结构化知识

一句话：**先记录，再归档，再提炼。**

## 1. 今天先跑“最小可用版”（推荐）

### Step 1：创建仓库与目录

新建私有仓库后，创建两个目录：

- `logs/`
- `knowledge/`

### Step 2：创建一个 Daily Log Issue

手动新建一个 Issue，标题例如：

- `Daily Log`

创建后记下编号（例如 `#12`），后面会用到。

### Step 3：手机记录（两种方式）

**方式 A（最简单）**：直接打开这个 Issue，手动评论。

**方式 B（更快）**：用快捷指令调用 GitHub API 发评论。

API 地址（把参数替换成你自己的）：

- `https://api.github.com/repos/<用户名>/<仓库>/issues/<DailyIssue编号>/comments`

请求头：

- `Accept: application/vnd.github+json`
- `Authorization: Bearer <你的PAT>`

Body 示例：

```json
{
  "body": "今天做了什么...\n\n记录时间: 2026-02-28 22:30"
}
```

### Step 4：每天自动归档为 Markdown

在仓库创建文件：`.github/workflows/daily-log-to-md.yml`

> 只需要改 1 处：把 `DAILY_ISSUE_NUMBER` 换成你的编号。

```yaml
name: Daily Log to Markdown

on:
  schedule:
    - cron: '0 15 * * *'
  workflow_dispatch:

permissions:
  contents: write
  issues: read

jobs:
  archive:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set env
        run: |
          echo "TODAY=$(date +'%Y%m%d')" >> $GITHUB_ENV
          echo "DAILY_ISSUE_NUMBER=12" >> $GITHUB_ENV

      - name: Fetch comments
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          gh issue view $DAILY_ISSUE_NUMBER --comments --json comments --jq '.comments[].body' > comments.txt

      - name: Archive to logs
        run: |
          if [ -s comments.txt ]; then
            mkdir -p logs
            {
              echo "# ${TODAY}"
              echo
              cat comments.txt
            } > logs/${TODAY}.md
          fi

      - name: Commit
        run: |
          git config user.name "github-actions"
          git config user.email "github-actions@github.com"
          git add logs/
          git commit -m "chore(logs): archive ${TODAY}" || echo "No changes"
          git push
```

完成后你会得到：

- `logs/20260228.md`
- `logs/20260301.md`
- ...

## 2. AI 如何参与（最实用）

每周/每月让模型读取 `logs/*.md`，输出到 `knowledge/`。

可直接用这个 Prompt：

```text
请读取最近一个月的 logs/*.md，输出一份 Markdown 回顾，包含：
1) 本月完成了什么（按任务/学习/生活分组）
2) 重复出现的问题（最多 5 条）
3) 下月最重要的 3 个行动（按优先级排序）
文件名：YYYY-MM-回顾.md
```

建议输出文件名：

- `knowledge/2026-02-回顾.md`

## 4. 常见问题（简版）

### Q1：为什么只用一个 Daily Issue？

因为输入最快，手机评论最省事；归档后一样能按日期管理。

### Q2：PAT 会不会不安全？

会，前提是你泄露。建议：

- PAT 只给最小权限
- 不写进仓库
- 放手机密码管理器

### Q3：Actions 没有生成 logs 文件？

按顺序检查：

1. `DAILY_ISSUE_NUMBER` 是否正确
2. 当天是否真的有新评论
3. Workflow 权限是否有 `contents: write`
4. 在 Actions 页面手动点一次 `Run workflow`

## 5. 一周执行计划（照做即可）

- Day 1：建仓库 + Daily Issue + 手动评论 3 条
- Day 2：接入快捷指令（或保持手动）
- Day 3：启用 `daily-log-to-md.yml`
- Day 4-6：每天至少记录 1 条
- Day 7：让 AI 生成第一份周总结到 `knowledge/`
