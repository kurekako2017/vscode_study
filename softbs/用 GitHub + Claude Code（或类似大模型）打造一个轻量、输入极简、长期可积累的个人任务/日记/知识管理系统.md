# 用 GitHub + Claude Code（或类似大模型）打造个人 AI 管理系统

## 概述
目标：用 GitHub（Issues + Projects + Actions）配合 Claude Code（或任意 LLM），搭建一个轻量、输入极简且可长期回溯的个人系统，管理三件事：任务、微日记、个人知识库。

设计原则：
- 输入要低摩擦：手机一键（语音或文本）即可记录；
- 数据要可积累：所有记录都存入 Git，可检索与版本回溯；
- 可用 AI 提炼：定期把零散记录交给 LLM 进行结构化整理。

## 整体流程（推荐闭环）
- 手机快捷指令（或语音）→ 提交到 GitHub Issue（新建或评论）
- 在 GitHub Projects（看板）整理、优先级管理
- 每日/定期由 GitHub Actions 将 Issue 评论归档为 `logs/*.md`
- 用 Claude Code 或其他 LLM 读取仓库并提炼为结构化知识，保存到 `knowledge/`
- 周期性回顾（每月/每年）并形成复盘文档

## 步骤 1：创建仓库与基础配置
1. 新建仓库（建议私有），例如 `life-manager` 或 `my-second-brain`。
2. 创建 Issue 模板（放在 `.github/ISSUE_TEMPLATE/`）：

**优先度**：高 / 中 / 低

(在此直接输入内容)

3. 手动创建一个专用的 `Daily Log` Issue（记下编号，例如 `#1`），作为所有日记/快速记录的集中位置（在该 Issue 下追加评论）。

## 步骤 2：手机端实现“超低摩擦输入”（关键）
方式 A：新任务（快速打开 Issue 新建页面）
- iPhone 快捷指令示例：
  1. 使用“询问文本”获取标题；
  2. 构造 URL：`https://github.com/<用户名>/<仓库>/issues/new?title=...&labels=task&template=task.md`；
  3. 在 Safari 中打开该 URL。

方式 B（推荐）：把所有日记/灵感丢进同一 `Daily Issue`（通过 API POST 评论）
- 生成 PAT（Personal Access Token）：GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)，勾选 `repo` 权限。
- 快捷指令示例（POST 评论）：
  - URL: `https://api.github.com/repos/<用户名>/<仓库>/issues/<DailyIssue编号>/comments`
  - Header:
    - `Accept: application/vnd.github+json`
    - `Authorization: Bearer <你的PAT>`
  - Body (JSON):

```json
{
  "body": "{{快捷指令输入文本}}\n\n(记录时间: {{当前时间}})"
}
```

- 可选：操作成功后播放提示音或显示确认。

## 步骤 3：用 GitHub Actions 每日把评论生成 md 文件
在 `.github/workflows/daily-log-to-md.yml` 中添加如下工作流：

```yaml
name: Daily Log → Markdown
on:
  schedule:
    - cron: '0 15 * * *'   # UTC 15:00 = JST 0:00
  workflow_dispatch:

jobs:
  archive:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4

      - name: Set today date
        run: echo "TODAY=$(date +'%Y%m%d')" >> $GITHUB_ENV

      - name: Get comments from daily issue
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          gh issue view 123 --comments --json comments --jq '.comments[].body' > comments.txt
          # 将 123 替换为你的 Daily Issue 编号

      - name: Create markdown file if has content
        run: |
          if [ -s comments.txt ]; then
            mkdir -p logs
            echo "# ${{ env.TODAY }}" > logs/${{ env.TODAY }}.md
            echo "" >> logs/${{ env.TODAY }}.md
            cat comments.txt >> logs/${{ env.TODAY }}.md
            git config user.name "GitHub Actions"
            git config user.email "actions@github.com"
            git add logs/
            git commit -m "Add daily log ${{ env.TODAY }}" || echo "No changes"
            git push
          fi
```

结果：按调度将当天评论合并为 `logs/YYYYMMDD.md`，便于后续检索与 LLM 处理。

## 步骤 4：自动生成重复/周期任务（可选）
示例（每周自动创建习惯任务）：

```yaml
name: Recurring Tasks
on:
  schedule:
    - cron: '0 0 * * 1'   # 每周一

jobs:
  create:
    runs-on: ubuntu-latest
    steps:
      - name: Create habit issue
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
        run: |
          WEEK=$(date +"%Y年第%V周")
          if ! gh issue list --label "habit" | grep -q "Gym $WEEK"; then
            gh issue create --title "Gym $WEEK" --label "habit" --body "- [ ] Mon\n- [ ] Wed\n- [ ] Fri"
          fi
```

## 步骤 5：用 Claude Code（或任意 LLM）把日志与 Issue 提炼成知识
- 用途：每月/季度把 `logs/*.md` 与相关 Issue 输入 LLM，提取“本期学到的事、需要改进的点、习惯变化”等结构化信息，保存到 `knowledge/`。
- 示例 Prompt（中文）：

```
请读取仓库 <github 链接> 中最近一个月的 logs/*.md 与相关 Issue，按以下三项分别总结：
1) 技术/工作学习：重要知识点、实践与反思
2) 生活习惯与健康：习惯改变与建议
3) 情绪、人际：重要事件与感受变化

以 Markdown 输出，包含要点、行动建议与优先级，文件名格式：YYYY-MM-回顾.md
```

- 若使用 Claude Code，可直接授权模型访问仓库并生成文件；若使用 API，则先 `git` 拉取并将文本传给模型，最后把结果提交回 `knowledge/`。

## 安全与注意事项
- PAT 请仅授予必要权限（`repo`），并存入手机的安全存储或 iOS 密码管理器；不要把密钥泄露或提交到仓库；
- Actions 使用 `${{ secrets.GITHUB_TOKEN }}` 时注意权限边界，若需要更高权限可把自定义 PAT 存入仓库 Secrets；
- 日志若包含敏感信息，请在提交前脱敏或将仓库设为 Private。

## 常见问题
- Q: 为什么使用同一 Issue 记录每日日志？
  - A: 单 Issue 评论便于手机端快速追加和聚合，后续由 Actions 按天归档为文件，同时保持输入低摩擦；
- Q: Claude Code 能否直接读取私有仓库？
  - A: 取决于工具支持和权限；常见做法是把仓库克隆到执行环境，再把文本交给模型；部分 LLM 平台支持仓库访问授权。
