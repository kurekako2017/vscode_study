# 典型项目结构与开发流程

## 1. 传统项目结构

在对日项目中，你可能看到以下几类结构。

### 传统 Eclipse Dynamic Web Project

```text
project/
|-- src/
|   `-- java/
|       `-- com/company/project/
|           |-- action/
|           |-- service/
|           |-- dao/
|           `-- entity/
|-- WebContent/
|   |-- WEB-INF/
|   |   |-- web.xml
|   |   `-- jsp/

`-- lib/
```

|   |-- main/
|   |   |-- java/
|   |   |-- resources/
|   |   `-- webapp/
|   `-- test/
|       `-- java/

## 2. 结合你给出的示意图来理解

一个典型流程如下:

1. 用户打开 `userlogin.jsp`
2. 表单提交到 `UserController` 或旧项目中的 `Action`
3. 控制层进行参数接收和基础校验
4. 调用 `UserService.userLogin()`
5. 业务层调用 `UserDao`
6. `UserDao` 访问数据库
7. 返回结果给页面或跳转到下一画面

这条链路对应的职责划分:

- `jsp/html`: 页面展示
- `controller/action`: 请求入口
- `service`: 业务判断
- `dao/repository`: 数据库访问
- `entity/model`: 对象封装

## 3. 典型文件说明

### 启动类

`DemoApplication.java`

作用:

- 启动 Spring Boot 应用
- 加载 Spring 容器

### 配置文件

- `application.properties`
- `application.yml`
- 老项目中也可能看到 `.xml` 配置

作用:

- 端口配置
- 数据库连接
- 日志设置
- 环境差异配置

### 页面文件

- `jsp`
- `html`
- `thymeleaf` 模板

### 测试目录

`src/test/java`

作用:

- 单元测试
- 集成测试

## 4. 实际开发流程

### 新功能开发

1. 阅读设计书
2. 确认输入输出
3. 确认 DB 表和接口
4. 修改页面
5. 修改 Controller
6. 修改 Service
7. 修改 DAO/SQL
8. 自测
9. 编写测试项结果

### 缺陷修复

1. 复现问题
2. 看日志
3. 查 SQL
4. 定位责任层
5. 修正代码
6. 回归验证
7. 记录影响范围

## 5. 初学者需要重点建立的习惯

- 看懂项目目录，不要只会写一个类
- 明确每层职责，不要把 SQL 写进 Controller
- 明确配置文件位置
- 学会跟踪请求链路
- 学会通过日志和断点定位问题
