# 学习路线

## 第一阶段: Java 入门与基础

目标是能独立写出控制台程序，并理解 Java 的核心语法。

重点内容:

- JDK 安装与环境变量配置
- `javac`、`java`、`jar` 的基本使用
- 变量、数据类型、运算符、流程控制
- 数组、字符串、方法
- 类与对象、封装、继承、多态
- 接口、抽象类、内部类
- 异常处理
- 集合框架
- IO 和文件操作
- 泛型、Lambda、Stream 基础

建议结果:

- 能写一个学生管理系统或图书管理系统的控制台程序。

## 第二阶段: Java 进阶与工程能力

目标是理解企业开发需要的代码组织方式和常见基础能力。

重点内容:

- 多线程与并发基础
- 日志框架使用
- 单元测试
- Maven 或 Gradle 构建
- Git 或 SVN 版本管理
- 常见代码规范与静态检查

建议结果:

- 能独立创建一个标准 Maven Java 项目并完成打包、运行、测试。

## 第三阶段: Java Web 开发

目标是进入企业项目主战场。

重点内容:

- HTTP 基础
- Servlet、JSP、Filter、Listener
- MVC 模式
- Tomcat 部署与调试
- Spring、Spring MVC、Spring Boot
- MyBatis、Hibernate、JPA
- 表单提交、参数校验、会话管理

建议结果:

- 能完成登录、列表、查询、增删改查等基础 Web 功能。

## 第四阶段: 数据库与业务系统开发

目标是具备真实项目的数据处理能力。

重点内容:

- SQL 基础与多表查询
- 事务
- 索引
- 存储过程基础
- JDBC
- ORM 框架
- 数据库连接工具使用

建议结果:

- 能看懂业务表结构并完成报表、查询、导入导出等功能。

## 第五阶段: 对日项目专项能力

目标是能适应日本项目现场的实际要求。

重点内容:

- 常见日语 IT 术语
- 设计书阅读
- 画面项定义理解
- 単体テスト、結合テスト、総合テスト
- 障害分析和修正流程
- 提问和汇报方式
- 老旧项目技术栈维护

建议结果:

- 能理解传统项目结构，按设计书编码，按测试项执行，按缺陷单修复。

## 推荐学习节奏

- 第 1 周到第 3 周: Java 基础
- 第 4 周到第 6 周: Java 进阶和工具链
- 第 7 周到第 10 周: Java Web 和 Spring
- 第 11 周到第 13 周: SQL 和数据库访问
- 第 14 周以后: 对日项目模拟练习

## 推荐练习项目

- 控制台版员工管理系统
- JSP/Servlet 版登录与用户管理系统
- Spring Boot 版用户管理系统
- 带数据库的订单管理系统
- 模拟对日项目的画面 CRUD 系统
