# Spring Boot 部署运维专题

Spring Boot 的学习不只停留在写接口，还要继续看打包、部署、监控、日志和环境管理。这个专题就是把“能跑”继续推进到“能稳定上线和维护”。

## 1. 这个专题是什么 / このテーマは何か

- 中文：Spring Boot 部署运维专题关注打包、环境配置、日志、健康检查、监控和上线方式。
- 日本語：Spring Boot のデプロイ運用テーマは、パッケージ化、環境設定、ログ、ヘルスチェック、監視、リリース方法を扱います。

## 2. 核心知识点 / 核心ポイント

| 中文概念 | 日本語概念 | 说明 |
|---|---|---|
| Build | ビルド | 生成可部署产物 |
| Jar / War | Jar / War | 常见打包格式 |
| Profile | プロファイル | 区分开发、测试、生产环境 |
| Actuator | Actuator | 健康检查、指标、运行状态 |
| Log | ログ | 运行日志与排障信息 |
| Docker | Docker | 容器化部署 |
| Nginx | Nginx | 反向代理与静态资源分发 |
| ECS / Fargate | ECS / Fargate | 云上容器运行环境 |

## 3. 典型部署链路 / 典型的なデプロイフロー

1. 本地开发完成后执行打包。
2. 将配置按环境区分开来。
3. 在测试环境进行启动和自测。
4. 通过 Docker 或服务器脚本发布。
5. 使用日志和 Actuator 观察运行状态。
6. 出现问题时回滚或切换版本。

日本語：
1. ローカル開発の後にビルドする。
2. 設定を環境ごとに分ける。
3. テスト環境で起動と自己確認を行う。
4. Docker またはサーバースクリプトで配布する。
5. ログと Actuator で稼働状態を確認する。
6. 問題があればロールバックまたはバージョン切り替えを行う。

## 4. 适合搭配什么 / どんな組み合わせに向くか

- 中文：Spring Boot + Docker + Nginx
- 日本語：Spring Boot + Docker + Nginx
- 中文：Spring Boot + ECS / Fargate
- 日本語：Spring Boot + ECS / Fargate
- 中文：Spring Boot + Actuator + Prometheus / Grafana
- 日本語：Spring Boot + Actuator + Prometheus / Grafana
- 中文：Spring Boot + GitHub Actions / CI CD
- 日本語：Spring Boot + GitHub Actions / CI CD

## 5. 开发重点 / 開発ポイント

- 中文：不同环境的配置不能混在一起。
- 日本語：環境ごとの設定を混在させない。
- 中文：日志要足够支持排障，但不要泄露敏感信息。
- 日本語：ログは十分に残しつつ、機密情報を出さない。
- 中文：健康检查要能真实反映服务状态。
- 日本語：ヘルスチェックは実際の稼働状態を反映する必要がある。
- 中文：部署步骤要能重复执行。
- 日本語：デプロイ手順は繰り返し実行できるようにする。

## 6. 常见坑 / よくある落とし穴

- 中文：只会本地启动，不会部署上线。
- 日本語：ローカル起動はできるが、本番へ出せない。
- 中文：配置文件混乱，开发和生产环境互相污染。
- 日本語：設定ファイルが混乱し、開発環境と本番環境が干渉する。
- 中文：没有监控和日志，出问题后很难定位。
- 日本語：監視やログがなく、障害時に原因特定できない。

## 7. 一句话总结 / 一言まとめ

- 中文：Spring Boot 部署运维专题关注的是“从能写出来到能稳定跑起来并持续维护”。
- 日本語：Spring Boot のデプロイ運用は、「書ける」から「安定稼働して継続運用できる」へ進めるためのテーマです。
