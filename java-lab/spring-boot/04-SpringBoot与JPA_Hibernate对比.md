# Spring Boot 与 JPA / Hibernate 对比

Spring Boot + JPA / Hibernate 适合 CRUD 较多、实体关系较清晰的系统。它和 MyBatis 的核心区别在于：一个更偏对象映射，一个更偏 SQL 可控。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Spring Boot 负责应用框架，JPA / Hibernate 负责对象与数据库表之间的映射。
- 日本語：Spring Boot がアプリ基盤を担当し、JPA / Hibernate がオブジェクトとテーブルのマッピングを担当します。

## 2. 核心知识点 / 核心ポイント

| 中文概念 | 日本語概念 | 说明 |
|---|---|---|
| Entity | エンティティ | 对应数据库表的对象 |
| ORM | ORM | 对象关系映射 |
| Repository | リポジトリ | 数据访问接口 |
| JPQL / HQL | JPQL / HQL | 面向对象的查询语言 |
| Lazy Loading | 遅延ロード | 延迟加载 |
| Cascade | カスケード | 级联操作 |
| FetchType | フェッチタイプ | 读取策略 |
| Dirty Checking | 変更検知 | 自动变更检测 |

## 3. 典型开发链路 / 典型的な開発フロー

1. 先定义 Entity 和关联关系。
2. Repository 提供基础增删改查。
3. Service 组织业务流程。
4. 必要时通过 JPQL、Specification 或原生 SQL 处理复杂查询。
5. 返回 DTO 给 Controller。

日本語：
1. まず Entity と関連を定義する。
2. Repository が基本 CRUD を提供する。
3. Service が業務フローを組み立てる。
4. 必要に応じて JPQL、Specification、ネイティブ SQL を使う。
5. Controller へ DTO を返す。

## 4. 和 MyBatis 的区别 / MyBatis との違い

| 维度 | JPA / Hibernate | MyBatis |
|---|---|---|
| 思路 | 面向对象映射 | 面向 SQL 映射 |
| SQL 可控性 | 中等 | 很高 |
| 开发速度 | 快 | 中等 |
| 复杂查询 | 需要额外处理 | 更直观 |
| 实体关系 | 更自然 | 更手工 |
| 日本企业常见度 | 中高 | 很高 |

## 5. 适合做什么 / どんな場面に向くか

- 中文：实体关系比较清晰的管理系统。
- 日本語：エンティティ関係が明確な管理システム。
- 中文：CRUD 多、业务规则相对标准化的项目。
- 日本語：CRUD が多く、業務ルールが標準化された案件。
- 中文：希望减少样板代码的项目。
- 日本語：ボイラープレートを減らしたい案件。

## 6. 常见坑 / よくある落とし穴

- 中文：实体关系设计不好，导致加载和查询复杂化。
- 日本語：Entity 設計が悪く、読み込みやクエリが複雑になる。
- 中文：懒加载在事务外使用时报错。
- 日本語：トランザクション外で遅延ロードを使ってエラーになる。
- 中文：自动生成 SQL 虽然方便，但复杂报表场景不一定合适。
- 日本語：自動生成 SQL は便利だが、複雑な帳票には向かないことがある。

## 7. 一句话总结 / 一言まとめ

- 中文：JPA / Hibernate 更适合实体关系清晰、CRUD 密集的系统；MyBatis 更适合 SQL 需要高度可控的系统。
- 日本語：JPA / Hibernate はエンティティ関係が明確で CRUD が多いシステムに向き、MyBatis は SQL を強く制御したいシステムに向いています。
