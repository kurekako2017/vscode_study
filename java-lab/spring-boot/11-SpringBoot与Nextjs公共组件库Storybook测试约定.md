# Spring Boot 与 Next.js 公共组件库 Storybook / 测试约定

这页把公共组件库如何写 Storybook、如何测、如何评审统一起来，让组件规范真正进入开发流程。

## 1. 这页的目标 / このページの目的

- 中文：把公共组件的“写法规范”落到“验证规范”。
- 日本語：共通コンポーネントの「書き方」を「検証のルール」まで落とし込むためのページです。

## 2. 组件开发流程 / コンポーネント開発フロー

```mermaid
flowchart LR
    D[设计组件 API] --> T[编写组件实现]
    T --> S[补充 Storybook]
    S --> U[添加单元测试]
    U --> V[可选视觉回归测试]
    V --> M[review & merge]
```

### 推荐顺序 / 推奨順序

1. 先设计 props 和状态边界。
2. 再写组件本体。
3. 然后补 Storybook。
4. 接着补单元测试。
5. 最后做视觉确认和代码评审。

## 3. Storybook 约定 / Storybook 規約

### 什么时候必须写 Storybook

- 中文：公共 UI 原子组件必须写 Storybook。
- 日本語：共通 UI 原子コンポーネントには必ず Storybook を書く。
- 中文：组合组件如果会被多个页面复用，也建议写 Storybook。
- 日本語：複数ページで再利用される複合コンポーネントにも Storybook を付けるとよい。
- 中文：只在单一页面使用、且业务变化很快的组件，可以先不写。
- 日本語：単一ページ限定で業務変化が激しい部品は、最初は省略してもよい。

### Storybook 里写什么

- 中文：展示默认态、禁用态、错误态、加载态和长文本态。
- 日本語：デフォルト、無効、エラー、ローディング、長文などの状態を表示する。
- 中文：把尺寸、变体、主题差异也拆成不同 stories。
- 日本語：サイズ、バリエーション、テーマ差分も別 story に分ける。
- 中文：故事名要一眼能看出使用场景。
- 日本語：story 名は用途が分かる名前にする。

### Storybook 命名建议

- 中文：`Primary`、`Disabled`、`WithError`、`Loading`、`LongText`。
- 日本語：`Primary`、`Disabled`、`WithError`、`Loading`、`LongText` のようにする。

## 4. 测试约定 / テスト規約

### 测什么

- 中文：测用户能看见的行为，不测实现细节。
- 日本語：実装詳細ではなく、ユーザーから見える振る舞いをテストする。
- 中文：测点击、输入、显示、禁用、回调触发、错误提示。
- 日本語：クリック、入力、表示、無効状態、コールバック、エラー表示を検証する。

### 不怎么测什么

- 中文：不直接测试内部 state 变量名。
- 日本語：内部 state 変数名を直接テストしない。
- 中文：不测试与业务无关的渲染细节。
- 日本語：業務に関係ない描画詳細はテストしない。

### 推荐测试层级

| 层级 | 中文职责 | 日本語の役割 |
|---|---|---|
| 单元测试 | 验证组件输入输出和交互 | 入出力と操作を検証する |
| 集成测试 | 验证多个组件协作 | 複数コンポーネントの連携を確認する |
| 视觉回归 | 验证样式是否意外变化 | 見た目の意図しない変化を確認する |

## 5. 常见测试场景 / よくあるテストケース

### Button

- 中文：点击后是否触发 `onClick`。
- 日本語：クリックで `onClick` が呼ばれるか。
- 中文：disabled 时是否无法点击。
- 日本語：disabled 時にクリックできないか。
- 中文：不同 variant 是否渲染正确样式。
- 日本語：variant ごとに正しい見た目になるか。

### Input

- 中文：输入值是否能正确同步。
- 日本語：入力値が正しく反映されるか。
- 中文：错误态是否显示提示文案。
- 日本語：エラー時に補助文言が表示されるか。
- 中文：前缀、后缀、清空按钮是否正常工作。
- 日本語：前後アイコンやクリアボタンが正しく動くか。

### Modal

- 中文：打开和关闭是否生效。
- 日本語：開閉が正しく動作するか。
- 中文：点击遮罩是否关闭。
- 日本語：オーバーレイクリックで閉じるか。
- 中文：Esc 键是否关闭。
- 日本語：Esc キーで閉じられるか。

## 6. 目录中的测试文件建议 / テストファイルの置き方

```text
button/
|-- button.tsx
|-- button.types.ts
|-- button.test.tsx
|-- button.stories.tsx
`-- index.ts
```

### 命名建议 / 命名の推奨

- 中文：单元测试文件和组件同名。
- 日本語：テストファイル名はコンポーネント名と揃える。
- 中文：Storybook 文件统一使用 `.stories.tsx`。
- 日本語：Storybook ファイルは `.stories.tsx` に統一する。
- 中文：如果组件有辅助函数，最好拆到 `*.utils.ts` 并单独测。
- 日本語：補助関数があるなら `*.utils.ts` に分けて単独でテストする。

## 7. 评审约定 / レビュー規約

- 中文：先看组件 API 是否足够小。
- 日本語：まずコンポーネント API が小さいか確認する。
- 中文：再看 Storybook 是否覆盖了主要状态。
- 日本語：次に Storybook が主要状態を覆っているか確認する。
- 中文：最后看测试是否覆盖了关键交互。
- 日本語：最後にテストが重要な操作を覆っているか確認する。
- 中文：如果没有 Storybook，也没有测试，说明这个公共组件还没准备好进入共享层。
- 日本語：Storybook もテストもないなら、その部品はまだ共通化の準備ができていない。

## 8. 工具选型建议 / ツール選定の目安

- 中文：Storybook 用来人工预览和协作沟通。
- 日本語：Storybook は目視確認とチーム共有に使う。
- 中文：单元测试用来防止组件行为回归。
- 日本語：単体テストは振る舞いの回帰防止に使う。
- 中文：视觉回归测试适合 UI 稳定性要求高的组件。
- 日本語：ビジュアル回帰テストは UI の安定性が重要な部品に向いている。

## 9. 一句话总结 / 一言まとめ

- 中文：这页的重点，是让公共组件库不仅“有目录”，而且“可验证、可回归、可协作”。
- 日本語：このページの重点は、共通コンポーネントを「あるだけ」ではなく、「検証できて、回帰を防げて、共同作業しやすい」状態にすることです。
