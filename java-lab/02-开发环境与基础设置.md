# 开发环境与基础设置

## 1. 目标

这一篇只解决一件事:

- 把 Java 开发最基础的环境搭起来
- 知道项目运行前要检查哪些设置

工具的详细展开放到 `03-对日项目常用工具总览.md`，这里不重复铺开。

## 2. JDK

### 作用

JDK 是 Java 开发工具包，包含:

- `java`
- `javac`
- `jar`
- `javadoc`

### 常见版本

- `JDK 8`: 传统对日项目最常见
- `JDK 11`: 长期支持版本
- `JDK 17`: 现代项目常见

### 必须会的基础操作

- 安装 JDK
- 配置 `JAVA_HOME`
- 配置 `Path`
- 用 `java -version` 确认版本
- 用 `javac -version` 确认编译器可用

## 3. Maven

Maven 是 Java 项目最常见的构建工具。

### 你至少要会

- 安装 Maven
- 配置 `MAVEN_HOME`
- 配置 `Path`
- 使用 `mvn -version`
- 理解 `pom.xml`

### 常用命令

- `mvn clean`
- `mvn compile`
- `mvn test`
- `mvn package`
- `mvn spring-boot:run`

## 4. Git

现在大多数项目都至少会用 Git。

### 基础设置

- 安装 Git
- 配置用户名和邮箱
- 学会 `clone`
- 学会 `pull`
- 学会 `add / commit / push`

### 常用确认命令

- `git --version`
- `git status`
- `git log --oneline`

## 5. IDE

### IntelliJ IDEA

现代 Java / Spring Boot 项目最常见。

至少要会:

- 打开 Maven 项目
- 配置 SDK
- 运行主类
- 看控制台日志
- Debug 调试

### Eclipse

传统对日项目里仍然很多。

至少要会:

- 导入 Existing Projects
- 配置 JDK
- 配置 Tomcat Server
- Debug 调试

## 6. 数据库客户端

开发环境通常还需要一个数据库连接工具。

常见选择:

- `DBeaver`
- `A5:SQL Mk-2`
- `SQL Developer`

至少要会:

- 建连接
- 跑 `SELECT`
- 跑 `INSERT / UPDATE / DELETE`
- 查看表结构

## 7. Tomcat

如果是传统 Java Web 项目，经常还需要 Tomcat。

### 你要理解

- Tomcat 是 Servlet/JSP 容器
- 项目可能以 `war` 方式部署到 Tomcat
- 传统项目常通过 IDE 里的 Server 视图启动 Tomcat

### 需要会的基础项

- 下载和解压 Tomcat
- 确认端口
- 启动和停止
- 看启动日志

## 8. 常见基础设置

### 编码

- 推荐统一使用 `UTF-8`
- 对日老项目有时会碰到 `Shift-JIS` 或 `Windows-31J`
- 如果出现乱码，优先检查:
  - 文件编码
  - 页面编码
  - Tomcat 编码
  - 数据库编码

### 换行与格式

- 不要随意全量格式化旧项目
- 先确认团队 formatter 规则

### 环境变量

常见要确认:

- `JAVA_HOME`
- `MAVEN_HOME`
- `Path`

### 代理与仓库

有些公司网络环境会涉及:

- Maven 私服
- 代理设置
- 证书问题

如果依赖下不下来，优先检查这些。

## 9. 环境区分

项目通常会区分:

- 开发环境
- 测试环境
- 预生产环境
- 生产环境

常见差异:

- 数据库连接不同
- 配置文件不同
- 日志级别不同
- 外部接口地址不同

## 10. 最小可用学习环境

如果你现在是自己学 Java，最小可用组合建议是:

- `JDK 17`
- `IntelliJ IDEA Community`
- `Maven`
- `Git`
- `MySQL` 或 `PostgreSQL`

如果你准备进入传统对日项目，再补这些:

- `Eclipse`
- `Tomcat`
- `SVN`
- `Oracle` 连接工具
- `A5:SQL Mk-2`

## 11. 启动项目前的检查清单

- JDK 版本是否正确
- Maven 是否可用
- Git 是否已拉到最新代码
- 数据库连接是否正常
- 配置文件是否指向正确环境
- 端口是否被占用
- 日志目录是否可写

把这几项养成习惯，能少掉很多低级环境问题。
