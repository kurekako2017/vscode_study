# 日本现场应用与案件关键词

## 1. 更常见的应用形态

当前日本现场更常见的生成 AI 应用，通常是：

- 社内问答
- 社内搜索
- `RAG` 知识库问答
- 摘要
- 翻译
- 邮件草拟
- 特定流程助手

而不是一开始就做强自治的复杂 Agent。

## 2. 常见案件关键词

- `Python`
- `RAG`
- `FastAPI`
- `LangChain`
- `LlamaIndex`
- `Azure OpenAI`
- `AWS Bedrock`
- `生成AIアプリ`
- `社内検索`
- `PoC`

## 3. 对应学习优先级

如果按日本案件来学，建议优先顺序是：

1. 模型调用
2. 结构化输出
3. `RAG`
4. `FastAPI`
5. PDF / 文档处理
6. 评估
7. Agent

## 4. 作品集建议

如果你要做作品集，最值得优先准备的是：

- 一个最小 `RAG CLI`
- 一个 `RAG API`
- 一份简化版需求概要
- 一份简化版基本设计
- 一份测试观点或测试用例表

## 5. 和 Agent 的关系

不是不学 Agent，而是：

- 先学 `LLM 应用开发`
- 再学 `Agent 开发`

这更贴近日本现场的实际需求。
