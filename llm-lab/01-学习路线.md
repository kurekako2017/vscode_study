# 学习路线

## 1. 总目标

如果目标是贴近日本 IT 现场与派遣案件，优先要建立的是 `LLM 应用开发` 能力，而不是一开始就冲复杂 Agent。

开始具体路线前，建议先看一遍：

- [LLM应用框架系统知识.md](./LLM应用框架系统知识.md)

这份文档会先解释：LLM 应用有哪些分层、角色是什么意思、RAG/Embedding/Prompt/Eval 分别负责什么。

这条路线的核心能力是：

- 模型调用
- 结构化输出
- `RAG / 社内検索`
- `FastAPI` 后端整合
- 评估与运维
- 基础前后端联动

这条路线的基本顺序是：

1. 主线先学 `Python`
2. 先做 `LLM + RAG + FastAPI`
3. 后面再补 `TypeScript`
4. 最后再进入 Agent 进阶

## 2. 先说结论

如果只记最重要的路线，可以记成下面这句：

- `Python -> LLM 应用 -> RAG -> FastAPI -> TypeScript -> Agent`

不要一开始并行硬学太多东西。

更现实的做法是：

- 先把 `Python` 学到能独立改 demo、写接口、处理文件
- 再做 2 到 4 个最小可运行项目
- 再补 `JavaScript / TypeScript`
- 再做带前端页面的 AI 应用

## 4. 8 周执行路线

这份版本比 12 周版更适合现在直接执行。

本周目标：

- schema
- `model_dump()`
- 字段校验

本周输出物：

- 跑通 `structured_output_demo`

### 第 3 周：文档处理与最小 RAG

- 关键词匹配

### 第 4 周：FastAPI 基础

本周目标：

- 能处理基本请求和响应

要学的内容：

- `FastAPI` 路由
- `GET / POST`
- 请求体
- 响应 JSON

### 第 5 周：RAG API 化

本周目标：

- 把问答能力包装成可调用服务

- 日志
- 异常处理

## 5. 教程与示例怎么搭配

建议不要只看教程，也不要只抄代码。

更好的节奏是：

1. 先看一小段教程
2. 跑一个最小示例
3. 自己改一版
4. 再进入下一个知识点

也就是说，教程和示例要交替进行。

推荐理解方式：

- 教程：负责建立概念
- 示例：负责建立手感
- 自改代码：负责真正学会

## 7. 学到什么程度再补 TypeScript

建议至少满足下面这些条件，再把 `TypeScript` 提到主线上来：

- 能读懂基础 `Python` 项目代码
- 能自己改 demo
- 能自己写最小接口
- 能处理 JSON 和文件
- 能看懂基础 `Pydantic`
- 能写简单 `FastAPI`

如果这些还没稳定，不建议把精力过早分散到前端。

## 8. 最重要的原则

- 先做日本现场更常见的应用，再学复杂 Agent。
- 先做 `RAG + FastAPI`，再做 Tool Calling。
- 先做可以交付的应用，再做炫技型自动化。
- 先把 `Python` 主线跑通，再补 `TypeScript`。
- 每一阶段都要有最小输出物，不要只停留在看教程。
