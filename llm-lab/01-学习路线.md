# 学习路线

## 1. 总目标

如果目标是贴近日本 IT 现场与派遣案件，优先要建立的是 `LLM 应用开发` 能力，而不是一开始就冲复杂 Agent。

这条路线的核心能力是：

- 模型调用
- 结构化输出
- `RAG / 社内検索`
- `FastAPI` 后端整合
- 云平台对接
- 评估与运维

## 2. 推荐阶段

### 阶段 1: 模型调用

- 学消息结构
- 学模型参数
- 学基本错误处理

输出物:

- `chat_cli`

### 阶段 2: 结构化输出

- 学 JSON 输出
- 学 schema 约束
- 学结果校验

输出物:

- `structured_output_demo`

### 阶段 3: RAG / 社内検索

- 学文档切分
- 学检索
- 学引用来源
- 学知识问答

输出物:

- `doc_qa_agent`

### 阶段 4: API 化与企业集成

- 用 `FastAPI` 包装能力
- 处理配置、异常、日志
- 让前端或内部系统可调用

输出物:

- `rag_api_demo`

### 阶段 5: 评估与运维

- 设计测试问题集
- 记录命中质量
- 监控成本与延迟
- 理解上线风险

输出物:

- `rag_eval_notes`
- `operations_notes`

### 阶段 6: Agent 进阶

- Tool Calling
- 单 Agent
- 多 Agent

输出物:

- 进入 [agent-lab/README.md](D:/dev/source_code/vscode_study/agent-lab/README.md)

## 3. 12 周建议节奏

### 第 1 到 2 周

- 模型调用
- Prompt 基础
- 命令行样例

### 第 3 到 4 周

- 结构化输出
- JSON / schema
- 分类与任务清单

### 第 5 到 6 周

- `RAG`
- 本地资料检索
- 带出处回答

### 第 7 到 8 周

- `FastAPI`
- `RAG API`
- PDF 资料支持

### 第 9 到 10 周

- 评估
- 质量验证
- 成本和延迟意识

### 第 11 到 12 周

- 整理作品集
- 写需求概要、基本设计、测试观点
- 再进入 Agent 进阶

## 4. 最重要的原则

- 先做日本现场更常见的应用，再学复杂 Agent。
- 先做 `RAG + FastAPI`，再做 Tool Calling。
- 先做可以交付的应用，再做炫技型自动化。
