# RAG

## 1. 为什么 `RAG` 是主线能力

如果按日本企业现场和派遣案件来看，`RAG` 往往比复杂 Agent 更优先。

因为企业最常见的落地方向是：

- 社内検索
- ナレッジ検索
- 规程问答
- 设计资料检索
- FAQ 问答

## 2. 这一阶段要掌握什么

- 文档切分
- 检索
- 引用来源
- 问答生成
- 检索质量意识

## 3. 推荐学习顺序

1. 先做最小本地检索
2. 再做带出处回答
3. 再做 API 化
4. 再考虑向量检索
