# 评估与运维

## 1. 为什么这一块不能省

很多生成 AI 样例“能跑”，但不代表“能上线”。

现场真正关心的是：

- 命中率如何
- 回答稳不稳
- 成本高不高
- 延迟能不能接受
- 出错时怎么处理

## 2. 这一阶段要掌握什么

- 测试问题集
- 命中质量观察
- 失败案例记录
- 成本意识
- 运行日志

## 3. 评估时常见关注点

- 是否引用正确来源
- 无命中时是否明确说明
- PDF 抽取质量是否稳定
- 关键词检索是否容易漏召回

## 4. 日本现场高频关键词

- `評価`
- `精度`
- `運用`
- `監視`
- `コスト`

## 5. 注意点

- 很多 PoC 死在“没有评估”。
- 如果想做作品集，最好至少有测试观点和简单测试用例表。
