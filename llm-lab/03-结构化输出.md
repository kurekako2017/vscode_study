# 结构化输出

## 1. 为什么这一步非常重要

在日本现场，很多生成 AI 应用真正需要的不是“自由聊天”，而是：

- 分类结果
- 任务清单
- 标签判断
- 固定 JSON
- 可供后端系统接收的结构化结果

所以结构化输出通常比 Agent 更早用到。

## 2. 这一阶段要掌握什么

- 固定 JSON 输出
- schema 约束
- 结果校验
- 失败重试

## 3. 典型场景

- 邮件分类
- 问题分类
- FAQ 路由
- 任务拆解
- 表单内容整理

- 这一步很适合和后端 API 结合。
