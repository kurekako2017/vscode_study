# 岗位与技能要求对照

## 1. 为什么要看这个

如果你的学习目标是：

- 找日本现场相关案件
- 对应派遣岗位要求
- 让学习内容和简历、面试对得上

那就不能只按技术主题学习，还要按“岗位需要什么能力”来整理。

## 2. 常见岗位类型

### 2.1 生成 AI 应用开发工程师

这是当前最贴 `llm-lab` 主线的岗位。

常见关键词：

- `Python`
- `RAG`
- `FastAPI`
- `Azure OpenAI`
- `AWS Bedrock`
- `生成AIアプリ`

优先要学的内容：

- 模型调用
- 结构化输出
- `RAG`
- `FastAPI`
- 企业集成

推荐优先案例：

- `structured_output_demo`
- `doc_qa_agent`
- `rag_api_demo`

### 2.2 社内搜索 / 知识库问答开发

这是日本现场非常常见的一类方向。

常见关键词：

- `社内検索`
- `ナレッジ検索`
- `FAQ`
- `RAG`
- `PDF`

优先要学的内容：

- 文档读取
- 文本切分
- 引用来源
- PDF 处理
- 检索质量意识

推荐优先案例：

- `doc_qa_agent`
- `rag_api_demo`

### 2.3 后端 API / 系统集成型岗位

这类岗位不一定直接写成“AI Engineer”，但会要求：

- 把大模型能力接进企业系统

常见关键词：

- `FastAPI`
- `API連携`
- `バックエンド`
- `認証`
- `運用`

优先要学的内容：

- API 设计
- 错误处理
- 文档重载
- 配置管理
- 云平台接入

推荐优先案例：

- `rag_api_demo`

### 2.4 Agent / 自动化进阶岗位

这是更偏后段的一类岗位，热度在上升，但不是最广泛的入口。

常见关键词：

- `Agent`
- `Tool Calling`
- `LangGraph`
- `Workflow`
- `Automation`

优先要学的内容：

- 在 `llm-lab` 主线打牢基础
- 再进入 `agent-lab`

更现实的理解是：

- 日本现场会谈 `AI Agent`
- 但很多岗位真正要的，仍然是“能做可控自动化”的人

也就是说，岗位里即使写了 `Agent`，很多时候也不是要你一上来就做高度自治系统。

更常见的实际要求仍然会落到：

- 结构化输出
- 工具调用
- 企业 API 对接
- 检索和知识库利用
- 流程编排

推荐先掌握的表述方式是：

- 先会做 `RAG + API + Tool Calling`
- 再把它包装成 Agent 场景

推荐优先案例：

- `tool_agent_demo`
- 后续的 `workflow_agent`

## 3. 不同岗位的学习优先级

### 如果你的目标是更容易拿到案件

优先顺序建议：

1. 生成 AI 应用开发
2. 社内搜索 / 知识库问答
3. 后端 API / 企业集成
4. Agent / 自动化进阶

如果岗位 JD 里同时出现了：

- `AI Agent`
- `RAG`
- `FastAPI`
- `社内検索`

通常更应该解读为：

- 先做企业可落地应用
- 再往自动化执行扩展

而不是：

- 一开始就做复杂多 Agent 系统

### 如果你的目标是技术进阶

优先顺序建议：

1. 模型调用
2. 结构化输出
3. `RAG`
4. `FastAPI`
5. 评估与运维
6. Agent

## 4. 简历和面试怎么对照

如果你要让学习成果更容易写进简历或项目经历，建议能对应出这些表达：

- `Python による生成AIアプリ試作`
- `RAG を用いた社内検索 PoC`
- `FastAPI による問い合わせ API 開発`
- `PDF 文書を対象としたナレッジ検索`
- `構造化出力を利用した分類機能`

如果你后面补了 Agent 方向，也更建议写成这种偏现场风格的表达：

- `構造化出力と Tool Calling を利用した業務支援エージェント試作`
- `社内ナレッジ検索と連携した問い合わせ支援エージェント`
- `社内 API 連携を前提とした業務補助 Agent PoC`

这些表达比单纯写：

- `Multi-Agent 開発`

更容易让人理解你做的东西和业务场景有什么关系。

## 6. 成本与注意点

- 不是每个岗位都会立刻要求复杂 Agent。
- 对日本现场来说，`RAG + Python + FastAPI` 往往比“多 Agent”更实用。
- 如果时间有限，先把：
  - `doc_qa_agent`
  - `rag_api_demo`
这两个打磨好，比平均铺开更有效。
