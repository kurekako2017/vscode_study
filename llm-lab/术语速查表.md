# LLM 应用术语速查表

这份表用于快速查名词。建议配合 [LLM应用框架系统知识.md](./LLM应用框架系统知识.md) 使用。

| 术语 | 中文理解 | 它的角色 | 主要作用 | 推荐学习 |
| --- | --- | --- | --- | --- |
| LLM | 大语言模型 | 生成能力核心 | 根据上下文生成回答、摘要、JSON、判断 | `02-模型调用基础.md` |
| Prompt | 提示词 / 指令 | 任务说明书 | 告诉模型做什么、按什么规则输出 | `02-模型调用基础.md` |
| System message | 系统消息 | 行为边界 | 规定模型身份、风格、安全边界 | `02-模型调用基础.md` |
| User message | 用户消息 | 用户输入 | 表示这次请求的实际问题 | `02-模型调用基础.md` |
| Assistant message | 助手消息 | 模型历史输出 | 在多轮对话里保留模型之前说过什么 | `02-模型调用基础.md` |
| Token | 文本计量单位 | 成本和长度单位 | 影响上下文长度、价格、速度 | `06-评估与运维.md` |
| Context Window | 上下文窗口 | 模型可见范围 | 决定一次请求最多能放多少内容 | `04-RAG.md` |
| Temperature | 随机性参数 | 输出风格控制 | 控制答案稳定或发散 | `02-模型调用基础.md` |
| Structured Output | 结构化输出 | 输出约束 | 让模型返回程序能解析的 JSON | `03-结构化输出.md` |
| Schema | 数据结构约定 | 输出格式说明 | 定义字段、类型、是否必填 | `03-结构化输出.md` |
| Pydantic | Python 校验库 | 数据模型层 | 定义模型、校验 JSON、转换数据 | `03-结构化输出.md` |
| Embedding | 文本向量 | 检索表示 | 把文本变成可计算相似度的向量 | `04-RAG.md` |
| Chunk | 文档片段 | 检索单位 | 把长文拆成小块，便于检索 | `04-RAG.md` |
| Retriever | 检索器 | 资料查找模块 | 根据问题找到相关 chunk | `04-RAG.md` |
| RAG | 检索增强生成 | 知识接入架构 | 先查资料，再让模型基于资料回答 | `04-RAG.md` |
| Citation | 引用来源 | 可信度证据 | 告诉用户答案依据哪段资料 | `04-RAG.md` |
| FastAPI | Python Web 框架 | 服务化入口 | 把模型能力封装成 HTTP API | `05-FastAPI与企业集成.md` |
| Eval | 评估 | 质量检查 | 判断答案准确性、引用、稳定性 | `06-评估与运维.md` |
| Latency | 延迟 | 性能指标 | 衡量请求响应时间 | `06-评估与运维.md` |
| Cost | 成本 | 运营指标 | 估算 token 和模型调用费用 | `06-评估与运维.md` |
| Hallucination | 幻觉 | 风险类型 | 模型编造不存在的信息 | `06-评估与运维.md` |
| Guardrail | 护栏 | 安全边界 | 限制敏感信息、危险输出和越权行为 | `06-评估与运维.md` |
