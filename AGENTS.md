# 工作区治理规则

本文件适用于 `vscode_study` 根目录及其所有子目录。子目录存在更严格的 `AGENTS.md` 时，必须同时遵守。

任何 AI Agent，包括：

- Codex
- Claude Code
- OpenHands
- Cursor
- Gemini CLI
- Continue

执行以下操作前：

- `git add`
- `commit`
- `push`
- PR
- 发布
- GitHub 同步

必须检查：

1. `工作区总索引.md`
2. `不上传GitHub文件清单.md`
3. `密钥与敏感信息管理规范.md`
4. `.gitignore`

---

如果发现：

- API Key
- Token
- Password
- SSH Key
- 数据库账号
- 云平台账号
- 敏感资料

立即停止。

必须询问用户。

禁止自动提交。

---

默认原则：

不确定是否上传时，不要上传，先询问用户。

---

GitHub 提交检查流程：

1. 检查工作区总索引。
2. 检查不上传 GitHub 文件清单。
3. 检查密钥规范。
4. 扫描敏感信息。
5. 用户确认。
6. 才允许提交。

未获得用户明确要求时，不得自动执行 `commit`、`push`、创建 PR 或发布。不得删除现有文件来规避检查。

## 目录保护规则

所有 AI Agent 在读取、修改、删除、重命名、执行 Git 操作或发布前，必须优先检查 `工作区总索引.md` 中目标目录的“目录等级”，再根据等级决定是否允许操作。

目录等级与其他权限发生冲突时，执行更严格的规则。子目录未单独登记时继承最近父目录等级；S 级父目录的保护规则覆盖全部子目录，除非用户针对当前操作给出明确授权。

### S 级：最高保护目录

- 默认只读。
- 任何修改必须先询问用户并获得明确同意。
- 禁止自动提交或同步 GitHub。
- 禁止批量修改。
- 禁止删除。
- 禁止重命名。

`softbs` 为 S 级目录，是个人知识库与业务资料库，可能包含税务资料、freee 资料、投资资料、API 配置、客户资料、网站维护资料和私人笔记。以上 S 级规则适用于 `softbs` 及其全部子目录。

### A 级：核心维护目录

- 允许在用户任务范围内新增文档、更新教程和补充知识库。
- 删除、重命名、大范围重构或改变目录结构前，必须询问用户。

`web-projects` 为 A 级目录，是 Web 开发知识库。允许新增文档、更新教程和补充知识库；涉及删除和重构必须先询问用户。

### B 级：常规项目目录

- 允许完成用户明确要求范围内的新增和修改。
- 删除、批量改动、重命名或结构调整前，必须询问用户。

### C 级：实验与练习目录

- 允许进行可恢复的小范围实验和验证。
- 仍须遵守敏感信息检查、禁止自动提交和用户确认规则。

无法确定目标目录或目录等级时，按更高保护等级处理，停止修改并询问用户。
