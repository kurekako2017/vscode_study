# Agent 术语速查表

这份表用于快速查 Agent 相关名词。建议配合 [Agent系统框架与术语.md](./Agent系统框架与术语.md) 使用。

| 术语 | 日语 | 是什么 | 核心作用 | 推荐学习 |
| --- | --- | --- | --- | --- |
| Agent | エージェント | 围绕目标自动推进任务的 LLM 应用系统 | 思考、调用工具、输出结果 | `05-Agent工作流.md` |
| Goal | 目標 / ゴール | Agent 要完成的任务目标 | 定义最终要达成什么 | `05-Agent工作流.md` |
| Planner | プランナー / 計画担当 | 负责拆解任务的模块 | 把目标拆成步骤，决定下一步 | `05-Agent工作流.md` |
| Executor | 実行担当 / 実行器 | 负责执行具体动作的模块 | 调用模型、工具或业务接口 | `05-Agent工作流.md` |
| Tool | ツール / 外部ツール | Agent 可以使用的外部能力 | 搜索、读文件、调 API、查数据库 | `03-Tool Calling.md` |
| Tool Calling | ツール呼び出し | 模型请求程序调用工具的机制 | 让模型选择工具并给出参数 | `03-Tool Calling.md` |
| Function Calling | 関数呼び出し | 把工具定义成函数名和参数的调用方式 | 让工具调用更结构化、更容易校验 | `03-Tool Calling.md` |
| Tool Schema | ツールスキーマ | 工具名称、参数和说明的结构定义 | 告诉模型工具怎么用 | `03-Tool Calling.md` |
| Observation | 観測結果 / 実行結果 | 工具执行后返回的真实结果 | 回填给 Agent 继续判断 | `03-Tool Calling.md` |
| Agent Loop | エージェントループ | 思考、行动、观察、再思考的循环 | 支持多步任务持续推进 | `05-Agent工作流.md` |
| State | 状態 / ステート | 当前任务过程中的数据状态 | 保存步骤、历史、变量、错误次数 | `05-Agent工作流.md` |
| Memory | メモリ / 記憶 | 保存上下文或历史信息的机制 | 让 Agent 能利用之前的信息 | `05-Agent工作流.md` |
| Termination | 終了条件 | 判断循环何时停止的规则 | 防止无限循环和成本失控 | `05-Agent工作流.md` |
| Retry | リトライ / 再試行 | 失败后重新尝试的机制 | 提高失败处理能力 | `05-Agent工作流.md` |
| Workflow | ワークフロー | 固定步骤组成的处理流程 | 用明确步骤控制任务执行 | `workflow_agent` |
| Multi-Agent | マルチエージェント | 多个 Agent 分工协作的架构 | 不同角色分别负责不同任务 | 进阶主题 |
| Evaluator | 評価器 / 評価担当 | 判断输出质量的模块 | 判断结果是否合格、是否需要重做 | `05-Agent工作流.md` |
| Guardrail | ガードレール / 安全制御 | 限制风险行为的安全规则 | 限制危险工具、越权请求、敏感信息 | `05-Agent工作流.md` |
| Human-in-the-loop | 人間確認 / 人手確認 | 人参与确认的控制点 | 关键步骤需要人确认后执行 | `05-Agent工作流.md` |
| RAG Tool | RAG ツール / 検索ツール | 用于检索企业资料的工具 | 让 Agent 能查本地或社内知识 | `04-RAG.md` |

## 日本现场常见说法

| 场景 | 中文说法 | 日语表达 |
| --- | --- | --- |
| 说明 Agent 方案 | 使用 Agent 自动处理多步骤任务 | AI エージェントで複数ステップの業務を自動化します |
| 说明工具调用 | 模型判断后调用外部 API | モデルの判断に基づいて外部 API を呼び出します |
| 说明安全控制 | 高风险操作需要人工确认 | 重要な操作には人手確認を入れます |
| 说明状态管理 | 保存中间结果并传给下一步 | 中間結果を状態として保持し、次の処理に渡します |
| 说明 RAG | 基于社内文档检索后回答 | 社内文書を検索し、その結果に基づいて回答します |
