# 学习路线

## 1. 总目标

如果你的学习目标是贴近日本 IT 现场和派遣案件，这条路线的重点不应该是先冲复杂 Agent，而应该先建立这些能力：

- 会调用模型
- 会约束输出
- 会做 `RAG / 社内検索`
- 会把能力接进 Python 后端
- 会设计工具
- 会做任务循环
- 会做结果校验

## 2. 推荐阶段

### 阶段 1: 模型调用

- 学消息结构
- 学参数控制
- 学结构化输出

日本案件高频关键词:

- `Python`
- `OpenAI API`
- `Azure OpenAI`
- `構造化出力`

输出物:

- `chat_cli`
- `structured_output_demo`

### 阶段 2: RAG / 社内検索

- 文本切分
- 检索
- 引用来源
- 企业资料问答

日本案件高频关键词:

- `RAG`
- `社内検索`
- `ナレッジ検索`
- `LangChain`
- `LlamaIndex`

输出物:

- `doc_qa_agent`

### 阶段 3: 后端整合

- 把模型能力接进 API
- 处理认证、日志、配置
- 和业务系统或内部资料对接

日本案件高频关键词:

- `FastAPI`
- `API連携`
- `AWS Bedrock`
- `Azure OpenAI`

输出物:

- `rag_api_demo`

### 阶段 4: Tool Calling

- 设计工具接口
- 处理参数校验
- 处理工具结果回填

输出物:

- `tool_agent_demo`

### 阶段 5: Agent 工作流

- 计划
- 执行
- 重试
- 终止条件

输出物:

- `workflow_agent`

### 阶段 6: 评估与上线

- 日志
- 成本
- 成功率
- 安全边界

输出物:

- `agent_eval_report.md`

## 3. 建议节奏

### 第 1 到 2 周

- 模型调用
- Prompt 基础
- 结构化输出

### 第 3 到 4 周

- `RAG`
- 本地知识库问答

### 第 5 到 6 周

- Python 后端整合
- API 化

### 第 7 到 8 周

- Tool Calling
- 文件、HTTP、数据库等基础工具

### 第 9 到 12 周

- 单 Agent 工作流
- 评估
- 监控
- Demo 收尾

## 4. 最重要的原则

- 先学日本现场更常见的 `RAG + Python + 企业集成`，再学复杂 Agent。
- 先把简单链路跑通，再加复杂能力。
- 先自己手写基本流程，再用框架。
- 每个阶段都要有可运行成果。
