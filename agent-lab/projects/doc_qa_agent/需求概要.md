# doc_qa_agent 需求概要

## 1. 背景

在日本 IT 现场，生成 AI 的典型落地方向之一是：

- 社内検索
- 规程问答
- 手顺书问答
- 设计资料检索

很多场景一开始不需要复杂 Agent，而是先希望做一个“能读本地资料并回答问题”的最小 `RAG` 样例，用于验证可行性。

## 2. 目的

本项目的目的是实现一个最小可运行的本地文档问答程序，用于验证以下内容：

- 本地资料是否可以作为知识源
- 能否根据用户问题检索相关片段
- 能否基于检索结果生成带出处的回答

## 3. 目标用户

- 生成 AI 初学者
- 想贴近日本现场学习 `RAG` 的工程师
- 需要做 `社内検索` / `ナレッジ検索` PoC 的开发者

## 4. 使用场景

- 用户指定一个本地文档目录
- 程序读取其中的 `md` / `txt` 文件
- 用户输入问题
- 系统检索相关文档片段
- 模型基于片段回答，并输出来源

## 5. 功能要求

### 5.1 文档读取

- 支持读取本地目录中的 `md`
- 支持读取本地目录中的 `txt`
- 递归扫描子目录

### 5.2 文档切分

- 对文本按固定长度切分
- 为每个片段附加来源标识

### 5.3 检索

- 根据用户问题做本地关键词检索
- 选出得分较高的片段作为上下文

### 5.4 回答生成

- 将检索结果作为上下文传给模型
- 模型只能基于检索结果回答
- 如果检索结果不足，要明确说明

### 5.5 来源输出

- 输出命中的来源片段标识
- 方便用户核对回答依据

## 6. 非功能要求

- 本地命令行运行
- 结构尽量简单，便于学习
- 不依赖向量数据库
- 不依赖外部检索平台

## 7. 范围外事项

本样例暂不包含：

- PDF 支持
- 向量检索
- API 化
- 认证
- 日志持久化
- 检索评估

## 8. 前提条件

- Python 环境可用
- `OPENAI_API_KEY` 已配置
- 本地存在可读的 `md` / `txt` 文件

## 9. 预期成果

- 一个可运行的命令行 `RAG` 样例
- 能回答基于本地资料的问题
- 能输出简单引用来源

## 10. 成功判定

满足以下条件可视为完成：

- 能读取本地文档目录
- 能返回回答
- 能列出来源片段
- 在找不到依据时能明确说明
