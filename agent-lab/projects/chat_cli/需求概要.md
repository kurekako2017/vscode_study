# 需求概要

## 1. 背景

在学习大模型应用开发时，第一步通常不是直接做复杂系统，而是先确认最基础的模型调用链路是否成立。

因此需要一个最小案例，用来验证：

- 环境变量是否配置正确
- Python 程序是否能成功调用模型
- 输入和输出链路是否正常

## 2. 目的

实现一个最小可运行的命令行聊天示例，使开发者可以：

- 在命令行直接输入问题
- 调用模型获取回答
- 体验最基础的 `Responses API` 调用流程

## 3. 目标用户

- 刚开始学习 LLM 应用开发的人
- 需要先打通 API 调用环境的人
- 准备继续学习结构化输出、RAG、Tool Calling 的开发者

## 4. 功能要求

- 支持一次性提问
- 支持交互式输入
- 支持指定模型名
- 支持从环境变量读取 `OPENAI_API_KEY`

## 5. 非功能要求

- 实现尽量简单
- 便于阅读和修改
- 本地命令行可直接运行

## 6. 范围外事项

当前版本不包含：

- 多轮历史记忆
- 工具调用
- 结构化输出
- RAG
- Web API 化

## 7. 成功标准

- 能成功调用模型
- 能输出文本回答
- 缺少 `OPENAI_API_KEY` 时能明确报错

## 8. 成本与注意点

- 这是最小学习样例，不是正式业务系统。
- 每次请求都会产生 API 成本。
- 当前只适合验证调用链路，不适合展示复杂业务能力。
