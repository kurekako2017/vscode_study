# 需求概要

## 1. 背景

在很多实际应用里，开发者需要的不是自由文本聊天，而是稳定的结构化结果。

例如：

- 需求整理
- 分类结果
- 功能清单
- 风险列表

因此需要一个最小案例，验证模型能否按固定 schema 返回 JSON。

## 2. 目的

实现一个最小可运行的结构化输出示例，使开发者可以：

- 输入自然语言需求
- 调用模型
- 得到稳定 JSON 结果
- 用代码对结果进行校验

## 3. 目标用户

- 已完成最基础 API 调用学习的人
- 想学习结构化输出的人
- 想为后续 Tool Calling、RAG、后端 API 做准备的开发者

## 4. 功能要求

- 接收自然语言项目需求
- 返回固定 JSON 结构
- 使用 schema 校验结果
- 支持指定模型名

## 5. 非功能要求

- 本地命令行可运行
- 结构清晰，便于理解
- 结果格式稳定，便于后续程序使用

## 6. 范围外事项

当前版本不包含：

- 文件写入
- Web API
- 数据库存储
- 工具调用
- RAG

## 7. 成功标准

- 能稳定输出符合 schema 的 JSON
- 程序能解析并打印结构化结果
- 模型调用失败时能明确报错

## 8. 成本与注意点

- 这是学习版示例，不是正式企业系统。
- API 调用会产生实际成本。
- 结构化输出稳定度仍会受模型和提示词影响。
