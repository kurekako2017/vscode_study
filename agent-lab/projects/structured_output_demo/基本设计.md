# 基本设计

## 1. 系统概要

`structured_output_demo` 是一个最小结构化输出示例。

系统目标是把自然语言需求转换为固定结构的 JSON，并由程序完成解析和校验。

## 2. 模块构成

| 模块 | 作用 |
|---|---|
| 参数解析 | 接收需求文本和模型名 |
| Client 初始化 | 读取 `OPENAI_API_KEY` 并创建客户端 |
| Pydantic Schema | 定义输出字段和约束 |
| 模型调用 | 使用 `responses.parse` 获取结构化结果 |
| 输出模块 | 打印美化 JSON 和原始 JSON |

## 3. 输入输出设计

### 3.1 输入

- `prompt`
  - 自然语言需求说明
- `--model`
  - 模型名

### 3.2 输出

- Pretty JSON
- Raw JSON

主要字段包括：

- `goal`
- `user_type`
- `core_capabilities`
- `tools`
- `deliverables`
- `risks`

## 4. 处理流程

1. 解析命令行参数
2. 初始化 OpenAI Client
3. 根据 Pydantic schema 调用 `responses.parse`
4. 解析模型返回结果
5. 输出结构化 JSON

## 5. 错误处理

- 未配置 `OPENAI_API_KEY` 时退出
- 模型调用失败时输出错误信息
- 解析失败时退出

## 6. 设计取舍

- 不做历史对话
- 不做文件输出
- 不做 Web API

当前重点是验证“结构化输出是否可稳定用于后续程序”。

## 7. 后续扩展方向

- 将结果写入文件
- 将结果作为后端 API 输出
- 把结构化结果接入 Tool Calling 或 Workflow

## 8. 成本与注意点

- 当前版本是学习版。
- 结果虽有 schema 约束，但仍需要关注模型输出稳定性。
- API 调用会产生实际成本。
