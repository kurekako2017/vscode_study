# 模型调用基础

## 1. 这一阶段要解决什么

先不要急着做 Agent。

第一阶段只需要掌握：

- 如何向模型发送请求
- 如何组织消息
- 如何控制输出格式
- 如何处理错误和重试

如果按日本现场和案件要求来看，这一阶段对应的是最基础的 `Python + LLM API` 能力，是后面做 `RAG`、业务助手、企业集成的前提。

## 2. 必学内容

### 消息结构

至少要理解：

- system message
- user message
- assistant message

### 常见参数

- `temperature`
- `max_tokens`
- `top_p`
- 响应格式控制

### 结构化输出

这一阶段非常重要，建议优先练：

- 输出固定 JSON
- 输出任务清单
- 输出分类结果

因为在日本现场里，很多生成 AI 应用最终都要接到：

- 后端 API
- 业务流程
- 数据库
- 企业内系统

这时候“稳定输出结构化结果”比“自由聊天”更有价值。

## 3. 建议练习

- 做一个命令行问答程序
- 做一个“输入需求，输出任务清单”的程序
- 做一个“输入文档，输出摘要”的程序

如果是案件导向，建议优先做：

- 需求整理 JSON 输出
- FAQ 分类输出
- 文档摘要输出

## 4. 完成标志

- 能稳定拿到模型响应
- 能把响应保存为文件
- 能让输出格式基本稳定

## 5. 常见坑

- 只会自然语言输出，不会结构化输出
- 没有处理超时和失败
- Prompt 改一点，输出格式就崩
- 只会做聊天 demo，和业务系统接不上
