# LLM Agent 学习路径与计划

## 1. 目标

这份文档的目标不是只学会“调用一下大模型接口”，而是建立一条从:

- 大语言模型基础
- Prompt 设计
- `RAG / 社内検索`
- Python 后端整合
- 工具调用
- 工作流编排
- Agent 设计
- 评估与上线

到真正能做出可用 Agent 的学习路线。

## 2. 先明确什么是 Agent

如果只让模型回答问题，那更像:

- `Chat`
- `Completion`
- `Q&A`

如果让模型在目标驱动下自己决定下一步，并且能:

- 调用工具
- 读取文件
- 查询数据库
- 搜索资料
- 分解任务
- 反复执行直到完成

这才更接近 `Agent`。

所以学习 Agent，不只是学模型本身，还要学:

- 上下文管理
- 工具设计
- 状态管理
- 任务拆解
- 结果验证
- 安全边界

## 3. 学习前置基础

开始学 Agent 之前，建议至少具备这些基础。

### 编程基础

- `Python` 或 `TypeScript / JavaScript`
- 基本文件操作
- HTTP 请求
- JSON
- 异常处理

### 后端基础

- `REST API`
- 鉴权
- 日志
- 配置管理
- 环境变量

### AI 基础

- Token
- Context Window
- Temperature
- Prompt
- Embedding
- RAG
- Function Calling / Tool Calling

### 工程基础

- Git
- 虚拟环境
- 包管理
- 测试
- Docker 基础

## 4. 推荐学习顺序

如果你的目标是根据日本 IT 工作现场、企业落地场景和派遣案件要求来学习，那么学习顺序不应该按“技术炫酷程度”排，而应该按“最常见、最容易落地、最容易对应案件要求”的顺序排。

### 阶段 1: 模型调用入门

目标:

- 会调用模型
- 会写系统提示词
- 会处理结构化输出

要做的事:

- 用 Python 或 Node.js 调一次模型接口
- 学会消息结构
- 让模型输出 JSON
- 学会处理失败重试和超时

完成标志:

- 能做一个命令行问答工具
- 能把结果保存到文件

### 阶段 2: Prompt 与结构化输出

目标:

- 会稳定约束模型输出格式
- 会把自然语言任务转换成结构化步骤

要做的事:

- 学习 system / user / developer prompt 的角色
- 学习 few-shot 示例
- 学习 JSON schema 或固定输出格式
- 学习怎么减少幻觉和格式漂移

完成标志:

- 能做一个“输入需求，输出任务清单”的小工具

### 阶段 3: RAG 与知识库

目标:

- 让模型基于企业资料、文档和知识库回答问题

要做的事:

- 学文本切分
- 学向量检索
- 学检索后再回答
- 学引用来源
- 做企业资料问答或社内搜索 demo

完成标志:

- 能做一个“基于本地文档回答问题”的知识 Agent

### 阶段 4: Python 后端整合

目标:

- 把模型能力接到真实后端服务里

要做的事:

- 用 `FastAPI` 或类似框架包一层 API
- 处理配置、日志、异常和环境变量
- 处理和业务系统、内部服务、资料目录的集成

完成标志:

- 能做一个可调用的 `RAG API` 或知识助手后端

### 阶段 5: Tool Calling

目标:

- 让模型不只是说，还能调用有限工具完成动作

要做的事:

- 设计 3 到 5 个简单工具
- 例如:
  - 读文件
  - 搜索文本
  - 调内部 API
  - 查数据库
- 学会处理工具参数校验
- 学会把工具结果再喂回模型

完成标志:

- 能做一个“会查文件、会总结内容、会生成结果”的小 Agent

### 阶段 6: 单 Agent 工作流

目标:

- 做出一个能持续完成任务的 Agent

要做的事:

- 增加任务循环
- 增加 memory / scratchpad
- 增加计划和执行分离
- 增加失败重试
- 增加终止条件

完成标志:

- 能做一个“读目录 -> 找目标文件 -> 修改内容 -> 输出总结”的 Agent

### 阶段 7: 多工具与多步骤任务

目标:

- 让 Agent 能处理真实工作流

要做的事:

- 结合搜索、文件、数据库、代码执行
- 增加状态存储
- 增加任务阶段控制
- 增加结果校验

完成标志:

- 能做一个“查询资料 -> 整理摘要 -> 输出 Markdown 报告”的 Agent

### 阶段 8: 多 Agent 与协作

目标:

- 学会把复杂任务拆给不同角色的 Agent

要做的事:

- 设计 planner / worker / reviewer 模式
- 控制上下文传递
- 控制权限边界
- 避免重复工作

完成标志:

- 能做一个“规划 -> 执行 -> 审核”的多 Agent 样例

### 阶段 9: 评估、监控与上线

目标:

- 从 demo 走向可用系统

要做的事:

- 设计评估集
- 记录输入、输出、工具调用链
- 监控失败率、耗时、成本
- 增加安全限制
- 增加人工确认点

完成标志:

- 能稳定运行一个内部使用的小 Agent

## 5. 推荐实战项目

按学习顺序，推荐做这些项目。

### 项目 1: 命令行问答助手

目标:

- 学会最基础的模型调用

### 项目 2: 文档摘要助手

目标:

- 输入 Markdown 或 PDF 文本
- 输出摘要和关键点

### 项目 3: RAG 知识库问答

目标:

- 基于本地文档回答问题
- 给出引用出处

### 项目 4: RAG API / 社内検索 API

目标:

- 用 Python 后端把知识检索能力接口化

### 项目 5: 文件整理 Agent

目标:

- 读取目录
- 找指定文件
- 生成整理报告

### 项目 6: 代码辅助 Agent

目标:

- 搜索代码
- 定位文件
- 生成修改建议
- 输出变更说明

### 项目 7: 浏览器或办公 Agent

目标:

- 调用浏览器自动化
- 读取表格
- 生成日报或周报

## 6. 12 周学习计划

### 第 1 到 2 周

- 学模型 API 基础
- 学 Prompt 基础
- 完成一个简单聊天程序

输出物:

- `chat_cli`
- `prompt_notes.md`

### 第 3 到 4 周

- 学结构化输出
- 学 JSON 校验
- 做任务拆解小工具

输出物:

- `task_planner`
- `structured_output_demo`

- 学 RAG
- 接入向量库或简单检索
- 做本地文档问答

输出物:

- `doc_qa_agent`
- `rag_notes.md`

### 第 5 到 6 周

- 学 Python 后端整合
- 做 `RAG API`
- 处理配置、日志、异常

输出物:

- `rag_api_demo`
- `api_notes.md`

### 第 7 到 8 周

- 学 tool calling
- 做文件读写、搜索、HTTP 请求工具
- 做一个单 Agent 循环

输出物:

- `tool_agent_demo`
- `file_summary_agent`

### 第 9 到 10 周

- 学多步骤工作流
- 增加任务状态、失败重试、日志
- 做一个真实一点的业务 Agent

输出物:

- `workflow_agent`
- `agent_log_viewer`

### 第 11 到 12 周

- 学评估、监控、安全
- 做一个完整 demo
- 写项目总结

输出物:

- `agent_eval_report.md`
- `final_agent_demo`

## 7. 每个阶段要特别注意的坑

### 不要只会聊天

很多人学了半天，其实只会:

- 写 prompt
- 调一次接口

但不会:

- 设计工具
- 管理状态
- 控制输出
- 处理失败

如果按日本现场和案件要求来看，还常会缺这些：

- `RAG`
- 后端 API 化
- 企业资料接入
- 云平台对接

### 不要过早上复杂框架

一开始不要先迷失在框架名里。

先自己手写明白:

- 消息怎么传
- 工具怎么调用
- 结果怎么回填
- 循环怎么终止

再去学框架会更稳。

同样也不要过早沉迷复杂多 Agent。

日本现场更常见的优先级通常是：

- `RAG`
- 企业搜索
- API 化
- 半自动 Agent

### 不要忽视评估

Agent 最大的问题不是“能不能跑”，而是:

- 结果稳不稳
- 错了能不能发现
- 成本高不高
- 是否可控

### 不要忽视安全边界

如果 Agent 能:

- 执行命令
- 写文件
- 访问数据库
- 调外部 API

那就必须明确:

- 哪些工具能用
- 哪些目录能写
- 哪些操作要人工确认

## 8. 推荐知识地图

你可以把 Agent 学习分成这 6 条线并行推进:

- 模型调用线
- Prompt 线
- Tool Calling 线
- RAG 线
- Workflow / Agent 线
- Eval / Safety 线

## 9. 学完之后应该具备的能力

如果这条路线走完，你应该能做到:

- 独立写一个调用大模型的服务
- 给模型设计稳定的结构化输出
- 给 Agent 设计工具接口
- 做一个单 Agent 工作流
- 做一个基于知识库的问答 Agent
- 设计基本的评估与日志方案
- 把 Agent 从 demo 推到可内部使用的程度

## 10. 建议的输出节奏

学习 Agent 时，最好不要只记笔记，要持续产出。

建议每一阶段至少有:

- 1 个可运行 demo
- 1 份总结文档
- 1 份问题清单

这样你最后会形成:

- 代码
- 文档
- 项目经验

而不是只停留在概念层。

## 11. 下一步怎么落地到当前工作区

如果你想在这个 `vscode_study` 工作区里继续展开，建议下一步这样做:

1. 新建一个 `agent-lab/` 或 `llm-lab/` 目录
2. 先做 `chat_cli`
3. 再做 `structured_output_demo`
4. 再做 `doc_qa_agent`
5. 再做 `rag_api_demo`
6. 再做 `tool_agent_demo`
7. 最后做一个带评估的完整 Agent demo

这样这条学习线会和现在的 `java-lab`、`sap-lab` 一样，逐步形成自己的资料和项目结构。
