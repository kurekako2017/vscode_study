# 密钥与敏感信息管理规范

## 禁止上传

- `.env`
- `.env.local`
- `.env.production`
- `*.key`
- `*.pem`
- `*.p12`
- `credentials.json`
- `secrets.json`
- `apikey*`
- `token*`
- `secret*`

以上规则适用于工作区内所有层级。除密钥外，密码、Cookie、会话信息、数据库连接串、云平台凭据、SSH 私钥、个人资料和公司内部资料也不得上传。

## API Key 管理规范

推荐使用项目本地的 `.env` 存储：

- OpenRouter
- ChatNVIDIA
- Gemini
- OpenAI
- GitHub Token

只允许在 `.env.example` 中提供变量名和无效示例值，例如 `OPENAI_API_KEY=请在本地填写`。禁止把 Key 直接写入代码、Markdown、测试数据、日志、截图、Shell 历史或 Git 提交信息。

读取密钥时应使用环境变量。不同项目、不同环境尽量使用不同密钥，并遵循最小权限、定期轮换和可随时撤销原则。

## GitHub 提交前检查

检查：

- API Key
- Token
- Password
- SSH Key
- 数据库账号密码

还要查看暂存区差异、未跟踪文件、配置文件、日志、导出数据和文档示例。发现后立即停止提交，不得用遮掩、改名或强制提交绕过规则；必须询问用户。

如果敏感信息曾经进入提交历史，应立即停止推送，撤销或轮换对应凭据，并由用户决定历史清理方案。
