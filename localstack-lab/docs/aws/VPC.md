# VPC 学习笔记（中日对照）

VPC 是 AWS 网络的基础。只要你在 AWS 上部署多个资源，就几乎一定会接触到 VPC。

## 1. 这个服务是什么 / このサービスは何か

- 中文：VPC 是你在 AWS 里自己定义的一块隔离网络空间。
- 日本語：VPC は AWS 上で自分が定義する分離されたネットワーク空間です。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| VPC | 虚拟私有云 | 仮想プライベートクラウド |
| Subnet | 子网 | サブネット |
| Route Table | 路由表 | ルートテーブル |
| Internet Gateway | 互联网网关 | インターネットゲートウェイ |
| NAT Gateway | NAT 网关 | NAT ゲートウェイ |
| Security Group | 安全组 | セキュリティグループ |
| NACL | 网络 ACL | ネットワーク ACL |
| CIDR | 网段表示法 | CIDR 表記 |

## 3. 学习重点 / 学習ポイント

- 中文：理解 VPC、Subnet、Route Table、Gateway 的关系。
- 日本語：VPC、Subnet、Route Table、Gateway の関係を理解する。
- 中文：理解公有子网和私有子网。
- 日本語：パブリックサブネットとプライベートサブネットを理解する。
- 中文：理解安全组和 NACL 的差别。
- 日本語：セキュリティグループと NACL の違いを理解する。

## 4. 架构直觉 / アーキテクチャ感覚

- 中文：VPC 关注的是“网络边界”和“通信路径”。
- 日本語：VPC は「ネットワーク境界」と「通信経路」を考えるサービスです。
- 中文：Subnet 关注的是“资源放在哪里”。
- 日本語：Subnet は「リソースをどこに置くか」を表します。
- 中文：Route Table 关注的是“流量往哪里走”。
- 日本語：Route Table は「通信をどこへ送るか」を定義します。

## 5. LocalStack 里怎么学 / LocalStack での学び方

- 中文：重点是把 VPC 相关概念和其他服务连起来理解，不必一开始追求真实网络的全部细节。
- 日本語：最初は VPC の概念を他サービスとつなげて理解し、実ネットワークの細部まで追いすぎないことが大事です。

## 6. 常见坑 / よくある落とし穴

- 中文：只记 CIDR，忘了路由和网关。
- 日本語：CIDR だけ覚えてルートやゲートウェイを忘れる。
- 中文：把安全组和路由表当成同一种东西。
- 日本語：セキュリティグループとルートテーブルを同一視する。
