# ECR 学习笔记（中日对照）

ECR 是 AWS 的容器镜像仓库，和 ECS、EKS、Fargate 经常一起出现。

## 1. 这个服务是什么 / このサービスは何か

- 中文：ECR 用来存放、管理和分发 Docker 镜像。
- 日本語：ECR は Docker イメージを保存、管理、配信するためのレジストリです。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| Repository | 镜像仓库 | リポジトリ |
| Image | 容器镜像 | コンテナイメージ |
| Tag | 镜像标签 | タグ |
| Push / Pull | 推送 / 拉取 | プッシュ / プル |

## 3. 学习重点 / 学習ポイント

- 中文：理解镜像仓库、标签、推送、拉取的基本流程。
- 日本語：イメージの保存、タグ付け、プッシュ、プルの基本フローを理解する。
- 中文：理解 ECR 在容器交付链路中的位置。
- 日本語：ECR がコンテナ配信フローのどこに位置するかを理解する。
- 中文：理解“构建镜像”和“运行容器”是不同阶段。
- 日本語：イメージのビルドとコンテナ実行は別フェーズであることを理解する。

## 4. LocalStack 里怎么练 / LocalStack での練習方法

- 中文：先练镜像构建，再练 push/pull。
- 日本語：まずイメージのビルド、その後 push/pull を練習する。
- 中文：把 ECR 和 ECS/Fargate 放在同一条流水线上理解。
- 日本語：ECR を ECS/Fargate と同じデリバリーパイプラインで理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：把镜像仓库和容器运行环境混为一谈。
- 日本語：イメージレジストリとコンテナ実行環境を混同する。
- 中文：标签管理混乱，导致部署错镜像。
- 日本語：タグ管理が乱れると誤ったイメージをデプロイしてしまう。
