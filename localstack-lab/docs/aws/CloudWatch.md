# CloudWatch 学习笔记（中日对照）

CloudWatch 是 AWS 的监控、日志和告警服务，适合学习可观测性基础。

## 1. 这个服务是什么 / このサービスは何か

- 中文：CloudWatch 用于收集指标、日志，并设置告警。
- 日本語：CloudWatch はメトリクスとログを収集し、アラームを設定するためのサービスです。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| Metric | 指标 | メトリクス |
| Log | 日志 | ログ |
| Alarm | 告警 | アラーム |
| Dashboard | 监控看板 | ダッシュボード |
| Namespace | 指标命名空间 | 名前空間 |
| Event | 事件 | イベント |

## 3. 学习重点 / 学習ポイント

- 中文：理解指标、日志、告警三者之间的关系。
- 日本語：メトリクス、ログ、アラームの関係を理解する。
- 中文：理解云上监控的基本思路：采集 -> 可视化 -> 告警。
- 日本語：クラウド監視の基本は「収集 -> 可視化 -> 通知」です。
- 中文：理解 CloudWatch 和 EventBridge、SNS 的配合。
- 日本語：CloudWatch と EventBridge、SNS の連携を理解する。

## 4. LocalStack 里怎么练 / LocalStack での練習方法

- 中文：先看日志和指标，再练告警触发。
- 日本語：まずログとメトリクスを確認し、その後アラーム発報を練習する。
- 中文：把应用日志、系统日志和告警联动起来看。
- 日本語：アプリログ、システムログ、アラームを連携して確認する。

## 5. 常见坑 / よくある落とし穴

- 中文：只看告警，不看日志和指标。
- 日本語：アラームだけ見てログやメトリクスを見ない。
- 中文：指标命名混乱，导致后面很难排查。
- 日本語：メトリクス命名が混乱すると後で追跡しにくい。
