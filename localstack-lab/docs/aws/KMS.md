# KMS 学习笔记（中日对照）

KMS 是 AWS 的密钥管理服务，适合学习加密、密钥轮换、数据保护这些安全基础。

## 1. 这个服务是什么 / このサービスは何か

- 中文：KMS 用于创建和管理加密密钥，保护 AWS 上的数据安全。
- 日本語：KMS は暗号鍵を作成・管理し、AWS 上のデータを保護するためのサービスです。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| Key | 密钥 | 鍵 |
| Customer Managed Key | 客户自主管理密钥 | 顧客管理キー |
| AWS Managed Key | AWS 托管密钥 | AWS 管理キー |
| Encryption | 加密 | 暗号化 |
| Decryption | 解密 | 復号 |
| Key Rotation | 密钥轮换 | 鍵のローテーション |

## 3. 学习重点 / 学習ポイント

- 中文：理解对称加密和密钥管理的基本思路。
- 日本語：対称暗号と鍵管理の基本を理解する。
- 中文：理解 KMS 与 S3、EBS、RDS、Secrets Manager 的关系。
- 日本語：KMS と S3、EBS、RDS、Secrets Manager の関係を理解する。
- 中文：理解密钥轮换为什么重要。
- 日本語：鍵のローテーションが重要な理由を理解する。

## 4. LocalStack 里怎么练 / LocalStack での練習方法

- 中文：先理解“谁在加密、谁在解密、谁管理密钥”。
- 日本語：「誰が暗号化し、誰が復号し、誰が鍵を管理するか」を理解する。
- 中文：把 KMS 放到存储和数据库的安全设计里一起看。
- 日本語：KMS をストレージや DB のセキュリティ設計と一緒に考える。

## 5. 常见坑 / よくある落とし穴

- 中文：把 KMS 当成普通的密码保存仓库。
- 日本語：KMS を単なるパスワード保管庫だと考える。
- 中文：忽略密钥权限，导致应用无法解密数据。
- 日本語：鍵の権限を無視して、アプリが復号できなくなる。
