# Route 53 学习笔记（中日对照）

Route 53 是 AWS 的 DNS 和域名解析服务，适合学习域名、解析记录、流量分发这些概念。

## 1. 这个服务是什么 / このサービスは何か

- 中文：Route 53 用来管理域名和 DNS 记录，把域名解析到具体资源。
- 日本語：Route 53 はドメイン名と DNS レコードを管理し、名前を実際のリソースへ解決するサービスです。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| Domain | 域名 | ドメイン |
| Hosted Zone | 托管区域 | ホストゾーン |
| Record | 解析记录 | レコード |
| A Record | 指向 IP 的记录 | IP を指すレコード |
| CNAME | 别名记录 | 別名レコード |
| TTL | 缓存时间 | キャッシュ時間 |

## 3. 学习重点 / 学習ポイント

- 中文：理解域名、解析记录、TTL 的关系。
- 日本語：ドメイン、レコード、TTL の関係を理解する。
- 中文：理解 Route 53 的健康检查和路由策略概念。
- 日本語：Route 53 のヘルスチェックとルーティングポリシーを理解する。
- 中文：理解 DNS 在服务入口中的作用。
- 日本語：DNS がサービスの入口で果たす役割を理解する。

## 4. LocalStack 里怎么练 / LocalStack での練習方法

- 中文：先理解“域名如何映射到服务入口”。
- 日本語：「ドメインがサービスの入口にどう対応するか」を理解する。
- 中文：把 Route 53 和 ALB / EC2 / API 入口结合起来看。
- 日本語：Route 53 を ALB / EC2 / API エンドポイントと合わせて考える。

## 5. 常见坑 / よくある落とし穴

- 中文：把 DNS 当成只是“把名字变成 IP”。
- 日本語：DNS を単に「名前を IP に変えるだけ」と考える。
- 中文：忽略 TTL 导致修改后短时间内仍然访问旧记录。
- 日本語：TTL を無視して、更新後もしばらく古いレコードにアクセスしてしまう。
