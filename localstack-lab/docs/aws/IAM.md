# IAM 学习笔记（中日对照）

IAM 是 AWS 权限体系的核心。只要涉及访问 AWS 服务，就绕不开 IAM。

## 1. 这个服务是什么 / このサービスは何か

- 中文：IAM 负责用户、组、角色和策略管理，用来控制谁能访问什么资源。
- 日本語：IAM はユーザー、グループ、ロール、ポリシーを管理し、誰が何にアクセスできるかを制御します。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| User | IAM 用户 | IAM ユーザー |
| Group | 用户组 | グループ |
| Role | 角色，通常给服务或临时身份使用 | ロール。主にサービスや一時的な権限付与に使う |
| Policy | 权限策略 | ポリシー |
| Permission | 允许/拒绝某些操作的能力 | 許可・拒否の能力 |
| MFA | 多因素认证 | 多要素認証 |

## 3. 学习重点 / 学習ポイント

- 中文：区分 User、Group、Role 的使用场景。
- 日本語：User、Group、Role の使い分けを理解する。
- 中文：理解 Policy 的语法和作用范围。
- 日本語：Policy の文法と適用範囲を理解する。
- 中文：理解“最小权限原则”。
- 日本語：「最小権限の原則」を理解する。

## 4. 典型认知 / 典型的な理解

- 中文：人登录通常用 User + MFA。
- 日本語：人のログインは通常 User + MFA を使う。
- 中文：服务间访问通常用 Role。
- 日本語：サービス間アクセスは通常 Role を使う。
- 中文：开发环境和生产环境最好分开权限。
- 日本語：開発環境と本番環境の権限は分けるのが基本。

## 5. 本地练习建议 / ローカル練習のすすめ

- 中文：LocalStack 里重点练 CLI 的 profile、access key、secret key、endpoint-url 组合。
- 日本語：LocalStack では CLI の profile、access key、secret key、endpoint-url の組み合わせを練習する。
- 中文：先学会给命令“赋权”，再学习给服务“授信”。
- 日本語：まずコマンドに対する権限付与を理解し、その後サービスへの権限付与を学ぶ。

## 6. 常见坑 / よくある落とし穴

- 中文：把 User 和 Role 混为一谈。
- 日本語：User と Role を混同する。
- 中文：策略写得太宽，导致权限过大。
- 日本語：ポリシーが広すぎて権限が過剰になる。
