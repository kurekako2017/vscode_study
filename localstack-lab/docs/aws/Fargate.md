# Fargate 学习笔记（中日对照）

Fargate 是容器的无服务器运行方式，适合学习“只关心容器，不管理服务器”的思路。

## 1. 这个服务是什么 / このサービスは何か

- 中文：Fargate 让你运行容器时不必自己管理底层服务器。
- 日本語：Fargate はコンテナを実行する際に、基盤となるサーバー管理を不要にします。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| Serverless Compute | 无服务器计算 | サーバーレスコンピューティング |
| Task | 任务 | タスク |
| CPU / Memory | 资源配置 | CPU / メモリ設定 |
| Platform Version | 平台版本 | プラットフォームバージョン |

## 3. 学习重点 / 学習ポイント

- 中文：理解 Fargate 和 ECS 的关系。
- 日本語：Fargate と ECS の関係を理解する。
- 中文：理解“只管容器、不要管服务器”的价值。
- 日本語：「コンテナだけに集中し、サーバー管理をしない」価値を理解する。
- 中文：理解资源配置和成本控制的重要性。
- 日本語：リソース設定とコスト管理の重要性を理解する。

## 4. LocalStack 里怎么练 / LocalStack での練習方法

- 中文：把 Fargate 当成容器执行模式来学习，不必一开始追求所有底层细节。
- 日本語：Fargate はコンテナ実行方式として学び、最初から下層の細部まで追いすぎない。
- 中文：结合 ECS、ECR 看完整容器链路。
- 日本語：ECS、ECR と合わせてコンテナの全体フローを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：把 Fargate 误认为编排器本身。
- 日本語：Fargate をオーケストレーターそのものだと誤解する。
- 中文：不关注资源配置，导致成本或性能不合适。
- 日本語：リソース設定を軽視してコストや性能が合わなくなる。
