# Secrets Manager 学习笔记（中日对照）

Secrets Manager 是 AWS 的敏感信息管理服务，适合学习配置管理、密钥保护、运行时取值这些概念。

## 1. 这个服务是什么 / このサービスは何か

- 中文：Secrets Manager 用来保存数据库密码、API Key、Token 等敏感信息。
- 日本語：Secrets Manager はデータベースのパスワード、API キー、トークンなどの機密情報を保存するサービスです。

## 2. 核心概念 / 基本概念

| 英文 | 中文说明 | 日本語説明 |
|---|---|---|
| Secret | 密文对象 | シークレット |
| Secret Version | 密文版本 | シークレットのバージョン |
| Rotation | 自动轮换 | 自動ローテーション |
| Retrieval | 读取密文 | シークレットの取得 |

## 3. 学习重点 / 学習ポイント

- 中文：理解 Secrets Manager 和 KMS 的区别。
- 日本語：Secrets Manager と KMS の違いを理解する。
- 中文：理解“配置放代码里”和“配置放密钥服务里”的差别。
- 日本語：設定をコードに直書きするのと、シークレットサービスで管理する違いを理解する。
- 中文：理解运行时读取 secret 的思路。
- 日本語：実行時に secret を読み込む考え方を理解する。

## 4. LocalStack 里怎么练 / LocalStack での練習方法

- 中文：把数据库密码、第三方 API Key 这类配置想象成独立资源。
- 日本語：DB パスワードや外部 API キーを独立したリソースとして考える。
- 中文：配合 KMS 一起理解“加密存储 + 安全读取”。
- 日本語：KMS と組み合わせて「暗号化保存 + 安全な読み取り」を理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：把 secret 直接写进代码仓库。
- 日本語：secret をソースコードに直接書く。
- 中文：不做轮换和权限控制。
- 日本語：ローテーションと権限管理を行わない。
