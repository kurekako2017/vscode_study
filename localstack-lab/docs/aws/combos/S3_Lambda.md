# S3 + Lambda 学习笔记（中日对照）

S3 + Lambda 是典型的数据处理组合，适合学习“文件上传后触发自动处理”的事件驱动模式。
## 0. 说明概要 / 説明概要

- 中文：S3 存储文件，Lambda 在对象事件触发时自动处理，适合学习文件上传后的事件驱动加工链路。
- 日本語：S3 がファイルを保存し、Lambda がオブジェクトイベントで自動処理します。ファイルアップロード後のイベント駆動処理を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：S3 存放文件，Lambda 在对象创建等事件发生时自动执行处理逻辑。
- 日本語：S3 にファイルを保存し、オブジェクト作成などのイベントで Lambda が自動実行されます。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| S3 | 文件/对象存储 | ファイル・オブジェクト保存先 |
| Lambda | 自动处理函数 | 自動処理関数 |
| Event Notification | 事件通知 | イベント通知 |
| Trigger | 触发器 | トリガー |

## 3. 典型流程 / 典型フロー

1. 用户或系统把文件上传到 S3。
2. S3 产生对象创建事件。
3. 事件触发 Lambda。
4. Lambda 读取对象并处理。
5. 处理结果写回 S3、数据库或日志系统。

日本語：
1. ユーザーまたはシステムが S3 にファイルをアップロードする。
2. S3 でオブジェクト作成イベントが発生する。
3. そのイベントで Lambda が起動する。
4. Lambda がオブジェクトを読み取り処理する。
5. 処理結果を S3、DB、ログなどへ出力する。

## 4. 学习重点 / 学習ポイント

- 中文：理解“文件存储”和“自动处理”如何解耦。
- 日本語：「保存」と「自動処理」を分離する考え方を理解する。
- 中文：理解事件触发而不是轮询。
- 日本語：ポーリングではなくイベント駆動で動くことを理解する。
- 中文：理解函数的幂等性很重要。
- 日本語：関数の冪等性が重要であることを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：上传触发后处理重复执行，没有做幂等控制。
- 日本語：アップロード後の処理が重複実行されるのに冪等性がない。
- 中文：Lambda 处理时间过长，超出预期。
- 日本語：Lambda の処理時間が長くなりすぎる。
