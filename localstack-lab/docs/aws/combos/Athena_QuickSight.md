# Athena + QuickSight 学习笔记（中日对照）

Athena + QuickSight 适合学习“查询结果可视化”链路，是把无服务器分析结果直接呈现给业务人员的常见组合。
## 0. 说明概要 / 説明概要

- 中文：Athena 负责查询 S3 上的数据，QuickSight 负责把查询结果可视化，适合学习分析结果的展示链路。
- 日本語：Athena が S3 データを検索し、QuickSight が結果を可視化します。分析結果の表示と共有の流れを学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Athena 负责查询 S3 上的数据，QuickSight 负责把查询结果做成图表和仪表盘。
- 日本語：Athena が S3 上のデータをクエリし、QuickSight がその結果をグラフやダッシュボードにします。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Athena | 无服务器 SQL 查询 | サーバーレス SQL クエリ |
| QuickSight | 商业智能可视化 | BI 可視化 |
| S3 | 数据来源 | データソース |
| Dashboard | 可视化看板 | ダッシュボード |

## 3. 典型流程 / 典型フロー

1. 数据先保存在 S3。
2. Athena 对 S3 数据执行 SQL 查询。
3. QuickSight 连接 Athena 作为数据源。
4. 业务用户查看图表、报表和仪表盘。
5. 报表可随数据变化自动更新。

日本語：
1. データが S3 に保存される。
2. Athena が S3 データに SQL を実行する。
3. QuickSight が Athena をデータソースとして接続する。
4. 業務ユーザーがグラフ、レポート、ダッシュボードを見る。
5. レポートはデータの更新に合わせて自動更新できる。

## 4. 学习重点 / 学習ポイント

- 中文：理解查询层和展示层的分离。
- 日本語：クエリ層と表示層を分離して考える。
- 中文：理解业务人员最终看到的是图表，不是原始 SQL。
- 日本語：最終的に業務担当者が見るのは SQL ではなく可視化である。
- 中文：理解 Athena 是 QuickSight 的常见数据源之一。
- 日本語：Athena が QuickSight の一般的なデータソースであることを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：查询太复杂，导致仪表盘刷新慢。
- 日本語：クエリが複雑すぎてダッシュボード更新が遅い。
- 中文：数据源和可视化的刷新节奏不一致。
- 日本語：データソースと可視化の更新タイミングが合わない。
