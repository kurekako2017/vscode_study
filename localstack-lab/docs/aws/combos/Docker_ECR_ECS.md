# Docker / ECR / ECS 路线（中日对照）

这个页面用“容器化应用交付链路”来串起 AWS 的常见组合，适合学习 Docker 打包、ECR 存镜像、ECS 运行服务的完整流程。

## 0. 说明概要 / 説明概要

- 中文：先把应用打成 Docker 镜像，推送到 ECR，再由 ECS 启动任务或服务运行容器。
- 日本語：アプリを Docker イメージにして ECR に push し、ECS がタスクやサービスとして実行します。

## 1. 这个路线是什么 / このルートは何か

- 中文：这是一条典型的容器化交付路线，适合学习“打包成镜像 -> 存到镜像仓库 -> 在容器平台运行”。
- 日本語：これは典型的なコンテナ配信ルートで、「イメージ化 -> レジストリ保存 -> コンテナ実行」を学ぶのに向いています。

## 1.1 适合哪些 application / どんな application に向くか

| application 类型 | 适合度 | 中文说明 | 日本語説明 |
|---|---|---|---|
| Web 应用 | 高 | 适合容器化部署的前后端 Web 服务，尤其是需要持续运行的 API 或站点。 | コンテナ化した Web サービスや API に向いています。 |
| API 服务 | 高 | 适合长期运行、需要平滑发布和滚动更新的服务。 | 常駐型で、段階的リリースやローリング更新が必要なサービスに向きます。 |
| Batch 应用 | 高 | 适合把批处理打成镜像后在 ECS Task 里按需执行。 | バッチ処理をイメージ化し、ECS Task で必要時に実行できます。 |
| Worker / 消费者 | 高 | 适合消息队列消费者、异步处理程序、后台 worker。 | キューのコンシューマーや非同期処理 worker に向きます。 |
| 定时任务 | 中-高 | 适合配合 EventBridge Scheduler / Cron 定时拉起容器任务。 | EventBridge Scheduler や Cron で定期起動するジョブに向きます。 |
| 静态站点 | 低 | 一般不优先走这条路线，通常会用 S3 + CloudFront。 | 通常は S3 + CloudFront の方が適しています。 |

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Docker | 应用容器化工具 | コンテナ化ツール |
| ECR | 容器镜像仓库 | コンテナイメージレジストリ |
| ECS | 容器编排服务 | コンテナオーケストレーション |
| Task Definition | 任务定义 | タスク定義 |
| Service | 持续运行服务 | 常駐サービス |
| CloudWatch | 日志与监控 | ログと監視 |
| IAM | 镜像推送与运行权限 | push / 実行権限 |

## 3. 典型流程 / 典型フロー

1. 开发者先把应用代码打成 Docker 镜像。
2. 镜像推送到 ECR。
3. ECS 读取 Task Definition，拉取镜像并启动任务或服务。
4. 运行中的容器把日志送到 CloudWatch。
5. 通过更新镜像版本和重新部署完成发布。

日本語：
1. 開発者がアプリを Docker イメージにする。
2. イメージを ECR に push する。
3. ECS が Task Definition を読み、イメージを pull してタスクやサービスを起動する。
4. 実行中コンテナのログを CloudWatch に送る。
5. イメージの更新と再デプロイでリリースする。

## 4. 学习重点 / 学習ポイント

- 中文：理解 Docker、ECR、ECS 各自的职责分工。
- 日本語：Docker、ECR、ECS の役割分担を理解する。
- 中文：理解容器化后，应用部署从“拷贝文件”变成“发布镜像”。
- 日本語：コンテナ化後は、デプロイが「ファイル配置」から「イメージ配信」になることを理解する。
- 中文：理解这种路线既适合 Web 服务，也适合 Batch、Worker 和定时任务。
- 日本語：このルートは Web サービスだけでなく、バッチ、worker、定期ジョブにも向いていることを理解する。
