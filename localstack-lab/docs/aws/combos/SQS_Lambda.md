# SQS + Lambda 学习笔记（中日对照）

SQS + Lambda 是非常经典的异步处理组合，适合学习事件驱动、削峰填谷和解耦。
## 0. 说明概要 / 説明概要

- 中文：SQS 负责缓冲消息，Lambda 负责异步消费处理，适合学习事件驱动和解耦系统的基础链路。
- 日本語：SQS がメッセージをバッファし、Lambda が非同期で処理します。イベント駆動と疎結合の基本を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：SQS 负责存消息，Lambda 负责消费和处理消息。
- 日本語：SQS はメッセージを保持し、Lambda はそれを取り出して処理します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| SQS | 消息队列 | メッセージキュー |
| Lambda | 无服务器处理器 | サーバーレスの処理関数 |
| Producer | 发送消息的一方 | メッセージ送信側 |
| Consumer | 处理消息的一方 | メッセージ処理側 |

## 3. 典型流程 / 典型フロー

1. 上游系统把消息写入 SQS。
2. Lambda 监听队列。
3. 有消息时 Lambda 被触发执行。
4. 处理成功后消息被移除或确认。
5. 失败时可以进入重试或死信队列。

日本語：
1. 上流システムが SQS にメッセージを送る。
2. Lambda がキューを監視する。
3. メッセージが来ると Lambda が起動する。
4. 成功すればメッセージは処理済みになる。
5. 失敗時は再試行や DLQ に送られる。

## 4. 学习重点 / 学習ポイント

- 中文：理解异步解耦和削峰填谷。
- 日本語：非同期の分離とピーク平準化を理解する。
- 中文：理解可见性超时、重试和死信队列。
- 日本語：可視性タイムアウト、再試行、DLQ を理解する。
- 中文：理解 Lambda 适合短时无状态任务。
- 日本語：Lambda は短時間・ステートレス処理に向いていることを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：消息处理超时，导致重复消费。
- 日本語：処理時間が長すぎて重複消費が起きる。
- 中文：没有设计 DLQ，失败消息难排查。
- 日本語：DLQ がないと失敗メッセージを追跡しにくい。
