# Redshift + QuickSight 学习笔记（中日对照）

Redshift + QuickSight 适合学习经典的数据仓库 BI 链路，重点是“仓库查询结果如何服务业务展示”。
## 0. 说明概要 / 説明概要

- 中文：Redshift 负责仓库级分析，QuickSight 负责仪表盘展示，适合学习经典 BI 分析与共享链路。
- 日本語：Redshift が DWH 分析を担当し、QuickSight がダッシュボード表示を担います。典型的な BI 分析と共有の流れを学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Redshift 负责仓库级分析，QuickSight 负责将查询结果可视化给业务用户。
- 日本語：Redshift がデータウェアハウス分析を担当し、QuickSight が結果を業務ユーザー向けに可視化します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Redshift | 数据仓库 | データウェアハウス |
| QuickSight | BI 可视化 | BI 可視化 |
| Dataset | 数据集 | データセット |
| Dashboard | 仪表盘 | ダッシュボード |

## 3. 典型流程 / 典型フロー

1. 数据加载到 Redshift。
2. Redshift 执行聚合、筛选和分析查询。
3. QuickSight 连接 Redshift 作为数据集来源。
4. 业务人员查看报表和仪表盘。
5. 分析结果用于经营判断和共享阅读。

日本語：
1. データが Redshift に取り込まれる。
2. Redshift で集計、絞り込み、分析クエリを実行する。
3. QuickSight が Redshift をデータセット元として接続する。
4. 業務担当者がレポートやダッシュボードを見る。
5. 分析結果を経営判断や共有閲覧に使う。

## 4. 学习重点 / 学習ポイント

- 中文：理解数据仓库和 BI 工具的职责边界。
- 日本語：DWH と BI ツールの責務境界を理解する。
- 中文：理解快速共享的是分析结果，而不是底层建模细节。
- 日本語：共有されるのはモデルの細部ではなく分析結果である。
- 中文：理解报表层应该尽量服务业务问题。
- 日本語：レポート層はできるだけ業務課題に寄せるべきである。

## 5. 常见坑 / よくある落とし穴

- 中文：让 BI 直接承担过多计算，导致卡顿。
- 日本語：BI に計算を持たせすぎて重くなる。
- 中文：仓库建模和报表口径不统一。
- 日本語：DWH モデリングとレポートの指標定義が一致しない。
