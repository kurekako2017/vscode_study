# S3 + CloudFront 学习笔记（中日对照）
## 0. 说明概要 / 説明概要

- 中文：S3 负责静态资源存储，CloudFront 负责全球加速分发，适合学习静态网站和边缘缓存的基础链路。
- 日本語：S3 が静的リソースを保存し、CloudFront がグローバル配信を担当します。静的サイトとエッジキャッシュの基本構成を学ぶのに適しています。

S3 + CloudFront 是最常见的静态资源分发组合，适合学习“对象存储 + CDN 加速”的整体思路。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：S3 负责存储静态文件，CloudFront 负责把内容缓存到边缘节点并加速访问。
- 日本語：S3 は静的ファイルを保存し、CloudFront はそれをエッジでキャッシュして高速配信します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| S3 | 静态资源源站 | 静的ファイルのオリジン |
| CloudFront | CDN、边缘缓存 | CDN、エッジキャッシュ |
| Origin | 源站 | オリジン |
| Cache Policy | 缓存策略 | キャッシュポリシー |

## 3. 典型流程 / 典型フロー

1. 用户访问域名。
2. 请求先到 CloudFront。
3. CloudFront 命中缓存则直接返回。
4. 未命中时回源到 S3。
5. 结果返回后写入边缘缓存。

日本語：
1. ユーザーがドメインへアクセスする。
2. リクエストはまず CloudFront に届く。
3. キャッシュがあればそのまま返す。
4. なければ S3 をオリジンとして取りに行く。
5. 返却後、エッジキャッシュに保存する。

## 4. 学习重点 / 学習ポイント

- 中文：理解“源站”和“边缘缓存”的分工。
- 日本語：「オリジン」と「エッジキャッシュ」の役割分担を理解する。
- 中文：理解缓存命中和回源的区别。
- 日本語：キャッシュヒットとオリジンフェッチの違いを理解する。
- 中文：理解静态网站发布和全球加速的价值。
- 日本語：静的サイト配信とグローバル高速化の価値を理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：缓存策略配错，导致更新不及时。
- 日本語：キャッシュ設定を誤って更新が反映されない。
- 中文：把 S3 公开桶和 CloudFront 分发策略混在一起理解。
- 日本語：S3 の公開設定と CloudFront の配信設定を混同する。
