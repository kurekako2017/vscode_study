# Kinesis + Lambda 学习笔记（中日对照）

Kinesis + Lambda 适合学习实时流处理，尤其是“持续写入、事件触发、近实时加工”的分析链路。
## 0. 说明概要 / 説明概要

- 中文：Kinesis 提供实时数据流，Lambda 负责近实时消费和处理，适合学习流式计算和事件驱动加工。
- 日本語：Kinesis がリアルタイムストリームを提供し、Lambda が準リアルタイムで処理します。ストリーム処理とイベント駆動加工を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Kinesis 提供实时数据流，Lambda 作为消费者对流数据做处理。
- 日本語：Kinesis がリアルタイムデータストリームを提供し、Lambda がコンシューマーとして処理します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Kinesis | 实时数据流 | リアルタイムストリーム |
| Lambda | 流处理函数 | ストリーム処理関数 |
| Shard | 分片 | シャード |
| Consumer | 消费者 | コンシューマー |

## 3. 典型流程 / 典型フロー

1. 应用持续把事件写入 Kinesis。
2. Kinesis 按分片保存和分发数据。
3. Lambda 订阅流并处理记录。
4. 处理结果写入 S3、数据库或监控系统。
5. 通过 CloudWatch 观察吞吐、错误和延迟。

日本語：
1. アプリが継続的にイベントを Kinesis に送る。
2. Kinesis がシャード単位でデータを保持・配信する。
3. Lambda がストリームを購読してレコードを処理する。
4. 結果を S3、DB、監視基盤へ出力する。
5. CloudWatch でスループット、エラー、遅延を確認する。

## 4. 学习重点 / 学習ポイント

- 中文：理解流处理和批处理的区别。
- 日本語：ストリーム処理とバッチ処理の違いを理解する。
- 中文：理解分片会影响并行度和吞吐量。
- 日本語：シャードが並列度とスループットに影響することを理解する。
- 中文：理解 Lambda 可以作为近实时消费层。
- 日本語：Lambda を準リアルタイムの消費層として使えることを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：没有考虑分片设计，导致吞吐不足。
- 日本語：シャード設計を考えず、スループットが不足する。
- 中文：处理函数太慢，造成积压。
- 日本語：処理関数が遅く、滞留が発生する。
