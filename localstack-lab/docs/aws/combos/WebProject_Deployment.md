# Web Project 部署流程（中日对照）

这个页面用“传统 Web 项目部署链路”来串起 AWS 的常见组合，适合学习源码管理、构建打包、域名解析、负载均衡和 EC2 部署的完整流程。

## 0. 说明概要 / 説明概要

- 中文：以传统 Web 项目为例，先通过源码管理保存代码，再构建打包并部署到 EC2，配合 Route 53、ALB、Auto Scaling 和 CloudWatch 组成标准链路。
- 日本語：典型的な Web プロジェクトを例に、ソース管理でコードを保管し、ビルドして EC2 に配置し、Route 53、ALB、Auto Scaling、CloudWatch で標準構成を学びます。

## 1. 这个流程是什么 / このフローは何か

- 中文：用户访问域名后，请求先到 ALB，再分发到后端 EC2；代码先经过源码管理和构建打包，再部署到实例上，配合 Auto Scaling 做扩缩容。
- 日本語：ユーザーがドメインにアクセスすると、リクエストは ALB に届き、背後の EC2 に振り分けられます。コードはソース管理とビルドを経てインスタンスへ配置され、Auto Scaling で増減します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Route 53 | 域名解析入口 | DNS 入口 |
| Source Control / CodeCommit / GitHub | 源码管理入口 | ソース管理の入口 |
| Build / Package | 构建与打包 | ビルドとパッケージング |
| ALB | HTTP/HTTPS 负载均衡 | L7 ロードバランサー |
| EC2 | Web 应用运行实例 | Web アプリの実行先 |
| Auto Scaling | 自动扩缩容 | 自動スケーリング |
| CloudWatch | 监控与告警 | 監視とアラーム |
| Security Group | 网络访问控制 | ネットワーク制御 |

## 3. 典型流程 / 典型フロー

1. 先在 GitHub 或 CodeCommit 管理源代码。
2. 在 CI 或手动流程中完成构建和打包。
3. 用 Route 53 绑定域名，ALB 作为统一入口。
4. ALB 根据规则把流量分发到健康的 EC2 实例。
5. Auto Scaling 根据负载调整实例数量，CloudWatch 监控响应时间、错误率和资源使用率。

日本語：
1. Route 53 でドメインを紐付ける。
2. ALB を共通入口としてリクエストを受ける。
3. ALB が正常な EC2 にトラフィックを振り分ける。
4. Auto Scaling が負荷に応じて台数を調整する。
5. CloudWatch が応答時間、エラー率、リソース使用率を監視する。

## 4. 学习重点 / 学習ポイント

- 中文：理解“源码管理 → 构建打包 → 域名 → 负载均衡 → 应用实例”的部署主链路。
- 日本語：「ソース管理 → ビルド → ドメイン → ロードバランサー → アプリ実行先」の流れを理解する。
- 中文：理解健康检查和自动扩缩容如何协同工作。
- 日本語：ヘルスチェックと自動スケーリングの連携を理解する。
- 中文：理解传统 Web 项目通常不走 Docker / ECR / ECS，而是源码管理 + 构建 + EC2 部署。
- 日本語：従来型の Web デプロイでは Docker / ECR / ECS を使わず、ソース管理 + ビルド + EC2 配置で考えることを理解する。
