# QuickSight + Lake Formation 学习笔记（中日对照）

QuickSight + Lake Formation 适合学习“可视化访问受控数据”的场景，重点是业务共享与数据安全的平衡。
## 0. 说明概要 / 説明概要

- 中文：Lake Formation 控制数据访问，QuickSight 负责共享报表和仪表盘，适合学习企业共享分析中的权限边界。
- 日本語：Lake Formation がデータアクセスを制御し、QuickSight が共有レポートとダッシュボードを提供します。企業共有分析の権限境界を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Lake Formation 负责授权，QuickSight 负责把被授权的数据做成报表和仪表盘。
- 日本語：Lake Formation が認可を担当し、QuickSight が許可されたデータをレポートやダッシュボードにします。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Lake Formation | 访问控制 | アクセス制御 |
| QuickSight | BI 可视化 | BI 可視化 |
| Athena / Redshift | 数据查询来源 | クエリ元 |
| Dashboard | 共享看板 | 共有ダッシュボード |

## 3. 典型流程 / 典型フロー

1. Lake Formation 为数据资产设置权限。
2. QuickSight 连接 Athena 或 Redshift 等受控数据源。
3. 用户只能基于授权数据创建报表。
4. 仪表盘面向团队或业务部门共享。
5. 安全边界保持在数据层，而不是只靠报表层隐藏。

日本語：
1. Lake Formation でデータ資産の権限を設定する。
2. QuickSight が Athena や Redshift などの制御されたデータソースに接続する。
3. ユーザーは許可されたデータだけでレポートを作成する。
4. ダッシュボードをチームや部門へ共有する。
5. セキュリティ境界はレポート層だけでなくデータ層に置く。

## 4. 学习重点 / 学習ポイント

- 中文：理解 BI 工具和数据治理是两层控制。
- 日本語：BI ツールとデータガバナンスは二層の制御である。
- 中文：理解共享看板不等于共享所有原始数据。
- 日本語：共有ダッシュボードは全生データ共有ではない。
- 中文：理解授权要尽量前置到数据层。
- 日本語：権限はできるだけデータ層で先に制御する。

## 5. 常见坑 / よくある落とし穴

- 中文：只限制报表下载，不限制底层数据访问。
- 日本語：レポートのダウンロードだけを制限して、元データのアクセスを止めない。
- 中文：团队共享后权限边界不清晰。
- 日本語：チーム共有後に権限境界が曖昧になる。
