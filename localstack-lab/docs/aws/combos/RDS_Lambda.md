# RDS + Lambda 学习笔记（中日对照）

RDS + Lambda 常见于数据库自动化、轻量数据处理和定时运维任务，适合学习“数据库 + 计算函数”的联动方式。
## 0. 说明概要 / 説明概要

- 中文：RDS 提供结构化数据存储，Lambda 负责查询、同步或轻量转换，适合学习数据库与函数计算的联动方式。
- 日本語：RDS が構造化データを保存し、Lambda が検索・同期・軽量変換を担当します。DB と関数計算の連携を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：RDS 保存结构化数据，Lambda 负责查询、同步、转换或触发后续动作。
- 日本語：RDS が構造化データを保存し、Lambda が検索、同期、変換、後続処理を担当します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| RDS | 托管关系型数据库 | マネージド RDB |
| Lambda | 计算函数 | コンピュート関数 |
| Event Source | 事件源 | イベントソース |
| JDBC / SDK | 数据访问方式 | データアクセス手段 |

## 3. 典型流程 / 典型フロー

1. 数据写入 RDS。
2. Lambda 通过计划任务、事件或调用读取数据。
3. Lambda 执行转换、清洗或同步逻辑。
4. 处理结果写回 RDS、S3 或外部系统。
5. CloudWatch 记录执行日志并辅助排障。

日本語：
1. データが RDS に書き込まれる。
2. Lambda が定期実行、イベント、または呼び出しでデータを読む。
3. Lambda が変換、整形、同期の処理を行う。
4. 結果を RDS、S3、外部システムへ書き戻す。
5. CloudWatch が実行ログを残して調査を助ける。

## 4. 学习重点 / 学習ポイント

- 中文：理解 Lambda 可以作为轻量数据处理层。
- 日本語：Lambda を軽量なデータ処理層として使えることを理解する。
- 中文：理解数据库访问的连接和权限问题。
- 日本語：DB 接続と権限の問題を理解する。
- 中文：理解计算函数不一定直接替代应用层服务。
- 日本語：Lambda が必ずしもアプリケーション層を完全に置き換えるわけではないことを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：Lambda 连接数据库时没有控制连接数。
- 日本語：Lambda から DB へ接続するときに接続数を制御しない。
- 中文：函数职责太重，导致超时。
- 日本語：関数の責務が重すぎてタイムアウトする。
