# SQS + SNS 学习笔记（中日对照）

SQS + SNS 是典型的消息分发组合，适合学习广播通知、解耦消费和异步处理。
## 0. 说明概要 / 説明概要

- 中文：SNS 负责广播消息，SQS 负责排队消费，适合学习消息分发和异步处理的分层设计。
- 日本語：SNS がメッセージをブロードキャストし、SQS がキューイングして消費します。メッセージ配信と非同期処理の分離を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：SNS 负责广播通知，SQS 负责排队处理，二者经常一起用。
- 日本語：SNS は通知のブロードキャストを担当し、SQS はキュー処理を担当します。両者はよく組み合わせて使われます。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| SNS | 广播通知 | 通知の配信 |
| SQS | 消息队列 | メッセージキュー |
| Publisher | 发布方 | 発行元 |
| Subscriber | 订阅方 | 購読先 |
| Queue | 队列 | キュー |

## 3. 典型流程 / 典型フロー

1. 发布方把消息发到 SNS Topic。
2. SNS 将消息广播给多个订阅者。
3. 一部分订阅者可以是 SQS 队列。
4. SQS 负责把消息排队。
5. 消费者按自己的节奏从 SQS 拉取并处理。

日本語：
1. 発行元が SNS Topic にメッセージを送る。
2. SNS が複数の購読先へメッセージを配信する。
3. 購読先の一部は SQS キューになる。
4. SQS がメッセージをキューに積む。
5. コンシューマーが自分のペースで取り出して処理する。

## 4. 学习重点 / 学習ポイント

- 中文：理解“广播”和“排队”的分工。
- 日本語：「ブロードキャスト」と「キューイング」の役割分担を理解する。
- 中文：理解一个 Topic 可以连接多个队列或订阅端。
- 日本語：1 つの Topic から複数のキューや購読先に接続できることを理解する。
- 中文：理解消息分发和消息消费是两层问题。
- 日本語：メッセージ配信とメッセージ消費は別レイヤーの問題であると理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：把 SNS 当成单纯的消息队列。
- 日本語：SNS を単なるメッセージキューだと考える。
- 中文：把 SQS 当成广播工具。
- 日本語：SQS をブロードキャストツールだと考える。
