# Athena + Glue 学习笔记（中日对照）

Athena + Glue 是查询与元数据管理的经典组合，适合学习“表定义、目录编目和无服务器查询”之间的关系。
## 0. 说明概要 / 説明概要

- 中文：Glue 提供目录和元数据，Athena 基于这些目录直接查询 S3 数据，适合学习查询与编目联动。
- 日本語：Glue がカタログとメタデータを提供し、Athena がその定義で S3 を直接検索します。クエリとカタログの連携を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Glue 负责发现和管理数据目录，Athena 通过这些目录直接查询 S3 上的数据。
- 日本語：Glue がデータカタログを検出・管理し、Athena はそのカタログを使って S3 上のデータを直接検索します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Glue | 数据目录与 ETL | データカタログと ETL |
| Athena | 无服务器查询 | サーバーレスクエリ |
| Catalog | 元数据目录 | メタデータカタログ |
| S3 | 数据存储 | データ保存先 |

## 3. 典型流程 / 典型フロー

1. 数据先进入 S3。
2. Glue Crawler 扫描目录并更新 Catalog。
3. Athena 读取 Glue Catalog 中的表定义。
4. 用户执行 SQL 查询 S3 中的数据。
5. 查询结果可继续输出到 S3 或报表系统。

日本語：
1. データが S3 に保存される。
2. Glue Crawler がディレクトリを走査し Catalog を更新する。
3. Athena が Glue Catalog のテーブル定義を読む。
4. ユーザーが SQL で S3 のデータを検索する。
5. 結果を S3 やレポート基盤に出力できる。

## 4. 学习重点 / 学習ポイント

- 中文：理解 Glue 是 Athena 的元数据来源之一。
- 日本語：Glue が Athena のメタデータ源のひとつであることを理解する。
- 中文：理解“表定义”和“数据文件”不是同一个东西。
- 日本語：テーブル定義とデータファイルは別物だと理解する。
- 中文：理解 Catalog 是分析链路的关键基础设施。
- 日本語：Catalog が分析基盤の重要な土台であることを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：文件路径变化后忘记更新目录。
- 日本語：ファイルパスが変わったのにカタログを更新しない。
- 中文：表结构和实际数据格式不一致。
- 日本語：テーブル構造と実データ形式が一致しない。
