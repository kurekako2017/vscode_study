# Lake Formation + Glue 学习笔记（中日对照）

Lake Formation + Glue 适合学习数据治理和数据目录管理，重点在“权限控制、目录编目、数据共享前的准备”。
## 0. 说明概要 / 説明概要

- 中文：Lake Formation 负责权限治理，Glue 负责编目和 ETL，适合学习数据共享前的治理与准备。
- 日本語：Lake Formation が権限ガバナンスを担い、Glue がカタログ化と ETL を担います。共有前のガバナンスと準備を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Lake Formation 负责数据治理和权限控制，Glue 负责目录编目和 ETL。
- 日本語：Lake Formation がデータガバナンスと権限制御を担当し、Glue がカタログ化と ETL を担当します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Lake Formation | 数据治理与权限管理 | データガバナンスと権限管理 |
| Glue | 数据目录与 ETL | データカタログと ETL |
| Catalog | 元数据目录 | メタデータカタログ |
| S3 | 数据存储 | データ保存先 |

## 3. 典型流程 / 典型フロー

1. 数据先进入 S3。
2. Glue 扫描并更新 Catalog。
3. Lake Formation 对数据库、表和列设置访问权限。
4. 共享用户或团队通过受控权限访问数据。
5. 下游分析工具仅看到被授权的数据资产。

日本語：
1. データがまず S3 に保存される。
2. Glue が走査して Catalog を更新する。
3. Lake Formation が DB、テーブル、列単位で権限を設定する。
4. 共有先のユーザーやチームが制御された権限でデータにアクセスする。
5. 下流の分析ツールは許可されたデータ資産だけを見る。

## 4. 学习重点 / 学習ポイント

- 中文：理解治理层和编目层的区别。
- 日本語：ガバナンス層とカタログ層の違いを理解する。
- 中文：理解权限不是只针对文件，也可以针对表和列。
- 日本語：権限はファイルだけでなく、テーブルや列にも適用できる。
- 中文：理解数据共享前要先做结构和权限整理。
- 日本語：データ共有の前に構造と権限を整理する必要がある。

## 5. 常见坑 / よくある落とし穴

- 中文：只建目录，不设治理规则。
- 日本語：カタログだけ作ってガバナンスを設定しない。
- 中文：共享粒度过粗，导致越权访问风险。
- 日本語：共有粒度が粗すぎて権限逸脱のリスクがある。
