# S3 + SNS 学习笔记（中日对照）

S3 + SNS 适合学习“对象上传后发出通知”的事件驱动模式，常见于文件分发、处理通知和后续异步任务触发。
## 0. 说明概要 / 説明概要

- 中文：S3 负责存储对象，SNS 负责把对象事件通知给订阅者，适合学习上传后通知和后续异步联动。
- 日本語：S3 がオブジェクトを保存し、SNS がオブジェクトイベントを購読者へ通知します。アップロード後の通知と後続の非同期連携を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：S3 负责存储对象，SNS 负责把对象事件通知给订阅者。
- 日本語：S3 がオブジェクトを保存し、SNS がオブジェクトイベントを購読者へ通知します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| S3 | 对象存储 | オブジェクトストレージ |
| SNS | 通知发布 | 通知の配信 |
| Event Notification | 事件通知 | イベント通知 |
| Subscriber | 订阅者 | 購読者 |

## 3. 典型流程 / 典型フロー

1. 文件上传到 S3。
2. S3 产生对象创建事件。
3. 事件发送到 SNS Topic。
4. SNS 将消息推送给多个订阅者。
5. 订阅者继续执行告警、同步或异步处理。

日本語：
1. ファイルが S3 にアップロードされる。
2. S3 でオブジェクト作成イベントが発生する。
3. そのイベントが SNS Topic に送られる。
4. SNS が複数の購読者へ配信する。
5. 購読者が通知、同期、非同期処理を続ける。

## 4. 学习重点 / 学習ポイント

- 中文：理解对象事件如何变成通知。
- 日本語：オブジェクトイベントが通知に変わる流れを理解する。
- 中文：理解 SNS 适合多订阅者分发。
- 日本語：SNS が複数購読者への配信に向いていることを理解する。
- 中文：理解上传、通知、后续处理是分层的。
- 日本語：アップロード、通知、後続処理が層になっていることを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：只设置存储，不设置事件通知。
- 日本語：保存だけしてイベント通知を設定しない。
- 中文：把 SNS 当成存储层使用。
- 日本語：SNS を保存層のように扱う。
