# SQS + Lambda + DLQ 学习笔记（中日对照）

SQS + Lambda + DLQ 是非常典型的异步事件处理组合，适合学习“消费失败后的兜底与重试”设计。
## 0. 说明概要 / 説明概要

- 中文：SQS 负责消息缓冲，Lambda 负责消费，DLQ 负责兜底失败消息，适合学习异步处理的重试和补偿链路。
- 日本語：SQS がメッセージをバッファし、Lambda が処理し、DLQ が失敗メッセージを受け止めます。再試行と補償を含む非同期処理を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：SQS 提供消息缓冲，Lambda 负责消费处理，DLQ 负责接住多次失败的消息。
- 日本語：SQS がメッセージをバッファし、Lambda が処理し、DLQ が失敗メッセージを受け止めます。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| SQS | 主队列 | メインキュー |
| Lambda | 消费函数 | コンシューマー関数 |
| DLQ | 死信队列 | デッドレターキュー |
| Retry | 重试 | リトライ |
| Visibility Timeout | 可见性超时 | 可視性タイムアウト |

## 3. 典型流程 / 典型フロー

1. 生产者发送消息到 SQS。
2. Lambda 从 SQS 拉取消息并处理。
3. 如果处理失败，消息按重试策略再次投递。
4. 超过最大重试次数后，消息进入 DLQ。
5. 运维或开发者查看 DLQ 排查原因并补偿处理。

日本語：
1. 送信元が SQS にメッセージを送る。
2. Lambda が SQS からメッセージを取得して処理する。
3. 処理に失敗すると、再試行ポリシーに従って再配信される。
4. 最大再試行回数を超えると DLQ に移る。
5. 運用者や開発者が DLQ を確認して原因調査と補償処理を行う。

## 4. 学习重点 / 学習ポイント

- 中文：理解 DLQ 是异步系统的重要保险丝。
- 日本語：DLQ が非同期システムの重要な保険であることを理解する。
- 中文：理解重试、超时和幂等性必须一起设计。
- 日本語：リトライ、タイムアウト、冪等性を同時に設計する必要がある。
- 中文：理解主队列和死信队列的职责不同。
- 日本語：メインキューと DLQ の役割が違うことを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：没有 DLQ，失败消息直接丢失或反复堆积。
- 日本語：DLQ がなく、失敗メッセージが失われるか再処理され続ける。
- 中文：Lambda 处理函数没有幂等性。
- 日本語：Lambda の処理に冪等性がない。
