# RDS + CloudWatch 学习笔记（中日对照）

RDS + CloudWatch 是数据库运维里非常常见的组合，适合学习“托管数据库 + 监控告警”的基础架构。
## 0. 说明概要 / 説明概要

- 中文：RDS 提供托管数据库，CloudWatch 提供指标、日志和告警，适合学习数据库运维监控的基础链路。
- 日本語：RDS がマネージド DB を提供し、CloudWatch がメトリクス・ログ・アラームを担います。DB 運用監視の基礎を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：RDS 负责提供托管数据库，CloudWatch 负责监控指标、日志和告警。
- 日本語：RDS はマネージドデータベースを提供し、CloudWatch はメトリクス・ログ・アラームを監視します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| RDS | 托管数据库 | マネージド DB |
| CloudWatch | 监控与告警 | 監視とアラーム |
| Metric | 监控指标 | メトリクス |
| Alarm | 告警 | アラーム |
| Log | 日志 | ログ |

## 3. 典型流程 / 典型フロー

1. 应用读写 RDS。
2. RDS 产生性能指标或错误信号。
3. CloudWatch 收集指标和日志。
4. 指标超过阈值时触发告警。
5. 告警可以联动 SNS 或运维通知。

日本語：
1. アプリが RDS を読み書きする。
2. RDS から性能メトリクスやエラーの兆候が出る。
3. CloudWatch がメトリクスとログを収集する。
4. 閾値を超えるとアラームが発報する。
5. アラームは SNS や運用通知と連携できる。

## 4. 学习重点 / 学習ポイント

- 中文：理解数据库和监控是一起设计的。
- 日本語：データベース設計と監視設計はセットで考える。
- 中文：理解告警不是目的，定位问题才是目的。
- 日本語：アラーム自体が目的ではなく、原因特定が目的である。
- 中文：理解 CloudWatch 指标、日志、告警之间的关系。
- 日本語：CloudWatch のメトリクス、ログ、アラームの関係を理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：只部署数据库，不做监控和告警。
- 日本語：DB だけ置いて監視やアラームを設定しない。
- 中文：告警阈值过于宽松或过于敏感。
- 日本語：アラーム閾値が緩すぎたり厳しすぎたりする。
