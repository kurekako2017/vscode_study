# ECS + ECR + Fargate 学习笔记（中日对照）

ECS + ECR + Fargate 是容器应用上线的典型组合，适合学习“构建镜像 -> 存储镜像 -> 运行容器”的完整链路。
## 0. 说明概要 / 説明概要

- 中文：ECR 负责镜像仓库，ECS + Fargate 负责无服务器容器运行，适合学习容器交付与运行的完整链路。
- 日本語：ECR がイメージ保管庫を担い、ECS + Fargate がサーバーレスでコンテナを実行します。コンテナの配布と実行の流れを学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：ECR 存镜像，ECS 负责编排，Fargate 负责无服务器运行容器。
- 日本語：ECR はイメージを保存し、ECS はオーケストレーションを担当し、Fargate はサーバーレスでコンテナを実行します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| ECR | 镜像仓库 | イメージレジストリ |
| ECS | 容器编排 | コンテナオーケストレーション |
| Fargate | 容器运行方式 | コンテナ実行方式 |
| Task Definition | 任务定义 | タスク定義 |
| Cluster | 集群 | クラスター |

## 3. 典型流程 / 典型フロー

1. 开发者构建镜像。
2. 镜像推送到 ECR。
3. ECS 读取任务定义。
4. Fargate 按任务定义启动容器。
5. 对外提供服务。

日本語：
1. 開発者がイメージをビルドする。
2. イメージを ECR に push する。
3. ECS がタスク定義を参照する。
4. Fargate が定義に従ってコンテナを起動する。
5. サービスとして公開する。

## 4. 学习重点 / 学習ポイント

- 中文：理解“构建、存储、运行”三段式链路。
- 日本語：「ビルド、保存、実行」の 3 段階を理解する。
- 中文：理解 ECS 和 Fargate 的职责边界。
- 日本語：ECS と Fargate の役割の違いを理解する。
- 中文：理解镜像版本和任务定义的关系。
- 日本語：イメージのバージョンとタスク定義の関係を理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：镜像推了新版本，但任务定义还在用旧标签。
- 日本語：新しいイメージを push しても、タスク定義が古いタグのまま。
- 中文：把 ECR、ECS、Fargate 的职责混在一起。
- 日本語：ECR、ECS、Fargate の役割を混同する。
