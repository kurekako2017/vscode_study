# Lake Formation + Athena 学习笔记（中日对照）

Lake Formation + Athena 适合学习“受控共享下的无服务器查询”，重点是权限边界和可见数据范围。
## 0. 说明概要 / 説明概要

- 中文：Lake Formation 负责权限控制，Athena 负责受控查询，适合学习企业共享场景下的数据访问边界。
- 日本語：Lake Formation が権限制御を担い、Athena が制御された範囲で検索します。企業共有におけるデータアクセス境界を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：Lake Formation 负责数据权限控制，Athena 负责在授权范围内查询 S3 数据。
- 日本語：Lake Formation がデータ権限を制御し、Athena が許可範囲内で S3 データを検索します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| Lake Formation | 数据权限边界 | データ権限の境界 |
| Athena | 无服务器查询 | サーバーレスクエリ |
| Glue Catalog | 元数据目录 | メタデータカタログ |
| S3 | 数据存储 | データ保存先 |

## 3. 典型流程 / 典型フロー

1. 数据存入 S3 并由 Glue 建立目录。
2. Lake Formation 配置数据库、表或列级权限。
3. Athena 通过目录查询数据。
4. 用户只能看到被授权的数据集。
5. 查询结果用于共享分析或报表。

日本語：
1. データが S3 に保存され、Glue でカタログ化される。
2. Lake Formation で DB、テーブル、列単位の権限を設定する。
3. Athena がカタログ経由でデータを検索する。
4. ユーザーは許可されたデータセットだけを見る。
5. 結果を共有分析やレポートに使う。

## 4. 学习重点 / 学習ポイント

- 中文：理解查询工具也要受权限边界约束。
- 日本語：クエリツールも権限境界の制約を受けることを理解する。
- 中文：理解目录权限和数据权限要一起设计。
- 日本語：カタログ権限とデータ権限をセットで設計する。
- 中文：理解共享分析不是把所有数据都开放。
- 日本語：共有分析は全データを開放することではない。

## 5. 常见坑 / よくある落とし穴

- 中文：目录能看到，但表或列没权限。
- 日本語：カタログは見えても、テーブルや列に権限がない。
- 中文：只配置 Athena，不处理 Lake Formation 权限。
- 日本語：Athena だけ設定して Lake Formation 権限を整備しない。
