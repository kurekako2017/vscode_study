# EC2 + ALB 学习笔记（中日对照）

EC2 + ALB 是最典型的 Web 服务部署组合，适合学习“负载均衡 + 多实例后端”的架构。
## 0. 说明概要 / 説明概要

- 中文：EC2 提供计算实例，ALB 负责把流量分发到后端目标，适合学习应用扩容和负载均衡的基础架构。
- 日本語：EC2 が計算インスタンスを提供し、ALB がトラフィックを後段へ振り分けます。アプリのスケールと負荷分散の基礎を学ぶのに向いています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：ALB 接收用户请求并把流量分发到后端 EC2 实例。
- 日本語：ALB はユーザーのリクエストを受け取り、背後の EC2 インスタンスへ振り分けます。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| EC2 | 应用运行实例 | アプリを動かすインスタンス |
| ALB | 应用层负载均衡 | アプリケーションロードバランサー |
| Target Group | 目标组 | ターゲットグループ |
| Health Check | 健康检查 | ヘルスチェック |

## 3. 典型流程 / 典型フロー

1. 用户访问入口域名。
2. 请求先到 ALB。
3. ALB 根据规则转发到目标组。
4. 目标组中健康的 EC2 处理请求。
5. 如果某台机器异常，ALB 会避开它。

日本語：
1. ユーザーが入口ドメインへアクセスする。
2. リクエストは ALB に届く。
3. ALB がルールに従ってターゲットグループへ振り分ける。
4. ターゲットグループ内の正常な EC2 が処理する。
5. 異常なインスタンスはヘルスチェックで除外される。

## 4. 学习重点 / 学習ポイント

- 中文：理解 ALB 为什么比直接访问 EC2 更适合生产环境。
- 日本語：直接 EC2 にアクセスするより ALB が本番向きな理由を理解する。
- 中文：理解健康检查和流量切换。
- 日本語：ヘルスチェックとトラフィック切り替えを理解する。
- 中文：理解水平扩展的基本思路。
- 日本語：水平スケールの基本を理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：只开了 ALB，没有正确配置安全组。
- 日本語：ALB を作っただけでセキュリティグループを正しく設定しない。
- 中文：健康检查路径写错，导致实例一直不健康。
- 日本語：ヘルスチェックパスの設定ミスで常に不健康判定になる。
