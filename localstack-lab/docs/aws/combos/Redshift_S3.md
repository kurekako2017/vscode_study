# Redshift + S3 学习笔记（中日对照）

Redshift + S3 是典型的数据仓库与数据湖组合，适合学习“海量数据落地、装载、分析”的离线分析链路。
## 0. 说明概要 / 説明概要

- 中文：S3 负责数据落地和中转，Redshift 负责大规模结构化分析，适合学习数据仓库与数据湖的配合方式。
- 日本語：S3 がデータの保存と中継を担い、Redshift が大規模な構造化分析を担当します。データレイクと DWH の連携を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：S3 负责保存原始或中间数据，Redshift 负责大规模结构化分析。
- 日本語：S3 が生データや中間データを保存し、Redshift が大規模な構造化分析を担当します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| S3 | 数据湖/文件中转 | データレイク / ファイル置き場 |
| Redshift | 数据仓库 | データウェアハウス |
| ETL | 数据加载与转换 | データロードと変換 |
| Query | 分析查询 | 分析クエリ |

## 3. 典型流程 / 典型フロー

1. 数据先落到 S3。
2. ETL 作业清洗并准备数据。
3. 数据从 S3 导入 Redshift。
4. 分析人员在 Redshift 上执行 SQL。
5. 结果用于报表、看板或后续分析。

日本語：
1. データがまず S3 に保存される。
2. ETL ジョブがデータを整形する。
3. S3 から Redshift にデータを取り込む。
4. 分析担当者が Redshift 上で SQL を実行する。
5. 結果をレポート、ダッシュボード、追加分析に使う。

## 4. 学习重点 / 学習ポイント

- 中文：理解 S3 和 Redshift 分别承担什么。
- 日本語：S3 と Redshift の役割を分けて理解する。
- 中文：理解批量导入和查询分析是两条不同链路。
- 日本語：バッチ取り込みと分析クエリは別の流れだと理解する。
- 中文：理解数据仓库并不替代数据湖。
- 日本語：データウェアハウスはデータレイクを置き換えるものではない。

## 5. 常见坑 / よくある落とし穴

- 中文：把原始文件直接当成分析仓库使用。
- 日本語：生ファイルをそのまま分析基盤として使う。
- 中文：导入前没有统一数据格式。
- 日本語：取り込み前にデータ形式を揃えていない。
