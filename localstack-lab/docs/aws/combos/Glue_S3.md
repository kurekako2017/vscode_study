# Glue + S3 学习笔记（中日对照）

Glue + S3 适合学习 ETL 和数据准备流程，尤其是“从原始文件到可分析数据”的转换链路。
## 0. 说明概要 / 説明概要

- 中文：S3 存放原始数据，Glue 负责编目和 ETL，适合学习从原始文件到可分析数据的准备链路。
- 日本語：S3 が生データを保存し、Glue がカタログ化と ETL を担います。生ファイルから分析可能データへの準備を学ぶのに適しています。

## 1. 这个组合是什么 / この組み合わせは何か

- 中文：S3 保存原始数据，Glue 负责发现、转换和编目数据。
- 日本語：S3 が生データを保存し、Glue が検出、変換、カタログ化を担当します。

## 2. 组合中的角色 / 役割分担

| 组件 | 中文说明 | 日本語説明 |
|---|---|---|
| S3 | 原始数据存储 | 生データ保存先 |
| Glue | ETL 与编目 | ETL とカタログ |
| Crawler | 爬虫/探测器 | クローラー |
| Catalog | 数据目录 | データカタログ |
| Job | 转换任务 | 変換ジョブ |

## 3. 典型流程 / 典型フロー

1. 原始数据上传到 S3。
2. Glue Crawler 扫描数据并生成或更新目录。
3. Glue Job 对数据做清洗、转换或分区整理。
4. 处理后的数据写回 S3 的目标目录。
5. 上层分析工具再读取整理后的数据。

日本語：
1. 生データが S3 にアップロードされる。
2. Glue Crawler がデータを走査してカタログを更新する。
3. Glue Job がデータをクレンジング、変換、パーティション整理する。
4. 処理済みデータを S3 の出力先に書き戻す。
5. 上位の分析ツールが整形済みデータを読む。

## 4. 学习重点 / 学習ポイント

- 中文：理解 Glue 是数据准备层，不只是调度工具。
- 日本語：Glue は単なるスケジューラではなく、データ準備層でもある。
- 中文：理解目录、爬虫、作业三者的分工。
- 日本語：カタログ、クローラー、ジョブの分担を理解する。
- 中文：理解 S3 和 Glue 往往一起构成数据湖 ETL。
- 日本語：S3 と Glue がデータレイク ETL を構成することを理解する。

## 5. 常见坑 / よくある落とし穴

- 中文：原始数据格式不稳定，Crawler 结果不可靠。
- 日本語：生データの形式が不安定で、Crawler の結果が安定しない。
- 中文：转换后目录层次混乱，后续查询困难。
- 日本語：変換後のディレクトリ構成が乱れて、後続クエリが難しくなる。
