## 🎯 一键启动

### 方法1：使用 Maven（推荐）

```bash
# 在项目根目录执行
mvn spring-boot:run
```

就这么简单！不需要任何额外参数。

### 方法2：使用 IDEA

1. 打开项目
2. 找到 `JtSpringProjectApplication.java`
3. 右键 → Run 'JtSpringProjectApplication'

### 方法3：运行 JAR 文件

```bash
# 先打包
mvn clean package

# 运行
java -jar target/JtSpringProject-0.0.1-SNAPSHOT.jar
```

## 🔧 配置说明

### 已禁用的测试

测试类位于备份目录（不会被编译）：
```
src/test_disabled_backup/
├── java/
│   └── com/
│       └── jtspringproject/
│           └── JtSpringProject/
│               ├── JtSpringProjectApplicationTests.java
│               └── controller/
│                   ├── AdminControllerTest.java
│                   └── UserControllerTest.java
```

### pom.xml 配置

已在 `pom.xml` 中配置：
```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-surefire-plugin</artifactId>
    <version>3.2.5</version>
    <configuration>
        <skipTests>true</skipTests>
    </configuration>
</plugin>
```

## ⚠️ 如果启动失败

### 端口被占用
```bash
# 查找占用8080端口的进程
netstat -ano | findstr :8080

# 结束进程（替换 PID）
taskkill /F /PID <进程ID>
```

### 数据库问题
删除数据库文件重新初始化：
```bash
Remove-Item -Path "data\ecommjava.mv.db" -Force
```

### Maven 缓存问题
```bash
mvn clean install -U
```

## 🔄 如需重新启用测试

如果将来想重新启用测试类：

### 1. 恢复测试目录
```bash
Rename-Item -Path "src\test_disabled_backup" -NewName "test"
```

### 2. 修改 pom.xml
```xml
<configuration>
    <skipTests>false</skipTests>  <!-- 改为 false -->
</configuration>
```

### 3. 运行测试
```bash
mvn test
```
