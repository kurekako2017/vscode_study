## 1. 打开项目

### 方法A：从欢迎界面打开
1. 启动IDEA
2. 在欢迎界面点击 **Open**
3. 导航到：`D:\dev\source_code\vscode_study\java-projects\JtProject`
4. 选择这个文件夹，点击 **OK**
5. 等待IDEA索引（右下角会显示进度）

### 方法B：从菜单打开
1. IDEA已打开时，点击 **File → Open**
2. 选择项目目录
3. 如果询问"Open in new window?"，选择你想要的方式

### ⚠️ 重要提示
- 第一次打开项目时，IDEA会下载Maven依赖，**这可能需要几分钟**
- 右下角会显示"Downloading..."和"Indexing..."
- **等待所有进度完成后再进行操作**

## 2. 理解项目结构

在左侧的项目面板（**Project** 视图），你会看到：

```
JtProject/                           # 项目根目录
├── .idea/                          # IDEA配置文件（不要手动修改）
├── src/                            # 源代码目录
│   ├── main/                       # 主代码
│   │   ├── java/                   # Java代码
│   │   │   └── com/jtspringproject/JtSpringProject/
│   │   │       ├── JtSpringProjectApplication.java  # 主入口类 ⭐
│   │   │       ├── controller/     # 控制器（处理HTTP请求）
│   │   │       ├── models/         # 数据模型
│   │   │       ├── dao/            # 数据访问层
│   │   │       └── ...
│   │   ├── resources/              # 资源文件
│   │   │   ├── application.properties  # 配置文件 ⭐
│   │   │   └── ...
│   │   └── webapp/                 # Web资源
│   │       └── views/              # JSP页面
│   └── test/                       # 测试代码
├── target/                         # 编译输出目录（自动生成）
├── pom.xml                         # Maven配置文件 ⭐
└── ...
```

### 关键文件说明

| 文件 | 作用 | 需要修改吗 |
|------|------|----------|
| **JtSpringProjectApplication.java** | 项目启动类 | ❌ 一般不需要 |
| **application.properties** | 配置文件（数据库、端口等） | ✅ 可能需要 |
| **pom.xml** | Maven依赖管理 | ✅ 添加依赖时需要 |
| **controller/** | 业务逻辑控制器 | ✅ 经常修改 |
| **models/** | 数据模型 | ✅ 经常修改 |

## 3. 运行项目

### 🎯 最简单的方法（推荐）

#### 步骤1：打开启动类
双击打开：`src/main/java/com/jtspringproject/JtSpringProject/JtSpringProjectApplication.java`

#### 步骤2：找到main方法
看到第41行的：
```java
public static void main(String[] args) {
```

#### 步骤3：运行
**选择以下任一方式：**

**方式1：点击绿色按钮**
- 看main方法左侧（行号旁边）
- 有个**绿色三角形** ▶️
- 点击它，选择 **Run 'JtSpringProjectApplication'**

**方式2：快捷键（最快）**
- 光标放在文件任意位置
- 按 **Ctrl + Shift + F10**

**方式3：右键菜单**
- 在 `main` 方法内右键
- 选择 **Run 'JtSpringProjectApplication.main()'**

**方式4：顶部按钮**
- 确认右上角显示 "JtSpringProjectApplication"
- 点击绿色运行按钮 ▶️

#### 步骤4：查看结果
- 底部会弹出 **Run** 面板
- 看到绿色的日志输出
- 显示 "JT电商系统启动成功！"
- 浏览器访问：`http://localhost:8080`

### 📊 运行成功的标志
```

...

...
```

## 4. 停止项目

### 方法A：使用停止按钮
1. 查看底部 **Run** 面板
2. 点击左侧的**红色方块** 🟥 （停止按钮）

### 方法B：快捷键
按 **Ctrl + F2**

### 方法C：关闭标签页
点击 Run 面板标签页上的 **×**

### ⚠️ 注意
如果项目没有正常停止，可能端口8080会被占用。解决方法：
```powershell
# 在PowerShell中运行
netstat -ano | findstr :8080
# 找到PID，然后
taskkill /F /PID [PID号]
```

## 5. 调试项目

### 什么是调试？
调试可以让你：
- 暂停程序执行
- 查看变量的值
- 一步一步执行代码
- 找出Bug

### 如何调试？

#### 步骤1：设置断点
1. 打开想调试的文件（比如某个Controller）
2. 找到想要暂停的那一行代码
3. 点击行号旁边的空白处
4. 会出现一个**红色圆点** 🔴

#### 步骤2：启动调试
**选择以下任一方式：**

**方式1：点击绿色虫子按钮**
- main方法左侧有个**绿色小虫** 🐞
- 点击它，选择 **Debug 'JtSpringProjectApplication'**

**方式2：快捷键**
按 **Ctrl + Shift + F9**

**方式3：顶部按钮**
右上角点击**绿色小虫按钮** 🐞

#### 步骤3：触发断点
1. 用浏览器访问对应的页面
2. 当代码执行到断点时，IDEA会自动切换到 **Debug** 视图
3. 程序会暂停在断点处

#### 步骤4：调试操作

| 按钮 | 快捷键 | 功能 |
|------|--------|------|
| ▶️ Resume | **F9** | 继续执行到下一个断点 |
| ⬇️ Step Over | **F8** | 执行当前行，不进入方法内部 |
| ⤵️ Step Into | **F7** | 进入方法内部 |
| ⤴️ Step Out | **Shift + F8** | 跳出当前方法 |
| 🟥 Stop | **Ctrl + F2** | 停止调试 |

#### 步骤5：查看变量
- 左下角 **Variables** 面板显示当前所有变量的值
- 鼠标悬停在代码上的变量，也能看到值
- 右键变量 → **Evaluate Expression** 可以执行表达式

## 6. 查看日志

### Run面板
- 项目运行时，底部的 **Run** 面板显示所有日志
- 滚动查看历史日志
- 使用过滤器（漏斗图标）筛选日志级别

### 日志级别
| 级别 | 颜色 | 含义 |
|------|------|------|
| ERROR | 红色 | 错误，需要立即处理 |
| WARN | 黄色 | 警告，可能有问题 |
| INFO | 白色 | 普通信息 |
| DEBUG | 灰色 | 调试信息 |

### 保存日志
1. 右键点击日志输出区域
2. 选择 **Export to Text File**
3. 选择保存位置

## 7. 修改代码

### 基本编辑

#### 自动完成（代码提示）
- 输入时会自动弹出提示
- 按 **Enter** 或 **Tab** 接受建议
- 按 **Ctrl + Space** 手动触发

#### 格式化代码
- 选中代码（或全选 **Ctrl + A**）
- 按 **Ctrl + Alt + L**
- 代码会自动格式化

#### 查找和替换
| 操作 | 快捷键 |
|------|--------|
| 查找 | **Ctrl + F** |
| 替换 | **Ctrl + R** |
| 全局查找 | **Ctrl + Shift + F** |
| 全局替换 | **Ctrl + Shift + R** |

#### 注释代码
| 操作 | 快捷键 |
|------|--------|
| 单行注释/取消 | **Ctrl + /** |
| 多行注释 | **Ctrl + Shift + /** |

### 导入类
如果代码显示红色波浪线（未导入的类）：
- 按 **Alt + Enter**
- 选择 **Import class**
- 或选择 **Auto-import**

### 重构代码

#### 重命名
1. 光标放在要改名的变量/方法/类上
2. 按 **Shift + F6**
3. 输入新名称
4. 按 **Enter**
5. IDEA会自动修改所有引用

#### 提取方法
1. 选中要提取的代码
2. 按 **Ctrl + Alt + M**
3. 输入方法名
4. 按 **Enter**

## 8. 热部署

### 什么是热部署？
修改代码后，不用重启项目，改动就能生效。

### 本项目已配置热部署
使用了 **spring-boot-devtools**

### 使用方法
1. 项目运行中
2. 修改Java代码
3. 按 **Ctrl + F9**（Build Project）
4. 项目会自动重启，几秒钟后改动生效

### 哪些改动需要重启？
- ✅ 修改方法内容 → 自动生效
- ✅ 添加/删除方法 → 自动生效
- ❌ 修改配置文件 → 需要手动重启
- ❌ 添加Maven依赖 → 需要手动重启
