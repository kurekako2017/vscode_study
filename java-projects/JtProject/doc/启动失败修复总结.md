## 📝 技术要点说明

### UserController vs AdminController 的会话管理

| 特性 | UserController | AdminController |
|------|----------------|-----------------|
| **会话管理方式** | Cookie + Session | 静态变量 |
| **登录状态存储** | Cookie ("username") | `static int adminlogcheck` |
| **用户信息存储** | Session attribute | `static String usernameforclass` |
| **线程安全性** | ✅ 安全（每个请求独立） | ⚠️ 不安全（静态变量全局共享） |
| **并发支持** | ✅ 支持多用户同时登录 | ❌ 多管理员会互相覆盖 |
| **适用场景** | 多用户电商系统 | 单管理员后台 |

### 为什么AdminController使用静态变量？

**原因**：
1. 简化实现：避免使用Session
2. 单管理员场景：假设只有一个管理员使用
3. 跨请求状态保持：静态变量在应用运行期间一直存在

**缺点**：
- ⚠️ 多线程不安全
- ⚠️ 多管理员无法同时登录
- ⚠️ 应用重启后状态丢失
- ⚠️ 不符合RESTful设计原则

## 🎯 后续改进建议

### 1. 改进AdminController的会话管理

**当前实现（有风险）**：
```java
static int adminlogcheck = 0;
static String usernameforclass = "";

@PostMapping("/admin/loginvalidate")
public ModelAndView adminlogin(...) {
    adminlogcheck = 1;  // ⚠️ 全局变量，线程不安全
    usernameforclass = username;
}
```

**建议改进为（更安全）**：
```java
@PostMapping("/admin/loginvalidate")
public ModelAndView adminlogin(
    @RequestParam String username,
    @RequestParam String password,
    HttpSession session  // ← 使用Session
) {
    User user = userService.checkLogin(username, password);
    if("ROLE_ADMIN".equals(user.getRole())) {
        // 存储到Session而不是静态变量
        session.setAttribute("adminUser", user);
        session.setAttribute("isAdmin", true);
        // ...
    }
}
```

### 2. 添加登录拦截器

创建 `LoginInterceptor.java`：
```java
@Component
public class LoginInterceptor implements HandlerInterceptor {
    @Override
    public boolean preHandle(HttpServletRequest request, 
                           HttpServletResponse response, 
                           Object handler) throws Exception {
        HttpSession session = request.getSession();
        
        // 检查管理员登录状态
        if(request.getRequestURI().startsWith("/admin/")) {
            Object isAdmin = session.getAttribute("isAdmin");
            if(isAdmin == null || !(Boolean)isAdmin) {
                response.sendRedirect("/admin/login");
                return false;
            }
        }
        return true;
    }
}
```

### 3. 统一会话管理

建议所有Controller都使用相同的会话管理方式：
```java
// 推荐使用Session + Cookie组合
@PostMapping("/login")
public ModelAndView login(
    @RequestParam String username,
    @RequestParam String password,
    HttpSession session,
    HttpServletResponse response
) {
    User user = userService.checkLogin(username, password);
    if(user != null) {
        // 1. Session存储（服务器端，安全）
        session.setAttribute("user", user);
        session.setAttribute("isLoggedIn", true);
        
        // 2. Cookie存储（客户端，方便）
        Cookie cookie = new Cookie("username", username);
        cookie.setMaxAge(7 * 24 * 60 * 60);  // 7天
        cookie.setPath("/");
        response.addCookie(cookie);
    }
}
```
