## 方法1：PowerShell手动启动（逐步执行）

### 准备工作

1. **打开PowerShell**
   - 按 `Win + X`
   - 选择 "Windows PowerShell" 或 "Windows PowerShell (管理员)"

### 步骤1：进入项目目录

```powershell
cd D:\dev\source_code\vscode_study\java-projects\JtProject
```

### 步骤2：检查JAR文件是否存在

```powershell
ls .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

**如果文件不存在**，需要先编译：

```powershell
mvn clean package -Dmaven.test.skip=true
```

### 步骤3：停止所有Java进程

```powershell
# 查看当前Java进程
Get-Process -Name java -ErrorAction SilentlyContinue

# 停止所有Java进程
Get-Process -Name java -ErrorAction SilentlyContinue | Stop-Process -Force

# 确认已停止
Get-Process -Name java -ErrorAction SilentlyContinue
```

### 步骤4：检查端口是否被占用

```powershell
netstat -ano | findstr ":8082"
```

**如果有输出**（端口被占用），执行：

```powershell
# 从输出中找到PID（最后一列的数字），然后执行：
taskkill /F /PID <替换为实际的PID>
```

### 步骤5：等待端口释放

```powershell
Start-Sleep -Seconds 5
```

### 步骤6：启动应用

**方式A：使用JDK 21**

```powershell
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

**方式B：使用JDK 11（如果有）**

```powershell
& "C:\Program Files\Eclipse Adoptium\jdk-11.0.28.6-hotspot\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

**方式C：如果Java在PATH中**

```powershell
java -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

**方式D：使用8083端口启动（如果8082有问题）**

```powershell
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar --server.port=8083
```

### 步骤7：观察启动日志

等待看到以下关键日志：

```
✅ Tomcat initialized with port(s): 8082 (http)
✅ Started JtSpringProjectApplication in X.XXX seconds
✅ JT电商系统启动成功！
```

### 步骤8：访问应用

打开浏览器，访问：

```
http://localhost:8082
```

或（如果用8083端口）：

```
http://localhost:8083
```

### 步骤9：停止应用

在PowerShell窗口中按 `Ctrl + C` 停止应用

## 方法3：IntelliJ IDEA启动（最推荐）

### 步骤1：打开IDEA并导入项目

1. 启动IntelliJ IDEA
2. 点击菜单栏 **File** → **Open**（或在欢迎界面点击 **Open** 按钮）
3. 在弹出的**文件选择对话框**中：
   - 对话框会显示文件浏览器界面
   - 导航到路径：`D:\dev\source_code\vscode_study\java-projects\JtProject`
   - **单击选中** `JtProject` 文件夹（文件夹会高亮/蓝底显示）
   - **不需要右键**，也不需要双击进入文件夹
4. 选中文件夹后，点击对话框右下角的 **"OK"** 按钮
5. IDEA会开始加载项目（右下角显示进度）

### 步骤2：等待项目加载

- 右下角会显示 "Indexing..."
- 等待进度条完成（可能需要2-5分钟）

### 步骤3：配置JDK（如果需要）

1. 按 `Ctrl + Alt + Shift + S` 打开项目结构（Project Structure）对话框
2. 在左侧导航栏选择 **"Project"**
3. 在右侧 **"Project SDK"** 下拉菜单中选择 JDK 11 或 JDK 21
4. 如果列表中没有合适的SDK：
   - 点击 **"Add SDK"** → **"Download JDK"**（下载JDK）
   - 或点击 **"Add SDK"** → **"JDK"**（添加本地已安装的JDK）
5. 配置完成后，点击右下角的 **"Apply"** 按钮应用更改
6. 最后点击 **"OK"** 按钮关闭对话框

### 步骤4：刷新Maven

1. 点击右侧的 **"Maven"** 标签
2. 点击刷新图标 🔄（**Reload All Maven Projects**）
3. 等待依赖下载完成

### 步骤5：运行应用

**方式A：直接运行主类**

1. 按 `Ctrl + N` 打开查找类
2. 输入：`JtSpringProjectApplication`
3. 按回车打开文件
4. 找到 `main` 方法
5. 点击方法左侧的 **绿色▶️图标**
6. 选择 **"Run 'JtSpringProjectApplication.main()'"**

**方式B：使用运行配置**

1. 顶部工具栏找到运行配置下拉框
2. 确保选中 **"JtSpringProjectApplication"**
3. 点击绿色 ▶️ 运行按钮
4. 或按快捷键 `Shift + F10`

### 步骤6：查看启动日志

- 底部会打开 **"Run"** 控制台
- 查看彩色日志输出
- 等待看到启动成功信息

### 步骤7：访问应用

浏览器打开：`http://localhost:8082`

### 步骤8：停止应用

- 点击控制台的 **红色⏹️按钮**
- 或按快捷键 `Ctrl + F2`

## 故障排查

### 问题1：找不到Java命令

**症状**：

```
java : 无法将"java"项识别为 cmdlet
```

**解决方法**：

使用完整路径：

```powershell
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

或配置环境变量PATH

### 问题2：端口持续被占用

**症状**：

```
Port 8082 is already in use
```

**解决方法**：

#### 方法A：暴力清理

```powershell
# 停止所有Java进程
Get-Process | Where-Object {$_.ProcessName -eq 'java'} | Stop-Process -Force

# 查找并杀死占用8082的进程
$port = netstat -ano | findstr ":8082"
if ($port) {
    $port -split "`n" | ForEach-Object {
        if ($_ -match "\s+(\d+)\s*$") {
            taskkill /F /PID $Matches[1]
        }
    }
}

# 等待5秒
Start-Sleep -Seconds 5
```

#### 方法B：重启电脑

最彻底的方法，释放所有端口

#### 方法C：更换端口

修改 `application.properties` 添加：

```properties
server.port=8083
```

或启动时指定：

```powershell
java -jar xxx.jar --server.port=8083
```

### 问题3：启动后浏览器显示404

**症状**：

```
Whitelabel Error Page
There was an unexpected error (type=Not Found, status=404)
```

**原因分析**：

1. ✅ 应用已经启动（能看到错误页面说明Tomcat在运行）
2. ❌ 访问的路径不对
3. ❌ 或者JSP视图没有正确部署

**解决方法**：

#### 尝试不同的URL：

```
http://localhost:8082/          # 用户登录页
http://localhost:8082/register  # 注册页面
http://localhost:8082/admin/login  # 管理员登录
http://localhost:8082/actuator/health  # 健康检查
```

#### 检查启动日志：

查看是否有类似错误：

```
JSP servlet class not found
Could not resolve view with name 'userLogin'
```

#### 重新打包：

```powershell
cd D:\dev\source_code\vscode_study\java-projects\JtProject
mvn clean package -Dmaven.test.skip=true
java -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

### 问题4：Maven命令不可用

**症状**：

```
'mvn' 不是内部或外部命令
```

**解决方法**：

使用Maven Wrapper：

```powershell
.\mvnw.cmd spring-boot:run
```

或：

```powershell
.\mvnw.cmd clean package -Dmaven.test.skip=true
```

### 问题5：启动很慢或卡住

**可能原因**：

- Maven下载依赖
- 数据库初始化
- 防火墙阻止

**解决方法**：

1. **耐心等待**（首次启动可能需要5-10分钟）
2. **配置Maven国内镜像**（加速依赖下载）
3. **暂时关闭防火墙**（测试是否被阻止）

## 快速命令参考

### 一键启动命令（复制粘贴）

```powershell
# 进入项目目录
cd D:\dev\source_code\vscode_study\java-projects\JtProject

# 停止旧进程
Get-Process -Name java -ErrorAction SilentlyContinue | Stop-Process -Force

# 启动应用
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

### 切换端口启动

```powershell
cd D:\dev\source_code\vscode_study\java-projects\JtProject
Get-Process -Name java -ErrorAction SilentlyContinue | Stop-Process -Force
Start-Sleep -Seconds 5
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar --server.port=8083
```

### 后台启动（不占用终端）

```powershell
cd D:\dev\source_code\vscode_study\java-projects\JtProject
Start-Process -FilePath "C:\Program Files\jdk-21.0.2\bin\java.exe" -ArgumentList "-jar",".\target\JtSpringProject-0.0.1-SNAPSHOT.jar" -WindowStyle Hidden
```

### 检查应用状态

```powershell
# 检查Java进程
Get-Process -Name java

# 检查端口
netstat -ano | findstr ":8082"

# 测试健康状态
curl http://localhost:8082/actuator/health
```

### 停止应用

```powershell
# 优雅停止
Get-Process -Name java | Stop-Process

# 强制停止
Get-Process -Name java | Stop-Process -Force
```

## 最佳实践建议

### 开发环境

1. ✅ **使用IntelliJ IDEA** - 最稳定、功能最强大
2. ✅ **配置JDK 11** - 与项目版本匹配
3. ✅ **启用热部署** - 代码修改自动重启
4. ✅ **使用Debug模式** - 支持断点调试

### 命令行启动

1. ⚠️ **仅用于测试** - 不如IDEA方便
2. ⚠️ **注意端口冲突** - 每次启动前检查
3. ⚠️ **查看完整日志** - 不要后台运行
4. ⚠️ **及时停止** - 不用时停止进程

### 生产部署

1. ✅ **使用JAR包** - 独立运行
2. ✅ **配置日志文件** - 便于追踪问题
3. ✅ **使用systemd** - 开机自启动（Linux）
4. ✅ **反向代理** - Nginx + 域名
