## 2. 目的

本書は、`BAT-003 テストデータリセット` の詳細な処理フロー、トランザクション方針、例外制御、実行後確認、テスト観点を定義するための資料である。

開発・検証時にデータを一定状態へ戻すため、以下を重視する。
- リセット対象テーブルの削除順序
- 初期データ再投入の再現性
- 実行結果の可観測性
- 異常時の復旧容易性

## 3. 概要

### 3.1 バッチ概要

| 項目 | 内容 |
|---|---|
| バッチ ID | `BAT-003` |
| バッチ名 | テストデータリセット |
| 起動クラス | `TestDataResetBatchApplication` |
| 実行クラス | `TestDataResetBatchService` |
| 実行方式 | Spring Boot 非 Web 起動 |
| プロファイル | `batch` |
| 入力 | 既存 DB 状態、`data.sql` |
| 出力 | リセット済み DB 状態、実行ログ |
| 終了コード | `0`: 正常、`1`: システム異常 |

### 3.2 処理概要

```
開始
  ↓
[フェーズ 1] 既存データ削除
  ├─ CART_PRODUCT
  ├─ CART
  ├─ PRODUCT
  ├─ CATEGORY
  └─ CUSTOMER
  ↓
[フェーズ 2] 初期データ再投入
  └─ classpath:data.sql を実行
  ↓
[フェーズ 3] 実行結果ログ出力
  └─ テーブル別削除件数
  ↓
終了コード返却
```

## 4. 処理詳細

### 4.1 フェーズ 1: 既存データ削除

#### 4.1.1 削除順序

外部キー整合性を保つため、依存子から親の順で削除する。

| 順序 | テーブル | 理由 |
|---|---|---|
| 1 | `CART_PRODUCT` | `CART` / `PRODUCT` 参照を先に解消 |
| 2 | `CART` | `CUSTOMER` 参照の前に削除 |
| 3 | `PRODUCT` | `CATEGORY` 参照の前に削除 |
| 4 | `CATEGORY` | 親マスタ |
| 5 | `CUSTOMER` | 親マスタ |

#### 4.1.2 実行 SQL

```sql
DELETE FROM CART_PRODUCT;
DELETE FROM CART;
DELETE FROM PRODUCT;
DELETE FROM CATEGORY;
DELETE FROM CUSTOMER;
```

### 4.2 フェーズ 2: 初期データ再投入

#### 4.2.1 処理内容

`ResourceDatabasePopulator` を使用し、`data.sql` を実行して初期データを再投入する。

```java
ResourceDatabasePopulator populator =
    new ResourceDatabasePopulator(new ClassPathResource("data.sql"));
populator.setContinueOnError(true);
DatabasePopulatorUtils.execute(populator, dataSource);
```

#### 4.2.2 冪等性考慮

- 本バッチは「削除 -> 再投入」を前提とするため、実行前状態に依存しない。
- 再実行時も同じ結果を期待できる。

### 4.3 フェーズ 3: 実行結果ログ

- テーブル別削除件数を `Map<String, Integer>` で集約
- INFO ログで最終結果を出力

## 5. エラーハンドリング戦略

### 5.1 エラー分類

| 区分 | 例 | 終了コード |
|---|---|---|
| システム異常 | SQL 実行失敗、接続失敗、初期化スクリプト失敗 | `1` |
| 正常終了 | 削除・再投入完了 | `0` |

### 5.2 例外処理

```java
try {
    Map<String, Integer> deleted = cleanupTables();
    initializeData();
    return 0;
} catch (Exception e) {
    return 1;
}
```

## 6. パフォーマンス・運用

### 6.1 パフォーマンス方針

- 対象は開発・試験データを想定
- 全件削除を採用し、処理単純化を優先
- 大量件数時は `TRUNCATE` への切替を検討

### 6.2 運用注意点

- 本番環境での直接実行は禁止
- 実行前に対象 DB 接続先（`batch` profile）を確認
- 他ユーザー試験中の同時実行を避ける

## 7. 出力仕様

### 7.1 ログ出力

| レベル | 内容 |
|---|---|
| INFO | 開始、削除件数、完了 |
| ERROR | 例外内容、スタックトレース |

### 7.2 終了コード

| コード | 意味 |
|---|---|
| 0 | 正常終了 |
| 1 | システム異常 |

## 8. テスト戦略

### 8.1 単体テスト観点

- `cleanupTables()` が順序通り実行される
- `initializeData()` が `data.sql` を実行する
- 例外発生時に終了コード `1` を返す

### 8.2 統合テスト観点

- 実行前後で `CUSTOMER` / `CATEGORY` / `PRODUCT` 件数が期待値になる
- 複数回実行で同一結果となる
