## 2. 目的

本書は、`BAT-007 DBバックアップ` のファイルバックアップ処理、命名規則、例外制御、運用上の注意点を定義する。

狙い:
- 学習用 H2 ファイル DB の退避
- 障害調査時の復元元確保
- バッチ運用に必要な証跡管理

## 3. 概要

### 3.1 バッチ概要

| 項目 | 内容 |
|---|---|
| バッチ ID | `BAT-007` |
| バッチ名 | DB バックアップ |
| 起動クラス | `DatabaseBackupBatchApplication` |
| 実行クラス | `DatabaseBackupBatchService` |
| 実行方式 | Spring Boot 非 Web 起動 |
| プロファイル | `batch` |
| 入力 | `batch-work/jtproject-batch.mv.db` |
| 出力 | `batch-backup/jtproject-batch-{timestamp}.mv.db` |
| 付帯 | trace ファイルも存在時にコピー |
| 終了コード | `0`: 正常、`2`: 対象なし、`1`: システム異常 |

### 3.2 処理概要

```
開始
  ↓
バックアップ元ファイル存在チェック
  ├─ 無し: WARN + 終了コード 2
  └─ 有り: 続行
      ↓
バックアップ先ディレクトリ作成
      ↓
mv.db コピー
      ↓
trace.db が存在すればコピー
      ↓
完了ログ出力
      ↓
終了コード返却
```

## 4. 処理詳細

### 4.1 対象ファイル

| 種別 | 既定値 | 備考 |
|---|---|---|
| DB本体 | `batch-work/jtproject-batch.mv.db` | 必須 |
| Trace | `batch-work/jtproject-batch.trace.db` | 任意 |

### 4.2 出力先

| 項目 | 既定値 |
|---|---|
| バックアップディレクトリ | `batch-backup` |
| 命名規則 | `jtproject-batch-yyyyMMdd_HHmmss.mv.db` |

### 4.3 コピー処理

```java
Files.createDirectories(backupDirPath);
Files.copy(source, target, StandardCopyOption.REPLACE_EXISTING);
if (Files.exists(trace)) {
    Files.copy(trace, traceTarget, StandardCopyOption.REPLACE_EXISTING);
}
```

### 4.4 設定項目

| プロパティ | 意味 | 既定値 |
|---|---|---|
| `batch.db.file` | バックアップ元 DB ファイル | `batch-work/jtproject-batch.mv.db` |
| `batch.db.trace-file` | trace ファイル | `batch-work/jtproject-batch.trace.db` |
| `batch.db.backup.dir` | バックアップ出力先 | `batch-backup` |

## 5. エラーハンドリング戦略

| 区分 | 条件 | 終了コード |
|---|---|---|
| 正常 | コピー成功 | `0` |
| 業務通知 | 元ファイルなし | `2` |
| システム異常 | `IOException` など | `1` |

## 6. パフォーマンス・運用

### 6.1 パフォーマンス方針

- ファイルコピー主体のため I/O 依存
- 大容量化に備え、夜間帯実行を推奨

### 6.2 運用注意点

- バックアップ先容量監視を実施
- 復元手順とペアで管理
- 本番 DB は別途専用ツール（mysqldump 等）を利用

## 7. 出力仕様

### 7.1 生成物

- `batch-backup/jtproject-batch-{timestamp}.mv.db`
- `batch-backup/jtproject-batch-{timestamp}.trace.db`（存在時）

### 7.2 ログ

| レベル | 内容 |
|---|---|
| INFO | 開始、コピー完了、出力先 |
| WARN | 元ファイル未存在 |
| ERROR | 例外発生 |

## 8. テスト戦略

### 8.1 単体テスト観点

- 元ファイル有無で `0/2` が返る
- 例外発生で `1` が返る
- trace 有無でコピー分岐が動作する

### 8.2 統合テスト観点

- 実ファイル環境でバックアップ生成を確認
- 連続実行時にタイムスタンプで重複回避できることを確認
