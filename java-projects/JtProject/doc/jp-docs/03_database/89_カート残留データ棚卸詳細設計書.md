## 2. 目的

本書は、`BAT-006 カート残留データ棚卸` の抽出ロジック、削除モード、出力仕様、異常制御を定義する。

主な目的:
- 不整合な `CART` / `CART_PRODUCT` の可視化
- 影響調査のための CSV 証跡出力
- 任意の削除モードによる運用簡略化

## 3. 概要

### 3.1 バッチ概要

| 項目 | 内容 |

| バッチ名 | カート残留データ棚卸 |

### 3.2 処理概要

```
開始
  ↓
[フェーズ 1] 空カート抽出
  ↓
[フェーズ 2] 孤児 CART_PRODUCT 抽出
  ↓
[フェーズ 3] CSV 出力
  ↓
[フェーズ 4] (deleteMode=true) 削除実行
  ├─ 孤児 CART_PRODUCT 削除
  └─ 空 CART 削除
  ↓
終了コード返却
```

## 4. 処理詳細

### 4.1 フェーズ 1: 空カート抽出

```sql
SELECT c.id AS cart_id, c.customer_id AS customer_id
FROM CART c
LEFT JOIN CART_PRODUCT cp ON cp.cart_id = c.id
WHERE cp.id IS NULL
ORDER BY c.id;
```

### 4.2 フェーズ 2: 孤児明細抽出

```sql
SELECT cp.id AS cart_product_id, cp.cart_id, cp.product_id,
       CASE WHEN c.id IS NULL THEN 'CART_MISSING' ELSE '' END AS cart_status,
       CASE WHEN p.product_id IS NULL THEN 'PRODUCT_MISSING' ELSE '' END AS product_status
FROM CART_PRODUCT cp
LEFT JOIN CART c ON cp.cart_id = c.id
LEFT JOIN PRODUCT p ON cp.product_id = p.product_id
WHERE c.id IS NULL OR p.product_id IS NULL
ORDER BY cp.id;
```

### 4.3 フェーズ 3: CSV 出力

ヘッダー:

```text
check_timestamp,record_type,cart_id,customer_id,cart_product_id,product_id,status,messages
```

`record_type`:
- `EMPTY_CART`
- `ORPHAN_CART_PRODUCT`

### 4.4 フェーズ 4: 削除モード

#### 4.4.1 削除 SQL

```sql
DELETE FROM CART_PRODUCT
WHERE cart_id NOT IN (SELECT id FROM CART)
   OR product_id NOT IN (SELECT product_id FROM PRODUCT);

DELETE FROM CART
WHERE id NOT IN (SELECT DISTINCT cart_id FROM CART_PRODUCT);
```

#### 4.4.2 運用ルール

- 既定値は `deleteMode=false`（棚卸のみ）
- `true` 時のみ削除を実施
- 削除前に必ず CSV を出力する

## 6. パフォーマンス・運用

### 6.1 パフォーマンス方針

- `LEFT JOIN` で抽出を集約し、往復回数を最小化
- CSV は逐次行追加でシンプルに出力

### 6.2 運用注意点

- 削除モードは検証済み環境でのみ有効化
- 本番適用時は削除対象件数の事前確認を必須化

## 8. テスト戦略

### 8.1 単体テスト観点

- 空カート抽出クエリの期待件数
- 孤児明細抽出時の `CART_MISSING` / `PRODUCT_MISSING` メッセージ
- `deleteMode` 切替時の削除有無

### 8.2 統合テスト観点

- サンプルデータで `0/2/1` の各終了コードを確認
- 削除モード実行後に残留件数が減ることを確認
