## 2. 目的

本書は、`JtProject` におけるログ出力方針、出力対象、確認観点、障害解析時の利用方法を整理することを目的とする。

## 3. ログ設計方針

- 障害発生時に一次切り分けできる情報を残すこと。
- 画面操作起点、認証処理、DB 接続異常、起動停止イベントは追跡可能とすること。
- 個人情報や認証情報は平文出力しないこと。
- 開発環境ではコンソールログ中心、本番想定ではファイル出力併用を前提とすること。

## 4. ログ一覧

| ログID | 区分 | 出力契機 | 主な出力内容 | 出力先 | レベル | 備考 |
|---|---|---|---|---|---|---|
| LOG-01 | 起動ログ | アプリ起動時 | 起動開始、ポート、起動完了 | Console / File | INFO | 起動確認用 |
| LOG-02 | 停止ログ | アプリ停止時 | 停止開始、停止完了 | Console / File | INFO | 手動停止時も記録対象 |
| LOG-03 | 認証ログ | 一般利用者ログイン時 | ユーザ名、成功/失敗結果 | Console / File | INFO / WARN | パスワード出力禁止 |
| LOG-04 | 管理者認証ログ | 管理者ログイン時 | 管理者 ID、認証結果 | Console / File | INFO / WARN | 権限系障害確認用 |
| LOG-05 | 画面遷移ログ | 主画面表示時 | 画面名、利用者区分、遷移元 | Console | INFO | 必要に応じて簡易出力 |
| LOG-06 | DB 接続異常ログ | DAO 例外発生時 | SQL/HQL 実行箇所、対象テーブル、例外概要 | Console / File | ERROR | 例外詳細確認用 |
| LOG-07 | 業務例外ログ | 業務チェック異常時 | 機能名、入力条件、異常理由 | Console / File | WARN | 入力不正、重複等 |
| LOG-08 | システム例外ログ | 想定外例外発生時 | クラス名、メソッド名、スタックトレース | File | ERROR | 障害票連携対象 |

## 5. 主要ログ詳細

### 5.1 認証ログ

- 対象: `UserController.userlogin()`, `AdminController.loginvalidate()`
- 目的: ログイン失敗多発、認証系障害、権限誤設定の切り分け
- 注意点: パスワード、セッション ID をログに残さない

### 5.2 DB 異常ログ

- 対象: `UserDao`, `ProductDao`, `CategoryDao`, `CartDao`, `CartProductDao`
- 目的: 接続異常、更新失敗、データ整合性問題の切り分け
- 注意点: 実データ全文ではなく、キー項目と異常概要を優先記録する

### 5.3 業務例外ログ

- 対象: 重複登録、ログイン失敗、権限外アクセス、存在しないデータ更新
- 目的: 障害票化前の運用確認、再現条件整理

## 6. ログ確認観点

| 観点 | 確認内容 |
|---|---|
| 起動確認 | 起動完了ログが出ているか |
| 認証確認 | 成功/失敗が追えるか |
| DB 異常 | DAO 例外時に対象処理が分かるか |
| 障害解析 | スタックトレースと操作起点が追えるか |
| 情報漏洩防止 | パスワードや不要な個人情報が出力されていないか |

## 7. 運用上の留意点

- 障害調査時は `監視・障害対応手順書` とあわせて参照すること。
- ログローテーションや保持期間は本番運用時に別途定義すること。
- ログレベル変更時は `設定値一覧` も更新すること。
