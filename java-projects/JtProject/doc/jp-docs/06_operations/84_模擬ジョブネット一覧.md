# 84. 模擬ジョブネット一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 模擬ジョブネット一覧 |
| 対象システム | JtProject |
| 文書区分 | 学習用ジョブネット資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、`BAT-LAB-001 商品在庫整合チェック` を `JP1` 風のジョブネットとして整理し、ジョブ依存、戻りコード、異常時停止の考え方を学ぶための資料である。

## 3. ジョブネット概要

| 項目 | 内容 |
|---|---|
| ジョブネット ID | `JN-LAB-01` |
| ジョブネット名 | 商品在庫整合チェックジョブネット |
| 実行契機 | 手動実行、または日次定時実行想定 |
| 実行方式 | `bat` 起動 |
| 起動スクリプト | `scripts/start/run-product-inventory-check.cmd` |

## 4. ジョブ構成

| 順序 | ジョブ ID | ジョブ名 | 内容 | 正常終了条件 |
|---|---|---|---|---|
| 1 | JOB-01 | 実行前ディレクトリ確認 | `logs/batch`、`batch-output` 利用確認 | 作業ディレクトリ参照可能 |
| 2 | JOB-02 | 商品在庫整合チェック起動 | バッチ本体を起動する | 終了コード `0` または `2` |
| 3 | JOB-03 | 結果ファイル確認 | CSV が生成されていることを確認する | 出力ファイル存在 |
| 4 | JOB-04 | 実行結果記録 | 実施結果を Excel / CSV 管理表へ転記する | 実績記録完了 |

## 5. 戻りコード設計

| 戻りコード | 意味 | 後続動作 |
|---|---|---|
| `0` | 全件 OK | 後続へ進む |
| `2` | 業務 NG あり | 結果確認のうえ後続へ進む |
| `1` | システム異常 | 後続停止、原因調査 |

## 6. 運用イメージ

```text
JOB-01 実行前確認
  -> JOB-02 商品在庫整合チェック
      -> 正常(0,2) の場合: JOB-03 結果ファイル確認
      -> 異常(1) の場合: ジョブネット異常終了
  -> JOB-04 実行結果記録
```

## 7. 学習ポイント

- `JP1` ではバッチ本体だけでなく、前後確認もジョブとして分割することが多い
- 戻りコード `0` 以外も業務上許容する設計がある
- 異常時の後続停止条件を事前に決めておく
- 実行結果を Excel 管理表に残す運用が多い
