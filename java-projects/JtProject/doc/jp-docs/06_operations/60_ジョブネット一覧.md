# 60. ジョブネット一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | ジョブネット一覧 |
| 対象システム | JtProject |
| 文書区分 | 運用・定期処理資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、`JtProject` におけるジョブネット、定期実行想定処理、運用作業単位の依存関係を整理し、将来のバッチ運用、日次作業、保守作業に備えることを目的とする。

## 3. 前提

- 現行アプリには本格的なバッチ機能は未実装である。
- ただし、現場資料としては「未実装」であってもジョブネット想定を持つことが多いため、運用・試験・保守観点で整理する。

## 3.1 JP1 学習向け補足

- 本プロジェクトでは `JP1/AJS` のような商用ジョブ管理製品は導入していない。
- 学習時は「ジョブネット」という考え方だけを先に文書化し、実行は `bat`、`PowerShell`、Windows タスク スケジューラで代替できる。
- まずはジョブ依存、戻りコード、異常時停止、再実行の考え方を身につけることを優先する。

## 4. ジョブネット一覧

| ジョブネットID | 名称 | 実行契機 | 構成ジョブ | 目的 | 現状 |
|---|---|---|---|---|---|
| JN-01 | 初期データ準備ジョブネット | 環境構築時 | DB 作成 -> DDL 実行 -> 初期データ投入 | 開発・試験開始準備 | 手動実施 |
| JN-02 | テストデータ再作成ジョブネット | 試験前 | 既存データ確認 -> データクリア -> テストデータ投入 | 試験前状態統一 | 手動実施 |
| JN-03 | アプリ起動確認ジョブネット | 起動時 | 設定確認 -> DB 接続確認 -> アプリ起動 -> 画面疎通確認 | 起動可否確認 | 手順書ベース |
| JN-04 | 障害一次対応ジョブネット | 障害発生時 | 障害受付 -> ログ確認 -> 接続確認 -> 復旧判定 | 一次切り分け | 手順書ベース |
| JN-05 | リリース反映ジョブネット | リリース時 | バックアップ -> 配備 -> 起動確認 -> 画面確認 -> 結果記録 | 反映作業標準化 | 手順書ベース |

## 5. ジョブ詳細

### 5.1 初期データ準備ジョブネット

| 順序 | ジョブ名 | 内容 |
|---|---|---|
| 1 | DB 作成 | 対象 DB の存在確認または新規作成 |
| 2 | DDL 実行 | テーブル、制約作成 |
| 3 | 初期データ投入 | 顧客、分類、商品等の初期データ投入 |
| 4 | 動作確認 | 主要画面から表示確認 |

### 5.2 リリース反映ジョブネット

| 順序 | ジョブ名 | 内容 |
|---|---|---|
| 1 | 反映前確認 | 設定値、資材、版数確認 |
| 2 | バックアップ | DB / 資材退避 |
| 3 | 配備 | アプリ反映 |
| 4 | 起動確認 | ログ、画面、DB 接続確認 |
| 5 | 実施記録 | 結果、問題有無、障害票採番 |

## 6. 運用上の留意点

- 将来バッチ実装時は、ジョブ名、スケジュール、異常時再実行方針を追記すること。
- ジョブネット変更時は `バッチ・定期処理一覧` と整合を取ること。
- 監視対象ジョブが増えた場合は、監視手順書にも反映すること。

## 7. 模擬ジョブネット学習案

| 模擬ジョブネット | 構成ジョブ | 学べる内容 |
|---|---|---|
| JN-LAB-01 テストデータ再作成 | 既存データ退避 -> データ削除 -> 初期データ投入 -> 疎通確認 | 前後関係、異常時停止 |
| JN-LAB-02 日次集計 | アプリ疎通確認 -> 集計処理 -> CSV 出力 -> 実行結果記録 | 日次実行、結果ファイル |
| JN-LAB-03 棚卸処理 | 入力チェック -> 抽出 -> レポート出力 -> 完了通知 | 戻りコード、通知、再実行 |

推奨する模擬方法:

- 業務処理は Java プログラムまたは SQL スクリプトで作る。
- ジョブ連結は `bat` または `PowerShell` で行う。
- 実行管理は Windows タスク スケジューラで行う。
- 結果管理は Excel / CSV に残す。
