# 65. 画面項目定義書

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 画面項目定義書 |
| 対象システム | JtProject |
| 文書区分 | 画面詳細設計資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、主要画面の入力項目、表示項目、ボタン、遷移先、チェック観点を画面単位で整理することを目的とする。

## 3. 画面項目定義

### 3.1 一般ログイン画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 入力 | ユーザ名 | `username` | 必須 | 一般利用者ログイン ID |
| 入力 | パスワード | `password` | 必須 | 一般利用者ログインパスワード |
| ボタン | ログイン | `submit` | - | 認証処理実行 |
| リンク | 新規登録 | `register` | - | ユーザ登録画面へ遷移 |
| 表示 | エラーメッセージ | `message` | - | 認証失敗時表示 |

### 3.2 ユーザ登録画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 入力 | ユーザ名 | `username` | 必須 | 利用者識別子 |
| 入力 | パスワード | `password` | 必須 | ログイン用情報 |
| 入力 | メールアドレス | `email` | 任意/将来 | 連絡先 |
| ボタン | 登録 | `submit` | - | 新規登録処理 |
| 表示 | 結果メッセージ | `message` | - | 登録結果表示 |

### 3.3 商品管理画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 表示 | 商品 ID | `id` | - | 更新対象識別 |
| 入力 | 商品名 | `name` | 必須 | 商品名称 |
| 入力 | 価格 | `price` | 必須 | 商品価格 |
| 入力 | 分類 ID | `categoryId` | 必須 | 紐付分類 |
| ボタン | 追加 | `add` | - | 商品追加 |
| ボタン | 更新 | `update` | - | 商品更新 |
| ボタン | 削除 | `delete` | - | 商品削除 |

### 3.4 分類管理画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 表示 | 分類 ID | `id` | - | 管理用識別子 |
| 入力 | 分類名 | `name` | 必須 | 分類名称 |
| ボタン | 追加 | `add` | - | 分類追加 |
| ボタン | 更新 | `update` | - | 分類更新 |
| ボタン | 削除 | `delete` | - | 分類削除 |

### 3.5 管理者ホーム画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 表示 | 画面タイトル | `title` | - | 管理者ホーム表示 |
| 表示 | お知らせ領域 | `messageArea` | - | 管理向け案内表示 |
| ボタン/リンク | 分類管理 | `categories` | - | 分類管理画面へ遷移 |
| ボタン/リンク | 商品管理 | `products` | - | 商品管理画面へ遷移 |
| ボタン/リンク | 顧客一覧 | `customers` | - | 顧客一覧画面へ遷移 |
| ボタン/リンク | プロフィール更新 | `profile` | - | プロフィール更新画面へ遷移 |
| ボタン/リンク | ログアウト | `logout` | - | ログアウト処理実行 |

### 3.6 顧客一覧画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 表示 | 顧客 ID | `id` | - | 顧客識別子 |
| 表示 | ユーザ名 | `username` | - | 顧客ログイン名 |
| 表示 | メール/連絡先 | `email` / `contact` | - | 顧客連絡情報 |
| 表示 | 権限 | `role` | - | 一般/管理者区分 |
| ボタン/リンク | 詳細 | `detail` | - | 顧客詳細参照 |
| ボタン/リンク | 戻る | `back` | - | 管理者ホームへ戻る |

### 3.7 プロフィール更新画面

| 項目区分 | 項目名 | 物理名 | 必須 | 内容 |
|---|---|---|---|---|
| 入力 | ユーザ名 | `username` | 必須 | 管理者表示名またはログイン名 |
| 入力 | パスワード | `password` | 必須/方針依存 | 更新対象認証情報 |
| 入力 | メール/連絡先 | `email` / `contact` | 任意 | 連絡先情報 |
| ボタン | 更新 | `update` | - | プロフィール更新処理 |
| ボタン/リンク | 戻る | `back` | - | 管理者ホームへ戻る |
| 表示 | 結果メッセージ | `message` | - | 更新成功/失敗表示 |

## 4. 設計観点

- 項目名、パラメータ名、Controller 受取名を一致させること。
- ボタン押下時の URL と遷移先を `URL一覧` と整合させること。
- 入力チェックは `入力項目一覧`、`入力チェック一覧` と一致させること。
- 画面部品の構成、ボタン/リンク/メッセージ領域の部品 ID は `80_画面部品一覧` と整合させること。
