## 2. システム概要

`JtProject` は、一般ユーザー向け商品閲覧・カート操作機能と、管理者向け商品・カテゴリ・顧客管理機能を持つ Web アプリケーションである。  
本システムは学習用サンプルであるが、対日案件で一般的な「画面中心の業務システム」を意識した構成を採用している。

## 3. 開発目的

- Java Web アプリケーションの基本構造を学習する
- Spring MVC ベースの画面遷移と業務処理の流れを理解する
- 伝統的な JSP 画面構成と Controller / Service / DAO の責務分離を学ぶ
- 日本案件でよく要求される設計書、テスト仕様書、DB 資料の作成練習を行う

## 4. 利用者区分

| 利用者 | 役割 | 利用機能 |
|---|---|---|
| 一般ユーザー | 商品閲覧、登録、ログイン、カート利用 | ログイン、商品一覧、カート追加・削除 |
| 管理者 | システム運用・商品管理 | 管理者ログイン、カテゴリ管理、商品管理、顧客一覧、プロフィール更新 |
| 開発者 | 学習・保守・改修 | テスト画面、設定、調査 |

## 5. システム範囲

### 5.1 対象範囲

- ユーザー登録
- ユーザーログイン
- 商品一覧表示
- カート追加、表示、削除
- 管理者ログイン
- カテゴリ登録、更新、削除
- 商品登録、更新、削除
- 顧客一覧表示
- 管理者プロフィール更新

### 5.2 対象外

- 注文確定
- 決済処理
- 在庫引当
- 配送管理
- メール通知
- 外部 API 連携

## 6. システム構成概要

| 層 | 採用技術 | 概要 |
|---|---|---|
| 画面層 | JSP / JSTL | 画面表示、フォーム送信 |
| 制御層 | Spring MVC Controller | リクエスト受付、画面遷移制御 |
| 業務層 | Service | 業務ロジック実行 |
| データアクセス層 | DAO / JPA / Hibernate | DB 操作 |
| DB | H2 / MySQL | 開発用・切替用 DB |
| 実行基盤 | Spring Boot + Embedded Tomcat | Web 実行環境 |

## 7. 主要業務フロー

### 7.1 一般ユーザー

```text
ログイン
  -> 商品トップ表示
  -> 商品一覧確認
  -> カート追加
  -> カート確認
  -> カート内商品削除
```

### 7.2 管理者

```text
管理者ログイン
  -> 管理者ホーム
  -> カテゴリ管理
  -> 商品管理
  -> 顧客一覧確認
  -> プロフィール更新
```

## 9. DB 概要

主要テーブル：

- `CUSTOMER`: 顧客情報管理、ログイン認証、権限判定に使用
- `CATEGORY`: 商品カテゴリ管理に使用
- `PRODUCT`: 商品情報管理、商品一覧表示、商品管理に使用
- `CART`: 顧客ごとのカートヘッダ管理に使用
- `CART_PRODUCT`: カート内商品明細管理に使用

関係概要：

- 顧客はカートを持つ
- 商品はカテゴリに属する
- カートと商品は中間テーブルで関連づく

## 10. 非機能要件概要

| 項目 | 内容 |
|---|---|
| 認証 | 簡易ログイン認証 |
| 権限 | 一般ユーザー / 管理者 |
| 可用性 | 学習用のため高可用性は対象外 |
| 性能 | 小規模データ前提 |
| 保守性 | 3 層構造により改修しやすい形を目指す |
| 文字コード | UTF-8 前提 |

## 11. 運用・保守上の留意点

- 一般ユーザー系と管理者系でセッション管理方式が異なる
- 一部メッセージが英語中心であり、対日案件向けには日本語化余地がある
- 実案件向けには入力チェック、監査ログ、例外処理統一が必要
