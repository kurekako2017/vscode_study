## 2. 目的

本書は、主要画面を構成する入力欄、ボタン、リンク、一覧領域、メッセージ領域などの画面部品を一覧化し、部品単位での管理、設計参照、入力チェック資料との対応を容易にすることを目的とする。

## 4. 運用ルール

- 画面部品の追加、削除、名称変更が発生した場合は、本書、`65_画面項目定義書.md`、`75_画面イベント一覧.md` を同時に見直すこと。
- 入力欄とプルダウンは `62_入力項目一覧.md`、`18_入力チェック一覧.md` の項目名と一致させること。
- 共通利用されるメッセージ領域、セッション連動項目、共通ボタン制御は `54_共通部品一覧.md` と整合させること。
- 命名揺れが存在する項目は、実装寄り名称と設計寄り名称を備考に併記し、今後の統一対象として管理すること。

## 5. 本書の位置付け

| 文書 | 主な役割 | 本書との違い |
|---|---|---|
| `14_画面一覧.md` | 画面単位の総覧 | 本書は画面内の部品単位で管理する |
| `65_画面項目定義書.md` | 項目の意味、内容、必須区分の定義 | 本書は部品 ID、イベント、関連資料まで管理する |
| `62_入力項目一覧.md` | 入力パラメータの設計 | 本書はボタン、リンク、一覧、メッセージ領域も含める |
| `18_入力チェック一覧.md` | バリデーション / 入力チェック設計 | 本書はどの部品がどのチェック資料に紐づくかを示す |
| `54_共通部品一覧.md` | 複数機能で共通利用される部品の管理 | 本書は画面単位の UI 部品を管理する |
