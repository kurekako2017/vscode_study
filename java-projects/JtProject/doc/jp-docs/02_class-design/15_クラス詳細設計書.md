# クラス詳細設計書

## 1. 目的

本書は `JtProject` のクラス設計資料全体の入口である。  
日本案件でよくあるように、クラス詳細設計は 1 ファイルに集約せず、層ごとに分割して管理する。

## 2. 分割方針

| 区分 | 文書 | 主な内容 |
|---|---|---|
| Controller | `15a_Controller詳細設計書.md` | 画面要求受付、入力値受領、画面遷移、セッション制御 |
| Service | `15b_Service詳細設計書.md` | 業務ロジック、判定条件、DAO 呼出、戻り値制御 |
| DAO | `15c_DAO詳細設計書.md` | DB アクセス、検索条件、更新方式、戻り値 |
| Model | `15d_Model詳細設計書.md` | Entity 構造、属性、テーブル対応、業務上の意味 |

## 3. レイヤ構成概要

```text
JSP / Request
  -> Controller
      -> Service
          -> DAO
              -> DB
```

補足:

- Controller では HTTP 要求受付と画面制御を担う
- Service では業務ロジック、分岐、整合制御を担う
- DAO では Hibernate / JPA を用いた永続化処理を担う
- Model は業務データと DB 構造の対応単位となる

## 4. 対象パッケージ

| パッケージ | 内容 |
|---|---|
| `controller` | `UserController`、`AdminController` |
| `services` / `services.impl` | `UserService`、`ProductService`、`CategoryService`、`CartService` と実装クラス |
| `dao` / `dao.impl` | `UserDao`、`ProductDao`、`CategoryDao`、`CartDao`、`CartProductDao` と実装クラス |
| `models` | `User`、`Product`、`Category`、`Cart`、`CartProduct` |

## 6. 読書順総ナビゲーション

推奨順:

1. [15a_Controller詳細設計書.md](15a_Controller詳細設計書.md)
2. [15b_Service詳細設計書.md](15b_Service詳細設計書.md)
3. [15c_DAO詳細設計書.md](15c_DAO詳細設計書.md)
4. [15d_Model詳細設計書.md](15d_Model詳細設計書.md)

読み方:

- まず Controller 層で画面要求と遷移を把握する。
- 次に Service 層で業務ロジックと判定条件を確認する。
- その後 DAO 層で DB アクセス方式と検索条件を追う。
- 最後に Model 層でエンティティ構造とテーブル対応を固める。

## 7. レイヤ別の読む理由

| レイヤ | 先に読む理由 | 読了後に得られること |
|---|---|---|
| Controller | 画面要求と URL 起点で全体像をつかみやすい | 入口画面、主要操作、遷移が分かる |
| Service | 実際の業務判定が集中している | 認証、商品、カテゴリ、カートの処理意図が分かる |
| DAO | 永続化の実装差異や HQL / EntityManager 利用が見える | DB アクセス設計と改善余地が分かる |
| Model | 最後にデータ構造を整理すると理解が安定する | テーブル対応、関連、属性意味が固まる |

## 8. クラス単位文書への入り方

- Controller 層は `UserController` から読み、次に `AdminController` を確認する。
- Service 層は `UserService` から読み、次に `Product`、`Category`、`Cart` の順で読む。
- DAO 層は `UserDao` から始め、次に商品系、最後にカート系を読む。
- Model 層は `User`、`Category`、`Product`、`Cart`、`CartProduct` の順で読む。

## 8.1 処理フロー図の配置方針

クラス詳細設計で扱う `処理フロー図` は、クラス単位またはメソッド単位の詳細処理を対象とする。

対象例:

- `UserController.userlogin()`
- `ProductService.updateProduct()`
- `CartProductDao.getProductByCartID()`

記載内容:

- メソッド開始から終了までの判定分岐
- 呼出先クラス、戻り値、更新対象
- 例外発生点、戻り先、再処理有無

住み分け:

- 業務全体または画面機能全体の流れは [03_詳細設計書.md](../01_design/03_詳細設計書.md) に置く。
- 機能単位の流れは [13_機能設計書.md](../01_design/13_機能設計書.md) に置く。
- 本文書配下では「コードを書くための処理順」を示す。

## 9. 運用方針

- 機能追加時は該当レイヤの文書を更新する。
- 大規模改修時は `13_機能設計書.md`、`03_詳細設計書.md`、`20_シーケンス図.md` と整合を取る。
- クラス単位の更なる詳細化が必要な場合は、各層文書配下でメソッド別に追記する。
