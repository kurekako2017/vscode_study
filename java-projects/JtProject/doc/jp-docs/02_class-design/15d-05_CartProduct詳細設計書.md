## 2. クラス概要

| 項目 | 内容 |
|---|---|
| 役割 | カートと商品の関連明細を表す Entity |
| 対応テーブル | `CART_PRODUCT` |
| 主キー | `id` |
| 主な利用箇所 | カート商品追加、商品一覧再構成、カート商品削除 |

## 3. 属性一覧

| No | 属性名 | 型 | DB カラム | 内容 |
|---|---|---|---|---|
| 1 | `id` | `int` | `id` | 明細 ID、主キー、自動採番 |
| 2 | `cart` | `Cart` | `cart_id` | 所属カート |
| 3 | `product` | `Product` | `product_id` | 対象商品 |

## 4. 業務上の意味

- カート内にどの商品が入っているかを表す中間明細エンティティ
- `Cart` と `Product` の多対多関係を解消するための関連テーブル
- カート画面表示時の商品一覧生成元データ

## 5. 業務ルール

- 現行実装では数量カラムを持たないため、1 レコード 1 商品関連として扱う。
- 同一商品複数個をどう表現するかは追加設計事項である。
- 削除時は `cart_id` と `product_id` の組み合わせで対象候補を特定できる。

## 6. 関連

| 関連先 | 関係 | 内容 |
|---|---|---|
| `Cart` | N : 1 | 親カート |
| `Product` | N : 1 | 対象商品 |

## 7. 設計上の注意

- デフォルトコンストラクタで `product = null` を明示設定しているが、業務上の意味は限定的である。
- 実案件では数量、登録日時、単価スナップショット、削除フラグの追加が想定される。
- カート内明細順序や重複制御は現状 Entity では管理していない。
