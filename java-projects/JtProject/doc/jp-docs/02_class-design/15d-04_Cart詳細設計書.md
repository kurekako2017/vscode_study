## 2. クラス概要

| 項目 | 内容 |
|---|---|
| 役割 | 顧客ごとのカートヘッダを表す Entity |
| 対応テーブル | `CART` |
| 主キー | `id` |
| 主な利用箇所 | カート存在判定、カート生成、カート明細親情報 |

## 3. 属性一覧

| No | 属性名 | 型 | DB カラム | 内容 |
|---|---|---|---|---|
| 1 | `id` | `int` | `id` | カート ID、主キー、自動採番 |
| 2 | `customer` | `User` | `customer_id` | カート所有顧客 |

## 4. 業務上の意味

- 顧客単位の買い物かご本体を表すヘッダエンティティ
- カート内商品は `CartProduct` 明細を通じて表現する
- ログインユーザーごとに保持される想定の業務データ

## 5. 業務ルール

- 1 顧客 1 カートを前提とした運用が想定される。
- 現行実装ではカート状態、作成日時、更新日時、合計金額は保持しない。
- 商品一覧は `Cart` 自身では持たず、中間表 `CartProduct` 経由で解決する。

## 6. 関連

| 関連先 | 関係 | 内容 |
|---|---|---|
| `User` | N : 1 | カート所有者 |
| `CartProduct` | 1 : N 相当 | カート配下明細 |

## 7. 設計上の注意

- 旧多対多定義はコメントアウトされており、現在は中間表方式に移行済みである。
- カート配下商品を直接保持するコレクションがないため、取得時は DAO で再構成する。
- 実案件では失効カートのクリーンアップ、最終更新日時、セッション連携情報を持たせることが多い。
