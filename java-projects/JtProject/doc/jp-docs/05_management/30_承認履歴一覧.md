## 2. 目的

本書は、各設計書・試験資料について、作成、レビュー、承認の履歴を一覧管理するための文書である。  
日本案件で必要となる「誰が確認し、いつ承認したか」を追跡可能にすることを目的とする。

## 3. 承認フロー想定

| ステップ | 役割 | 主な作業 |
|---|---|---|
| 1 | 作成者 | 文書作成、自己点検 |
| 2 | レビュアー | 内容レビュー、指摘票記録 |
| 3 | 修正担当 | 指摘反映、再提出 |
| 4 | 承認者 | 最終確認、承認 |

## 6. ステータス定義

| ステータス | 意味 |
|---|---|
| 新規作成 | 作成直後、レビュー未実施 |
| レビュー済 | 指摘反映完了、承認待ち |
| 承認待ち | 承認者確認待ち |
| 承認済 | 正式版として採用済み |
| 差戻し | 修正が必要 |
| 参照資料 | 承認対象外の管理資料 |

## 7. 運用ルール

- 版数更新時は本書の対象文書版数も更新する。
- 承認済み後に変更した場合は、再レビューと再承認を行う。
- 管理台帳類は必要に応じて承認対象外として扱う。
- 実案件では承認者氏名、所属、承認方法、承認証跡 URL 等を追加する。

## 8. 備考

- 現在は学習用資料のため承認者欄を仮置きとしている。
- 実務へ寄せる場合は、押印欄または電子承認フロー番号を追加する。
