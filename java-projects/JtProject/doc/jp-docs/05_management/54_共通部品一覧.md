# 共通部品一覧

## 1. 目的

本書は `JtProject` の実装をもとに、複数機能・複数画面で横断利用されている設定、共通ロジック、共通データ項目を整理する。

## 2. 本プロジェクトにおける共通部品の考え方

本プロジェクトでは、厳密な `common` パッケージや共通 JSP 部品は未整備である。
そのため、以下のような「横断利用されている実装要素」を共通部品として扱う。

- Spring / Hibernate の共通設定
- 複数 Controller から利用される Service / DAO
- Session / Cookie の共通キー
- 画面横断で使われるロール値、メッセージキー
- 管理画面・利用者画面で繰り返し使われる UI パターン

## 4. 実際の共通処理

| 処理 ID | 処理名 | 実装箇所 | 利用箇所 | 概要 |
|---|---|---|---|---|
| CP-001 | ログインユーザー名解決 | `UserController.resolveLoggedInUsername()` | `/`, `/index`, `/products/addtocart`, `/user/cart`, `/user/cart/delete` | Session と Cookie の両方から一般ユーザー名を解決 |
| CP-002 | 管理者ログイン判定 | `AdminController.isAdminLoggedIn()` | 管理系画面全般 | `adminLoggedIn` による管理画面入場制御 |
| CP-003 | 管理者表示名取得 | `AdminController.getAdminUsername()` | `/admin/index`, `/admin/profileDisplay` | Session から管理者名を取得 |
| CP-004 | カート検索処理 | `UserController.addToCart()` / `showCart()` / `deleteFromCart()` 内 | カート関連機能 | `CartService.getCarts()` の結果から対象ユーザーのカートを探索 |
| CP-005 | 入力空チェック | `InputCheckUtil.hasText()` | 登録、カテゴリ更新、商品更新、プロフィール更新 | 必須入力の共通判定 |
| CP-006 | 文字列 trim 共通化 | `InputCheckUtil.trimToNull()` / `trimToEmpty()` | 登録、カテゴリ、商品、プロフィール更新 | 前後空白除去と null/空文字正規化 |
| CP-007 | 型変換共通化 | `TypeConversionUtil.toTrimmedString()` | Session / Cookie 値取得 | `Object -> String` の安全変換 |

## 5. 共通メッセージ / 共通取扱項目

| 項目 | 内容 | 主な利用箇所 |
|---|---|---|
| `msg` | 画面一般メッセージ格納キー | ユーザーログイン、登録、管理者ログイン、商品一覧 |
| `cartMsg` | カート結果メッセージ格納キー | カート追加、カート表示、カート削除 |
| `Please login first` | 未ログイン時メッセージ | `/products/addtocart`, `/user/cart/delete` |
| `Add Success` | カート追加成功メッセージ | `/products/addtocart` |
| `Delete Success` | カート削除成功メッセージ | `/user/cart/delete` |
| `ROLE_NORMAL` | 一般ユーザーロール | 利用者登録、一般導線 |
| `ROLE_ADMIN` | 管理者ロール | 管理者ログイン、カート利用抑止 |

## 6. 実装上の注意

- `application-h2.properties` は現時点では存在しない。利用可能な profile 用設定は `local`、`mysql`、`remote`、`test` である。
- 共通 UI は JSP include 化されておらず、ナビゲーションや CDN 読み込みは各 JSP に重複記述されている。
- 利用者向け共通判定は `UserController` 内 private method、管理者向け共通判定は `AdminController` 内 private method に留まっているが、文字列判定や Session キーは `common` 配下へ切り出し済みである。
- `CartProductDao` は Controller から直接利用されており、典型的な層分離としては `CartProductService` の新設余地がある。
- `HibernateConfiguration` は無効化済みで、実際の共通設定として機能していない。
