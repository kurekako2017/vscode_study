## 2. 目的

本書は、発生した障害を一覧で管理し、発生日、影響範囲、優先度、対応状況を横断的に把握するための台帳である。

## 3. 管理方針

- 個別の詳細は `08_障害票.md` に記録する。
- 本書では障害全体の一覧性を重視する。
- 障害の調査状況、改修状況、再発防止要否を確認できるようにする。

## 5. 優先度定義

| 優先度 | 内容 |
|---|---|
| 高 | 主要機能停止、セキュリティ重大影響、リリース阻害 |
| 中 | 一部機能や運用に影響、計画対応が必要 |
| 低 | 軽微不具合、改善対象 |

## 6. 状況定義

| 状況 | 内容 |
|---|---|
| 未対応 | 障害登録のみで未着手 |
| 調査中 | 原因調査中 |
| 対応中 | 改修または回避策対応中 |
| 対応済み | 改修済み、再確認待ちまたは完了 |
| 管理中 | 直近改修対象外だが継続監視中 |

## 7. 利用ルール

- 新規障害発生時は本書へ 1 行追加する。
- 詳細内容は `08_障害票.md` と `10_調査票.md` に記載する。
- 改修完了時は `09_改修票.md` と状態を合わせる。
- 再発防止事項は `31_課題管理表.md`、`32_リスク一覧.md` に反映する。

## 8. 備考

- 実案件では検知日時、検知者、復旧日時、恒久対応日、障害レベルを追加する。
- Excel 管理の場合はフィルタ・集計しやすい列順に再配置する。
