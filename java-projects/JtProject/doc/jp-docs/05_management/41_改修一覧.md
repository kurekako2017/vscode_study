## 2. 目的

本書は、実施済みまたは予定されている改修案件を一覧で管理し、対象機能、目的、影響範囲、対応状況を把握するための台帳である。

## 3. 管理方針

- 個別改修の詳細は `09_改修票.md` に記載する。
- 本書では複数改修を横断的に管理する。
- 障害起因か改善起因かを区別して記録する。

## 5. 区分定義

| 区分 | 内容 |
|---|---|
| 障害対応 | 発生済み不具合に対する修正 |
| 設計改善 | 設計書、構成、整理方針の改善 |
| 運用改善 | 起動、運用、保守手順等の改善 |
| セキュリティ改善 | 権限、認証、秘匿情報保護の改善 |
| 品質改善 | 試験、レビュー、証跡整備の改善 |

## 6. 状況定義

| 状況 | 内容 |
|---|---|
| 未着手 | 改修登録のみ |
| 対応中 | 設計、実装、反映のいずれか進行中 |
| 対応済み | 改修完了 |
| 保留 | 方針待ち、後続対応 |
| 継続中 | 継続的な整備対象 |

## 7. 利用ルール

- 新規改修案件は本書へ 1 行追加する。
- 改修詳細は `09_改修票.md` に記録する。
- 障害起因の場合は `40_障害一覧.md` と対応関係を確認する。
- 影響が大きい改修は試験仕様書と結果報告書を更新する。

## 8. 備考

- 実案件では工数、担当者、レビュー結果、リリース日を追加する。
- 台帳用途のため、時系列で追記する運用を推奨する。
