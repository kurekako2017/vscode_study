# 69. 設計レビュー観点一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 設計レビュー観点一覧 |
| 対象システム | JtProject |
| 文書区分 | レビュー管理資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、設計レビュー時に確認すべき主要観点を整理し、レビュー品質のばらつきを抑えることを目的とする。

## 3. レビュー観点一覧

| 観点区分 | 確認内容 | 対象文書 |
|---|---|---|
| 要件整合 | 要求事項と設計内容が一致しているか | 概要/基本/詳細設計書 |
| 画面整合 | 画面一覧、画面遷移、画面項目定義の矛盾がないか | 画面関連文書 |
| 入出力整合 | 入力項目、出力項目、URL、メッセージが整合しているか | 入出力関連文書 |
| 権限整合 | 権限一覧、権限別画面一覧、URL 制御が整合しているか | 権限関連文書 |
| DB 整合 | ER、表定義、DDL、DAO 設計に矛盾がないか | DB 関連文書 |
| クラス整合 | Controller / Service / DAO の責務分離が守られているか | クラス詳細設計書 |
| 異常系設計 | 例外条件、ログ、利用者メッセージが整理されているか | 例外/ログ関連文書 |
| 試験観点 | 設計内容が試験項目に落ちているか | 試験仕様書 |
| 保守性 | 変更時の追跡が可能な構成か | 台帳・変更管理文書 |

## 4. レビュー実施時の留意点

- 指摘は再現可能な根拠付きで記録すること。
- 文書単体ではなく、関連文書との整合で見ること。
- 実装済コードがある場合は、設計書との差分も確認すること。
- 指摘内容は `レビュー記録票`、`レビュー指摘一覧` に反映すること。
