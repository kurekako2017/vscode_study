## 文档样式特点

日本项目里常见的文档特点：

- 文档编号明确
- 改版履历明确
- 表格化表达
- 输入、输出、前提条件、异常情况写清楚
- 测试项强调 `観点`、`期待結果`、`実績結果`
- 障害与改修强调 `発生条件`、`原因`、`対応内容`、`影響範囲`
