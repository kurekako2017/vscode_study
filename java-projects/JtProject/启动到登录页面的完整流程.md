## 概述

当你启动应用并访问 `http://localhost:8080/` 时，页面显示登录界面的过程涉及：
- ✅ Spring Boot 应用启动
- ✅ Spring MVC 路由映射
- ✅ Controller 处理请求
- ✅ JSP 视图解析
- ✅ 页面渲染返回

## 详细步骤解析

### 🔹 步骤1-3：应用启动阶段

**JtSpringProjectApplication.main()**
```java
@SpringBootApplication(exclude = HibernateJpaAutoConfiguration.class)
public class JtSpringProjectApplication {
    public static void main(String[] args) {
        // 这是程序的入口点
        SpringApplication.run(JtSpringProjectApplication.class, args);
    }
}
```

**作用**：
- 初始化 Spring 容器
- 扫描并注册所有 Bean
- 配置 Spring MVC 组件
- 启动嵌入式 Tomcat 服务器

### 🔹 步骤4-8：请求路由阶段

**用户访问 `http://localhost:8080/`**

Spring MVC 的 **DispatcherServlet** 负责接收所有 HTTP 请求：

```java
// Spring 内部逻辑（简化版）
public class DispatcherServlet {
    protected void doDispatch(HttpServletRequest request) {
        // 1. 获取请求路径
        String uri = request.getRequestURI(); // "/"
        String method = request.getMethod(); // "GET"
        
        // 2. 查找匹配的处理器
        HandlerExecutionChain handler = getHandler(request);
        // 找到: UserController.userlogin()
        
        // 3. 调用处理器方法
        ModelAndView mv = handler.handle(request, response);
        
        // 4. 解析视图
        render(mv, request, response);
    }
}
```

### 🔹 步骤9：Controller 处理

**UserController.java**
```java
@Controller
public class UserController {
    
    @GetMapping("/")  // ← 映射根路径
    public String userlogin(Model model) {
        // 可以在这里添加数据到 Model
        // model.addAttribute("message", "欢迎登录");
        
        return "userLogin";  // ← 返回视图名称（不是文件路径）
    }
}
```

**关键点**：
- `return "userLogin";` 不是文件路径
- 这是一个**逻辑视图名称**
- 需要通过 ViewResolver 解析成实际的文件路径

### 🔹 步骤10：视图解析

**application.properties 配置**
```properties
spring.mvc.view.prefix=/views/
spring.mvc.view.suffix=.jsp
```

**视图解析过程**：
```
逻辑视图名: "userLogin"
         ↓
ViewResolver 解析:
  /views/ + userLogin + .jsp
         ↓
物理路径: /views/userLogin.jsp
         ↓
实际文件: src/main/webapp/views/userLogin.jsp
```

### 🔹 步骤11-13：JSP 渲染和响应

**userLogin.jsp 文件内容**
```jsp
<%@ page contentType="text/html;charset=UTF-8" language="java" %>
<!DOCTYPE html>
<html>
<head>
    <title>用户登录 - JT电商系统</title>
    <link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/.../bootstrap.min.css">
</head>
<body>
    <div class="container">
        <h2>User Login coke1</h2>
        <form action="userloginvalidate" method="post">
            <input type="text" name="username" placeholder="Username*" required>
            <input type="password" name="password" placeholder="Password*" required>
            <button type="submit">Login</button>
        </form>
        <a href="/register">Register here</a>
    </div>
</body>
</html>
```

**渲染结果**：生成完整的 HTML 页面返回给浏览器

## 关键配置说明

### 1. **Spring MVC 视图解析器配置**

```properties
# application.properties
spring.mvc.view.prefix=/views/
spring.mvc.view.suffix=.jsp
```

**作用**：告诉 Spring 如何将逻辑视图名转换为物理路径

### 2. **JSP 文件位置**

```
项目结构:
src/main/webapp/views/userLogin.jsp  ← 源文件位置

运行时编译后:
target/classes/META-INF/resources/views/userLogin.jsp  ← 运行时位置
```

### 3. **路由映射注解**

```java
@GetMapping("/")        // 等价于 @RequestMapping(value="/", method=RequestMethod.GET)
@PostMapping("/login")  // 等价于 @RequestMapping(value="/login", method=RequestMethod.POST)
```

## 总结

### 完整调用链（简化版）

```
JtSpringProjectApplication.main()
  ↓
SpringApplication.run()
  ↓
启动 Tomcat (监听 8080 端口)
  ↓
用户访问 http://localhost:8080/
  ↓
Tomcat 接收请求
  ↓
DispatcherServlet 路由
  ↓
查找匹配的 @GetMapping("/")
  ↓
调用 UserController.userlogin()
  ↓
返回 "userLogin"
  ↓
ViewResolver 解析: /views/userLogin.jsp
  ↓
渲染 JSP 生成 HTML
  ↓
返回 HTTP 响应
  ↓
浏览器显示登录页面
```

### 关键要点

1. **JtSpringProjectApplication** 只负责启动应用，不直接处理请求
2. **DispatcherServlet** 是 Spring MVC 的核心，负责路由分发
3. **@GetMapping("/")** 定义了路由规则
4. **UserController.userlogin()** 返回视图名称（不是文件路径）
5. **ViewResolver** 将视图名称解析为 JSP 文件路径
6. **JSP 引擎** 渲染页面生成 HTML
7. **浏览器** 最终显示登录界面
