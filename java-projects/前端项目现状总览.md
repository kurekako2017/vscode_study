# 前端项目现状总览

这份总览专门说明 `java-projects` 里几个“现代前端路线”项目当前到底是什么状态。

重点回答三个问题：

1. 现在这个项目是不是已经完全切到新前端了
2. JSP 还在不在运行链路里
3. 下一步如果要继续演进，应该往哪里走

## 总览表

| 项目 | 当前前端路线 | JSP 是否仍在 | 是否可直接删 JSP | 结论 |
| --- | --- | --- | --- | --- |
| `JtProject-Next` | Next.js + TypeScript | 是，仍保留在 `src/main/webapp/views/`，并有后端控制器路由引用 | 否 | 现在是混合结构，Next 是主学习线，JSP 主要做对照和兼容 |
| `JtProject-Vue` | Vue 3 + TypeScript | 是，仍保留在 `src/main/webapp/views/`，并有后端控制器路由引用 | 否 | 现在是混合结构，Vue 是主学习线，JSP 仍参与运行 |
| `JtProject-TypeScript` | Node.js + Express + React + TypeScript | 否 | 不适用 | 这是独立的纯 TypeScript 全栈路线，没有 JSP 这层 |

## 1. JtProject-Next

### 当前状态

- 后端仍是 Spring Boot
- 前端主线是 Next.js + TypeScript
- 项目里仍保留了 `src/main/webapp/views/`
- `WebMvcConfig` 和部分 Controller 仍然会把视图解析到 JSP

### 你应该怎么理解

这个项目不是“已经删掉 JSP 的纯 Next 项目”。

更准确地说：

- Next.js 是新的学习和展示主线
- JSP 仍然作为老页面和后端对照资料存在
- 如果想彻底移除 JSP，需要先把 Controller 的视图返回迁移掉

### next 步骤

如果后续要继续演进，建议两条线任选其一：

1. 继续把 Next 的前端学习补深，比如路由、Server Component、数据获取、表单和状态管理
2. 如果目标是纯前后端分离迁移，就把 JSP 路由逐步替换成 API + Next 页面

## 2. JtProject-Vue

### 当前状态

- 后端仍是 Spring Boot
- 前端主线是 Vue 3 + TypeScript + Vite
- 项目里仍保留了 `src/main/webapp/views/`
- 当前后端仍有 JSP 视图解析链路

### 你应该怎么理解

这个项目也不是“JSP 已经完全废弃”。

更准确地说：

- Vue 3 是主学习线
- JSP 仍然保留着，并且当前还在被后端路由引用
- 这套结构更像“新前端 + 老页面对照”的混合练习版

### next 步骤

如果后续要继续演进，建议：

1. 先继续补 Vue 的组件、组合式 API、状态管理和路由学习
2. 如果目标是清理遗留结构，再逐步把 JSP 页面迁移掉

## 3. JtProject-TypeScript

### 当前状态

- 这是独立的纯 TypeScript 全栈项目
- 后端是 Node.js + Express + TypeScript
- 前端是 React + Vite + TypeScript
- 共享类型放在 `packages/shared`
- 没有 JSP，也没有 Spring Boot 这层

### 你应该怎么理解

这个项目和前两个不一样：

- 它不是 Java Web 迁移版
- 它不是“保留 JSP 的混合版”
- 它是全新的 TypeScript 全栈路线

### next 步骤

如果后续要继续学习，建议顺着这条线补：

1. API 设计
2. 前后端共享类型
3. 组件和数据流
4. 构建与部署

## 一句话结论

- `JtProject-Next` 和 `JtProject-Vue` 都是 **新前端 + JSP 保留的混合项目**
- `JtProject-TypeScript` 是 **独立的纯 TypeScript 全栈项目**
- 如果目标是“删掉 JSP 只保留新前端”，那是迁移工程，不是删文件
