## ✅ 解决方案A：使用IntelliJ IDEA启动（强烈推荐）

### 为什么必须用IDEA？

命令行启动遇到的问题：
- 端口管理复杂
- 后台进程难以追踪
- 日志输出不完整
- 调试困难

IDEA的优势：
- ✅ 自动端口管理
- ✅ 一键启动/停止
- ✅ 完整日志显示
- ✅ 支持断点调试
- ✅ 热部署支持

### 详细步骤：

#### 1. 打开IDEA并导入项目

```
1. 启动IntelliJ IDEA
2. File → Open
3. 选择：D:\dev\source_code\vscode_study\java-projects\JtProject-Next
4. 点击 OK
5. 等待项目加载和索引完成（右下角进度条）
```

#### 2. 配置JDK（如果需要）

```
1. 按 Ctrl + Alt + Shift + S 打开项目结构
2. 左侧选择 "Project"
3. "Project SDK" 选择 JDK 11 或 JDK 21
4. 点击 OK
```

#### 3. 刷新Maven依赖

```
1. 点击右侧的 "Maven" 标签
2. 点击刷新图标 🔄（Reload All Maven Projects）
3. 等待依赖下载完成
```

#### 4. 启动应用

**方式1：运行主类（最简单）**

```
1. 按 Ctrl + N 打开查找类
2. 输入：JtSpringProjectApplication
3. 打开文件后，找到 main 方法
4. 点击方法左侧的绿色 ▶️ 图标
5. 选择 "Run 'JtSpringProjectApplication.main()'"
```

**方式2：使用运行配置**

```
1. 顶部工具栏找到运行配置下拉框
2. 选择 "JtSpringProjectApplication"
3. 点击绿色 ▶️ 运行按钮
4. 或按快捷键 Shift + F10
```

#### 5. 查看启动日志

```
底部会打开 "Run" 控制台
等待看到：
========================================
JT电商系统启动成功！
访问地址: http://localhost:8082
========================================
```

#### 6. 访问应用

```
浏览器打开：http://localhost:8082
应该看到用户登录页面
```

#### 7. 如果遇到端口占用

```
IDEA会弹出提示：
"Port 8082 is already in use"

点击提示中的选项：
1. "Stop the application" - 停止旧实例
2. 或手动停止：点击控制台的红色 ⏹️ 按钮
3. 然后重新运行
```

## ✅ 解决方案B：命令行启动（需要先彻底清理）

### 步骤1：彻底停止所有Java进程

打开PowerShell执行：

```powershell
# 方法1：停止所有Java进程
Get-Process -Name java -ErrorAction SilentlyContinue | Stop-Process -Force

# 方法2：如果方法1不行，找到具体PID
netstat -ano | findstr ":8082"
# 记下最后一列的PID，然后：
taskkill /F /PID <替换为实际的PID>

# 方法3：终极方案 - 重启电脑
# 如果以上都不行，重启电脑释放所有端口
```

### 步骤2：确认端口已释放

```powershell
netstat -ano | findstr ":8082"
# 应该没有任何输出
```

### 步骤3：启动应用

```powershell
cd D:\dev\source_code\vscode_study\java-projects\JtProject-Next

# 方式1：使用Maven（推荐）
mvn spring-boot:run -Dmaven.test.skip=true

# 方式2：直接运行JAR
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar

# 方式3：如果8082端口有问题，换端口
& "C:\Program Files\jdk-21.0.2\bin\java.exe" -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar --server.port=8083
```

### 步骤4：等待启动完成

```
等待看到日志：
- Tomcat initialized with port(s): 8082 (http)
- Started JtSpringProjectApplication in X.XXX seconds
```

### 步骤5：访问应用

```
http://localhost:8082
或
http://localhost:8083（如果换了端口）
```

## ✅ 解决方案C：修改默认端口（永久解决端口冲突）

### 修改配置文件

编辑文件：`src/main/resources/application.properties`

添加：

```properties
# 修改服务器端口为8083
server.port=8083
```

保存后重新启动，应用将运行在8083端口。

访问地址变为：`http://localhost:8083`

## 🎯 常见错误及解决

### 错误1：Whitelabel Error Page (404)

**原因**：
- 应用已启动，但访问路径错误
- 或JSP视图没有正确部署

**解决方法**：

1. **确认正确的访问路径**
   ```
   正确：http://localhost:8082/
   错误：http://localhost:8082/error
   ```

2. **查看可用的路径**
   - 用户登录：`/`
   - 管理员登录：`/admin/login`
   - 用户注册：`/register`

3. **如果首页404，尝试**：
   ```
   http://localhost:8082/register
   http://localhost:8082/admin/login
   ```

### 错误2：Port 8082 is already in use

**解决方法**：

```powershell
# 快速解决
Get-Process -Name java | Stop-Process -Force
Start-Sleep -Seconds 5
# 然后重新启动

# 如果不行，重启电脑
```

### 错误3：JSP不显示内容

**原因**：JSP文件没有正确打包到JAR中

**解决方法**：

```powershell
# 重新打包
mvn clean package -Dmaven.test.skip=true

# 然后运行新的JAR
java -jar .\target\JtSpringProject-0.0.1-SNAPSHOT.jar
```

## 🆘 如果所有方法都失败

### 终极解决方案：

1. **重启电脑**
   - 彻底释放所有端口
   - 清理所有进程

2. **重新下载项目**
   - 可能项目文件损坏
   - 重新从源克隆

3. **检查防火墙**
   - 防火墙可能阻止8082端口
   - 临时关闭防火墙测试

4. **检查杀毒软件**
   - 杀毒软件可能拦截
   - 添加项目目录到白名单
