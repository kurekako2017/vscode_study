# 83. 模擬バッチ設計書

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 模擬バッチ設計書 |
| 対象システム | JtProject |
| 文書区分 | 学習用バッチ設計資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、`JtProject-Next` に対して最小構成の模擬バッチを追加し、日本案件でよくあるバッチ設計、ジョブ管理、結果出力の流れを学習できるようにすることを目的とする。

## 3. バッチ概要

| 項目 | 内容 |
|---|---|
| バッチ ID | `BAT-LAB-001` |
| バッチ名 | 商品在庫整合チェック |
| 起動クラス | `ProductInventoryCheckBatchApplication` |
| 実行方式 | Spring Boot 非 Web 起動 + Maven Exec |
| プロファイル | `batch` |
| ワーク DB | `batch-work/jtproject-batch` |
| 出力先 | `batch-output/product_inventory_check_yyyyMMdd_HHmmss.csv` |
| ログ出力先 | `logs/batch/product-inventory-check.log` |
| 起動スクリプト | `scripts/start/run-product-inventory-check.cmd` |

## 4. 処理内容

対象テーブル:

- `PRODUCT`
- `CATEGORY`

チェック内容:

- 商品名が空でないこと
- 分類が設定されていること
- 在庫数が 0 以上であること
- 価格が 0 より大きいこと

## 5. 入出力

### 5.1 入力

| 項目 | 内容 |
|---|---|
| DB | `application-batch.properties` で指定した H2 ファイル DB |
| 初期データ | `data.sql` |

### 5.1.1 実行時ディレクトリ

| ディレクトリ | 役割 |
|---|---|
| `batch-work/` | batch profile 用の H2 ファイル DB を配置する作業ディレクトリ |
| `batch-output/` | 業務結果 CSV の出力先 |
| `logs/batch/` | batch 実行ログの出力先 |

補足:

- `batch-work/jtproject-batch.mv.db` は batch 実行時に生成される H2 データファイルである。
- `batch-work/jtproject-batch.trace.db` は H2 の trace 情報である。
- これらはソースコードではなく、batch 実行時の作業ファイルである。

### 5.2 出力

| 項目 | 内容 |
|---|---|
| 結果 CSV | 商品ごとの整合チェック結果 |
| アプリログ | 実行件数、NG 件数、出力ファイルパス |
| 終了コード | `0`: 正常、`2`: 整合性 NG あり、`1`: システム異常 |

### 5.3 CSV 出力項目

| 項目名 | 内容 |
|---|---|
| `check_timestamp` | 実行日時 |
| `product_id` | 商品 ID |
| `product_name` | 商品名 |
| `category_id` | 分類 ID |
| `category_name` | 分類名 |
| `quantity` | 在庫数 |
| `price` | 価格 |
| `status` | `OK` / `NG` |
| `check_messages` | エラー内容 |

## 6. 処理フロー

1. `batch` profile で Spring Boot を非 Web 起動する。
2. `PRODUCT` 一覧を取得する。
3. 商品ごとに整合チェックを行う。
4. 結果を CSV に出力する。
5. ログへ件数を出力する。
6. 終了コードを返却する。

## 7. 実行方法

### 7.1 Windows CMD

```cmd
scripts\start\run-product-inventory-check.cmd
```

### 7.2 IDE 実行

起動クラス:

```text
com.jtspringproject.JtSpringProject.batch.launcher.ProductInventoryCheckBatchApplication
```

Program Arguments 例:

```text
--batch.jobName=productInventoryCheck
```

## 8. 学習ポイント

- Web アプリとは別にバッチ専用の起動クラスを持つこと
- `bat` から起動できること
- 結果を CSV に出すこと
- 終了コードで正常 / 業務 NG / システム異常を分けること
- 後続で `JP1` や `Systemwalker` に載せる前提の形にしておくこと
