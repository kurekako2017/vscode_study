# インターフェース一覧

## 1. 目的

本書は `JtProject` で使用している公開インターフェース、主要入出力境界、外部公開 URL を一覧化し、呼出元と呼出先の責務を整理する。

## 2. インターフェース分類

| 分類 | 対象 | 内容 |
|---|---|---|
| 画面インターフェース | Controller URL | 画面からの HTTP リクエスト受付 |
| 業務インターフェース | Service Interface | Controller から業務処理を呼び出す契約 |
| データアクセスインターフェース | DAO Interface | Service から永続化処理を呼び出す契約 |
| 永続化境界 | Entity / DB | DB テーブルとの読み書き境界 |

## 6. インターフェース設計上の注意

- 画面インターフェースと Service / DAO インターフェースで責務分離はされているが、`UserController` から `CartProductDao` を直接参照する箇所が存在する。
- `deletProduct`、`deletCategory` のようにスペル揺れが残るメソッドがある。
- 実案件では DTO、Request、Response の専用インターフェースを追加し、境界をより明確にすることが多い。
- `/buy` に対応する Controller は存在するが、現時点では対応 JSP が配置されていないため本一覧では正式画面 IF から除外した。
- 外部システム連携 API は現状存在しないため、本書では内部画面 IF 中心に整理している。
