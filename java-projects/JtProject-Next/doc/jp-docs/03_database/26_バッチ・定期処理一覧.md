## 2. 目的

本書は、システムで使用するバッチ処理および定期処理の有無、役割、実行タイミング、入出力、異常時対応を整理するための資料である。

## 3. 現行方針

`JtProject` は学習用 Web アプリケーションであり、現行実装では独立した Spring Batch ジョブ、OS スケジューラ連携、夜間定期処理は実装されていない。  
ただし、日本案件の設計資料として必要となるため、本書では以下の区分で整理する。

- 現行実装済み処理
- 運用上想定される定期処理
- 今後追加候補となる処理

## 3.1 実装確認結果

| 確認項目 | 現状 |
|---|---|
| `Spring Batch` 依存関係 | なし |
| `@Scheduled` 定期実行 | なし |
| `Quartz` 利用 | なし |
| `CommandLineRunner` / `ApplicationRunner` による業務バッチ | なし |
| JP1 など外部ジョブ管理との連携 | なし |

補足:

- 現在存在する `mvnw.cmd` の batch は Maven Wrapper 用であり、業務バッチではない。
- 初期データ投入はアプリ起動時 SQL や手動 SQL 実行に近い扱いであり、ジョブ管理製品配下の定期バッチではない。
- 学習用の模擬 batch では、`batch-work/` に H2 作業 DB、`batch-output/` に結果 CSV、`logs/batch/` に batch ログを出力する構成を採用している。

## 4. バッチ・定期処理一覧

| No | 処理ID | 処理名 | 区分 | 実行契機 | 入力 | 出力 | 対象テーブル/ファイル | Javaクラス | 現状 |
|---|---|---|---|---|---|---|---|---|---|
| 1 | BAT-001 | アプリケーション起動初期化 | 起動処理 | サーバ起動時 | `application.properties`、初期設定 | Spring Boot 起動完了 | 設定ファイル、ログ | `com.jtspringproject.JtSpringProject.JtSpringProjectApplication` | 実装済み |
| 2 | BAT-002 | 初期データ投入 | 初期化処理 | 初回起動時または開発準備時 | DDL、初期マスタ、テストデータ | `CUSTOMER`、`CATEGORY`、`PRODUCT` 等へ初期データ投入 | DB テーブル | なし（`basedata.sql`、`data.sql`） | 実装済み相当 |
| 3 | BAT-003 | テストデータリセット | 開発支援処理 | 開発者手動実行 | 初期データ定義 | DB 初期化結果 | DB テーブル | `com.jtspringproject.JtSpringProject.batch.launcher.TestDataResetBatchApplication` / `TestDataResetBatchService` | 実装済み |
| 4 | BAT-004 | アプリケーションログローテーション | 定期処理 | 日次または容量閾値超過時 | ログファイル | 圧縮済みログ、現行ログ | ログファイル | `com.jtspringproject.JtSpringProject.batch.launcher.AppLogRotationBatchApplication` / `AppLogRotationBatchService` | 実装済み |
| 5 | BAT-005 | 商品・カテゴリマスタ整合チェック | 監視処理 | 日次 | `PRODUCT`、`CATEGORY` | 整合チェック結果レポート | DB テーブル | `com.jtspringproject.JtSpringProject.batch.launcher.ProductCategoryCheckBatchApplication` / `ProductCategoryCheckBatchService` | 実装済み |
| 6 | BAT-006 | カート残留データ棚卸 | 定期処理 | 日次または週次 | `CART`、`CART_PRODUCT` | 不要データ一覧、削除候補 | DB テーブル | `com.jtspringproject.JtSpringProject.batch.launcher.CartResidualInventoryBatchApplication` / `CartResidualInventoryBatchService` | 実装済み |
| 7 | BAT-007 | DB バックアップ | 運用処理 | 日次夜間 | DB 本体 | バックアップファイル | DB ファイル | `com.jtspringproject.JtSpringProject.batch.launcher.DatabaseBackupBatchApplication` / `DatabaseBackupBatchService` | 実装済み |

補足:

- 学習用の模擬バッチ（`BAT-LAB-001`）は、`BAT-005` とは別枠で以下のクラスを使用する。
- `com.jtspringproject.JtSpringProject.batch.launcher.ProductInventoryCheckBatchApplication`
- `com.jtspringproject.JtSpringProject.batch.service.ProductInventoryCheckBatchService`
- Quartz 学習用: `com.jtspringproject.JtSpringProject.batch.quartz.QuartzProductInventoryCheckApplication`

## 5. 各処理の概要

### 5.1 BAT-001 アプリケーション起動初期化

| 項目 | 内容 |
|---|---|
| 目的 | Web アプリケーションを起動し、Bean 初期化、DB 接続、画面提供を可能にする |
| 実行方式 | Spring Boot 起動 |
| 主処理 | 設定読込、DI 初期化、接続設定、Web コンテナ起動 |
| 正常終了条件 | ログイン画面またはトップ画面へアクセス可能であること |
| 異常時対応 | 起動ログ確認、設定値確認、DB 接続設定確認 |

### 5.2 BAT-002 初期データ投入

| 項目 | 内容 |
|---|---|
| 目的 | 開発・検証に必要な顧客、カテゴリ、商品データを準備する |
| 実行方式 | SQL 実行またはアプリ起動時初期化 |
| 入力 | DDL、テストデータ定義 |
| 出力 | 初期状態の `CUSTOMER`、`CATEGORY`、`PRODUCT`、`CART`、`CART_PRODUCT` |
| 注意点 | 再投入時は既存データ重複や採番競合に注意する |

### 5.3 BAT-003 テストデータリセット

| 項目 | 内容 |
|---|---|
| 目的 | 単体・結合・総合テスト前にデータ状態を初期化する |
| 実行方式 | 開発者による手動実行を想定 |
| 主処理 | カート系データ削除、商品・カテゴリ・顧客マスタ再投入 |
| 対象 | `CART_PRODUCT`、`CART`、`PRODUCT`、`CATEGORY`、`CUSTOMER` |
| 備考 | 実案件では専用スクリプト化を推奨する |

### 5.4 BAT-004 以降の運用想定処理

| 処理ID | 処理概要 | 必要性 | 補足 |
|---|---|---|---|
| BAT-004 | ログローテーション | 中 | 長期運用時にログ肥大化を防止する |
| BAT-005 | マスタ整合チェック | 中 | 商品とカテゴリの不整合検知に有効 |
| BAT-006 | カート残留データ棚卸 | 低〜中 | 学習用では不要だが実案件では有効 |
| BAT-007 | DB バックアップ | 高 | 本番運用では必須 |

## 6. バッチ・定期処理の設計観点

- 実行契機を明確にすること
- 手動処理と定期処理を区別すること
- 入力、出力、対象データを明記すること
- 異常時の通知先、復旧方法、再実行可否を定義すること
- 将来 Spring Batch や OS スケジューラへ移行しやすい粒度で整理すること

## 7. 学習用の模擬構成案

### 7.1 JP1 を使わずに学ぶ最小構成

| 要素 | 学習用の代替案 | 役割 |
|---|---|---|
| バッチ本体 | `Spring Boot` の `CommandLineRunner` または単独 `main` | 業務処理実行 |
| スケジューラ | Windows タスク スケジューラ / cron | 定刻起動 |
| ジョブネット管理 | `bat` / `ps1` / `sh` スクリプト | 前処理 -> 本処理 -> 後処理の連結 |
| 戻りコード管理 | `%ERRORLEVEL%` / `$LASTEXITCODE` | 正常・異常判定 |
| 実行ログ | ファイル出力 | 実行証跡 |
| 管理台帳 | Excel / CSV | 実行条件、結果、再実行履歴管理 |

### 7.2 JtProject で模擬しやすい題材

| 模擬バッチ | 内容 | 学習ポイント |
|---|---|---|
| 商品在庫整合チェック | `PRODUCT` の在庫・カテゴリ不整合を点検する | SQL、戻りコード、結果出力 |
| カート残留データ棚卸 | 古い `CART` / `CART_PRODUCT` を抽出する | バッチ検索、CSV 出力 |
| 日次商品件数集計 | 商品数、カテゴリ別件数を集計する | 集計 SQL、日次結果表 |
| テストデータ初期化 | 試験前にデータを作り直す | 再実行、前処理/後処理 |

### 7.3 学習ステップ

1. Java 単体プログラムまたは `Spring Boot` の CLI 実行処理を 1 本作る。
2. 実行結果をログファイルと CSV に出力する。
3. `bat` または `PowerShell` で前処理、本処理、後処理を連結する。
4. タスクスケジューラで定時実行する。
5. [60_ジョブネット一覧.md](../06_operations/60_ジョブネット一覧.md) のジョブネット形式で設計書を起こす。

### 7.4 JP1 学習観点として押さえる項目

- ジョブネット
- 実行順序制御
- 戻りコード判定
- 異常時中断と後続抑止
- 再実行
- 実行履歴管理
- 通知、監視、ログ確認

## 8. 留意事項

- 現在の `JtProject` では定期処理未実装のため、本書の一部は運用設計上の想定を含む。
- 実案件へ展開する場合は、実行ユーザー、ジョブネット、戻りコード、監視方法まで詳細化する。
