## 2. 目的

本書は、設計、試験、レビュー、リリース、引継ぎ時に確認すべき主要チェック項目を一覧化し、抜け漏れを防止するための資料である。

## 3. 利用方針

- 各工程の完了判定前に本書の該当チェックリストを確認する。
- 実案件ではチェック実施者、実施日、判定を記録する運用とする。
- 本書は総覧であり、個別の詳細チェック票は必要に応じて別紙化する。

## 4. チェックリスト一覧

### 4.1 設計書チェック

| No | チェック項目 |
|---|---|
| D-01 | 設計書番号、文書名、版数、作成日が記載されているか |
| D-02 | 機能概要、入力、出力、処理手順が記載されているか |
| D-03 | 異常系、例外系、権限制御が記載されているか |
| D-04 | 画面、URL、DB、クラス設計間に矛盾がないか |
| D-05 | 関連資料へのリンクが整備されているか |

### 4.2 DB 設計チェック

| No | チェック項目 |
|---|---|
| DB-01 | テーブル一覧、ER 図、テーブル定義書、DDL 一覧が整合しているか |
| DB-02 | PK / FK / NULL / UK が明記されているか |
| DB-03 | 業務上の用途説明があるか |
| DB-04 | テストデータ投入方法が明確か |
| DB-05 | 継続課題があれば課題管理表へ反映されているか |

### 4.3 試験チェック

| No | チェック項目 |
|---|---|
| T-01 | 単体、結合、総合テスト仕様書が揃っているか |
| T-02 | 前提条件、入力値、期待結果が具体化されているか |
| T-03 | テストデータ一覧が最新か |
| T-04 | 試験結果報告書に結果が反映されているか |
| T-05 | テスト実施記録一覧に実績が記録されているか |

### 4.4 レビューチェック

| No | チェック項目 |
|---|---|
| R-01 | レビュー記録票に実施履歴があるか |
| R-02 | レビュー指摘一覧に重要指摘が反映されているか |
| R-03 | 指摘の対応状況と再確認結果が更新されているか |
| R-04 | 承認履歴一覧と矛盾がないか |

### 4.5 リリースチェック

| No | チェック項目 |
|---|---|
| RL-01 | 試験完了または保留理由が明確か |
| RL-02 | DDL / 設定差分が整理されているか |
| RL-03 | 移行・リリース手順書が最新か |
| RL-04 | ロールバック手順が明記されているか |
| RL-05 | 監視・障害対応手順書が参照可能か |

### 4.6 引継ぎチェック

| No | チェック項目 |
|---|---|
| H-01 | 引継ぎ資料が最新か |
| H-02 | 重要文書への入口が明確か |
| H-03 | 未解決課題、リスク、障害一覧が最新か |
| H-04 | 保守運用 FAQ が整備されているか |
| H-05 | 用語集とドキュメント相関図が参照可能か |

## 5. 利用ルール

- 各工程の完了時に該当チェックリストを確認する。
- NG 項目があれば課題、障害、改修、レビュー指摘へ連携する。
- 将来、個別チェック票が必要になれば本書を親台帳として使う。

## 6. 備考

- 実案件ではチェック者、確認日、OK / NG 判定欄、コメント欄を追加する。
- 監査用途に使う場合は承認欄も追加する。
