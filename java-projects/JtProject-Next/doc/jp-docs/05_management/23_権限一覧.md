# 権限一覧

## 1. 目的

本書は `JtProject` における利用者ロール別の画面利用権限、操作権限を整理する。

## 2. ロール一覧

| ロール | 論理名 | 説明 |
|---|---|---|
| `ROLE_NORMAL` | 一般ユーザー | 商品閲覧、カート利用、一般ログイン利用者 |
| `ROLE_ADMIN` | 管理者 | 商品管理、カテゴリ管理、顧客一覧、管理者画面利用者 |

## 3. 権限一覧表

| No | 機能/画面 | 一般ユーザー | 管理者 | 備考 |
|---|---|---|---|---|
| 1 | ユーザーログイン | ○ | △ | 管理者でも一般ログイン自体は可能だが運用上は別導線 |
| 2 | ユーザー登録 | ○ | ○ | 実装上は制限なし |
| 3 | 商品トップ表示 | ○ | ○ | 管理者も遷移可能 |
| 4 | 一般商品一覧 | ○ | ○ | 管理者でも表示可能 |
| 5 | カート追加 | ○ | × | 管理者は明示的に禁止 |
| 6 | カート表示 | ○ | × | 管理者は管理画面へ誘導 |
| 7 | 管理者ログイン | × | ○ | 管理者専用 |
| 8 | 管理者ホーム | × | ○ | `adminLoggedIn=true` 前提 |
| 9 | カテゴリ管理 | × | ○ | 管理者専用 |
| 10 | 商品管理 | × | ○ | 管理者専用 |
| 11 | 顧客一覧表示 | × | ○ | 管理者専用 |
| 12 | プロフィール更新 | × | ○ | 管理者専用 |
| 13 | テスト画面 | 開発用途 | 開発用途 | 運用対象外 |

## 4. 権限制御方針

- 一般ユーザー系は `username` の Cookie / Session を利用
- 管理者系は Session の `adminLoggedIn`, `adminUsername` を利用
- 管理者による一般カート利用は Controller で抑止
- 本格案件では Spring Security 等による共通認可制御へ移行余地あり

## 5. 補足

- 現状は画面ごとに Controller 内で個別制御している
- URL ベースの共通認可フィルタは未実装
