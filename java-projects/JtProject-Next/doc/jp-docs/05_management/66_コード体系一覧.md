# 66. コード体系一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | コード体系一覧 |
| 対象システム | JtProject |
| 文書区分 | 管理標準資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、画面 ID、機能 ID、文書 ID、障害番号、改修番号など、プロジェクト内で用いるコード体系を統一することを目的とする。

## 3. コード体系一覧

| 区分 | 接頭辞 | 例 | 用途 |
|---|---|---|---|
| 画面 ID | `SCR-` | `SCR-01` | 画面識別 |
| 機能 ID | `FUN-` | `FUN-01` | 機能識別 |
| 文書 ID | `DOC-` | `DOC-15b-02` | 文書識別 |
| テスト ID | `UT-` / `IT-` / `ST-` | `UT-001` | 単体/結合/総合試験識別 |
| 障害 ID | `BUG-` | `BUG-001` | 障害管理 |
| 改修 ID | `CHG-` | `CHG-001` | 改修管理 |
| 調査 ID | `INV-` | `INV-001` | 調査管理 |
| リスク ID | `RSK-` | `RSK-001` | リスク管理 |
| 課題 ID | `ISSUE-` | `ISSUE-001` | 課題管理 |
| ログ ID | `LOG-` | `LOG-01` | ログ設計識別 |
| 設定 ID | `CFG-` | `CFG-01` | 設定値識別 |
| ジョブネット ID | `JN-` | `JN-01` | ジョブネット識別 |
| 帳票 ID | `RPT-` | `RPT-01` | 帳票識別 |

## 4. 採番ルール

- 同一カテゴリ内では重複採番しないこと。
- 廃止した番号は再利用しないこと。
- 文書番号と文書 ID は必要に応じて併用すること。
- 画面、機能、URL、テスト項目のトレーサビリティを意識して採番すること。

## 5. 運用上の留意点

- 新規台帳や一覧を追加した場合は、本書にコード体系を追記すること。
- 既存接頭辞と重複する新規命名を避けること。
- コード体系変更時は `用語集`、`文档一覧`、関連台帳を見直すこと。
