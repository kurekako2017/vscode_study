## 2. 目的

本書は、実施した調査テーマを一覧で整理し、調査対象、調査理由、結果、対応方針を管理するための台帳である。

## 3. 管理方針

- 個別調査の詳細は `10_調査票.md` に記載する。
- 本書は調査案件の一覧性を担保する。
- 調査のきっかけが障害、設計課題、改善要求のどれかを明記する。

## 5. 状況定義

| 状況 | 内容 |

| 未着手 | 調査登録のみ |
| 調査中 | 情報収集、確認中 |
| 継続中 | 一次結論あり、追加確認継続 |
| 完了 | 調査結果と対応方針確定 |

## 6. 利用ルール

- 新規調査テーマ発生時は本書へ登録する。
- 詳細なログ、コード位置、結論は `10_調査票.md` に記載する。
- 調査結果が改修へ繋がる場合は `41_改修一覧.md` と連携する。
- 障害起因の場合は `40_障害一覧.md` の対象レコードと対応付ける。

## 7. 備考

- 実案件では調査担当、期限、結論日、レビュー結果を追加する。
- 原因未確定のまま長期化する調査は週次レビュー対象とする。
