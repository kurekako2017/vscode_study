## 2. 目的

本書は、各レビューで発生した指摘事項を一覧で管理し、指摘内容、重要度、対応状況、再確認結果を追跡するための台帳である。

## 3. 管理方針

- レビュー実施記録は `29_レビュー記録票.md` に記載する。
- 本書では指摘単位で一覧化し、対応漏れを防止する。
- 指摘対応後は再レビュー結果を更新する。

## 5. 重要度定義

| 重要度 | 内容 |
|---|---|
| 高 | 設計品質、実装品質、試験品質に大きく影響する |
| 中 | 資料として改善が必要、または運用に影響する |
| 低 | 軽微な表現修正、補足で対応可能 |

## 6. 状況定義

| 状況 | 内容 |
|---|---|
| 未対応 | 指摘登録のみ |
| 対応中 | 修正作業中 |
| 対応済み | 修正完了 |
| 保留 | 方針待ち、後続対応 |

## 7. 利用ルール

- 新規レビュー指摘は本書へ 1 行追加する。
- 詳細レビュー実施記録は `29_レビュー記録票.md` に記載する。
- 文書改訂時は `28_変更履歴一覧.md` にも反映する。
- 未解決の指摘は必要に応じて `45_未解決課題一覧.md` へ引き継ぐ。

## 8. 備考

- 実案件ではレビュアー、期限、レビュー会議体、指摘起票者を追加する。
- Excel 管理の場合はフィルタしやすい列順へ調整する。
