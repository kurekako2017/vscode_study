## 2. 目的

本書は、システムの基本監視観点と、障害発生時の一次切り分け・記録・復旧手順を整理するための資料である。

## 3. 監視対象

| 区分 | 監視項目 | 監視内容 |
|---|---|---|
| アプリ | 起動状態 | Spring Boot アプリが起動しているか |
| アプリ | 主要画面表示 | ログイン画面、商品一覧、管理者画面が表示できるか |
| DB | 接続可否 | DB 接続エラーが出ていないか |
| 業務 | ログイン可否 | 一般ユーザー、管理者でログインできるか |
| 業務 | 商品表示 | 商品一覧が取得できるか |
| 業務 | カート操作 | カート追加、表示、削除が可能か |
| ログ | 異常ログ | 例外、スタックトレース、連続エラーが出ていないか |

## 4. 日常監視手順

1. アプリケーション起動状態を確認する。
2. トップ画面またはログイン画面へアクセスする。
3. 一般ユーザーでログイン確認する。
4. 管理者でログイン確認する。
5. 商品一覧、管理者商品管理画面を確認する。
6. 異常ログ有無を確認する。

## 5. 障害レベル区分

| レベル | 定義 | 例 |
|---|---|---|
| 障害A | システム全体利用不可 | 起動不可、全画面 500 エラー |
| 障害B | 主要機能利用不可 | ログイン不可、商品一覧取得不可 |
| 障害C | 一部機能不具合 | カート削除不可、カテゴリ更新不可 |
| 障害D | 軽微表示不具合 | メッセージ不備、画面文言不整合 |

## 6. 障害発生時の基本フロー

1. 事象受付
2. 影響範囲確認
3. 再現確認
4. ログ確認
5. 一次対処
6. 障害票起票
7. 調査票作成
8. 改修要否判定
9. 復旧確認

## 7. 一次切り分け手順

| No | 確認項目 | 内容 |
|---|---|---|
| 1 | 発生画面 | どの URL、どの画面か確認する |
| 2 | 発生条件 | どの操作、どの入力値で発生したか確認する |
| 3 | 再現性 | 毎回発生するか、一時的か確認する |
| 4 | ログ | アプリログ、コンソールログを確認する |
| 5 | DB | 接続エラー、対象データ存在有無を確認する |
| 6 | 権限 | 一般ユーザーか管理者か確認する |
| 7 | 影響範囲 | 単機能か全体か確認する |

## 8. 一次対処例

| 障害種別 | 一次対処 |
|---|---|
| 起動不可 | 設定確認、ポート競合確認、再起動 |
| ログイン不可 | 初期データ確認、`CUSTOMER` データ確認 |
| 商品表示不可 | `PRODUCT` データ確認、商品取得処理確認 |
| カート不具合 | `CART`、`CART_PRODUCT` データ確認 |
| 管理者画面不可 | Session、権限設定確認 |

## 9. 障害記録ルール

- 事象は `08_障害票.md` に記録する。
- 原因調査は `10_調査票.md` に記録する。
- 改修実施時は `09_改修票.md` に反映する。
- 再発防止が必要な場合は `31_課題管理表.md`、`32_リスク一覧.md` へ反映する。

## 10. 復旧確認

1. 障害発生機能が復旧していること
2. 関連画面に波及していないこと
3. ログに継続エラーが出ていないこと
4. 必要に応じて結合テスト、総合テストを再実施すること

## 11. 備考

- 実案件では監視ツール、アラート閾値、連絡体制、エスカレーション先を追加する。
- 本書は学習用環境を前提とした簡易監視・障害対応手順である。
