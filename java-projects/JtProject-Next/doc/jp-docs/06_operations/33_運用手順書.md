## 2. 目的

本書は、開発・検証環境における `JtProject-Next` の起動、停止、初期データ準備、試験前確認、障害時初動の基本手順を整理するものである。

## 3. 運用対象

- 開発者
- テスト実施者
- 引継ぎを受けた保守担当者

## 4. 前提環境

| 項目 | 内容 |
|---|---|
| OS | Windows 11 |
| Java | Java 11 |
| ビルドツール | Maven Wrapper (`mvnw`, `mvnw.cmd`) |
| フレームワーク | Spring Boot 2.7.18 |
| DB | H2 または開発用 MySQL |
| 配置場所 | `java-projects/JtProject-Next` |

## 5. 起動手順

### 5.1 事前確認

1. Java が利用可能であることを確認する。
2. プロジェクト直下に `mvnw.cmd` が存在することを確認する。
3. 使用ポートが競合していないことを確認する。
4. DB 接続先設定を確認する。

### 5.2 起動

1. `JtProject-Next` 直下へ移動する。
2. 以下のいずれかの方法で起動する。

```powershell
.\mvnw.cmd spring-boot:run
```

または

```powershell
.\scripts\start\run.ps1
```

### 5.3 起動確認

1. ブラウザでトップ画面またはログイン画面へアクセスする。
2. 一般ユーザー画面、管理者ログイン画面の双方が開けることを確認する。
3. エラーログが出ていないことを確認する。

## 6. 停止手順

1. 起動中のターミナルで `Ctrl + C` を実行する。
2. プロセスが残っている場合は対象 Java プロセスを停止する。
3. 使用ポートが解放されたことを確認する。

## 7. 初期データ準備手順

### 7.1 目的

試験および画面確認に必要な顧客、カテゴリ、商品データを準備する。

### 7.2 手順

1. `27_DDL一覧.md` を参照し、必要に応じてテーブルを作成する。
2. 初期データ DML またはプロジェクト既定の初期化処理を実行する。
3. `25_テストデータ一覧.md` の利用者、カテゴリ、商品が利用可能であることを確認する。

### 7.3 確認項目

- `admin01`、`user01` でログイン可能であること
- カテゴリが参照できること
- 商品一覧に初期商品が表示されること

## 8. 試験前確認手順

1. アプリケーションが正常起動していること
2. セッションが不要な状態で開始できること
3. テスト対象データが初期状態にあること
4. 変更対象機能に対応する設計書・試験仕様書が最新であること

## 9. 障害時初動手順

| No | 手順 | 内容 |
|---|---|---|
| 1 | 事象確認 | どの画面、どの操作、どの入力で発生したか確認する |
| 2 | ログ確認 | アプリケーションログ、コンソールログを確認する |
| 3 | 再現確認 | 同一条件で再現するか確認する |
| 4 | 影響確認 | 一般ユーザー系か管理者系か、他機能へ波及するか確認する |
| 5 | 起票 | 必要に応じて `08_障害票.md`、`10_調査票.md` を作成する |
| 6 | 一時対応 | 再起動、データ初期化、設定確認などを実施する |

## 10. 日常運用チェック

- 起動可否
- ログイン可否
- 商品一覧表示可否
- 管理者ログイン可否
- 商品管理画面表示可否
- 異常ログの有無

## 11. 引継ぎ時の確認ポイント

- 起動方法が共有されていること
- 使用する初期データが明確であること
- 試験仕様書とテストデータ一覧が更新されていること
- 既知課題とリスクが課題管理表、リスク一覧に記載されていること

## 12. 備考

- 実案件ではバックアップ、監視、権限管理、障害連絡先、運用カレンダーまで含める。
- 本書は開発・検証環境向けの簡易運用手順であり、本番運用手順ではない。
