# 81. 単体テスト観点一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 単体テスト観点一覧 |
| 対象システム | JtProject |
| 文書区分 | 単体試験観点整理資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、`JtProject` の単体テスト観点を「設計書ベース」で整理するための資料である。  
コードから直接思いつく観点ではなく、`詳細設計書`、`機能設計書`、`入力チェック一覧`、`例外設計一覧` などから観点を抽出し、単体テスト仕様書へ落とし込むことを目的とする。

## 3. 観点抽出元

| 設計資料 | 単体テストへ落とす観点 |
|---|---|
| `03_詳細設計書.md` | 処理手順、分岐、戻り値、前提条件 |
| `13_機能設計書.md` | 正常系、業務ルール、機能成立条件 |
| `18_入力チェック一覧.md` | 必須、桁数、形式、空白、不正値 |
| `21_例外設計一覧.md` | 異常系、対象なし、処理失敗、例外変換 |
| `65_画面項目定義書.md` | 入力項目、出力項目、Model 格納値 |
| `67_状態遷移一覧.md` | 処理前後の状態変化 |
| `74_入力項目・DB項目対応一覧.md` | 画面入力と DB 更新項目の対応 |

## 4. 基本テスト観点

### 4.1 共通観点

| 観点分類 | 内容 |
|---|---|
| 正常系 | 設計どおりの主処理が成立すること |
| 入力チェック | 必須、空値、空白、桁数、形式、不正値を確認すること |
| 境界値 | 最小値、最大値、境界直前、境界直後を確認すること |
| 分岐 | `if/else`、権限、データ有無、存在/不存在の分岐を確認すること |
| DB 連携 | 取得件数、更新件数、対象テーブル、更新対象項目を確認すること |
| 画面返却 | View 名、redirect 先、Model 値、メッセージを確認すること |
| Session/Cookie | 設定、取得、削除、引継ぎが正しいこと |
| 権限 | 一般ユーザーと管理者で制御が分かれていること |
| 例外系 | 異常終了せず、設計どおりのメッセージまたは戻り値になること |
| 状態変化 | 処理前後でオブジェクトやデータ状態が設計どおりに変化すること |

### 4.2 層別観点

| 層 | 主な単体テスト観点 |
|---|---|
| Controller | リクエスト受領、パラメータ受渡し、View 遷移、Model、Session、権限判定 |
| Service | 業務ルール、重複判定、事前条件、呼出順序、戻り値 |
| DAO | 検索条件、取得件数、更新件数、対象存在有無、DB 反映結果 |

## 5. 機能別単体テスト観点

### 5.1 ユーザー登録

対象:

- `UserController.newUseRegister()`
- `UserService.addUser()`
- `UserDao.saveUser()`

| 観点分類 | 観点内容 | 設計書との関係 |
|---|---|---|
| 正常系 | ユーザー名、メール、パスワードが正常時に登録成功する | `機能設計書` |
| 必須チェック | `username`、`email`、`password` が空のとき登録しない | `入力チェック一覧` |
| 空白 | 前後空白が trim されること | `詳細設計書` |
| 重複 | 既存ユーザー名がある場合に登録しない | `機能設計書` |
| 初期値 | `role` が `ROLE_NORMAL` で設定される | `詳細設計書` |
| DB | `CUSTOMER` に 1 件追加される | `入力項目・DB項目対応一覧` |
| 画面返却 | 成功時 `userLogin.jsp`、失敗時 `register.jsp` | `画面遷移図` |

### 5.2 ユーザーログイン

対象:

- `UserController.userlogin()`
- `UserService.checkLogin()`
- `UserDao.getUser()`

| 観点分類 | 観点内容 | 設計書との関係 |
|---|---|---|
| 正常系 | 正しいユーザー名 / パスワードでログイン成功 | `機能設計書` |
| 異常系 | 認証失敗時にログイン画面へ戻る | `例外設計一覧` |
| 権限 | 一般ユーザーとしてログイン後に通常導線へ入る | `権限別画面一覧` |
| Session | `username`、`userRole` が設定される | `詳細設計書` |
| Cookie | `username` Cookie が設定される | `詳細設計書` |
| 商品読込 | ログイン成功時に商品一覧が Model に設定される | `機能設計書` |
| データなし | 商品 0 件でも画面異常終了しない | `例外設計一覧` |

### 5.3 管理者ログイン

対象:

- `AdminController.adminlogin()`
- `UserService.checkLogin()`

| 観点分類 | 観点内容 | 設計書との関係 |
|---|---|---|
| 正常系 | 管理者アカウントでログイン成功する | `機能設計書` |
| 権限 | 一般ユーザーでは管理画面へ入れない | `権限別画面一覧` |
| Session | `adminLoggedIn`、`adminUsername` が設定される | `詳細設計書` |
| 画面返却 | 成功時 `adminHome.jsp`、失敗時 `adminlogin.jsp` | `画面遷移図` |

### 5.4 カテゴリ管理

対象:

- `AdminController.getcategory()`
- `AdminController.addCategory()`
- `AdminController.updateCategory()`
- `AdminController.removeCategoryDb()`

| 観点分類 | 観点内容 | 設計書との関係 |
|---|---|---|
| 一覧表示 | カテゴリ一覧が取得・表示される | `機能設計書` |
| 追加正常 | 正常入力でカテゴリ追加できる | `機能設計書` |
| 追加入力チェック | 空文字カテゴリ名では追加しない | `入力チェック一覧` |
| 更新正常 | 対象カテゴリが更新される | `詳細設計書` |
| 更新入力チェック | 空文字カテゴリ名では更新しない | `入力チェック一覧` |
| 削除正常 | 対象カテゴリが削除される | `詳細設計書` |
| 権限 | 未ログインでは `adminlogin.jsp` が返る | `権限別画面一覧` |

### 5.5 商品管理

対象:

- `AdminController.getproduct()`
- `AdminController.addProduct()`
- `AdminController.updateproduct()`
- `AdminController.updateProduct()`
- `AdminController.removeProduct()`

| 観点分類 | 観点内容 | 設計書との関係 |
|---|---|---|
| 一覧表示 | 商品一覧が表示される | `機能設計書` |
| 追加正常 | 正常入力で商品追加できる | `機能設計書` |
| 追加入力チェック | 商品名空では登録しない | `入力チェック一覧` |
| カテゴリ参照 | 登録・更新時にカテゴリが取得できる | `入力項目・DB項目対応一覧` |
| 更新画面表示 | 既存商品を取得して更新画面に表示する | `画面項目定義書` |
| 更新正常 | 既存商品が更新される | `詳細設計書` |
| 画像保持 | 新画像未指定時に既存画像を保持する | `詳細設計書` |
| 削除正常 | 対象商品が削除される | `詳細設計書` |
| データなし | 対象商品なしでも異常終了しない | `例外設計一覧` |

### 5.6 カート機能

対象:

- `UserController.addToCart()`
- `UserController.showCart()`
- `UserController.deleteFromCart()`

| 観点分類 | 観点内容 | 設計書との関係 |
|---|---|---|
| 正常系 | ログイン済み一般ユーザーが商品をカートに追加できる | `機能設計書` |
| 未ログイン | 未ログイン時は追加せずメッセージを返す | `例外設計一覧` |
| 権限 | 管理者はカート利用不可で管理画面へ戻す | `権限別画面一覧` |
| カート生成 | カート未作成時は新規 `CART` を作成する | `状態遷移一覧` |
| 商品なし | 商品 ID 不正時は追加しない | `例外設計一覧` |
| 一覧表示 | カート内商品一覧が取得される | `機能設計書` |
| 削除正常 | 対象明細を削除できる | `詳細設計書` |
| 明細なし | 対象明細なしでも異常終了しない | `例外設計一覧` |
| Session メッセージ | `cartMsg` が設定・消去される | `詳細設計書` |

## 6. 単体テスト仕様書への落とし込み方

単体テスト観点から単体テスト仕様書へ落とすときは、以下の順で整理する。

1. 設計資料から観点を抽出する
2. 観点ごとに正常系 / 異常系 / 境界値 / 権限 / 例外系へ分類する
3. 1 観点 1 テストケースに分解する
4. 前提条件、入力値、期待結果、確認対象を明記する

例:

| 観点 | テストケース化例 |
|---|---|
| 商品名必須 | 商品名空で商品追加時、登録されず追加画面へ戻る |
| 管理者カート利用不可 | 管理者で `/products/addtocart` 実行時、`/admin/products` へ戻る |
| 既存画像保持 | 画像未指定で商品更新時、DB の image 項目が変更されない |

## 7. 運用上の注意

- 単体テスト観点はコード変更時だけでなく、設計変更時にも更新すること。
- 単体テスト仕様書にない観点が本書に追加された場合は、仕様書側へ反映すること。
- 実装にだけ存在し設計にない分岐を見つけた場合は、設計書との不一致として管理すること。
