# 例外設計一覧

## 1. 目的

本書は `JtProject` における主要な異常条件と、その時の処理方針、ユーザー向けメッセージ、ログ方針を整理する。

## 3. 例外設計方針

### 3.1 Controller 層

- 可能な範囲で例外を捕捉し、画面崩壊を避ける
- 失敗時は元画面または安全な画面へ戻す
- ユーザーには必要最低限のメッセージを返却する

### 3.2 Service 層

- 業務処理失敗時はログ出力のうえ例外再送出を基本とする
- ビジネスルール違反は Controller 側で判定されている箇所が多い

### 3.3 DAO 層

- DB アクセス失敗時は ERROR ログ出力
- `UserDao.getUser()` のみ空 `User` 返却という業務寄り挙動を持つ
