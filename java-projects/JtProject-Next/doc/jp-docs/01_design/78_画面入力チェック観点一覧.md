# 78. 画面入力チェック観点一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 画面入力チェック観点一覧 |
| 対象システム | JtProject |
| 文書区分 | 入力設計・試験観点資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、画面ごとに確認すべき入力チェック観点を整理し、設計レビューや試験観点抽出の基礎資料とすることを目的とする。

## 3. 観点一覧

| 画面ID | 画面名 | 項目 | 観点 | 確認内容 |
|---|---|---|---|---|
| SCR-01 | ユーザーログイン画面 | `username` | 必須 | 未入力時メッセージ表示 |
| SCR-01 | `password` | 必須 | 未入力時メッセージ表示 |
| SCR-01 | 認証情報 | 整合性 | 認証失敗時のメッセージ表示 |
| SCR-02 | `username` | 重複 | 既存ユーザ名登録不可 |
| SCR-02 | `email` | 形式 | メール形式チェック |
| SCR-02 | `password` | 桁数/強度 | 将来強化対象 |
| SCR-08 | `name` | 必須 | 分類名未入力時エラー |
| SCR-08 | `name` | 重複 | 同一分類名重複時の扱い |
| SCR-09 | `price` | 数値 | 数値以外入力時エラー |
| SCR-09 | `price` | 範囲 | 負数/0 未満禁止 |
| SCR-09 | `categoryId` | 存在確認 | 存在しない分類指定不可 |
| SCR-13 | `username` | 必須 | 未入力時の更新不可 |
| SCR-13 | `email` | 形式 | 形式不正時エラー |

## 4. 留意点

- 入力観点は実装済み/未実装を区別して管理すること。
- 画面項目定義、入力項目一覧、入力チェック一覧と差異が出ないようにすること。
