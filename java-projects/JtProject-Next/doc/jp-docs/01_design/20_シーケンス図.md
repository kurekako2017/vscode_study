# シーケンス図

## 1. 目的

本書は `JtProject` の主要処理について、画面から Controller、Service、DAO、DB に至る呼出順序を明示する。

## 2. ユーザーログイン処理

[Mermaid source: 20_シーケンス図-mermaid-1.mmd](assets/20_シーケンス図-mermaid-1.mmd)

<details>
<summary>Mermaid source (editable)</summary>

```mermaid
sequenceDiagram
    actor User as 一般ユーザー
    participant JSP as userLogin.jsp
    participant C as UserController
    participant S as UserService
    participant D as UserDao
    participant DB as CUSTOMER
    participant PS as ProductService
    participant PDB as PRODUCT

    User->>JSP: ユーザー名/パスワード入力
    JSP->>C: POST /userloginvalidate
    C->>S: checkLogin(username, password)
    S->>D: getUser(username, password)
    D->>DB: usernameで検索
    DB-->>D: ユーザー情報返却
    D-->>S: User or 空User
    S-->>C: 認証結果返却
    alt 認証成功
        C->>PS: getProducts()
        PS->>PDB: 商品一覧取得
        PDB-->>PS: 商品一覧
        PS-->>C: 商品一覧
        C-->>JSP: index.jsp表示
    else 認証失敗
        C-->>JSP: userLogin.jsp + エラーメッセージ
    end
```
</details>



## 3. カート追加処理

[Mermaid source: 20_シーケンス図-mermaid-2.mmd](assets/20_シーケンス図-mermaid-2.mmd)

<details>
<summary>Mermaid source (editable)</summary>

```mermaid
sequenceDiagram
    actor User as 一般ユーザー
    participant UI as index.jsp / uproduct.jsp
    participant C as UserController
    participant US as UserService
    participant CS as CartService
    participant PS as ProductService
    participant CPD as CartProductDao
    participant DB1 as CUSTOMER/CART
    participant DB2 as PRODUCT
    participant DB3 as CART_PRODUCT

    User->>UI: 商品の追加操作
    UI->>C: /products/addtocart?id=productId
    C->>US: getUserByUsername(username)
    US->>DB1: 顧客取得
    DB1-->>US: 顧客情報
    US-->>C: 顧客情報
    C->>CS: getCarts()
    CS->>DB1: カート一覧取得
    DB1-->>CS: カート一覧
    CS-->>C: カート一覧
    alt カート未存在
        C->>CS: addCart(cart)
        CS->>DB1: CART登録
        DB1-->>CS: 登録結果
        CS-->>C: 新規カート
    end
    C->>PS: getProduct(productId)
    PS->>DB2: 商品取得
    DB2-->>PS: 商品情報
    PS-->>C: 商品情報
    C->>CPD: addCartProduct(cart, product)
    CPD->>DB3: CART_PRODUCT登録
    DB3-->>CPD: 登録結果
    CPD-->>C: 登録完了
    C-->>UI: /user/cartへリダイレクト
```
</details>



## 4. 管理者商品更新処理

[Mermaid source: 20_シーケンス図-mermaid-3.mmd](assets/20_シーケンス図-mermaid-3.mmd)

<details>
<summary>Mermaid source (editable)</summary>

```mermaid
sequenceDiagram
    actor Admin as 管理者
    participant UI as productsUpdate.jsp
    participant C as AdminController
    participant CS as CategoryService
    participant PS as ProductService
    participant PD as ProductDao
    participant DB as PRODUCT/CATEGORY

    Admin->>UI: 更新内容入力
    UI->>C: POST /admin/products/update/{id}
    C->>CS: getCategory(categoryId)
    CS->>DB: CATEGORY取得
    DB-->>CS: カテゴリ情報
    CS-->>C: カテゴリ
    C->>PS: getProduct(id)
    PS->>PD: getProduct(id)
    PD->>DB: PRODUCT取得
    DB-->>PD: 商品情報
    PD-->>PS: 既存商品
    PS-->>C: 既存商品
    C->>C: 画像未入力時は既存画像引継ぎ
    C->>PS: updateProduct(id, product)
    PS->>PD: updateProduct(product)
    PD->>DB: PRODUCT更新
    DB-->>PD: 更新結果
    PD-->>PS: 更新後商品
    PS-->>C: 更新結果
    C-->>UI: /admin/productsへリダイレクト
```
</details>
