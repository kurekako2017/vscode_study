# 67. 状態遷移一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 状態遷移一覧 |
| 対象システム | JtProject |
| 文書区分 | 設計補足資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、`JtProject-Next` における主要オブジェクトおよび利用者状態の遷移を整理し、画面・機能・業務処理の整合を明確化することを目的とする。

## 3. 対象状態

- 利用者ログイン状態
- カート状態
- 管理対象データの更新状態
- 障害・改修・調査台帳状態

## 4. 状態遷移一覧

### 4.1 利用者ログイン状態

| 現在状態 | イベント | 次状態 | 処理内容 |
|---|---|---|---|
| 未ログイン | 正常ログイン | ログイン済 | Session に利用者情報保持 |
| 未ログイン | 認証失敗 | 未ログイン | エラーメッセージ表示 |
| ログイン済 | ログアウト | 未ログイン | Session 破棄 |
| ログイン済 | Session 切れ | 未ログイン | 再ログイン要求 |

### 4.2 カート状態

| 現在状態 | イベント | 次状態 | 処理内容 |
|---|---|---|---|
| カート未作成 | 商品追加 | カート作成済 | CART 作成後に明細追加 |
| カート作成済 | 商品追加 | 明細あり | CART_PRODUCT 追加または更新 |
| 明細あり | 商品削除 | 明細あり/空 | 対象明細削除 |
| 明細あり | 全削除 | 空カート | 明細消去、ヘッダ残置は要方針 |

### 4.3 商品管理状態

| 現在状態 | イベント | 次状態 | 処理内容 |
|---|---|---|---|
| 未登録 | 商品追加 | 登録済 | PRODUCT 保存 |
| 登録済 | 商品更新 | 登録済 | 対象商品更新 |
| 登録済 | 商品削除 | 削除済 | 対象商品削除 |

### 4.4 管理台帳状態

| 台帳 | 現在状態 | イベント | 次状態 |
|---|---|---|---|
| 障害票 | 新規 | 調査開始 | 対応中 |
| 障害票 | 対応中 | 修正完了 | 確認待ち |
| 障害票 | 確認待ち | 再現なし | 完了 |
| 改修票 | 起票済 | 実装開始 | 対応中 |
| 改修票 | 対応中 | 反映完了 | 完了 |

## 5. 設計上の留意点

- 状態が変わる処理は、Controller と Service の責務を分けて管理すること。
- 状態遷移に伴うメッセージ表示は `メッセージ一覧` と整合させること。
- 状態遷移条件は `機能設計書`、`詳細設計書` と矛盾しないこと。
