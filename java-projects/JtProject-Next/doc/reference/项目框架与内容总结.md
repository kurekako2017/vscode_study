## 🎯 项目概述

### 项目简介
JT电商系统是一个基于 **Spring Boot 2.7.18** 的传统MVC架构电商应用，提供完整的电商业务功能，包括用户管理、商品管理、购物车、订单处理等核心模块。

### 项目特色
- ✅ **传统MVC架构** - 清晰的三层架构设计
- ✅ **手动配置Hibernate** - 完全控制ORM行为
- ✅ **JSP视图技术** - 服务端渲染，SEO友好
- ✅ **双数据库支持** - H2（开发）+ MySQL（生产）
- ✅ **完整业务流程** - 从浏览商品到下单支付

### 应用场景
- 📚 **学习项目** - 适合学习Spring Boot和传统Java Web开发
- 🏪 **小型电商** - 可直接用于小型电商网站
- 🔧 **二次开发** - 基础功能完善，易于扩展

### 架构分层说明

#### 1. **Controller层（控制器层）**
- **职责**: 处理HTTP请求，参数验证，返回视图
- **文件数**: 2个控制器，54个接口方法
- **特点**: 
  - 使用 `@Controller` 注解
  - `@GetMapping` / `@PostMapping` 处理请求
  - `ModelAndView` / `RedirectView` 返回视图

#### 2. **Service层（服务层）**
- **职责**: 业务逻辑处理，事务管理
- **文件数**: 4个接口 + 4个实现类
- **特点**:
  - 接口定义规范
  - 实现类使用 `@Service` 注解
  - 包含完整的CRUD操作

#### 3. **DAO层（数据访问层）**
- **职责**: 直接操作数据库，执行SQL
- **文件数**: 4个接口 + 4个实现类
- **特点**:
  - 使用Hibernate SessionFactory
  - 手动管理事务
  - HQL查询语言

#### 4. **Model层（实体模型层）**
- **职责**: 数据库表映射，实体关系定义
- **文件数**: 5个实体类
- **特点**:
  - JPA注解（`@Entity`, `@Table`, `@Column`）
  - Hibernate注解（`@ManyToOne`, `@OneToMany`）
  - 完整的getter/setter

#### 5. **View层（视图层）**
- **职责**: 页面渲染，用户交互
- **文件数**: 15个JSP页面
- **特点**:
  - JSP + JSTL标签
  - 服务端渲染
  - 表单提交

### 数据表详细设计

#### 1. User（用户表）
```sql
CREATE TABLE user (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(100) NOT NULL UNIQUE,
    email VARCHAR(100) NOT NULL UNIQUE,
    password VARCHAR(100) NOT NULL,
    role VARCHAR(20) DEFAULT 'ROLE_NORMAL',
    address TEXT,
    mobile VARCHAR(20)
);
```

**字段说明**:
- `id`: 主键，自增
- `username`: 用户名，唯一
- `email`: 邮箱，唯一
- `password`: 密码（建议加密存储）
- `role`: 角色（ROLE_NORMAL / ROLE_ADMIN）
- `address`: 收货地址
- `mobile`: 手机号

#### 2. Product（商品表）
```sql
CREATE TABLE product (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(200) NOT NULL,
    description TEXT,
    price DECIMAL(10, 2) NOT NULL,
    quantity INT NOT NULL DEFAULT 0,
    image VARCHAR(255),
    category_id INT,
    FOREIGN KEY (category_id) REFERENCES category(id)
);
```

**字段说明**:
- `id`: 主键，自增
- `name`: 商品名称
- `description`: 商品描述
- `price`: 商品价格
- `quantity`: 库存数量
- `image`: 商品图片路径
- `category_id`: 分类外键

#### 3. Category（分类表）
```sql
CREATE TABLE category (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL UNIQUE
);
```

**字段说明**:
- `id`: 主键，自增
- `name`: 分类名称，唯一

#### 4. Cart（购物车表）
```sql
CREATE TABLE cart (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_id INT NOT NULL,
    FOREIGN KEY (user_id) REFERENCES user(id)
);
```

**字段说明**:
- `id`: 主键，自增
- `user_id`: 用户外键（一对一关系）

#### 5. CartProduct（购物车商品关联表）
```sql
CREATE TABLE cart_product (
    id INT PRIMARY KEY AUTO_INCREMENT,
    cart_id INT NOT NULL,
    product_id INT NOT NULL,
    quantity INT NOT NULL DEFAULT 1,
    FOREIGN KEY (cart_id) REFERENCES cart(id) ON DELETE CASCADE,
    FOREIGN KEY (product_id) REFERENCES product(id)
);
```

**字段说明**:
- `id`: 主键，自增
- `cart_id`: 购物车外键
- `product_id`: 商品外键
- `quantity`: 商品数量

### 实体关系说明

1. **User ↔ Cart**: 一对一关系
   - 一个用户拥有一个购物车

2. **Cart ↔ CartProduct**: 一对多关系
   - 一个购物车包含多个购物车商品项

3. **Product ↔ CartProduct**: 一对多关系
   - 一个商品可以被添加到多个购物车

4. **Category ↔ Product**: 一对多关系
   - 一个分类包含多个商品

## 🚀 技术特点

### 1. 手动配置Hibernate（非Spring Data JPA）

**为什么不用Spring Data JPA？**
- ✅ **更灵活的控制**: 完全控制SessionFactory和事务
- ✅ **学习价值**: 理解Hibernate底层机制
- ✅ **性能优化**: 精确控制查询和缓存

**配置示例**:
```java
@Configuration
public class HibernateConfiguration {
    
    @Bean
    public LocalSessionFactoryBean sessionFactory() {
        LocalSessionFactoryBean sessionFactory = new LocalSessionFactoryBean();
        sessionFactory.setDataSource(dataSource());
        sessionFactory.setPackagesToScan("com.jtspringproject.JtSpringProject.models");
        sessionFactory.setHibernateProperties(hibernateProperties());
        return sessionFactory;
    }
    
    @Bean
    public HibernateTransactionManager transactionManager() {
        HibernateTransactionManager txManager = new HibernateTransactionManager();
        txManager.setSessionFactory(sessionFactory().getObject());
        return txManager;
    }
}
```

### 2. JSP视图技术

**为什么用JSP？**
- ✅ **服务端渲染**: SEO友好，首屏加载快
- ✅ **简单直接**: 无需前后端分离的复杂性
- ✅ **成熟稳定**: Java Web经典技术栈

**配置示例**:
```properties
spring.mvc.view.prefix=/views/
spring.mvc.view.suffix=.jsp
```

### 3. 多环境配置支持

**支持的环境**:
1. **默认环境（H2文件数据库）**
   - 开发便捷，数据持久化
   - 无需额外安装数据库

2. **本地MySQL环境**
   - 生产级别数据库
   - 本地开发测试

3. **远程MySQL环境**
   - 共享开发数据库
   - 团队协作

**切换方式**:
```bash
# H2数据库（默认）
mvn spring-boot:run

# 本地MySQL
mvn spring-boot:run -Dspring-boot.run.profiles=mysql

# 远程MySQL
mvn spring-boot:run -Dspring-boot.run.profiles=remote
```

### 4. 传统MVC三层架构

```
请求流程:
Browser → Controller → Service → DAO → Database
         ↓          ↓         ↓
         View ←── Model ←── Entity
```

**优点**:
- ✅ **职责清晰**: 每层职责明确
- ✅ **易于维护**: 代码结构清晰
- ✅ **便于测试**: 可单独测试每一层

## 🔧 常见问题

### Q1: 为什么不用Spring Data JPA Repository？
**A**: 本项目采用传统Hibernate配置，目的是：
- 更好地理解ORM底层原理
- 完全控制SessionFactory和事务
- 适合学习和教学场景

### Q2: 为什么用JSP而不是前后端分离？
**A**: JSP是传统Java Web技术，适合：
- 简单快速开发
- 服务端渲染，SEO友好
- 学习经典技术栈
- 如需前后端分离，可改造为REST API + Vue/React

### Q3: 数据库如何切换？
**A**: 修改 `application.properties`：
```properties
# H2数据库
hibernate.dialect=org.hibernate.dialect.H2Dialect
db.url=jdbc:h2:file:./data/ecommjava

# MySQL数据库
hibernate.dialect=org.hibernate.dialect.MySQL5Dialect
db.url=jdbc:mysql://localhost:3306/ecommjava
```

### Q4: 如何添加新功能？
**A**: 按MVC分层添加：
1. **Model**: 创建实体类
2. **DAO**: 创建DAO接口和实现
3. **Service**: 创建Service接口和实现
4. **Controller**: 创建Controller方法
5. **View**: 创建JSP页面

```java
// Controller层
@PostMapping("/register")
public String registerUser(@RequestParam String username, ...) {
    User user = new User();
    user.setUsername(username);
    // ...设置其他属性
    
    userService.register(user);
    return "redirect:/login";
}

// Service层
@Transactional
public void register(User user) {
    // 验证用户名是否存在
    if (userDao.findByUsername(user.getUsername()) != null) {
        throw new RuntimeException("用户名已存在");
    }
    
    user.setRole("ROLE_NORMAL");
    userDao.save(user);
    
    // 创建购物车
    Cart cart = new Cart();
    cart.setUser(user);
    cartDao.save(cart);
}

// DAO层
public void save(User user) {
    Session session = sessionFactory.getCurrentSession();
    session.save(user);
}
```

## 🔄 关键技术点说明

### 1. 事务管理

```java
@Service
@Transactional  // 类级别事务
public class UserServiceImpl implements UserService {
    
    @Override
    @Transactional(readOnly = true)  // 只读事务
    public User findById(int id) {
        return userDao.findById(id);
    }
    
    @Override
    @Transactional(propagation = Propagation.REQUIRED)
    public void register(User user) {
        userDao.save(user);
        cartDao.createForUser(user);
    }
}
```

### 2. Session管理

```java
@Repository
public class ProductDaoImpl implements ProductDao {
    
    @Autowired
    private SessionFactory sessionFactory;
    
    private Session getCurrentSession() {
        // Spring管理的Session，自动处理开启和关闭
        return sessionFactory.getCurrentSession();
    }
    
    @Override
    public Product findById(int id) {
        return getCurrentSession().get(Product.class, id);
    }
}
```

### 3. HTTP会话管理

```java
@PostMapping("/login")
public String login(String username, String password, HttpSession session) {
    User user = userService.authenticate(username, password);
    
    if (user != null) {
        // 存入Session
        session.setAttribute("username", username);
        session.setAttribute("userId", user.getId());
        session.setMaxInactiveInterval(1800); // 30分钟
        
        return "redirect:/user/home";
    }
    return "redirect:/login?error=true";
}
```
