# JtProject 数据访问层与调用链学习文档

这份文档用于回答两个核心问题：

1. `JtProject` 的数据访问层到底是 MyBatis、JPA、Hibernate，还是混合写法
2. 这个项目从 `Controller -> Service -> DAO -> 数据库 -> JSP` 的调用链到底是怎么串起来的

---

## 1. 先说结论

`JtProject` 不是 MyBatis 项目。

它更准确的定位是：

- `Spring Boot + JSP` 的 MVC Web 项目
- 实体映射使用 `JPA 注解`
- ORM 实现主要依赖 `Hibernate`
- DAO 层采用 `手写 DAO`
- 代码里混合使用了 `Hibernate SessionFactory/Session` 和 `JPA EntityManager`

所以，它不是：

- 纯 MyBatis
- 纯 Spring Data JPA Repository 风格
- 纯 Hibernate 原生单一写法

而是：

`JPA 实体 + Hibernate 实现 + 手写 DAO + SessionFactory / EntityManager 混合写法`

---

## 2. 对比表：MyBatis vs JPA vs Hibernate vs 本项目

| 项目/技术 | 它是什么 | 你主要写什么 | SQL 谁来负责 | 典型特征 | 适合什么场景 |
|---|---|---|---|---|---|
| MyBatis | 持久层框架 | `Mapper` 接口、XML 或注解 SQL | 开发者自己写 SQL | SQL 控制力强，语句很直观 | 复杂查询多、需要精细控制 SQL |
| JPA | Java 持久层规范 | 实体类注解、`EntityManager`、Repository 接口 | 规范本身不实现，通常交给底层框架 | 它是标准，不是具体 ORM 产品 | 想面向标准开发、减少厂商绑定 |
| Hibernate | ORM 框架，也是 JPA 的常见实现 | 实体类、HQL、`SessionFactory`、`Session` | Hibernate 负责大部分 ORM SQL 生成 | 功能完整、历史久、既支持 JPA 也支持原生 API | 典型 Java ORM 项目 |
| `JtProject` | 一个具体的电商项目 | 实体类、Service、手写 DAO、JSP 页面 | 大部分由 Hibernate 根据实体和 HQL 生成 | 不是 MyBatis，不是纯 `JpaRepository`，而是混合 DAO 风格 | 适合学习传统 Java Web 分层和 ORM 基础 |

再换一个更容易记忆的角度：

| 维度 | MyBatis | JPA | Hibernate | JtProject |
|---|---|---|---|---|
| 性质 | 框架 | 规范 | 框架/实现 | 业务项目 |
| 是否自己写 SQL | 通常要 | 不一定 | 不一定 | 大多不直接写 SQL |
| 是否有 `Mapper` | 有 | 没这个概念 | 没这个概念 | 没有 |
| 是否有 `EntityManager` | 没有 | 有 | 可支持 | 有一部分 |
| 是否有 `SessionFactory` | 没有 | 没有 | 有 | 有一部分 |
| 项目是否属于它 | 否 | 部分符合 | 很大程度符合 | 是 |

## 3. 本项目的数据访问层到底是什么风格

### 3.1 实体层是 JPA 注解风格

这些模型类使用了 `@Entity` 等注解：

- [User.java](../../src/main/java/com/jtspringproject/JtSpringProject/models/User.java)
- [Product.java](../../src/main/java/com/jtspringproject/JtSpringProject/models/Product.java)
- [Category.java](../../src/main/java/com/jtspringproject/JtSpringProject/models/Category.java)
- [Cart.java](../../src/main/java/com/jtspringproject/JtSpringProject/models/Cart.java)
- [CartProduct.java](../../src/main/java/com/jtspringproject/JtSpringProject/models/CartProduct.java)

这说明“对象如何映射表”这件事，走的是 JPA 注解思路。

### 3.2 DAO 主体是 Hibernate 风格

大部分 DAO 直接注入 `SessionFactory`，再拿 `Session` 去执行 HQL 或 CRUD：

- [UserDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/UserDaoImpl.java)
- [ProductDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/ProductDaoImpl.java)
- [CartDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CartDaoImpl.java)
- [CartProductDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CartProductDaoImpl.java)

这部分是明显的 Hibernate 原生 API 风格。

### 3.3 局部 DAO 又用了 JPA `EntityManager`

分类 DAO 使用的是 `EntityManager`：

- [CategoryDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CategoryDaoImpl.java)

这说明项目并不是“统一只用 SessionFactory”的纯 Hibernate 写法，而是混用了 JPA 标准接口。

### 3.4 没有走典型 Spring Data JPA Repository

虽然依赖里有：

- [pom.xml](../../pom.xml)

其中包含 `spring-boot-starter-data-jpa`，但代码里没有常见的：

- `interface XxxRepository extends JpaRepository<...>`
- `CrudRepository`

所以它不是现代常见的 `Spring Data JPA Repository` 项目，而是“手写 DAO”项目。

### 3.5 事务管理也偏 Hibernate 风格

项目自己配置了：

- [TransactionConfig.java](../../src/main/java/com/jtspringproject/JtSpringProject/TransactionConfig.java)
- [HibernateSessionFactoryConfig.java](../../src/main/java/com/jtspringproject/JtSpringProject/HibernateSessionFactoryConfig.java)

也就是说，这个项目虽然用了 Spring Boot，但持久层没有完全交给 Spring Boot 默认的 JPA 自动配置，而是保留了比较传统的 Hibernate 手动配置痕迹。

## 4. 分层结构总图

这是整个项目最适合记忆的结构：

```text
浏览器
  -> Controller
  -> Service
  -> DAO
  -> Hibernate / EntityManager
  -> 数据库
  -> 返回实体数据
  -> Service 继续处理
  -> Controller 放入 Model / ModelAndView
  -> JSP 页面渲染
  -> 返回 HTML 给浏览器
```

对应到本项目：

```text
Browser
  -> UserController / AdminController
  -> UserServiceImpl / ProductServiceImpl / CartServiceImpl / CategoryServiceImpl
  -> UserDaoImpl / ProductDaoImpl / CartDaoImpl / CartProductDaoImpl / CategoryDaoImpl
  -> SessionFactory / Session / EntityManager
  -> MySQL 或 H2
  -> JSP 视图
```

## 5. 数据访问流程图：从 Controller 到数据库

### 5.1 通用调用链

```text
HTTP Request
  ->
Controller
  负责接收请求参数、做页面跳转、组装 Model
  ->
Service
  负责业务逻辑、规则判断、组合多个 DAO
  ->
DAO
  负责执行 HQL / Hibernate CRUD / EntityManager 操作
  ->
Hibernate / JPA API
  负责对象和表的映射、生成 SQL
  ->
Database
  执行 SQL 并返回结果
  ->
DAO
  把结果封装成 Java 对象
  ->
Service
  继续做业务判断
  ->
Controller
  放入 ModelAndView
  ->
JSP
  渲染 HTML 页面
  ->
HTTP Response
```

## 6. 结合真实代码看 4 条典型链路

## 6.1 用户登录流程

核心代码位置：

- [UserController.java](../../src/main/java/com/jtspringproject/JtSpringProject/controller/UserController.java)
- [UserServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/UserServiceImpl.java)
- [UserDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/UserDaoImpl.java)

流程图：

```text
用户在 userLogin.jsp 提交用户名和密码
  ->
POST /userloginvalidate
  ->
UserController.userloginAlias(...)
  ->
UserController.userlogin(...)
  ->
userService.checkLogin(username, password)
  ->
UserServiceImpl.checkLogin(...)
  ->
userDao.getUser(username, password)
  ->
UserDaoImpl.getUser(...)
  ->
SessionFactory.getCurrentSession()
  ->
HQL: from User where username = :username
  ->
数据库查询 users 表
  ->
返回 User 对象
  ->
Service 判断 user.getId() > 0 是否登录成功
  ->
Controller 写入 Session / Cookie
  ->
Controller 再调用 productService.getProducts()
  ->
返回 index.jsp
```

你可以注意到一个细节：

- DAO 层先按用户名查用户
- 密码比较主要是在 Java 代码里完成
- 这说明它不是把用户名和密码完全都写进 SQL 条件里一次性校验

## 6.3 购物车流程

购物车流程更能体现“Service 负责业务编排”这个概念。

核心代码位置：

- [UserController.java](../../src/main/java/com/jtspringproject/JtSpringProject/controller/UserController.java)
- [CartServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/CartServiceImpl.java)
- [CartDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CartDaoImpl.java)
- [CartProductDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CartProductDaoImpl.java)

“加入购物车”的流程：

```text
用户点击 添加到购物车
  ->
GET/POST /products/addtocart?id=商品ID
  ->
UserController.addToCart(id, request)
  ->
userService.getUserByUsername(username)
  ->
cartService.getCarts()
  ->
如果该用户没有 Cart，则 cartService.addCart(cart)
  ->
productService.getProduct(id)
  ->
cartProductDao.addCartProduct(new CartProduct(cart, product))
  ->
Hibernate 保存 cart_product 关联数据
  ->
redirect:/user/cart
```

“查看购物车”的流程：

```text
浏览器访问 /user/cart
  ->
UserController.showCart(request)
  ->
userService.getUserByUsername(username)
  ->
cartService.getCarts()
  ->
找到当前用户对应的 Cart
  ->
cartProductDao.getProductByCartID(cart.getId())
  ->
DAO 查询 CartProduct，再提取 Product 列表
  ->
Controller 放入 ModelAndView
  ->
返回 cart.jsp
```

这条链说明：

- Controller 里有一部分流程编排
- Service 层不是每个功能都封装得非常完整
- 这个项目是“传统项目风格”，不是特别严格的现代分层

## 6.4 分类管理流程

分类模块更能看出 JPA `EntityManager` 的使用。

核心代码位置：

- [AdminController.java](../../src/main/java/com/jtspringproject/JtSpringProject/controller/AdminController.java)
- [CategoryServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/CategoryServiceImpl.java)
- [CategoryDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CategoryDaoImpl.java)

流程图：

```text
管理员访问 /admin/categories
  ->
AdminController
  ->
categoryService.getCategories()
  ->
CategoryServiceImpl.getCategories()
  ->
categoryDao.getCategories()
  ->
CategoryDaoImpl.getCategories()
  ->
EntityManager.createQuery("from Category", Category.class)
  ->
数据库 categories 表
  ->
返回 List<Category>
  ->
Controller 放入 ModelAndView
  ->
返回 categories.jsp
```

这条链和前面的 `SessionFactory` 风格不同，说明项目在 DAO 层确实是混合写法。

## 7. 项目里每一层各自负责什么

### Controller 层

代表类：

- [UserController.java](../../src/main/java/com/jtspringproject/JtSpringProject/controller/UserController.java)
- [AdminController.java](../../src/main/java/com/jtspringproject/JtSpringProject/controller/AdminController.java)

职责：

- 接收请求参数
- 处理 Session、Cookie、页面跳转
- 调用 Service
- 把结果放进 `Model` 或 `ModelAndView`

### Service 层

代表类：

- [UserServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/UserServiceImpl.java)
- [ProductServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/ProductServiceImpl.java)
- [CartServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/CartServiceImpl.java)
- [CategoryServiceImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/services/impl/CategoryServiceImpl.java)

职责：

- 处理业务规则
- 组合多个 DAO
- 在业务层做判断和数据整理

### DAO 层

代表类：

- [UserDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/UserDaoImpl.java)
- [ProductDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/ProductDaoImpl.java)

- [CartProductDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CartProductDaoImpl.java)
- [CategoryDaoImpl.java](../../src/main/java/com/jtspringproject/JtSpringProject/dao/impl/CategoryDaoImpl.java)

职责：

- 最靠近数据库
- 执行 `save`、`update`、`delete`
- 执行 HQL 查询
- 使用 `EntityManager` 或 `SessionFactory`

### 数据库访问实现层

关键配置：

- [HibernateSessionFactoryConfig.java](../../src/main/java/com/jtspringproject/JtSpringProject/HibernateSessionFactoryConfig.java)
- [TransactionConfig.java](../../src/main/java/com/jtspringproject/JtSpringProject/TransactionConfig.java)

职责：

- 创建 `SessionFactory`
- 管理事务

## 8. 你学习这个项目时最值得抓住的三个点

1. 这是一个典型“JSP + Spring MVC + 手写 DAO”的老派 Java Web 项目  
   学这个项目很适合建立传统分层理解。

2. 它的数据访问层不是单一风格，而是混合风格  
   你会同时看到 `@Entity`、`SessionFactory`、`EntityManager`。

3. 它的 Controller 有一部分业务编排逻辑  
   所以它不是教科书式的“绝对纯净分层”，更像真实维护中的项目。

---

## 9. 一句话总结

如果你后面要向别人解释这个项目，可以直接用这句话：

`JtProject 是一个 Spring Boot + JSP 的 MVC 电商项目，持久层不是 MyBatis，而是基于 JPA 实体和 Hibernate 的手写 DAO 混合实现，调用链总体是 Controller -> Service -> DAO -> Hibernate/JPA -> 数据库 -> JSP。`
