# JtProject-React 目录结构总览

这个项目是 **Spring Boot + React + JSP 混合结构**，不是纯 React 项目。

简单说：

- **React** 放在 `frontend/`
- **JSP 视图** 还保留在 `src/main/webapp/views/`
- **后端接口** 和传统 MVC 控制器都还在 `src/main/java/`

## 各部分分别负责什么

### `frontend/`

这里是新 React 前端。

主要内容：

- `src/main.tsx`：React 入口
- `src/App.tsx`：页面壳和组合逻辑
- `src/views/`：页面级组件
- `src/components/`：可复用组件
- `src/hooks/`：状态逻辑
- `src/services/`：请求后端 API

### `src/main/webapp/views/`

这里是老 JSP 页面。

这部分不是 React，而是传统服务器端渲染页面，而且**当前仍被后端控制器实际使用**，不能直接删除。

典型页面：

- `index.jsp`
- `userLogin.jsp`
- `adminlogin.jsp`
- `cart.jsp`
- `adminHome.jsp`
- `products.jsp`
- `categories.jsp`

### `src/main/java/`

这里是后端 Spring Boot 代码。

要特别注意两类控制器：

- `controller/AdminController.java`
- `controller/UserController.java`

它们保留了很多传统 MVC / JSP 路由。

另外还有：

- `controller/ApiController.java`

这个是给 React 前端调用的 REST API。

## JSP 和 React 的关系

这个项目不是“JSP 全部被 React 替换”。

更准确地说：

- **JSP 还保留着**，而且仍然参与运行
- **React 是新增的现代前端层**
- 两者都依赖同一个 Spring Boot 后端

所以你会看到：

- 老页面从 `src/main/webapp/views/` 出来
- 新页面从 `frontend/src/` 出来

## 能不能删除 JSP

现在不能直接删。

原因是后端控制器里仍然有大量返回视图名的逻辑，例如：

- `return "index"`
- `return "adminHome"`
- `return "userLogin"`
- `return "products"`
- `return "cart"`

这些返回值会通过 `spring.mvc.view.prefix=/views/` 和 `spring.mvc.view.suffix=.jsp` 解析成 JSP 页面。

如果要彻底删除 JSP，必须先完成下面这些迁移：

1. 把相关控制器改成返回 JSON 或重定向到 React 路由
2. 确认 React 已覆盖所有原 JSP 页面
3. 移除 JSP 视图解析配置
4. 再删除 `src/main/webapp/views/`

也就是说，**删 JSP 是一个迁移项目，不是一个文件清理动作**。
