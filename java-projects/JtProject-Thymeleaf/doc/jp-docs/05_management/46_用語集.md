## 2. 目的

本書は、`JtProject` および日本案件風資料で使用している用語を整理し、表記揺れや解釈差を防止するための資料である。

## 4. 表記ルール

- 文書では「一般ユーザー」と「管理者」を明確に区別する。
- DB テーブル名は大文字、Java クラス名は先頭大文字で表記する。
- `ROLE_NORMAL`、`ROLE_ADMIN` はコード表記を維持する。
- 票類と一覧類は区別して記載する。

## 5. 備考

- 実案件では業務用語、略語、禁止用語、顧客独自用語も追加する。
- 文書作成時に新しい重要用語が出た場合は本書へ追記する。
