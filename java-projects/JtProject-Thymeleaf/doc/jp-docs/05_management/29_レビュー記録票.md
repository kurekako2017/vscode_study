## 2. 目的

本書は、設計書・試験仕様書・DB 資料に対するレビュー実施状況、指摘事項、対応状況を記録するための票である。  
日本案件で一般的な「作成者レビュー」「相互レビュー」「承認前レビュー」を想定した形式で整理する。

## 3. レビュー運用方針

- 重要文書は作成後にレビューを実施する。
- 指摘は文書単位ではなく、章・表・項目単位で記録する。
- 指摘が改修に発展した場合は `09_改修票.md`、不具合であれば `08_障害票.md` と連携する。
- 対応完了後は再レビュー結果を明記する。

## 7. レビュー観点テンプレート

- 記載漏れがないか
- 設計書間で表記揺れや矛盾がないか
- 業務ロジック、異常系、権限制御が記載されているか
- 試験仕様として第三者が実施可能な粒度になっているか
- 実装との乖離がないか

## 8. 備考

- 実案件ではレビュアー、承認者、レビュー種別、指摘票番号をさらに細かく管理する。
- Excel 管理の場合は「指摘 No」「重要度」「期限」「是正完了日」を追加する運用が一般的である。
