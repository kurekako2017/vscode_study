## 2. 納品単位一覧

| 納品単位 ID | 納品単位名 | 内容 | 想定形式 | 納品可否 | 備考 |
|---|---|---|---|---|---|
| DELIV-001 | 設計書一式 | 概要、基本、詳細、機能、画面、クラス、DB 設計一式 | Markdown / PDF 化想定 | 可 | 顧客提示用に体裁調整余地あり |
| DELIV-002 | 試験資料一式 | 試験仕様、試験結果、実施記録、試験データ | Markdown / Excel 化想定 | 可 | 実施証跡は別途添付想定 |
| DELIV-003 | 管理台帳一式 | 障害、改修、調査、課題、レビュー、承認、リスク | Markdown / Excel 化想定 | 可 | 継続更新前提 |
| DELIV-004 | 運用保守資料一式 | 運用手順、FAQ、障害対応、引継ぎ資料 | Markdown / PDF 化想定 | 可 | 保守引継ぎ時に利用 |
| DELIV-005 | ソースコード一式 | `JtProject` 本体ソース、設定、起動関連 | Git / ZIP 想定 | 条件付き可 | 実行前に機密確認が必要 |
| DELIV-006 | DB 反映資料一式 | DDL、テーブル定義、DB 一覧、ER 図 | SQL / Markdown / PDF 化想定 | 可 | 実 DB 差分確認が必要 |

## 4. 納品時チェック観点

| No | チェック観点 | 内容 | 関連資料 |
|---|---|---|---|
| 1 | 版数整合 | 納品対象の版数が凍結されているか | `50_版数管理表` |
| 2 | 承認整合 | 承認状態が記録されているか | `30_承認履歴一覧` |
| 3 | 構成整合 | 納品対象に漏れや重複がないか | 本文書 |
| 4 | 変更整合 | 変更履歴が追跡可能か | `28_変更履歴一覧` |
| 5 | 成果物整合 | 成果物一覧と納品物構成が一致するか | `48_成果物一覧` |
| 6 | チェック実施 | 納品前チェックリストを実施済みか | `49_チェックリスト一覧` |

## 5. 納品対象外メモ

| 区分 | 対象 | 理由 |
|---|---|---|
| 作業途中資料 | 一時メモ、個人作業メモ | 納品対象外 |
| 実行ログ | ローカル試験ログ、IDE 出力 | 納品対象外 |
| 環境依存設定 | 個人環境向け設定ファイル | 顧客環境へそのまま適用不可 |
| 履歴作業補助 | 一時的な検証用ファイル | 納品対象外 |
