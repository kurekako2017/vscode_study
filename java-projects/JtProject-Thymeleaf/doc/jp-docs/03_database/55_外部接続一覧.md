## 2. 目的

本書は、`JtProject` が利用する外部接続先、ミドルウェア接続、開発時依存先を一覧化し、接続方式、用途、設定位置、運用上の注意点を整理することを目的とする。

## 3. 外部接続方針

- 本プロジェクトは外部 SaaS 連携を主目的とするシステムではなく、主な外部接続は `DB`、`Web サーバ`、`開発端末`、`ブラウザ` である。
- 本番想定では、アプリケーションは `Spring Boot` もしくは Servlet コンテナ上で動作し、`MySQL` 系 DB と接続する前提で整理する。
- 現状ソース上で明示的な外部 API 呼出は確認されていないため、将来拡張候補もあわせて記載する。

## 4. 外部接続一覧

| 接続ID | 接続先 | 区分 | 接続元 | 用途 | 接続方式 | 主な設定位置 | 備考 |

## 5. DB 接続詳細

### 5.1 接続用途

- `CUSTOMER`: 顧客情報、ログイン認証、権限判定
- `CATEGORY`: 商品分類管理
- `PRODUCT`: 商品一覧表示、商品管理
- `CART`: 顧客別カートヘッダ管理
- `CART_PRODUCT`: カート商品明細管理

### 5.2 接続時の注意事項

- 接続先 URL、ユーザ、パスワードは環境ごとに外出し管理すること。
- 開発環境と試験環境で DB 名が異なる場合、DDL とテストデータ整合性を維持すること。
- Hibernate 自動生成設定を利用する場合、既存テーブル破壊リスクに注意すること。

## 6. Web 接続詳細

| 接続先 | 利用者 | 主な画面 | 用途 |
|---|---|---|---|
| `/` | 一般利用者 | ログイン画面 | ログイン開始 |
| `/index` | 一般利用者 | 商品一覧画面 | 商品閲覧 |
| `/user/cart` | 一般利用者 | カート画面 | 購入候補確認 |
| `/admin/login` | 管理者 | 管理者ログイン画面 | 管理者認証 |
| `/admin/products` | 管理者 | 商品管理画面 | 商品追加・更新・削除 |
| `/admin/categories` | 管理者 | 分類管理画面 | 分類追加・更新・削除 |

## 7. 将来想定される外部接続

| 想定接続先 | 想定用途 | 必要となる追加設計 |
|---|---|---|
| メール送信サービス | 会員登録通知、障害通知 | メッセージ設計、再送制御、例外設計 |
| 決済 API | 注文・決済処理 | API 仕様書、外部 IF 試験、認証設計 |
| CDN / 静的配信基盤 | 商品画像最適配信 | ファイル管理、URL 設計、権限設計 |

## 8. 設計上の留意点

- 外部接続設定値はソース直書きを避け、環境設定ファイルで管理すること。
- 接続障害時は `例外設計一覧` と `監視・障害対応手順書` に従って一次切り分けを行うこと。
- 外部接続が増えた場合は、接続先ごとに認証方式、タイムアウト、リトライ方針を明記すること。
