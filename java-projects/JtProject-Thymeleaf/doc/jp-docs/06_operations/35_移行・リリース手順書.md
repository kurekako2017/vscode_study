## 2. 目的

本書は、開発・検証環境から対象環境へアプリケーションを反映する際の基本的な移行・リリース手順を整理するものである。

## 3. 前提

- 本資料は学習用プロジェクトを前提とした簡易版である。
- 本番運用を想定した厳密な承認フロー、停止調整、監視切替は含まない。
- リリース前に試験結果報告書で主要試験が完了していることを前提とする。

## 4. リリース対象

| 区分 | 対象 |
|---|---|
| アプリケーション | Spring Boot アプリケーション本体 |
| 設定 | `application.properties` 等の環境依存設定 |
| DB | 必要に応じた DDL / 初期データ / マスタデータ |
| 文書 | 設計書、試験結果、運用手順書 |

## 5. 事前確認

| No | 確認項目 | 確認内容 |
|---|---|---|
| 1 | ソース確定 | 対象版が確定していること |
| 2 | 試験完了 | 主要試験が完了していること |
| 3 | DB 差分 | DDL / データ差分が整理されていること |
| 4 | 設定差分 | 環境依存設定が明確であること |
| 5 | ロールバック | 障害時の戻し方が確認済みであること |

## 6. 移行手順

### 6.1 バックアップ取得

1. 現行アプリケーション資材を退避する。
2. DB を使用している場合は事前バックアップを取得する。
3. 設定ファイルを退避する。

### 6.2 アプリケーション反映

1. 対象ビルド成果物またはソースを配置する。
2. 必要に応じて Maven ビルドを実行する。
3. 環境依存設定を反映する。

### 6.3 DB 反映

1. `27_DDL一覧.md` を参照し、必要な DDL を適用する。
2. 初期データまたはマスタデータを反映する。
3. テーブル構造と起動整合性を確認する。

### 6.4 起動確認

1. アプリケーションを起動する。
2. ログイン画面表示を確認する。
3. 一般ユーザー導線、管理者導線の最小確認を行う。

## 7. リリース後確認

| 観点 | 確認内容 |
|---|---|
| 起動確認 | アプリケーションが正常起動すること |
| 画面確認 | ログイン画面、商品一覧、管理者画面が表示できること |
| DB 確認 | 商品、カテゴリ、ユーザーが参照できること |
| 権限制御 | 管理者画面へ一般ユーザーが入れないこと |
| ログ確認 | 異常ログが大量出力されていないこと |

## 8. ロールバック手順

1. 新規反映資材を停止する。
2. 退避しておいた旧資材へ切り戻す。
3. 必要に応じて DB バックアップを復元する。
4. 再起動して旧版動作を確認する。
5. 障害内容を `08_障害票.md`、`10_調査票.md` に記録する。

## 9. 注意事項

- 設定ファイルの環境差異を誤ると起動不可となる。
- DDL 適用有無は事前に確認し、重複実行を避ける。
- リリース後は最低限の疎通確認を即時に行う。
- 学習用プロジェクトのため、本番相当の承認・監視・ジョブ切替は別途補う必要がある。

## 10. 備考

- 実案件では移行判定会、リリース承認、作業担当、立会者、時刻管理を追加する。
- 本書は開発・検証環境向けの簡易移行・リリース手順である。
