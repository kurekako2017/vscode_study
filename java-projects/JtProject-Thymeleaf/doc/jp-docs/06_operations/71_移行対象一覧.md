# 71. 移行対象一覧

## 1. 文書情報

| 項目 | 内容 |
|---|---|
| 文書名 | 移行対象一覧 |
| 対象システム | JtProject |
| 文書区分 | 移行・リリース資料 |
| 版数 | v1.0 |

## 2. 目的

本書は、`JtProject` の環境移行、リリース、再構築時に対象となる資材、設定、データ、文書を一覧化し、移行漏れを防止することを目的とする。

## 3. 移行対象一覧

| 区分 | 移行対象 | 内容 | 移行要否 | 備考 |
|---|---|---|---|---|
| アプリ資材 | ソースコード/ビルド成果物 | Spring Boot 実行資材、JSP、設定ファイル | 要 | 主対象 |
| 設定値 | `application.properties` 等 | DB 接続、ポート、ログ等の設定 | 要 | 環境差異確認必要 |
| DB | テーブル定義 | `CUSTOMER`、`CATEGORY`、`PRODUCT`、`CART`、`CART_PRODUCT` | 要 | DDL 反映対象 |
| DB | 初期データ | 管理者、分類、商品等の初期データ | 要 | 環境ごとの差異管理 |
| 文書 | 設計・運用文書 | リリース、障害対応、設定、交接文書 | 要 | 運用引継ぎ用 |
| ログ | 運用ログ | 障害解析参考情報 | 任意 | 通常は移行対象外、必要時のみ |
| テストデータ | 試験用データ | 単体/結合/総合テスト用データ | 条件付 | 本番移行対象外が基本 |

## 4. 移行単位別観点

### 4.1 アプリ資材

- 実行対象 jar または war
- 静的資材、JSP、テンプレート
- 起動スクリプト、必要設定ファイル

### 4.2 DB 資材

- DDL
- 初期投入 SQL
- 必要に応じてマスタデータ

### 4.3 環境設定

- DB 接続先
- ポート番号
- ログ出力先
- セッション設定

## 5. 移行時の留意点

- 本番相当環境では試験用アカウント、試験データを混在させないこと。
- 環境差異がある設定値は `設定値一覧` を基準に確認すること。
- 移行実施前後で `移行・リリース手順書` の確認項目に従うこと。
- DDL とアプリ資材の版数整合性を `版数管理表` で確認すること。
