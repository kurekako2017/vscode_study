| 項目 | 内容 |
|---|---|
| 文書名 | Category 詳細設計書 |
| 対象クラス | `Category` |
| パッケージ | `models` |

## 2. クラス概要

| 項目 | 内容 |
|---|---|
| 役割 | 商品カテゴリ情報を表す Entity |
| 対応テーブル | `CATEGORY` |
| 主キー | `category_id` |
| 主な利用箇所 | 管理者カテゴリ管理、商品登録、商品更新 |

## 3. 属性一覧

| No | 属性名 | 型 | DB カラム | 内容 |
|---|---|---|---|---|
| 1 | `id` | `int` | `category_id` | カテゴリ ID、主キー、自動採番 |
| 2 | `name` | `String` | `name` | カテゴリ名称 |

## 4. 業務上の意味

- 商品を業務分類するためのマスタエンティティ
- 管理者がメンテナンスする分類情報
- 商品登録、商品更新画面での選択肢元データ

## 5. 業務ルール

- カテゴリ名称のみを保持するシンプルなマスタ構成である。
- 現行実装ではカテゴリコード、表示順、有効フラグは持たない。
- 削除時の参照整合性は上位設計で担保すべき項目である。

## 6. 関連

| 関連先 | 関係 | 内容 |
|---|---|---|
| `Product` | 1 : N 相当 | 商品所属カテゴリ |

## 7. 設計上の注意

- Entity 上は逆参照コレクションを持っていないため、カテゴリ配下商品一覧は `Product` 側から扱う。
- 実案件ではカテゴリコード、一意制約、論理削除、有効開始終了日を持たせることが多い。
