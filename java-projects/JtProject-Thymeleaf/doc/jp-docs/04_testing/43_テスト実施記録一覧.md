## 2. 目的

本書は、テスト実施の進捗と実施実績を一覧で管理し、どの試験がいつ、誰により、どの結果で実施されたかを把握するための台帳である。

## 3. 管理方針

- 試験仕様書はケース定義、本書は実施実績管理として使い分ける。
- 実施日、実施者、判定、障害有無を記録する。
- 障害発生時は `08_障害票.md` と紐づける。

## 5. 実施状況定義

| 状況 | 内容 |
|---|---|
| 未実施 | まだ試験していない |
| 実施中 | 一部実施中 |
| 実施済 | 実施完了 |
| 再試験待ち | 改修後の再確認待ち |

## 6. 判定定義

| 判定 | 内容 |
|---|---|
| OK | 期待結果どおり |
| NG | 不具合あり |
| 保留 | 判定不能、前提不足、再確認必要 |

## 7. 利用ルール

- 試験を実施したら本書へ実績を追記する。
- NG の場合は障害票番号を付与する。
- 試験結果サマリは `34_試験結果報告書.md` に反映する。
- 再試験時は同じ記録 No に追記するか、再試験用行を追加する。

## 8. 備考

- 実案件では実行環境、実行バージョン、証跡ファイル、承認欄を追加する。
- Excel 管理ではフィルタ、集計しやすい形式で管理する。
