## 2. 目的

利用者視点で業務シナリオが成立することを確認し、本システムが業務利用に耐えうる最低限の品質を満たすことを検証する。  
単体テスト、結合テストで確認済みの部品・連携を、利用者導線単位で通し確認する。

## 3. 対象範囲

- 一般ユーザーの利用開始からカート操作までの一連業務
- 管理者による商品・カテゴリ保守の一連業務
- 権限制御、エラーメッセージ、画面継続性の妥当性

## 4. 前提条件

| 項目 | 内容 |
|---|---|
| アプリ稼働状態 | Web アプリが起動済みであること |
| 初期データ | `25_テストデータ一覧.md` のデータが投入済みであること |
| ブラウザ | 一般ユーザー用と管理者用で別セッションを使用可能であること |
| 確認項目 | 画面表示、DB 反映、権限制御、メッセージ表示を確認すること |

## 5. 受入観点

- 一般ユーザー導線が途切れず成立していること
- 管理者導線が設計書どおり成立していること
- 商品、カテゴリ、カートデータの整合性が保たれること
- 想定異常時に画面が継続し、利用者向けメッセージが表示されること
- 権限外利用が抑止されること

## 6. 総合テストシナリオ

| No | 業務シナリオ | 前提条件 | 実施内容 | 期待結果 | 確認対象 |
|---|---|---|---|---|---|
| ST-01 | 新規ユーザー利用開始 | 新規登録可能、商品データあり | ユーザー登録 -> ログイン -> 商品一覧表示 | 正常に利用開始でき、商品一覧へ到達する | 登録、認証、初期導線 |
| ST-02 | 商品選択からカート確認まで | 一般会員ログイン済み | 商品一覧 -> 商品詳細または一覧で追加 -> カート確認 | 追加した商品がカートに表示され、数量・商品情報が一致する | カート業務成立 |
| ST-03 | カート操作中の異常系 | 一般会員ログイン済み | 不正商品 ID 指定、不正明細 ID 削除を実施 | 画面異常終了せず、一覧またはメッセージ表示で継続する | 例外制御 |
| ST-04 | 管理者による商品メンテ | 管理者ログイン可能、カテゴリあり | 管理者ログイン -> 商品追加 -> 商品更新 -> 商品削除 | 一連操作が正常終了し、一覧に整合反映される | 管理者商品保守 |
| ST-05 | 管理者によるカテゴリ管理 | 管理者ログイン済み | カテゴリ追加 -> 更新 -> 商品登録画面で参照確認 | カテゴリ変更内容が関連画面へ反映される | マスタ整合 |
| ST-06 | 管理者プロフィール更新 | 管理者ログイン済み | プロフィール更新実行 | DB と Session の管理者情報が一致する | セッション同期 |
| ST-07 | 権限制御確認 | 一般会員ログイン済みまたは未ログイン | 管理者 URL 直接アクセス、未ログインでカート追加 | 権限不足時は管理者画面を表示せず、ログイン画面またはエラーへ遷移する | 権限制御 |
| ST-08 | 障害時の画面継続性 | アプリ稼働中 | 誤ログイン、空入力、不正 ID 操作を複数実施 | システムが異常終了せず、業務継続可能な応答を返す | 業務継続性 |

## 7. 実施手順の基本方針

1. 初期データを確認する。
2. 一般ユーザー導線を実施し、DB 反映を確認する。
3. 管理者導線を実施し、マスタ更新結果を確認する。
4. 異常系・権限制御を実施し、画面継続性を確認する。
5. 実施結果を記録し、障害があれば `08_障害票.md` に起票する。
