## 2. 目的

画面、Controller、Service、DAO、DB の連携が詳細設計どおりに動作することを確認する。  
単体テストで確認済みの各部品を結合した際に、画面遷移、セッション維持、DB 更新、メッセージ表示が整合することを検証する。

## 3. テスト範囲

- 一般ユーザー機能
  - ログイン
  - 商品一覧参照
  - カート追加
  - カート参照・削除
- 管理者機能
  - 管理者ログイン
  - 商品登録・更新・削除
  - カテゴリ登録・更新・削除
  - プロフィール更新

## 4. テスト環境・前提条件

| 項目 | 内容 |
|---|---|
| アプリケーション | JtProject 開発環境ビルド |
| DB | 開発用 DB に初期データ投入済み |
| テストデータ | `25_テストデータ一覧.md` のユーザー、カテゴリ、商品を使用 |
| ブラウザ | Chrome または Edge |
| セッション | シナリオに応じて未ログイン / 一般会員 / 管理者で開始する |

## 5. 結合テスト観点

- 画面から Controller へのパラメータ受け渡しが正しいこと
- Controller から Service、DAO への呼出順序が想定どおりであること
- DB 更新後に再表示画面へ結果が反映されること
- エラー時に画面遷移が破綻せず、利用者向けメッセージが表示されること
- セッション情報により利用可能機能が制御されること

## 6. 結合テストケース

### 6.1 一般ユーザー機能

| No | シナリオ | 前提条件 | 入力値 | 操作手順 | 期待結果 | 確認対象 |
|---|---|---|---|---|---|---|
| IT-01 | ユーザーログインから商品トップ表示 | 一般会員 `user01` が登録済み | `user01/pass01` | ログイン画面で認証実行 | `index.jsp` に商品一覧が表示され、セッションに利用者情報が保持される | 画面遷移、セッション、商品一覧取得 |
| IT-02 | ログイン失敗時の画面復帰 | 一般会員あり | `user01/wrong` | ログイン実行 | `userLogin.jsp` に戻り、認証失敗メッセージを表示する | メッセージ、遷移先 |
| IT-03 | 商品トップからカート追加 | 一般会員ログイン済み、商品 ID=1001 存在 | `productId=1001` | 商品一覧画面で Add To Cart 実行 | カートに商品が追加され、カート画面で追加内容が確認できる | INSERT/更新、カート画面 |
| IT-04 | 未ログイン状態でのカート追加 | 未ログイン | `productId=1001` | Add To Cart 実行 | ログイン要求メッセージを表示し、ログイン画面へ誘導する | 権限制御、メッセージ |
| IT-05 | カート画面で削除 | ログイン済み、カート内に対象商品あり | `cartProductId=5001` | カート画面で削除実行 | 商品が一覧から消え、DB 明細も削除される | 表示結果、DELETE 結果 |
| IT-06 | 不正 ID によるカート削除 | ログイン済み | `cartProductId=9999` | カート削除実行 | 異常終了せず、一覧を再表示する | 例外制御、画面継続性 |

### 6.2 管理者機能

| No | シナリオ | 前提条件 | 入力値 | 操作手順 | 期待結果 | 確認対象 |
|---|---|---|---|---|---|---|
| IT-07 | 管理者ログインから商品一覧表示 | 管理者 `admin01` が登録済み | `admin01/admin01` | `/admin/login` でログイン | 管理画面トップまたは商品管理画面が表示される | 遷移、セッション |
| IT-08 | 一般会員による管理者ログイン失敗 | 一般会員 `user01` 存在 | `user01/pass01` | 管理者ログイン実行 | `adminlogin.jsp` に戻り、権限エラーを表示する | 権限制御、メッセージ |
| IT-09 | 商品追加から一覧反映 | 管理者ログイン済み、カテゴリあり | 商品名、価格、カテゴリ、画像 | 商品追加画面で登録 | 商品一覧に追加データが反映される | INSERT、一覧反映 |
| IT-10 | 商品更新から一覧反映 | 管理者ログイン済み、対象商品あり | 商品 ID、商品名、価格、画像未選択 | 商品更新実行 | 更新内容が一覧に反映され、既存画像は維持される | UPDATE、画像保持 |
| IT-11 | 商品削除 | 管理者ログイン済み、対象商品あり | `productId=1002` | 商品削除実行 | 商品一覧から対象商品が消える | DELETE、一覧反映 |
| IT-12 | カテゴリ追加から画面反映 | 管理者ログイン済み | 新カテゴリ名 | カテゴリ追加実行 | カテゴリ一覧に追加表示される | INSERT、一覧反映 |
| IT-13 | カテゴリ更新から商品画面反映 | 管理者ログイン済み、カテゴリあり | カテゴリ ID、名称変更 | 更新実行 | 管理画面および商品登録画面で新名称が表示される | UPDATE、参照整合 |
| IT-14 | プロフィール更新 | 管理者ログイン済み | 管理者名、メール等 | 更新送信 | DB 更新後、Session 上の管理者名も更新される | UPDATE、Session 同期 |

## 7. 実施時確認事項

- DB 更新前後の差分を確認すること
- エラー時にスタックトレースが画面へ露出しないこと
- 画面遷移図と実際の遷移に差異がないこと
- 入力チェック結果とメッセージ一覧の内容が整合していること
