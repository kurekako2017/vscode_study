# 機能設計書

## 1. 目的

本書は `JtProject` の機能単位の設計情報を整理し、入力、出力、業務ルール、異常系を明示することを目的とする。

## 2.1 処理フロー図の記載方針

本書では、機能単位で理解したい `処理フロー図` または処理手順を記載する。

対象:

- 1 機能の開始条件、入力、判定、出力
- 利用者視点で見た代表処理の流れ
- 正常系、異常系の主要分岐

向いている内容:

- `F-01 ユーザーログイン`
- `F-04 カート追加`
- `F-08 商品管理`

補足:

- システム全体寄りの処理フローは [03_詳細設計書.md](../../../../JtProject/doc/jp-docs/01_design/03_詳細設計書.md) に記載する。
- クラス、メソッド単位まで下げる場合は [15_クラス詳細設計書.md](../../../../JtProject/doc/jp-docs/02_class-design/15_クラス詳細設計書.md) 配下で管理する。
- 本書では機能観点の処理手順説明を主体とし、図が必要な場合も機能単位で簡潔に記載する。

## 3. 機能詳細

### 3.1 F-01 ユーザーログイン

| 項目 | 内容 |
|---|---|
| 機能概要 | 入力されたユーザー名・パスワードで認証し、成功時は商品トップへ遷移する |
| 入力 | `username`, `password` |
| 出力 | 成功時 `index.jsp`、失敗時 `userLogin.jsp` |
| 主処理 | `UserController.userlogin()` |
| 業務ルール | 認証成功時は Cookie と Session にユーザー名を保持する |
| 異常系 | 認証失敗、ユーザー未存在、例外発生 |

業務ロジック詳細：

1. 画面から `username` と `password` を受信する
2. `UserService.checkLogin(username, password)` を呼び出す
3. Service は `UserDao.getUser(username, password)` を呼び出す
4. DAO は `username` でユーザーを検索する
5. 検索結果が 0 件の場合は空 `User` オブジェクトを返す
6. 検索結果が存在する場合は DB 上のパスワードと入力値を比較する
7. 一致時は認証成功とし、`Cookie(username)`、`Session(username, userRole)` を設定する
8. `ProductService.getProducts()` により商品一覧を取得する
9. 商品一覧を `index.jsp` に引き渡して画面表示する
10. 認証失敗時は `userLogin.jsp` にエラーメッセージを設定して返却する

判定条件：

- `user.id > 0` の場合のみ認証成功とみなす
- `Objects.equals(username, user.getUsername())` によりユーザー名整合を確認する
- 商品一覧が取得できない場合でも、ログイン自体は成功として画面返却を継続する

### 3.2 F-04 カート追加

| 項目 | 内容 |
|---|---|
| 機能概要 | 指定商品をログインユーザーのカートへ追加する |
| 入力 | `id`（商品 ID） |
| 出力 | 成功時 `/user/cart` へリダイレクト |
| 主処理 | `UserController.addToCart()` |
| 業務ルール | 未ログイン時は追加不可、管理者はカート利用不可 |
| 異常系 | 未ログイン、ユーザーなし、商品なし、追加失敗 |

業務ロジック詳細：

1. Session または Cookie からログインユーザー名を取得する
2. ユーザー名が取得できない場合は `Please login first` を設定して商品一覧へ戻す
3. `UserService.getUserByUsername()` でユーザー情報を取得する
4. ユーザーが存在しない場合は再ログイン要求メッセージを返す
5. `ROLE_ADMIN` の場合は一般カートを利用させず、管理画面へ誘導する
6. `CartService.getCarts()` で全カートを取得し、対象ユーザーのカートを検索する
7. カートが存在しない場合は `new Cart()` を生成し `CartService.addCart()` で登録する
8. `ProductService.getProduct(id)` で商品存在確認を行う
9. 商品が存在する場合、`new CartProduct(cart, product)` を生成する
10. `CartProductDao.addCartProduct()` により中間表へ登録する
11. 完了後は `/user/cart` にリダイレクトする

判定条件：

- 一般ユーザーのみカート追加可能
- 商品 ID が不正な場合は追加しない
- カートが未作成でも処理中に自動生成する

### 3.3 F-08 商品管理

| 項目 | 内容 |
|---|---|
| 機能概要 | 管理者が商品の一覧、追加、更新、削除を行う |
| 入力 | 商品基本項目、カテゴリ ID、画像パス等 |
| 出力 | 商品一覧画面への遷移 |
| 主処理 | `AdminController.getproduct()`, `addProduct()`, `updateProduct()`, `removeProduct()` |
| 業務ルール | 管理者ログイン必須、更新時に画像未指定なら既存画像を引き継ぐ |
| 異常系 | 未ログイン、カテゴリ不整合、商品不存在 |

業務ロジック詳細：

- 一覧表示
1. 管理者 Session を確認する
2. 未ログイン時は `adminlogin.jsp` を返す
3. `ProductService.getProducts()` により商品一覧取得
4. 結果を `products.jsp` に引き渡す

- 商品追加
1. 画面から商品入力値を受け取る
2. `CategoryService.getCategory(categoryId)` でカテゴリ存在確認
3. `Product` オブジェクトを生成し各項目を設定
4. `ProductService.addProduct(product)` を実行
5. 登録後、商品一覧へリダイレクト

- 商品更新
1. 対象商品 ID で商品詳細取得
2. 更新値を新しい `Product` オブジェクトに設定
3. `productImage` 未入力時は既存商品の画像値を引き継ぐ
4. `ProductService.updateProduct(id, product)` を実行
5. 更新後、商品一覧へリダイレクト

- 商品削除
1. 商品 ID を受け取る
2. `ProductService.deleteProduct(id)` を実行
3. 一覧へリダイレクト

### 3.4 F-10 管理者プロフィール更新

| 項目 | 内容 |
|---|---|
| 機能概要 | 管理者が自分のユーザー情報を更新する |
| 入力 | `userid`, `username`, `email`, `password`, `address` |
| 出力 | `/index` へリダイレクト |
| 主処理 | `AdminController.updateUserProfile()` |
| 業務ルール | 更新後は Session の `adminUsername` も更新する |
| 異常系 | 未ログイン、対象ユーザー不存在 |

業務ロジック詳細：

1. 管理者ログイン状態を確認する
2. `userid` により対象ユーザーを取得する
3. 対象が存在しない場合はプロフィール画面へ戻す
4. 既存ユーザーに対し `username`, `email`, `password`, `address` を上書きする
5. `UserService.addUser(existingUser)` を呼び出し更新する
6. Session の `adminUsername` を新しいユーザー名で更新する
7. `/index` へリダイレクトする
