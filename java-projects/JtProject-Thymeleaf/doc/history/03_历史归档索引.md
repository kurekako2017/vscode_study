# 历史归档索引

## 1. 作用

本文件用于记录 `doc/history/` 中仍然保留的旧文档及其参考价值，避免继续产生“同主题多入口”的情况。

## 2. startup 归档

| 文件名 | 主题 | 当前用途 |
|---|---|---|
| `archive_STARTUP_SUCCESS.md` | 启动成功说明 | 原始过程记录 |
| `archive_启动项目-最佳方案.md` | 启动建议 | 旧版建议汇总 |

主入口已收敛到：

- [01_启动历史总结.md](../../../JtProject/doc/history/01_启动历史总结.md)

## 3. fixes 归档

| 文件名 | 主题 | 当前用途 |
|---|---|---|
| `archive_运行按钮问题已解决.md` | IDEA 运行按钮修复 | 原始修复记录 |
| `archive_✅登录404错误修复总结.md` | 登录路径 404 修复 | 原始修复记录 |
| `archive_启动问题解决.md` | Java 版本与启动问题 | 原始修复记录 |

主入口已收敛到：

- [02_问题修复历史总结.md](../../../JtProject/doc/history/02_问题修复历史总结.md)

## 4. notes 归档

| 文件名 | 说明 |
|---|---|
| `archive_文档导航.md` | 旧导航，已归档 |
| `archive_memo.md` | 临时记录 |

## 5. 后续维护规则

- 同一主题优先更新合并总结文档
- 只有在确实需要保留过程证据时，才增加新的历史单文档
- 新增历史文档后，应同步更新本索引
- 历史归档默认只保留少量代表性样本，不再保留同主题重复记录
