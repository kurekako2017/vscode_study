# 问题修复历史总结

## 1. 文档目的

本文件用于合并 `doc/history/fixes/` 中重复的修复总结文档，保留真正有持续价值的问题背景、修复类型和经验结论。

## 2. 历史高频问题

### 2.1 IDEA 运行按钮不显示

历史现象：

- IDEA 无法显示运行按钮
- Maven 项目未被正确识别
- `.idea` 运行配置缺失

历史处理方式：

- 创建或修复 `.idea/runConfigurations/JtSpringProjectApplication.xml`
- 更新 `workspace.xml`
- 提供脚本 `scripts/fixes/修复IDEA运行按钮.ps1`

当前建议：

- 优先使用 Maven Wrapper 命令启动，避免被 IDE 配置阻塞
- 仅在确定需要 IDEA 图形运行按钮时，再执行修复脚本
- 对应脚本位置：
- [修复IDEA运行按钮.ps1](D:/dev/source_code/vscode_study/java-projects/JtProject/scripts/fixes/修复IDEA运行按钮.ps1)

### 2.2 登录 404 或路径不一致

历史现象：

- JSP 表单 action 和 Controller 路由不一致
- 手工输入错误 URL 导致访问 404

经验结论：

- 先核对 JSP `form action`
- 再核对 `@RequestMapping` / `@GetMapping` / `@PostMapping`
- 最后核对浏览器实际访问路径

对当前项目的通用检查点：

- 登录表单提交地址
- 控制器实际路由
- 页面跳转后的 redirect 路径

### 2.3 启动失败并非业务代码本身问题

历史现象：

- DevTools 或测试代码导致启动失败
- 测试类错误引用静态字段
- 测试代码编译失败影响主程序启动

经验结论：

- 启动失败先看构建阶段
- 区分主代码、测试代码、IDE 配置、端口冲突
- 不要把所有失败都归因到 Spring Boot 本体

### 2.4 DAO / Controller 注释补充类文档

`✅DAO层注释完善总结.md` 这类文档本质上属于“过程记录”，不是运行故障修复。  
它更适合作为历史修改痕迹保留，而不是主指导文档。

## 3. 当前稳定处理建议

遇到问题时，优先按下面顺序排查：

1. 看 [README.md](D:/dev/source_code/vscode_study/java-projects/JtProject/README.md) 和 [doc/README.md](D:/dev/source_code/vscode_study/java-projects/JtProject/doc/README.md)
2. 用 `.\scripts\start\run.ps1` 或 `mvnw.cmd spring-boot:run` 验证是否能脱离 IDE 启动
3. 查端口、Java 版本、Maven 构建日志
4. 再查 JSP 路由、Controller 映射、测试代码、IDE 配置

## 5. 最终经验结论

- 配置问题和代码问题要分开处理
- 启动问题优先走命令行，少依赖 IDE 图形入口
- 路由问题优先对照 JSP 与 Controller
- 历史修复记录可以保留，但需要由统一总结文档收敛结论
