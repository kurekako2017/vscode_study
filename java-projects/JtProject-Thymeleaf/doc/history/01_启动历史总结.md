本文件用于合并 `doc/history/startup/` 中重复的启动成功、快速启动、IDEA 启动建议类文档，保留稳定结论，减少重复阅读。

## 2. 当前统一结论

- 项目当前默认访问地址为 `http://localhost:8082/`
- 推荐启动方式已经统一为 Maven Wrapper
- Windows PowerShell 推荐使用 `.\scripts\start\run.ps1`
- Windows CMD 推荐使用 `scripts\start\run.cmd`
- Linux / macOS 推荐使用 `./scripts/start/start.sh`

- 当前常用账号：
- 管理员：`admin / 123`
- 普通用户：`lisa / 765`

## 3. 启动问题历史归纳

历史启动文档中重复出现的核心问题主要有以下几类：

### 3.1 Java 版本不匹配

典型现象：

- 旧的 `target` 目录由 Java 21 编译
- 实际运行环境使用 Java 11
- 报错形态包含 `类文件具有错误的版本 65.0, 应为 55.0`

稳定处理方式：

```powershell
.\mvnw.cmd clean
.\mvnw.cmd spring-boot:run
```

说明：

- 当前项目 `pom.xml` 配置目标版本为 Java 11
- 统一使用 Maven Wrapper 可以减少本机 Maven / JDK 环境差异

### 3.2 端口占用

历史文档中既出现过 `8080`，也出现过 `8082`。  
按当前项目配置，应以 `8082` 为准。

排查方式：

```powershell
netstat -ano | findstr :8082
```

如需结束占用进程：

```powershell
Stop-Process -Id <PID> -Force
```

### 3.3 测试类影响启动

历史上曾出现测试代码编译失败导致应用无法正常启动的情况。

当前统一认知：

- 启动问题应先区分是主程序失败还是测试代码失败
- 需要先看 `pom.xml`、`src/test`、构建日志，而不是直接改业务代码
- 当前测试相关背景可参考 [测试类已禁用说明.md](D:/dev/source_code/vscode_study/java-projects/JtProject/doc/reference/测试类已禁用说明.md)

## 4. 当前推荐启动流程


### 4.2 Windows CMD

```cmd
cd /d D:\dev\source_code\vscode_study\java-projects\JtProject
scripts\start\run.cmd
```
