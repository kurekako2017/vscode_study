# JtProject-Thymeleaf 数据访问层与调用链

这个文档沿用原始 `JtProject` 的分析，但这里要额外记住一点：

`JtProject-Thymeleaf` 的主要变化在视图层，而不是业务层。

也就是说：

- `Controller` 还是原来的 Spring MVC Controller
- `Service` 还是原来的业务服务
- `DAO` 还是原来的 Hibernate / EntityManager 混合写法
- 变化主要是页面从 JSP 改成了 Thymeleaf 模板

## 1. 分层结构

```text
浏览器
  -> Controller
  -> Service
  -> DAO
  -> Hibernate / EntityManager
  -> 数据库
  -> Controller 把结果放到 Model / ModelAndView
  -> Thymeleaf 模板渲染
  -> 返回 HTML
```

## 2. 一条最典型的链路：商品列表

```text
GET /user/products
  ->
UserController.getproduct()
  ->
ProductServiceImpl.getProducts()
  ->
ProductDaoImpl.getProducts()
  ->
Hibernate HQL: from Product
  ->
数据库 products 表
  ->
返回 List<Product>
  ->
Thymeleaf 模板 uproduct.html 用 th:each 渲染
```

## 4. 和原项目的区别

| 维度 | 原始 JtProject | JtProject-Thymeleaf |
|---|---|---|
| 视图层 | JSP | Thymeleaf |
| Controller | Spring MVC | Spring MVC |
| Service | 一样 | 一样 |
| DAO | 一样 | 一样 |
| ORM | Hibernate / EntityManager 混合 | Hibernate / EntityManager 混合 |
| 数据库 | 原版默认远程 MySQL | 当前改成本地 H2 |

## 5. 一句话记忆

`JtProject-Thymeleaf = 原项目的业务层和数据层基本保留，只把页面表达方式切换成 Thymeleaf。`
